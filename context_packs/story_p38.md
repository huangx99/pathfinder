# P38 世界交互结算与可见后果 Context Pack

## 目标

实现 P38 时必须把 P37 的“能点按钮形成知识”升级为“动作会改变世界”。

核心体验：

```text
物品会减少或出现；
工具会损耗或恢复；
火源和火把会成为真实可用对象；
野兽威胁会推进、被反制、被结算；
同伴学会知识后能在危险中自动协助一次；野生 NPC/野兽也能基于观察、本能或经验产生一次自主行动。
```

## 实现硬约束

1. 后端是世界变化唯一权威，前端只显示 projection。
2. 所有按钮必须可执行，禁止 `feedback_not_found` 出现在玩家反馈里。
3. `使用野兽` 不能误判为 `用火把驱赶野兽`。
4. 对象定义和对象实例必须分开。
5. 吃、砍、点燃、制作、驱赶必须产生结构化 `WorldChange`。
6. 世界变化不能替代 P21 学习闭环；知识仍由学习链路形成。
7. 目标对象知识必须保留，例如 `torch + use @ beast_shadow`。
8. 同伴自动协助必须由知识、对象、信任和危险状态共同决定，不能写死。
9. 野生 NPC/野兽必须被视为可由 Agent 控制的 Actor，狼可以通过本能、观察或经验知道火有危险。
10. 野兽仍可以先以危险事件呈现，但不能阻碍后续升级为完整 Agent；P38 最小实现至少要预留观察事件、行动意图和规则结算入口。
10. 不做完整背包、地图、战斗和复杂 AI。

## 推荐子阶段

1. P38-A：世界对象实例最小状态。
2. P38-B：世界变更和应用器。
3. P38-C：动作到世界变化桥接。
4. P38-D：危险事件可见结算。
5. P38-E：Agent 最小自主行动，包括同伴协助和野生生物趋避。
6. P38-F：端到端和 H5 按钮门禁。

## 必测闭环

成功路线：

```text
吃红果 -> 红果减少
用斧头砍木头 -> 木头变化，斧头变钝
用火种点燃干草 -> 干草减少，火源出现
制作火把 -> 火把出现
教同伴 -> 同伴获得火把知识
等待到黄昏 -> 野兽威胁推进
同伴或玩家用火把驱赶，或野兽观察到火后主动避让 -> 野兽退去
撑过第一夜
```

失败路线：

```text
一直等待 -> 野兽靠近 -> 没有火把/火源/同伴协助 -> 第一夜失败或被迫逃离
```

## 后续扩展方向

P38 必须为以下内容预留：

- 毒蘑菇/普通蘑菇。
- 湿木头/雨天/火熄灭。
- 火把燃尽。
- 野兽学习火有危险、避让火光、被驯服、成为护卫或坐骑。
- 狼、蜘蛛、人类、外星人作为不同层级 Agent。
- 同伴采集、守夜、救援、反向教学。
- 地点卡和地图扩展。
- 文明能力读取世界变化。
