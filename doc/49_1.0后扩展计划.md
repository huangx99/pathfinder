# 《先驱者》1.0 后扩展计划

## 1. 定位

本文件记录“游戏全部完成后再考虑”的扩展玩法。

这里的内容不进入当前 `P0-P43` 完成路线，不作为 1.0 必须交付内容，也不影响当前最小可玩、MVP、Alpha、Beta、1.0 发布节奏。

它的作用是：

```text
保留高价值想法。
避免当前阶段范围膨胀。
等 1.0 主线完成、稳定、可玩后，再按优先级选择扩展。
```

## 2. 收录原则

只有满足下面条件的内容才放入本扩展计划：

```text
1. 玩法价值高，但不是 1.0 必须完成。
2. 依赖多个核心系统稳定后才适合做。
3. 现在做会扩大范围、拖慢主线完成。
4. 后续可以基于现有架构扩展，不应推翻现有设计。
```

## 3. 扩展项：族群知识驱动的 NPC 谋害识别

### 3.1 一句话结论

当某个知识成为族群知识后，NPC 可以基于该知识判断玩家行为是否危险，甚至判断玩家是否在害他。

示例：

```text
族群已经知道：毒蘑菇 + 吃 -> 中毒。
玩家把毒蘑菇给 NPC 吃。
NPC 查询个人/族群知识后，拒绝食用，并怀疑玩家意图。
```

### 3.2 玩法价值

这个扩展能让知识系统从“图鉴/记录”升级为“世界行为依据”。

它带来的玩法包括：

```text
NPC 会因为知识拒绝危险行为。
族群会因为共享知识避免灾难。
玩家可以救助、欺骗、误导、谋害 NPC。
玩家传播错误知识会造成社会后果。
不同族群因为知识差异产生交易、欺骗、冲突和战争。
```

这非常符合《先驱者》的核心主题：

```text
知识不是静态收藏，而是文明行动的基础。
```

### 3.3 为什么不放进 1.0 必做

这个扩展依赖多个系统同时稳定：

```text
知识查询。
族群知识。
NPC Agent 决策。
行为意图评估。
信任/关系系统。
事件传播。
社会后果。
```

如果在 1.0 前强行加入，会把当前目标从“完成核心学习与文明主线”扩大成“复杂社会模拟”。

因此它应作为 1.0 后扩展，而不是当前主线任务。

### 3.4 最小可行版本

1.0 后如果要做，第一版不要做复杂心理推理，只做最小闭环：

```text
前置知识：tribe_001 knows poison_mushroom + eat -> poison [active]
玩家行为：给同伴吃毒蘑菇
NPC 查询：个人知识 + 族群知识
判断结果：该物品食用后会伤害自己
NPC 反应：拒绝食用
社会结果：对玩家信任下降
事件记录：player_attempted_harm_with_known_poison
```

玩家看到：

```text
你把毒蘑菇递给同伴，示意他吃下去。
同伴后退了一步。
他想起族群早已传开的知识：毒蘑菇会让身体中毒。
同伴拒绝了你，并开始怀疑你的意图。
同伴对你的信任下降了。
```

### 3.5 后续增强方向

后续可分阶段增强：

```text
E1：NPC 知道危险就拒绝。
E2：拒绝后记录信任变化。
E3：判断玩家是否也知道该知识，区分误伤和谋害。
E4：NPC 把玩家危险行为传播给族群。
E5：加入欺骗系统，允许传播假知识，但会被验证和反噬。
E6：族群根据玩家信誉决定交易、合作、驱逐或攻击。
E7：不同族群拥有不同知识，产生知识差异博弈。
```

### 3.6 系统边界

这个扩展必须遵守架构红线：

```text
NPC 只能基于自己或族群已知知识行动，不能读取隐藏真相。
H5/UI 只展示后端结果，不能自己判断玩家是否有恶意。
Agent 是决策层，不是规则层。
信任变化必须通过规则/事件产生，不能写死在前端。
族群知识是主观知识，不等于真实世界真相。
```

### 3.7 未来验收标准

实现该扩展时，至少应满足：

```text
1. NPC 拥有或继承族群知识时，会拒绝已知危险食物。
2. NPC 不知道该知识时，不会凭空知道玩家在害他。
3. 玩家已知该危险知识但仍强迫 NPC 食用时，恶意评分更高。
4. 拒绝、信任下降、族群传播都有事件记录。
5. 全流程可复盘。
6. 调试报告能解释 NPC 为什么拒绝。
```

## 4. 扩展项：假消息、谣言与蓄意误导传播

### 4.1 一句话结论

游戏全部完成后，可以把 P20 的“错误知识可以传播”扩展成正式的假消息系统：角色不只会无意传播错误知识，也可以有目的地制造、包装、传播、放大或澄清假消息。

示例：

```text
玩家故意告诉敌对族群：蓝果可以治疗中毒。
敌对族群信任来源或缺乏验证，接受该知识。
之后成员中毒时误食蓝果，导致延误治疗。
如果事实被验证，玩家信誉下降，敌对族群可能报复。
```

### 4.2 玩法价值

这个扩展能把知识传播从“教学工具”升级为“社会博弈工具”。

它带来的玩法包括：

```text
玩家可以散布假消息误导敌人。
NPC 可以误传、夸大、隐瞒或恶意造谣。
族群会根据来源信誉、证据质量和传播路径决定是否相信。
错误知识可能造成实际行动后果。
真相被验证后，造谣者会承担信任、外交、冲突后果。
不同族群会因为信息不对称产生欺骗、恐慌、清算和宣传战。
```

它强化的核心主题是：

```text
知识不是天然可靠的资源，而是会被污染、争夺和武器化的社会力量。
```

### 4.3 为什么不放进 1.0 必做

P20 已经允许“错误知识、低置信知识可以传播”，但那只是安全传播主观知识，不等于完整假消息玩法。

正式假消息系统还依赖：

```text
稳定知识传播。
知识冲突与修正。
NPC/族群信任关系。
来源信誉。
意图识别。
族群事件传播。
社会惩罚和外交后果。
H5/UI 可读性。
```

如果在 1.0 前强行加入，会把当前主线从“学习、传播、文明能力”扩大成“复杂信息战与社会模拟”。

因此它应作为 1.0 后扩展，而不是当前主线任务。

### 4.4 最小可行版本

第一版只做“可追溯的低证据假消息传播”，不要做复杂宣传战。

流程：

```text
玩家行为：向 NPC/族群声明一条未验证知识。
系统输入：claim + declared_source + evidence_quality + intent_hint。
传播判断：按信任、通道质量、接收者已有知识决定是否接受。
接收者结果：形成 Rumor/Hypothesis 状态知识，而不是高置信 Active 知识。
后续验证：接收者亲身经历或获得反证后，进入 P19 修正。
社会结果：若证明为假，来源信誉下降，关系恶化，可能触发冲突事件。
```

红果示例：

```text
玩家告诉同伴：腐烂红果吃了能恢复体力。
同伴没有亲自验证，只形成“听玩家说过”的低置信谣言。
如果同伴真的吃下腐烂红果并中毒，知识被修正。
事件记录：false_claim_exposed / source_trust_decreased。
```

### 4.5 与 P20 错误知识传播的区别

P20 当前边界：

```text
传播已有主观知识。
错误知识可以传播。
接收者获得低置信、可追溯知识。
后续由 P19 修正。
```

假消息扩展新增：

```text
允许主动构造声明，而不一定来自发送者高置信知识。
记录传播意图：误传、夸大、隐瞒、蓄意误导。
记录来源信誉和伪装来源。
支持谣言扩散、澄清、反制和追责。
把“这条消息是假的吗”转化为可验证的社会事件，而不是直接读取 hidden truth。
```

### 4.6 系统边界

假消息扩展必须遵守：

```text
系统不能用 hidden truth 直接给 NPC 判定“这是假的”。
NPC 只能根据已有知识、证据链、来源信誉和后续经验判断可信度。
假消息不能绕过 KnowledgeClaim、Evidence、Propagation、Revision。
UI 只能展示后端给出的信任、来源、证据和冲突结果。
造谣惩罚必须由事件和规则产生，不能写死在前端。
蓄意误导是意图标签，不等于系统全知判真。
```

### 4.7 后续增强方向

后续可分阶段增强：

```text
E1：低证据传闻，接收者形成 Rumor/Hypothesis。
E2：来源信誉影响接受概率和初始置信度。
E3：假消息被亲身验证后触发信誉下降。
E4：NPC 可以主动澄清、反驳或二次传播。
E5：族群层出现恐慌、宣传、禁令和清算。
E6：加入伪装来源、嫁祸、反间、双重间谍。
E7：不同族群形成互相矛盾的信息生态。
```

### 4.8 未来验收标准

实现该扩展时，至少应满足：

```text
1. 玩家或 NPC 可以传播低证据声明。
2. 接收者不会把假消息直接当成高置信真理。
3. 假消息必须保留来源、通道、证据质量和传播链。
4. 后续经验可以证实、削弱、废弃或证伪该消息。
5. 被证伪后，来源信誉和关系会发生可追溯变化。
6. NPC 不知道真相时，不会凭空识破假消息。
7. 调试报告能解释为什么相信、为什么怀疑、为什么追责。
```

## 5. 扩展项：专家 NPC 与可教学 NPC

### 5.1 一句话结论

游戏全部完成后，可以加入“专家 NPC”系统，让部分 NPC 具备稳定知识、教学能力和课程行为，能够向玩家或其他 NPC 传授知识。

示例：

```text
驾校教练可以教玩家开车。
采集专家可以教玩家识别可食用植物。
铁匠可以教玩家打磨石器或制作工具。
猎人可以教玩家追踪、伏击和规避猛兽。
医生可以教玩家处理中毒、止血和辨认草药。
```

### 5.2 玩法价值

专家 NPC 能把“知识传播”从玩家单向传播扩展成双向学习网络：

```text
玩家可以向 NPC 学习。
NPC 可以训练其他 NPC。
族群可以培养专家岗位。
专家死亡、叛逃或迁移会造成知识断层。
玩家也可以成长为专家，再反向教学。
```

这能让文明发展更像真实社会：

```text
知识不只来自试错，也来自老师、学徒、职业、制度和传承。
```

### 5.3 为什么不放进 1.0 必做

专家 NPC 依赖多个系统稳定：

```text
NPC Agent。
知识传播。
个人/族群知识查询。
教学意愿和信任关系。
课程/训练任务。
技能或能力成熟度。
练习结果和规则验证。
```

如果现在加入，会把当前目标扩大成“职业训练与社会教育系统”，不适合在 1.0 主线前实现。

因此专家 NPC 属于 1.0 后扩展，而不是当前完成路线。

### 5.4 最小可行版本

第一版只做“专家 NPC 教玩家理论知识”，不要直接解锁能力。

流程：

```text
前置：NPC 拥有某类高置信知识。
玩家行为：向专家请教。
NPC 判断：是否愿意教、是否具备教学条件。
传播结果：NPC -> 玩家形成 shared/hypothesis 知识。
后续要求：玩家必须通过练习把知识转成 active/operational。
```

驾校教练示例：

```text
玩家：向教练学习开车。
教练：讲解方向盘、刹车、红灯、湿滑路面风险。
玩家获得：
- steering_wheel + use -> change_direction [shared]
- brake + use -> reduce_speed [shared]
- red_light + indicates -> stop_required [shared]
- car + use -> requires_driving_skill [shared]
```

然后玩家还需要：

```text
练习开车。
形成驾驶记忆。
修正错误认知。
通过基础驾驶验证。
最终获得基础驾驶能力。
```

### 5.5 教学和能力必须分离

专家 NPC 不能一句话直接让玩家获得完整能力。

正确边界：

```text
专家教学 = 传授知识和方法。
玩家练习 = 形成经验和记忆。
规则验证 = 判断是否能稳定执行。
能力系统 = 在证据足够后授予稳定能力。
```

错误做法：

```text
和教练对话一次 -> driving_skill = true
```

正确做法：

```text
教练讲解 -> 玩家获得理论知识 -> 玩家练习 -> 形成记忆 -> 强化/修正知识 -> 能力进度提升 -> 通过验证 -> 解锁基础能力
```

### 5.6 专家 NPC 类型示例

后续可以逐步加入：

```text
采集专家：教可食用植物、危险植物、季节采集。
狩猎专家：教追踪、伏击、协作狩猎、避险。
工具专家：教石器、火把、容器、基础制造。
医药专家：教止血、解毒、草药辨认。
战斗教官：教阵型、盾牌、防御、共同作战。
驾驶教练：教车辆操作、交通规则、风险规避。
外交导师：教语言、礼仪、谈判和欺骗识别。
```

### 5.7 系统边界

专家 NPC 扩展必须遵守：

```text
NPC 只能教授自己拥有或所属组织允许传播的知识。
NPC 不能教授隐藏真相，只能教授主观知识。
教学结果不能绕过记忆、知识、能力和规则验证。
玩家能力不能由前端直接赋值。
专家教学必须产生事件，可复盘、可审计。
错误教学、低可信教学、恶意教学都应允许存在，并能被后续经验修正。
```

### 5.8 未来验收标准

实现该扩展时，至少应满足：

```text
1. 专家 NPC 拥有明确知识集合和教学权限。
2. 玩家可以请求教学。
3. NPC 可以根据信任、代价、条件决定是否教学。
4. 教学后玩家获得 shared/hypothesis 知识，而不是直接获得完整能力。
5. 玩家练习后可以把教学知识提升为 active/operational。
6. 教学、拒绝教学、练习、能力提升都有事件记录。
7. 调试报告能解释 NPC 为什么教、为什么拒绝、玩家为什么还不能稳定执行。
```
