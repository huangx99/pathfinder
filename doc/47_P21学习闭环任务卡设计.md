# P21 学习闭环任务卡设计

## 1. 阶段定位

P21 是《先驱者》后端最小可玩主线的学习闭环编排阶段。

P15 到 P20 已经分别完成：

```text
P15：认知系统正式化，主体能从安全反馈形成主观认知。
P16：记忆系统基础，主体能把经历写成短期/中期/长期记忆。
P17：记忆压缩与检索，主体能把多次经历整理成可召回摘要。
P18：知识系统，主体能从记忆摘要形成 KnowledgeClaim。
P19：知识冲突与修正，主体能用新证据修正、特化、废弃旧知识。
P20：知识传播系统，主体能把主观知识传播给另一个主体。
```

P21 要解决的是：

```text
把 P15-P20 串成一条可复盘、可测试、可解释的后端学习故事。
```

一句话：

```text
P21 让“尝试一个未知物 → 得到反馈 → 形成认知 → 写入记忆 → 压缩召回 → 形成知识 → 修正知识 → 传播给别人 → 别人的可用知识发生变化”成为一个可验证闭环。
```

P21 不是新规则系统，也不是前端阶段。它是学习链路的编排层。

---

## 2. 本阶段目标

P21 的目标是建立 `learning` 模块，提供一个后端最小可玩学习闭环。

它必须做到：

1. 串联 P15 `cognition`、P16 `memory writer`、P17 `memory compression/recall`、P18 `knowledge formation`、P19 `knowledge revision`、P20 `knowledge propagation`。
2. 用一个安全输入描述“一次学习经历”，不能让学习闭环读取 hidden truth 或 raw GameState。
3. 输出完整 `LearningLoopResult`，明确每一步是否执行、跳过、失败、产生了什么结果。
4. 输出可读的 `LearningDebugReport`，供研发、测试、后续 H5/debug 面板复盘。
5. 支持最小可玩故事：红果从未知到可食用知识，再传播给另一个 Agent。
6. 支持纠错故事：腐烂红果导致旧知识修正，并把修正传播给别人。
7. 必须消费 P20 冲突传播信号：当传播结果要求 route to revision 时，P21 必须进入或安排 P19 修正流程，不能把传播误判为完成。
8. 所有 `Result::ok(dto)` 输出都必须通过 `validateBasic()`。
9. 所有步骤必须保留追溯链：反馈、认知、记忆、摘要、知识、传播、修正之间能互相引用。

核心原则：

```text
P21 只编排，不重写规则。
P21 只吃安全投影，不吃世界真相。
P21 只验证后端最小可玩，不做 UI。
P21 产出的是学习链路结果，不是 Agent 策略。
```

## 3. 玩家体验映射

虽然 P21 不做 H5，但它直接验证玩家未来能不能玩懂这个游戏。

### 3.1 玩家实际感受

```text
开局我不知道红果有什么用。
我让先驱者吃一口红果。
系统反馈：饥饿恢复。
先驱者记住：吃红果可能有益。
多次尝试后，系统形成知识：红果可食用，吃了恢复饥饿。
先驱者把这个知识教给另一个族人。
另一个族人再看到红果时，不再把它当未知危险，而知道“可以吃，但这是别人教的”。
```

### 3.2 P21 需要证明的体验链

P21 的测试和报告必须能证明：

```text
玩家行为不是白做的：一次尝试能进入认知和记忆。
重复经历不是白堆的：记忆能被压缩成证据摘要。
知识不是凭空来的：KnowledgeClaim 有来源摘要和来源记忆。
知识不是绝对真理：新反证能触发 P19 修正。
教学不是复制真相：接收者得到的是 Teaching 来源的主观知识。
```

## 5. 本阶段不做什么

P21 明确禁止：

```text
不做 H5。
不做 HTTP / WebSocket / JSON API。
不做存档 SaveManager。
不做族群政治、士气、分裂、共同作战。
不做文明科技树解锁。
不做对象反应扩展，例如火加木棍变火把。
不做完整 Agent 策略 AI。
不做强化学习 reward/done。
不让 AgentRuntime 或 Policy 成为规则权威。
不让 UI 自己推导学习结果。
不重写 P15-P20 已有规则。
不直接写 GameState。
不读取 ObjectDefinition hidden truth。
不读取 raw GameState / SaveGame / AgentTickRecord 作为学习依据。
```

P21 可以做：

```text
定义 learning 编排层 DTO。
调用已有服务。
返回状态变更草稿、事件草稿、知识草稿、报告草稿。
用测试数据构造安全反馈输入。
用 integration test 证明后端最小可玩链路成立。
```

## 8. 核心枚举设计

### 8.1 LearningLoopStoryKind

用途：标识这次闭环是哪类学习故事，方便测试、报告和后续 debug UI 分组。

| 枚举 | 字符串 | 中文翻译 | 说明 |
|---|---|---|---|
| `Unknown` | `unknown` | 未知 | 非法默认值 |
| `DirectDiscovery` | `direct_discovery` | 直接发现 | 通过亲身尝试形成新知识 |
| `RepeatedLearning` | `repeated_learning` | 重复学习 | 多次反馈累积形成知识 |
| `CorrectionAfterContradiction` | `correction_after_contradiction` | 反证纠错 | 新反馈推翻或特化旧知识 |
| `TeachingToRecipient` | `teaching_to_recipient` | 教学传播 | 把知识传播给接收者 |
| `FullLearningLoop` | `full_learning_loop` | 完整学习闭环 | 从反馈到传播全链路 |
| `TestOnly` | `test_only` | 测试专用 | 只允许测试显式开启 |

### 8.2 LearningLoopStage

用途：标识闭环步骤。每个步骤都必须产生 `LearningLoopStepTrace`。

| 枚举 | 字符串 | 中文翻译 | 说明 |

| `FeedbackCaptured` | `feedback_captured` | 捕获反馈 | 安全反馈输入已接收 |
| `CognitionUpdated` | `cognition_updated` | 认知更新 | P15 形成或更新认知 |
| `MemoryWritten` | `memory_written` | 写入记忆 | P16 写入经历记忆 |

## 9. DTO 设计

### 9.1 LearningActorRef

用途：表示学习者或接收者的安全主体引用。

```cpp
struct LearningActorRef {
    knowledge::KnowledgeOwner knowledge_owner;
    memory::MemoryOwner memory_owner;
    cognition::CognitionSubject cognition_subject;
    std::string display_key;
    std::vector<std::string> safe_tags;

    foundation::Result<void> validateBasic() const;
};
```

规则：

```text
knowledge_owner、memory_owner、cognition_subject 必须指向同一逻辑主体。
display_key 只能是安全展示 key，不能含 hidden truth。
safe_tags 不能包含 forbidden key。
```

### 9.2 LearningSafeFeedbackInput

用途：表示一次可学习反馈。它是 P21 唯一允许消费的“经历输入”。

```cpp
struct LearningSafeFeedbackInput {
    std::string feedback_key;
    cognition::CognitionTarget cognition_target;
    memory::MemorySubject memory_subject;
    knowledge::KnowledgeSubject knowledge_subject;
    cognition::CognitionActionContextKind action_context;
    foundation::ActionId action_id;
    std::string action_key;
    std::vector<cognition::CognitionOutcomeSignal> outcome_signals;
    std::string effect_key;
    std::vector<std::string> condition_keys;
    double utility_delta = 0.0;
    double risk_delta = 0.0;
    foundation::Tick observed_tick;
    std::vector<foundation::EventId> source_event_ids;
    std::vector<std::string> reason_keys;
    std::vector<std::string> warning_keys;

    foundation::Result<void> validateBasic() const;
};
```

字段中文说明：

```text
feedback_key：反馈键，用于追踪这次学习经历。
cognition_target：认知目标，例如某个世界物体或对象类别。
memory_subject：记忆主题，用于写入和检索记忆。
knowledge_subject：知识主题，用于生成 KnowledgeClaim。
action_context：行为上下文，例如吃、使用、触摸、组合。
action_id/action_key：行为标识。
outcome_signals：观察到的结果信号，例如需求改善、健康恶化、工具激活。
effect_key：安全效果 key，例如 restore_hunger、poison、ignite，不是真实隐藏数值。
condition_keys：安全条件 key，例如 decayed、wet、night。
utility_delta：主观效用变化，范围建议 [-1,1]。
risk_delta：主观风险变化，范围建议 [0,1]。
observed_tick：观察时间。
source_event_ids：来源事件 id。
reason_keys/warning_keys：安全原因与警告 key。
```

禁止：

```text
字段名或内容不得包含 hidden_truth、real_effect、edible_profile、hunger_delta、health_delta、raw_state 等真相语义。
P21 不把 utility_delta 当真实数值，只当主观反馈强度。
```

### 9.3 LearningLoopOptions

用途：控制闭环执行范围，支持单元测试和渐进开发。

```cpp
struct LearningLoopOptions {
    bool allow_test_only = false;
    bool enable_cognition = true;
    bool enable_memory_write = true;
    bool enable_memory_compression = true;
    bool enable_memory_recall = true;
    bool enable_knowledge_formation = true;
    bool enable_knowledge_revision = true;
    bool enable_knowledge_propagation = true;
    bool enable_recipient_projection_check = true;
    bool require_full_chain = true;
    bool route_propagation_conflict_to_revision = true;
    size_t max_step_traces = 64;
    size_t max_report_items = 128;

    foundation::Result<void> validateBasic() const;
};
```

规则：

```text
require_full_chain=true 时，必要步骤失败则整体 Failed。
require_full_chain=false 时，允许 PartialCompleted。
route_propagation_conflict_to_revision 默认必须为 true。
max_step_traces、max_report_items 必须 > 0。
```

### 9.4 LearningLoopInput

用途：一次学习闭环的总输入。

```cpp
struct LearningLoopInput {
    std::string loop_key;
    LearningLoopStoryKind story_kind = LearningLoopStoryKind::Unknown;
    LearningActorRef actor;
    std::optional<LearningActorRef> recipient;
    LearningSafeFeedbackInput feedback;
    std::vector<memory::MemoryRecord> actor_existing_memories;
    std::vector<memory::MemorySummary> actor_existing_summaries;
    std::vector<knowledge::KnowledgeClaim> actor_existing_claims;
    std::vector<knowledge::KnowledgeClaim> recipient_existing_claims;
    foundation::Tick loop_tick;
    std::vector<std::string> reason_keys;

    foundation::Result<void> validateBasic() const;
};
```

规则：

```text
loop_key 必填且只能是安全 key。
story_kind 不能是 Unknown/TestOnly，除非 allow_test_only=true。
actor 必填。
recipient 可选；没有 recipient 时不执行 P20。
existing_* 都是安全快照，由调用方提供，P21 不自己读取仓库或 GameState。
```

### 9.5 LearningEventChain

用途：把反馈、认知、记忆、知识、传播这些分散产物串成一条轻量事件链。它是路线图中 P21 的正式交付之一。

```cpp
struct LearningEventChain {
    std::string chain_key;
    std::string loop_key;
    LearningLoopStoryKind story_kind = LearningLoopStoryKind::Unknown;
    std::vector<foundation::EventId> source_event_ids;
    std::vector<cognition::CognitionEvidenceRecordId> cognition_evidence_ids;
    std::vector<cognition::CognitionClaimV2Id> cognition_claim_ids;
    std::vector<foundation::MemoryId> memory_ids;
    std::vector<memory::MemorySummaryId> memory_summary_ids;
    std::vector<foundation::KnowledgeId> knowledge_ids;
    std::vector<foundation::KnowledgeId> recipient_knowledge_ids;
    std::vector<std::string> chain_reason_keys;
    std::vector<std::string> warning_keys;

    foundation::Result<void> validateBasic() const;
};
```

规则：

```text
LearningEventChain 只保存 id 和安全 key，不保存大型对象。
每个 id 必须来自 P15-P20 的真实输出，不能伪造。
chain_key 必须稳定可复现，不能依赖随机数。
后续 H5、复盘、审计优先读 EventChain，而不是重新扫描所有大对象。
```

### 9.6 LearningLoopStepTrace

用途：记录每一步发生了什么，方便测试与调试。

```cpp
struct LearningLoopStepTrace {
    LearningLoopStage stage = LearningLoopStage::Unknown;
    LearningStepDecision decision = LearningStepDecision::Unknown;
    LearningLoopFailureReason failure_reason = LearningLoopFailureReason::None;
    std::string summary_key;
    std::vector<foundation::MemoryId> memory_ids;
    std::vector<memory::MemorySummaryId> memory_summary_ids;
    std::vector<foundation::KnowledgeId> knowledge_ids;
    std::vector<foundation::KnowledgeId> updated_knowledge_ids;
    std::vector<std::string> reason_keys;
    std::vector<std::string> warning_keys;

    foundation::Result<void> validateBasic() const;
};
```

规则：

```text
每个执行步骤必须有 trace。
失败步骤必须设置 failure_reason。
summary_key 只能使用安全枚举/字符串 key。
trace 不保存大型 DTO，避免上下文过大。
```

### 9.7 LearningLoopResult

用途：学习闭环的机器可读输出。

```cpp
struct LearningLoopResult {
    LearningLoopDecision decision = LearningLoopDecision::Unknown;
    LearningLoopFailureReason failure_reason = LearningLoopFailureReason::Unknown;
    std::optional<LearningEventChain> event_chain;
    std::vector<LearningLoopStepTrace> traces;

    std::optional<cognition::CognitionUpdateResult> cognition_result;
    std::optional<memory::MemoryWriteResult> memory_write_result;
    std::optional<memory::MemoryCompressionResult> memory_compression_result;
    std::optional<memory::MemoryRecallResult> memory_recall_result;
    std::optional<knowledge::KnowledgeFormationResult> knowledge_formation_result;
    std::optional<knowledge::KnowledgeRevisionResult> knowledge_revision_result;
    std::optional<knowledge::KnowledgePropagationResult> knowledge_propagation_result;
    std::optional<knowledge::KnowledgeProjection> recipient_projection;

    std::vector<knowledge::KnowledgeClaim> actor_claim_snapshot_after;
    std::vector<knowledge::KnowledgeClaim> recipient_claim_snapshot_after;
    std::vector<std::string> reason_keys;
    std::vector<std::string> warning_keys;

    bool ok() const;
    foundation::Result<void> validateBasic() const;
};
```

`ok()` 语义：

```text
Completed、PartialCompleted、Skipped、RoutedToRevision 都是合法业务结果。
Rejected、Failed 也是合法 DTO，但 ok() 可按业务约定返回 true，只要 validateBasic() 通过。
Unknown/TestOnly 默认不是 ok。
```

为保持 P20 一致性，推荐：

```text
Result<LearningLoopResult> 表示 C++ 调用是否成功。
LearningLoopResult::decision 表示玩法业务结果。
```

### 9.8 LearningDebugReport

```cpp
struct LearningDebugReport {

    std::vector<std::string> trace_summary_keys;
    std::vector<std::string> warning_keys;

```

规则：

```text

Report 不保存完整 MemoryRecord 或 KnowledgeClaim 大对象。
```

### 10.2 LearningLoopService

用途：执行闭环编排。

```cpp
class LearningLoopService {
public:
    foundation::Result<LearningLoopResult> run(
        const LearningLoopInput& input,
        const LearningLoopOptions& options = {}) const;
};
```

执行顺序必须固定：

```text
1. validate input/options
2. FeedbackCaptured
3. CognitionUpdated
4. MemoryWritten
5. MemoryCompressed
6. MemoryRecalled
7. KnowledgeFormed
8. KnowledgeRevised（当有冲突或新知识需要修正旧知识时）
9. KnowledgePropagated（当有 recipient 且有可传播知识时）
10. KnowledgeRevised（当 P20 assessment.should_route_to_revision=true 时，对 recipient 进入修正编排）
11. RecipientInfluenced
12. ReportBuilt
13. Completed / PartialCompleted / RoutedToRevision / Failed
```

注意：

```text
第 10 步是 P21 的关键要求。
P20 发现接收者已有同 subject/action/relation 但不同 effect 的知识时，会给出 should_route_to_revision。
P21 必须消费这个信号，不能把传播当普通完成。
```

## 11. 核心编排语义

### 11.1 正常学习路径

```text
LearningSafeFeedbackInput
→ CognitionEvidence / CognitionClaim
→ MemoryWriteInput / MemoryRecord
→ MemoryCompressionResult / MemorySummary
→ MemoryRecallResult
→ KnowledgeFormationInput / KnowledgeClaim
→ KnowledgeProjection
```

### 11.2 纠错路径

```text
新反馈与旧 KnowledgeClaim 冲突
→ KnowledgeRevisionInput
→ KnowledgeRevisionResult
→ updated_claims / created_claims / patches
→ actor_claim_snapshot_after
```

### 11.3 传播路径

```text
actor_claim_snapshot_after 中选择可传播 claim
→ KnowledgePropagationAttempt
→ KnowledgePropagationResult
→ recipient_claim_snapshot_after
→ KnowledgeProjectionBuilder 检查接收者可用投影
```

### 11.4 P20 route-to-revision 路径

当 P20 结果中任一 assessment 满足：

```text
assessment.should_route_to_revision == true
```

P21 必须：

```text
1. 添加 LearningLoopStepTrace：KnowledgeRevised / Routed。
2. 构造 recipient 视角的 KnowledgeRevisionInput。
3. 调用 P19 KnowledgeRevisionService 或在本阶段最小实现中生成明确 RoutedToRevision 结果。
4. 不得创建普通 recipient claim 来绕过冲突。
5. 最终 decision 应为 RoutedToRevision、PartialCompleted 或 Completed，具体取决于 revision 是否实际完成。
```

第一版推荐实际调用 P19，而不是只打标。

## 12. 最小可玩验收故事

### 12.1 未知红果学习传播

测试名建议：

```text
learning_loop_discovery_story_test
```

故事：

```text
Agent A 开始不知道 red_berry 的用途。
A 吃 red_berry。
安全反馈：NeedImproved，effect_key=restore_hunger。
P15 生成支持“red_berry 可食用/吃了有益”的认知。
P16 写入一条体验记忆。
P17 在足够样本或测试配置下生成 MemorySummary。
P18 形成 KnowledgeClaim：red_berry + eat -> restore_hunger。
P20 把该知识传播给 Agent B。
B 获得 Teaching 来源的主观 KnowledgeClaim。
B 的 KnowledgeProjection 可以看到 red_berry 可食用/有益。
```

验收：

```text
LearningLoopResult.decision == Completed。
traces 至少包含 CognitionUpdated、MemoryWritten、MemoryCompressed、KnowledgeFormed、KnowledgePropagated、RecipientInfluenced。
actor_claim_snapshot_after 至少有 1 条 red_berry/eat/restore_hunger claim。
recipient_claim_snapshot_after 至少有 1 条 Teaching evidence 的 claim。
接收者 claim owner 必须是 B，不得复用 A 的 owner。
没有 hidden truth/raw GameState 字段。
所有 ok DTO validateBasic 通过。
```

### 12.2 腐烂红果纠错传播

测试名建议：

```text
learning_loop_revision_story_test
```

故事：

```text
A 旧知识：red_berry + eat -> restore_hunger。
A 新反馈：decayed_red_berry + eat -> poison。
P19 应修正或特化知识：decayed 条件下吃 red_berry 有风险。
旧知识不能被无条件当作全局真理继续扩张。
A 把修正后的知识传播给 B。
如果 B 已有 red_berry + eat -> restore_hunger，P20 应 route_to_revision。
P21 必须消费这个路由，并触发或安排 B 的 P19 修正。
```

验收：

```text
traces 必须包含 KnowledgeRevised。
腐烂条件必须进入 KnowledgeCondition 或 safe condition key。
旧知识若被特化，必须有 supersedes/superseded 或 patch 追溯。
P20 should_route_to_revision=true 时，P21 不得创建普通重复 claim。
最终 recipient_claim_snapshot_after 不得出现两个同 subject/action/relation 但互相矛盾且无 revision trace 的 active claim。
```

### 12.3 错误知识可传播但可追溯

测试名建议：

```text
learning_loop_low_confidence_teaching_trace_test
```

故事：

```text
A 基于低样本形成低置信知识。
A 可以通过 Teaching 传播给 B，但 B 得到的是低置信、Teaching 来源知识。
B 后续亲身反证可以通过 P19 修正。
```

验收：

```text
传播后的 recipient claim 置信度不得超过 P20 上限。
recipient evidence_kind 必须是 Teaching。
source_knowledge_id、source owner、reason_keys 必须可追溯。
低置信知识不能被标记为 Operational/Teachable，除非 P18/P20 明确允许。
```
