# 28_P3未知对象首条规则闭环任务卡设计

## 1. 设计目标

本章把 P3 拆成可以直接交给 AI 编码执行的小任务卡。

P3 的目标是在 P0/P1/P2 地基之上，完成《先驱者》规则引擎的第一条真实可玩闭环：

```text
未知对象出现
-> 玩家执行 eat 尝试
-> 规则产生状态变化
-> 状态变化经过 StateChangeGate
-> 应用最小状态变化
-> 生成反馈事件
-> 反馈形成认知证据
-> 认知状态产生“这个东西可以吃/有影响”的初步认知
-> 测试可复盘验证
```

这不是完整游戏，也不是完整 RuleEngine。

一句话：

```text
P3 只证明一条规则链路跑通：未知果实 + 尝试吃 + 反馈 + 认知。
```

## 2. 玩家体验对应目标

P3 对应玩家体验是：

```text
玩家看到一个未知对象。
玩家不知道它有什么用。
玩家选择吃一口进行尝试。
系统反馈身体变化。
玩家获得“这个对象可食用/会产生某种影响”的认知证据。
再次查看时，不是视觉形态改变，而是用途认知发生变化。
```

P3 必须牢记早期设计修正：

```text
不是 ObjectRevealLevel。
不是揭示颜色、形状、外观。
不是从马赛克变成图片。
而是认知层变化：能否食用、食用后影响、风险、置信度。
```

P3 的最小闭环只关注：

```text
eat。
unknown_fruit。
actor hunger/health 的最小变化。
feedback event。
cognition evidence。
```

## 3. 本阶段解决什么问题

P3 要解决的是“抽象架构第一次落地成可验证规则”的问题。

没有 P3，会出现这些风险：

```text
P0/P1/P2 都只是地基，没有证明规则链路可跑通。
StateChange / EventRecord / PipelineResult 只是结构，没有真实组合测试。
认知系统仍停留在文档，没有证明反馈能进入认知。
后续 AI 可能不知道真实规则应该放在哪一层。
```

P3 必须证明：

```text
Command 不直接改状态。
Resolver 不绕过 StateChangeGate。
StateChange 先校验再 apply。
EventRecord 记录事实。
Cognition 不保存真实规则，只保存主体认知。
Pipeline 负责编排，不把所有逻辑写成巨大 if/else。
```

P3 严禁提前实现：

```text

完整存档复盘。

```

P3 允许：

```text

```


## 6. P3 最小闭环定义

### 6.1 初始状态

```text
GameState metadata 合法。
Actor actor_001 存在。
Actor 有最小 survival 状态：hunger、health。
WorldObject obj_unknown_fruit_001 存在。
WorldObject 指向 ObjectDefinition unknown_fruit。
ObjectDefinition 中声明 P3 fixture 真相：eat 后 hunger_delta = -20，health_delta = 0，可食用。
CognitionState 中 actor_001 对 unknown_fruit + eat 没有认知。
```

说明：

```text
hunger 数值越低表示越不饿。
P3 只用这个约定做测试，后续数值系统可扩展。
```

### 6.2 输入命令

```text
CommandEnvelope.command_id = cmd_try_eat_001。
source = Player 或 Test。
payload.action_id = eat。
payload.actor_id = actor_001。
payload.targets[0].target_type = object。
payload.targets[0].target_id = obj_unknown_fruit_001。
payload.intent = experiment。
```

### 6.3 规则结果

```text
StateChangeSet 至少包含：
1. actor_001 hunger 从 80 变为 60。
2. obj_unknown_fruit_001 被标记 consumed 或 quantity 减少。
3. cognition 中新增 unknown_fruit + eat 的证据或认知条目。

EventStream 至少包含：
1. ActionResolved 或 CommandAccepted 事实。
2. ActorStateChanged 或 StateChanged 事实。
3. FeedbackGenerated 事实。
4. CognitionEvidenceAdded 或 CognitionUpdated 事实。
```

P3 可简化事件名称，但必须能表达：

```text
吃了。
身体发生变化。
获得认知证据。
```

### 6.4 玩家认知结果

认知输出必须表达：

```text
对象定义 unknown_fruit。
行为 eat。
认知类型 edible 或 hunger_effect。
效果方向 beneficial。
置信度 > 0。
证据数量 = 1。
```

禁止表达：

```text
颜色被揭示。
形状被揭示。
外观从马赛克变具体。
```

## 29. 给 AI 编码执行者的最终提醒

```text
P3 是第一条真实规则闭环，不是完整游戏。
只能做 unknown fruit + eat。
Command 表达意图，不改状态。
Resolver 生成草稿，不直接改状态。
StateChangeGate 必须经过。
MinimalStateApplier 才能改 GameState。
EventRecord 记录事实。
Cognition 记录用途认知，不记录颜色形状。
Pipeline 只接入 P3 单条规则，不变成完整 RuleEngine。
不要做 use/combine。
不要做 Agent。
不要做 H5。
不要做传播和文明。
```
