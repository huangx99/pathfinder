# 先驱者：行为命令设计

## 1. 本文档目标

本文档用于设计《先驱者》规则引擎中的行为命令系统。

行为命令系统是玩家、AI、系统、脚本和复盘向规则引擎表达行动意图的统一入口。

一句话：

```text
Command 表达“想做什么”。
Rule 负责“能不能做、怎么结算”。
StateChange 表达“状态怎么变”。
Event 表达“发生了什么事实”。
```

行为命令系统必须兼容：

```text
01 基础类型与 ID。
02 通用枚举。
03 错误码与结果类型。
04 事件系统。
05 配置系统。
06 世界对象系统。
07 特性系统。
后续交互规则、反馈、认知、知识、传播、规则管线、复盘和强化学习。
```

## 2. 设计边界

### 2.1 本文档要解决的问题

```text
定义 ActionCommand。
定义 CommandEnvelope。
定义 CommandSource。
定义 CommandTarget。
定义 CommandParam。
定义 CommandIntent。
定义命令标准化、校验、路由和复盘记录。
定义命令如何接入配置、对象、特性、错误、事件、复盘和强化学习。
```

### 2.2 本文档暂不解决的问题

```text
不设计行为最终结算规则。
不设计对象反应具体规则。
不设计效果如何修改状态。
不设计完整反馈与认知流程。
不设计完整规则管线。
不设计前端 UI 交互细节。
```

这些会在后续交互规则、反馈系统、认知系统和规则管线设计中展开。

## 3. 核心原则

### 3.1 Command 是意图，不是结果

命令只表达调用方想做什么。

例如：

```text
玩家想吃 obj_000001。
AI 想采集 obj_000002。
系统想推进一帧等待。
复盘想重放 cmd_000001。
```

命令本身不保证成功。

成功、失败、阻断、部分成功由规则结算后通过 `RuleExecutionResult` 表达。

### 3.2 玩家、AI、系统、复盘使用同一入口

所有行动都应统一进入命令系统。

```text
玩家输入 -> Command
AI 决策 -> Command
系统行为 -> Command
复盘重放 -> Command
测试用例 -> Command
```

这样事件、错误、测试、复盘和强化学习都能统一追踪。

### 3.3 前端不直接调用规则函数

前端不应调用：

```text
eat(object_id)
collect(object_id)
combine(object_a, object_b)
```

前端只提交标准命令。

规则引擎负责校验、路由和结算。

### 3.4 命令必须可序列化、可校验、可重放

```text
可序列化：可以进入日志、存档、复盘和测试。
可校验：非法命令在规则结算前被拒绝。
可重放：同样命令在同样状态和随机流下得到同样结果。
```

### 3.5 命令不直接修改状态

命令只进入规则系统。

状态变化必须由规则管线生成 `StateChange`。

事件必须由事件系统生成 `EventRecord`。

## 4. 行为命令系统总架构

第一阶段行为命令系统包含：

| 模块 | 中文名 | 作用 |
|---|---|---|
| CommandEnvelope | 命令外壳 | 命令元信息，来源、时间、幂等键等 |
| ActionCommand | 行为命令 | 一次具体行动意图 |
| CommandSource | 命令来源 | 玩家、AI、系统、复盘等 |
| CommandTarget | 命令目标 | 对象、区域、实体、知识、族群等 |
| CommandParam | 命令参数 | 数量、风险选项、工具等 |
| CommandIntent | 命令意图 | 探索、利用、教学、避险等 |
| CommandValidationResult | 命令校验结果 | 命令是否合法 |
| CommandNormalizer | 命令标准化器 | 外部输入转内部标准命令 |
| CommandRouter | 命令路由器 | 将命令路由到规则处理流程 |
| CommandReplayRecord | 命令复盘记录 | 复盘、测试、AI 训练用记录 |

## 5. ActionDefinition 与 ActionCommand 的关系

### 5.1 ActionDefinition

`ActionDefinition` 来自配置系统。

它描述一种行为类型：

```text
eat 是接触范围、单对象目标、有风险、消耗时间的行为。
combine 是对象对目标、可能产生新对象的行为。
teach 是知识和族群目标、消耗注意力的行为。
```

### 5.2 ActionCommand

`ActionCommand` 是运行时一次具体行为请求。

例如：

```text
cmd_000001：玩家 player_000001 想让 ent_000001 吃 obj_000001。
cmd_000002：AI decision_000001 想让 ent_000002 采集 obj_000003。
cmd_000003：复盘系统重放一次 teach 命令。
```

### 5.3 边界

```text
ActionDefinition 是配置。
ActionCommand 是输入。
RuleExecutionResult 是输出。
EventRecord 是事实记录。
```

## 6. CommandEnvelope（命令外壳）

`CommandEnvelope` 包装命令元信息。

### 6.1 字段设计

| 字段 | 类型 | 中文名 | 必填 | 含义 |
|---|---|---|---|---|
| command_id | CommandId | 命令 ID | 是 | 命令唯一标识 |
| source | CommandSource | 命令来源 | 是 | 玩家、AI、系统、复盘等 |
| issued_tick | Tick | 发出帧 | 是 | 命令发出时刻 |
| issued_game_time | GameTime | 游戏时间 | 是 | 命令发出游戏时间 |
| idempotency_key | string | 幂等键 | 否 | 防止重复提交 |
| correlation_id | string | 关联链路 ID | 否 | 关联事件和日志 |
| decision_id | DecisionId | 决策 ID | 否 | AI / 强化学习预留 |
| payload | ActionCommand | 命令载荷 | 是 | 行为命令主体 |

### 6.2 设计注意事项

```text
command_id 由命令入口生成或由外部安全传入。
idempotency_key 用于处理前端重复提交。
decision_id 只在 AI 或训练场景需要。
correlation_id 应传递到 EventRecord。
```

## 7. ActionCommand（行为命令）

`ActionCommand` 表达一次具体行动意图。

### 7.1 字段设计

| 字段 | 类型 | 中文名 | 必填 | 含义 |
|---|---|---|---|---|
| action_id | ActionId | 行为定义 ID | 是 | 对应 ActionDefinition |
| action_type | ActionType | 行为类型 | 是 | eat、use、collect 等 |
| actor_id | EntityId | 行动者 | 是 | 谁执行行为 |
| targets | list<CommandTarget> | 目标列表 | 是 | 行为目标 |
| params | list<CommandParam> | 参数列表 | 否 | 行为参数 |
| intent | CommandIntent | 行为意图 | 否 | 探索、利用、教学等 |
| allow_risk_level | ActionRiskLevel | 可接受风险 | 否 | 调用方允许的最高风险 |
| client_prediction | bool | 是否前端预测 | 否 | 前端是否做过预测，仅用于协议 |

### 7.2 示例

吃红浆果：

```json
{
  "action_id": "eat",
  "action_type": "eat",
  "actor_id": "ent_000001",
  "targets": [
    { "target_type": "single_object", "object_id": "obj_000001", "role": "primary" }
  ],
  "intent": "experiment",
  "allow_risk_level": "unknown"
}
```

火加木棍：

```json
{
  "action_id": "combine",
  "action_type": "combine",
  "actor_id": "ent_000001",
  "targets": [
    { "target_type": "single_object", "object_id": "obj_fire_001", "role": "source" },
    { "target_type": "single_object", "object_id": "obj_branch_001", "role": "material" }
  ],
  "intent": "experiment"
}
```

传授知识：

```json
{
  "action_id": "teach",
  "action_type": "teach",
  "actor_id": "ent_000001",
  "targets": [
    { "target_type": "knowledge_claim", "knowledge_id": "know_000001", "role": "content" },
    { "target_type": "tribe", "tribe_id": "tribe_000001", "role": "receiver" }
  ],
  "intent": "teach"
}
```

## 8. CommandSource（命令来源）

`CommandSource` 描述命令由谁或什么系统提交。

### 8.1 字段设计

| 字段 | 类型 | 中文名 | 必填 | 含义 |
|---|---|---|---|---|
| source_type | string | 来源类型 | 是 | player、ai、system、script、replay、test |
| player_id | PlayerId | 玩家 ID | 条件必填 | 玩家来源时必填 |
| entity_id | EntityId | 实体 ID | 否 | 关联行动实体 |
| policy_version | string | 策略版本 | 否 | AI 来源时使用 |
| replay_id | string | 复盘 ID | 否 | 复盘来源时使用 |
| test_case_id | string | 测试用例 ID | 否 | 测试来源时使用 |

### 8.2 来源类型

```text
player：真实玩家输入。
ai：AI 或强化学习策略。
system：系统自动命令。
script：剧情或调试脚本。
replay：复盘重放。
test：测试用例。
```

### 8.3 注意事项

```text
source_type 第一阶段可以用字符串，后续如果稳定可提升为枚举。
AI 来源必须尽量携带 decision_id 或 policy_version。
复盘来源不应重新生成随机行为，应使用原始随机记录。
```

## 9. CommandTarget（命令目标）

`CommandTarget` 统一表达命令目标。

### 9.1 字段设计

| 字段 | 类型 | 中文名 | 必填 | 含义 |
|---|---|---|---|---|
| target_type | ActionTargetType | 目标类型 | 是 | 对象、区域、知识等 |
| role | string | 目标角色 | 是 | primary、source、material、receiver 等 |
| object_id | ObjectId | 对象 ID | 条件必填 | 目标为对象时使用 |
| region_id | RegionId | 区域 ID | 条件必填 | 目标为区域时使用 |
| entity_id | EntityId | 实体 ID | 条件必填 | 目标为实体时使用 |
| knowledge_id | KnowledgeId | 知识 ID | 条件必填 | 目标为知识时使用 |
| memory_id | MemoryId | 记忆 ID | 条件必填 | 目标为记忆时使用 |
| tribe_id | TribeId | 族群 ID | 条件必填 | 目标为族群时使用 |
| location_key | string | 位置键 | 否 | 抽象位置 |

### 9.2 role 设计

`role` 用于区分多个目标的语义。

示例：

```text
eat：primary
combine：source / material
teach：content / receiver
move：destination
use：tool / target
record：content / medium
```

### 9.3 设计注意事项

```text
CommandTarget 不直接保存对象快照。
CommandTarget 只引用 ID。
目标是否存在由 CommandValidator 检查。
目标是否可用由前置条件和规则系统检查。
```

## 10. CommandParam（命令参数）

`CommandParam` 表达命令额外参数。

### 10.1 字段设计

| 字段 | 类型 | 中文名 | 必填 | 含义 |
|---|---|---|---|---|
| key | string | 参数名 | 是 | 参数键 |
| value_type | string | 值类型 | 是 | number、string、bool、id、enum |
| value | any | 值 | 是 | 参数值 |
| source | string | 参数来源 | 否 | player、default、ai、system |

### 10.2 常见参数

```text
amount：数量。
use_tool_id：使用工具。
allow_high_risk：是否允许高风险。
teaching_scope：教学范围。
combine_order：组合顺序。
record_medium：记录媒介。
```

### 10.3 约束

```text
参数必须由 ActionDefinition 或 CommandValidator 白名单允许。
未知参数默认拒绝。
参数类型必须严格校验。
```

## 11. CommandIntent（命令意图）

`CommandIntent` 描述命令目的。

第一阶段建议作为字符串或小枚举，后续稳定后回填通用枚举。

### 11.1 建议值

| 值 | 中文名 | 含义 |
|---|---|---|
| experiment | 实验 | 尝试未知对象或未知行为 |
| exploit | 利用 | 使用已知有效知识 |
| teach | 教学 | 传播知识 |
| avoid | 避险 | 避开危险 |
| gather | 采集 | 收集资源 |
| craft | 制作 | 组合或转化对象 |
| move | 移动 | 改变位置 |
| rest | 休息 | 恢复状态 |
| debug | 调试 | 开发或测试用途 |

### 11.2 作用

```text
帮助事件和日志解释命令目的。
帮助 AI / 强化学习区分探索和利用。
帮助规则判断风险策略。
不应替代 action_type。
```

## 12. CommandValidationResult（命令校验结果）

命令进入规则结算前必须校验。

### 12.1 字段设计

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| valid | bool | 是否通过 | 命令是否合法 |
| normalized_command | CommandEnvelope | 标准化命令 | 校验通过后的命令 |
| errors | ErrorList | 错误列表 | 阻断问题 |
| warnings | ErrorList | 警告列表 | 非阻断问题 |

### 12.2 校验内容

```text
command_id 是否存在。
action_id 是否存在。
action_type 是否匹配 ActionDefinition。
actor_id 是否存在。
targets 是否满足 ActionTargetType。
目标 ID 是否存在。
params 是否在白名单内。
params 类型是否正确。
allow_risk_level 是否合法。
source 是否可信。
```

### 12.3 错误码映射

```text
命令格式错误：command_invalid_format。
未知命令类型：command_unknown_type。
缺少字段：command_missing_required_field。
参数非法：command_invalid_argument。
目标 ID 不存在：id_not_found。
目标类型不匹配：id_type_mismatch。
未知枚举：validation_enum_unknown。
```

## 13. CommandNormalizer（命令标准化器）

`CommandNormalizer` 将外部输入转换成内部标准命令。

### 13.1 职责

```text
填充默认参数。
统一目标顺序。
补齐 action_type。
规范 role。
移除不允许字段。
生成或校验 idempotency_key。
```

### 13.2 设计注意事项

```text
标准化不等于放行。
Normalizer 不访问世界状态。
Normalizer 不判断目标是否存在。
Normalizer 只处理格式层标准化。
```

## 14. CommandRouter（命令路由器）

`CommandRouter` 根据命令类型路由到后续规则流程。

### 14.1 输入

```text
CommandEnvelope
ConfigRegistry
```

### 14.2 输出

```text
Result<RuleRoute>
```

### 14.3 路由规则

```text
Observe / Touch / Eat / Use / Collect / Combine -> 交互规则流程。
Move / Wait / Rest -> 行为规则流程。
Teach / Record -> 知识与传播流程。
Avoid -> 风险规避流程。
Debug -> 调试流程。
```

### 14.4 注意事项

```text
Router 不执行规则。
Router 只决定交给哪个 Resolver。
找不到 Resolver 返回 rule_missing_resolver。
```

## 15. CommandReplayRecord（命令复盘记录）

`CommandReplayRecord` 用于复盘、测试和强化学习。

### 15.1 字段设计

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| command_id | CommandId | 命令 ID | 原始命令 |
| command | CommandEnvelope | 命令内容 | 可重放命令 |
| state_version_before | StateVersion | 前状态版本 | 执行前版本 |
| state_version_after | StateVersion | 后状态版本 | 执行后版本 |
| event_ids | list<EventId> | 事件 ID | 产生事件 |
| random_stream_position | object | 随机位置 | 随机复现信息 |
| result_type | RuleResultType | 结果类型 | 执行结果 |

### 15.2 强化学习用途

```text
CommandReplayRecord 可作为 action 记录。
decision_id 可关联策略决策。
EventStream 和 StateSnapshot 可生成 observation / next_observation。
RuleExecutionResult 可辅助 RewardModel 计算 reward。
```

## 16. 命令与配置系统关系

```text
ActionCommand.action_id 引用 ActionDefinition。
CommandValidator 通过 ConfigRegistry 查询 ActionDefinition。
CommandTarget 必须符合 ActionDefinition.target_type。
CommandParam 必须符合 ActionDefinition 允许的参数白名单。
```

## 17. 命令与世界对象 / 特性系统关系

```text
CommandTarget.object_id 引用 WorldObject。
CommandValidator 检查对象是否存在。
前置条件检查对象是否可交互。
后续交互规则通过 TraitResolver 判断目标特性。
```

示例：

```text
吃红浆果命令 -> target obj_000001 -> WorldObject -> red_berry -> trait_edible_fresh
火加木棍命令 -> targets fire_source + dry_branch -> TraitResolver -> combustible / fuel
```

## 18. 命令与错误系统关系

命令校验失败使用 `03_错误码与结果类型设计.md`。

```text
CommandValidationResult.valid=false。
errors 中包含 ErrorDetail。
RuleResultType 通常是 InvalidCommand 或 Blocked。
ErrorSeverity 通常是 UserError 或 Recoverable。
```

区分：

```text
InvalidCommand：命令格式、字段、目标类型错误。
Blocked：命令合法，但当前状态不允许执行。
Failed：命令执行后玩法失败。
SystemError：规则或数据错误。
```

## 19. 命令与事件系统关系

命令生命周期应产生事件。

建议事件：

```text
CommandReceived：命令收到。
ActionStarted：行动开始。
ActionBlocked：行动被阻止。
ActionResolved：行动已结算。
RuleErrorOccurred：命令或规则错误。
```

事件必须关联：

```text
command_id
correlation_id
rule_step
result_type
state_version
```

### 19.1 命令到事件的标准流程

```text
1. 收到 CommandEnvelope。
2. 生成 CommandReceived。
3. 标准化和校验。
4. 校验失败生成 RuleErrorOccurred 或 ActionBlocked。
5. 校验成功进入规则流程。
6. 行动开始生成 ActionStarted。
7. 结算完成生成 ActionResolved。
8. 所有事件进入 EventStream。
```

## 20. 命令与复盘 / 强化学习关系

### 20.1 复盘

```text
复盘重放 CommandReplayRecord。
复盘必须使用原始 CommandId、随机流位置和 StateVersion。
复盘不应重新解释玩家输入。
```

### 20.2 强化学习

```text
AI 通过同一套 CommandEnvelope 提交动作。
decision_id 关联策略决策。
CommandReplayRecord 记录 action。
EventStream + StateSnapshot 生成训练样本。
```

强化学习 action 可以直接映射：

```text
ActionType + CommandTarget + CommandParam
```

## 22. 组合测试样例

| 场景 | 输入命令 | 校验重点 | 预期结果 |
|---|---|---|---|
| 吃红浆果 | eat + obj_000001 | 单对象目标存在 | 通过校验，路由到交互规则 |
| 吃不存在对象 | eat + obj_missing | ObjectId 不存在 | id_not_found |
| 命令缺少 actor_id | eat + target only | 必填字段 | command_missing_required_field |
| 组合火和木棍 | combine + fire + branch | 多目标 role 正确 | 通过校验，路由到交互规则 |
| teach 传入 ObjectId | teach + obj_000001 | 目标类型错误 | id_type_mismatch 或 command_invalid_argument |
| teach KnowledgeClaim | teach + know_000001 + tribe | 知识和族群目标 | 通过校验，路由到传播流程 |
| 高风险行为被禁止 | eat unknown_mushroom + allow_risk=low | 风险策略 | Blocked |
| 重复提交命令 | 同 idempotency_key | 幂等处理 | command_duplicate 或返回原结果 |
| AI 提交命令 | source=ai + decision_id | AI 来源追踪 | 通过并记录 decision_id |
| 复盘重放命令 | source=replay | 保持原 CommandId | 进入复盘流程 |

测试结论：

```text
当前行为命令设计可以统一玩家、AI、系统、复盘和测试输入。
Command 只表达意图，不直接结算、不修改状态。
CommandTarget 可以覆盖对象、对象组合、知识、族群、区域等目标。
CommandValidationResult 可以复用错误系统。
CommandId 可以稳定连接事件、复盘和强化学习 action。
```

## 23. 研发实现注意事项

```text
禁止前端直接调用规则函数绕过 Command。
禁止 Command 直接修改状态。
禁止 CommandTarget 保存对象快照。
禁止 CommandRouter 执行具体规则。
禁止未知 CommandParam 静默通过。
新增 ActionType 或 ActionDefinition 必须补充命令校验和测试。
```

## 24. 第一阶段验收标准

```text
能构造 CommandEnvelope。
能表达 eat、touch、collect、combine、teach、move、wait、avoid。
能通过 CommandNormalizer 标准化命令。
能通过 CommandValidator 输出 ValidationResult。
能通过 CommandRouter 路由到后续流程。
命令失败能返回 ErrorCode 和 ErrorDetail。
命令能产生 CommandReceived / ActionBlocked / ActionResolved 等事件。
命令可进入 CommandReplayRecord。
AI 命令可携带 decision_id。
```
