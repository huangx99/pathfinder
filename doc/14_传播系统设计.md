# 先驱者：传播系统设计

## 1. 设计目标

传播系统负责把一个主体掌握的可传授知识，通过教学、演示、模仿、标记、记录、习俗等渠道，影响其他主体或族群的认知、记忆和知识状态。

传播系统回答以下问题：

```text
谁在传播？
传播给谁？
传播哪条 KnowledgeClaim？
通过什么渠道传播？
传播内容是否保留了条件、风险和证据？
接收者是否理解、记住、误解、拒绝或遗忘？
传播结果如何影响接收者认知、记忆和知识？
传播是否形成族群共享认知或文明前置？
AI 是否能学习更好的教学策略？
```

核心链路：

```text
KnowledgeClaim / PropagationKnowledgeCandidate
  ↓
PropagationResolver
  ↓
PropagationAttempt
  ↓
PropagationResult
  ↓
ReceiverCognitionCandidate / ReceiverMemoryCandidate / ReceiverKnowledgeCandidate
  ↓
KnowledgeShared / PropagationFailed EventDraft
  ↓
TribeState / CivilizationCapability / AI Observation
```

本系统必须保证：

- 传播不是复制 KnowledgeClaim。
- 传播可能成功、失败、部分成功、误解、拒绝、过期或有害扩散。
- 传播必须携带对象、行为、效果、条件、风险、证据摘要和置信度。
- 传播结果必须可复盘、可解释、可测试。
- 传播结果可以影响接收者认知、记忆和知识，但不直接越权改最终系统。
- 传播可以为族群状态和文明能力提供输入，但不直接解锁文明。
- 前端不自行计算传播结果。
- AI 不能通过传播读取未掌握知识或真实隐藏规则。

## 2. 不解决的问题

| 问题 | 原因 | 归属文档 |
|---|---|---|
| 判断知识真伪 | 知识有效性由知识系统负责 | `13_知识系统设计.md` |
| 创建最终接收者 KnowledgeClaim | 传播只输出候选，最终由知识系统处理 | `13_知识系统设计.md` |
| 直接修改族群行为 | 传播只输出族群影响候选 | `15_族群状态设计.md` |
| 直接解锁文明能力 | 传播只输出文明候选或传播证据 | `16_文明能力设计.md` |
| 修改世界对象真实状态 | 传播是社会学习过程，不是物理交互 | 世界对象 / 规则管线 |
| 前端计算传播结果 | 传播结果必须来自后端规则 | `20_前后端协议设计.md` |
| 黑盒语言生成教学内容 | 第一版使用结构化传播载荷 | 后续 AI 内容工具 |

关键边界：

```text
KnowledgeClaim：被传播的知识。
PropagationAttempt：一次传播尝试。
PropagationResult：这次传播传成什么样。
ReceiverCognitionCandidate：接收者可能形成的认知候选。
ReceiverMemoryCandidate：接收者可能形成的记忆候选。
ReceiverKnowledgeCandidate：接收者可能形成的知识候选。
TribeInfluenceCandidate：传播对族群状态的候选影响。
```

## 3. 核心概念

| 概念 | 中文名 | 含义 |
|---|---|---|
| PropagationSource | 传播来源 | 谁在传播，可能是先驱者、族人、族群、记录、环境标记 |
| PropagationTarget | 传播目标 | 谁接收传播，可能是个体、小队、族群 |
| PropagationChannel | 传播渠道 | 教学、演示、模仿、符号记录、环境标记等 |
| PropagationPayload | 传播载荷 | 被传播的结构化知识内容 |
| PropagationAttempt | 传播尝试 | 一次传播行为的完整输入记录 |
| PropagationResult | 传播结果 | 接收者接受、理解、误解、拒绝等结果 |
| PropagationEffect | 传播效果 | 对认知、记忆、知识、族群候选的影响 |
| PropagationResolver | 传播解析器 | 核心结算服务 |
| PropagationPolicy | 传播策略 | 配置化传播成功率、误解风险和渠道限制 |
| PropagationTrace | 传播解释链 | 说明为什么传播成功、失败或误解 |
| PropagationProjection | 传播投影 | 给前端、AI、族群系统的传播摘要 |
| TribeKnowledgeGapProjection | 族群知识差距投影 | 谁知道、谁不知道、哪里需要教学 |

### 3.1 传播不是复制

错误：

```text
Pioneer KnowledgeClaim 直接复制到 Tribe KnowledgeStore。
```

正确：

```text
KnowledgeClaim
  ↓
PropagationAttempt
  ↓
PropagationResult
  ↓
接收者认知候选 / 记忆候选 / 知识候选
  ↓
后续 Cognition / Memory / Knowledge 系统处理
```

原因：

```text
传播可能丢失条件、被误解、被拒绝、被旧经验反驳或形成有害扩散。
```

### 3.2 传播必须保留条件

传播载荷不能只说“红浆果能吃”。

必须表达：

```text
对象：red_berry
行为：Eat
效果：restore_hunger
条件：state:fresh
风险：state:decayed -> poison
证据：多次食用成功，腐烂样本中毒
置信度：Strong
范围：Personal / Tribe candidate
```

### 3.3 接收者不是空白容器

接收者可能已有：

```text
相同知识。
相反经验。
低信任。
记忆负担。
缺少共同语境。
缺少行为能力。
```

传播必须读取接收者上下文，而不是默认接受。

## 4. 枚举设计

本阶段主要使用 `02_通用枚举设计.md` 中已有传播相关枚举，不新增通用枚举。

### 4.1 使用的既有枚举

| 枚举 | 中文名 | 本系统用途 |
|---|---|---|
| PropagationChannelType | 传播渠道类型 | 教学、演示、模仿、标记等 |
| PropagationResultType | 传播结果类型 | 接受、理解、误解、失败等 |
| PropagationBlockReason | 传播阻断原因 | 为什么失败或不能开始 |
| KnowledgeStatus | 知识状态 | 判断知识是否 Teachable、Shared、Operational |
| KnowledgeScope | 知识范围 | 判断传播范围和接收范围 |
| CognitionOwnerType | 认知归属类型 | 表达传播来源和目标主体类型 |
| MemoryType | 记忆类型 | 输出 Teaching / Social MemoryCandidate |
| EvidenceType | 证据类型 | 传播形成 TribeReport 或 Teaching 证据 |
| ConfidenceChangeReason | 置信变化原因 | TeachingAccepted、TeachingFailed 等 |
| ActionType | 行为类型 | Teach、Record 等传播行为 |
| EventType | 事件类型 | KnowledgeShared、PropagationFailed 等 |
| EventVisibility | 事件可见性 | 传播事件展示范围 |
| EventImportance | 事件重要性 | 传播事件重要级别 |
| ErrorSeverity | 错误严重级别 | 错误处理 |

### 4.2 PropagationChannelType 使用规则

| 渠道 | 中文 | 设计重点 |
|---|---|---|
| DirectTeaching | 直接传授 | 成本低，但条件丢失风险较高 |
| Demonstration | 演示 | 成本较高，需要对象和环境，但理解可靠 |
| SharedExperience | 共同经历 | 多主体共同获得证据，可靠性强 |
| Imitation | 模仿 | 适合行为流程，但可能不理解原因 |
| WarningSignal | 警示信号 | 适合危险知识，传递“不要做” |
| SymbolRecord | 符号记录 | 适合长期留存，需要记录能力 |
| OralTradition | 口耳相传 | 长期传播，容易变形 |
| ToolPattern | 工具范式 | 通过工具用法隐性传播 |
| RitualPractice | 仪式实践 | 稳定行为，但可能丢失原理 |
| EnvironmentalMark | 环境标记 | 适合位置、危险、资源 |
| DelegatedTeaching | 委托传授 | 已学会者继续教别人 |
| PassiveDiffusion | 被动扩散 | 慢速自然扩散 |

### 4.3 PropagationResultType 使用规则

| 结果 | 中文 | 后续影响 |
|---|---|---|
| Accepted | 已接受 | 产生低强度认知候选 |
| Understood | 已理解 | 认知候选更完整 |
| Memorized | 已记住 | 产生 MemoryCandidate |
| Teachable | 可再传授 | 产生传播能力候选 |
| Operational | 可执行 | 输出族群状态候选 |
| PartiallyAccepted | 部分接受 | 产生不完整认知，需保留缺失字段 |
| Misunderstood | 误解 | 产生错误认知候选 |
| Ignored | 被忽略 | 可能形成弱记忆或无效果 |
| Rejected | 被拒绝 | 产生 TeachingFailed 记忆 |
| Failed | 失败 | 输出 PropagationFailed |
| HarmfulSpread | 有害扩散 | 输出风险候选和错误知识候选 |
| Contradicted | 遭遇反证 | 反馈知识系统和认知系统 |
| Expired | 已失效 | 不传播，记录条件过期 |

### 4.4 PropagationBlockReason 使用规则

阻断原因必须尽量具体，不要滥用 `RuleDenied`。

| 阻断 | 中文 | 使用场景 |
|---|---|---|
| KnowledgeNotTeachable | 知识不可传授 | KnowledgeStatus 未达到 Teachable |
| InsufficientConfidence | 置信度不足 | 传播者自己也不确定 |
| MissingSubject | 缺少知识对象 | 知识不完整 |
| MissingReceiver | 缺少接收者 | 没有目标 |
| ReceiverUnavailable | 接收者不可用 | 忙碌、昏迷、远离等 |
| ReceiverIncapable | 接收者能力不足 | 不能理解或执行 |
| NoSharedContext | 缺少共同语境 | 不理解对象、行为或条件 |
| ChannelUnavailable | 渠道不可用 | 没有记录、演示、标记能力 |
| MissingDemonstrationTarget | 缺少演示对象 | 演示条件不足 |
| TooDangerous | 过于危险 | 教学本身风险过高 |
| ContradictoryKnowledge | 既有知识冲突 | 接收者旧知识反驳 |
| LowTrust | 信任不足 | 不信传播者 |
| MemoryOverload | 记忆负担过高 | 暂时无法记住 |
| EnvironmentalNoise | 环境干扰 | 战斗、灾害、混乱 |
| TimeNotEnough | 时间不足 | 行动时间不足 |
| ConditionMismatch | 条件不匹配 | 知识条件与当前环境不符 |
| AlreadyKnown | 已经知道 | 接收者已掌握 |
| RuleDenied | 规则拒绝 | 兜底错误 |

| credibility | Ratio | 可信度 | 接收者视角下的可信度 |

### 5.4 PropagationPayload（传播载荷）

`PropagationPayload` 是被传播的结构化知识内容。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| payload_id | string | 载荷 ID | 临时标识 |
| knowledge_id | KnowledgeId | 知识 ID | 来源 KnowledgeClaim |
| subject_id | string | 主体 ID | 被传播对象或区域 |
| relation_summary | string | 关系摘要 | 可食用、危险、可反应等 |
| action_type | ActionType | 行为 | Eat、Use、Combine、Record 等 |
| effect_summary | string | 效果摘要 | restore_hunger、poison 等 |
| condition_summary | string | 条件摘要 | fresh、decayed、night 等 |
| risk_summary | string | 风险摘要 | 腐烂有毒、靠近危险等 |
| evidence_summary | string | 证据摘要 | 多次验证、族群报告等 |
| confidence | Ratio | 置信度 | 来源知识置信度 |
| status | KnowledgeStatus | 状态 | 来源知识状态 |
| scope | KnowledgeScope | 范围 | 来源知识范围 |
| required_context_keys | list<string> | 必要语境 | 接收者必须理解的概念 |
| loss_sensitive_fields | list<string> | 易丢字段 | condition、risk、scope 等 |

设计重点：

```text
condition_summary 和 risk_summary 不能省略。
传播失败和误解经常来自条件或风险丢失。
```

## 6. 接口设计

### 6.1 PropagationResolver

```text
resolve_propagation(context, attempt) -> Result<PropagationResolveResult>
```

| 项目 | 说明 |
|---|---|
| 输入 | PropagationContext、PropagationAttempt |
| 输出 | Result<PropagationResolveResult> |
| 是否修改 GameState | 不直接修改，由规则管线应用后续状态变化 |
| 是否产生事件 | 不直接写 EventStore，只输出 event_drafts |
| 是否读取配置 | 是，读取 PropagationPolicy |
| 是否使用随机数 | 第一版不使用随机数；误解风险用确定性规则或配置阈值 |
| 是否允许失败 | 是 |
| 失败返回 | ErrorDetail |
| 调用顺序 | KnowledgeResolver 之后，Cognition/Memory/Knowledge 后续更新之前 |

### 6.2 PropagationResolveResult

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| attempt | PropagationAttempt | 传播尝试 | 输入尝试 |

| receiver_knowledge_candidates | list<ReceiverKnowledgeCandidate> | 接收者知识候选 | 给知识系统 |
| tribe_influence_candidates | list<TribeInfluenceCandidate> | 族群影响候选 | 给族群系统 |

## 7. 传播生命周期

### 7.1 传播请求

```text
1. 玩家、AI 或系统发出 Teach / Record 等 ActionCommand。
2. CommandRouter 路由到传播系统。
3. KnowledgeProjection 提供可传播 KnowledgeClaim。
4. 构建 PropagationAttempt。
```

### 7.2 传播校验

```text
1. 检查 KnowledgeStatus 是否 Teachable 或更高。
2. 检查 confidence 是否达到传播阈值。
3. 检查接收者是否存在且可用。
4. 检查渠道是否可用。
5. 检查 payload 是否包含条件和风险。
6. 检查当前环境是否满足传播条件。
```

### 7.3 传播结算

```text
1. PropagationPolicy 命中。
2. ChannelResolver 计算渠道效果。
3. 接收者上下文影响结果。
4. 生成 Accepted / Understood / Misunderstood / Failed 等结果。
5. 记录 lost_fields 和 distorted_fields。
6. 生成 PropagationEffect。
```

### 7.4 成功传播

```text
Accepted / Understood / Memorized / Teachable / Operational
  ↓
ReceiverCognitionCandidate
ReceiverMemoryCandidate
ReceiverKnowledgeCandidate
TribeInfluenceCandidate
KnowledgeShared EventDraft
```

### 7.5 失败传播

```text
Failed / Ignored / Rejected / Expired
  ↓
TeachingFailed MemoryCandidate
PropagationFailed EventDraft
可能降低 source credibility 或 receiver trust
```

### 7.6 误解和有害扩散

```text
Misunderstood / HarmfulSpread
  ↓
ReceiverCognitionCandidate.misconception_hint = true
ReceiverKnowledgeCandidate.misconception_hint = true
TribeInfluenceCandidate.civilization_hint = risk
KnowledgeShared 或 PropagationFailed 事件视情况输出
```

### 7.7 遭遇反证

```text
Contradicted
  ↓
反馈给接收者认知系统
反馈给知识系统作为 Contradiction evidence
可能触发 KnowledgeContradicted
```

## 8. 事件输出

传播系统只输出事件草稿，不直接写 EventStore。

### 8.1 KnowledgeShared

触发：

```text
传播产生 Accepted、Understood、Memorized、Teachable 或 Operational 结果。
```

使用 `PropagationEventPayload`。

字段映射：

| Payload 字段 | 来源 |
|---|---|
| knowledge_id | PropagationPayload.knowledge_id |
| channel | PropagationChannel.channel_types |
| result | PropagationResult.result_type |
| block_reason | PropagationResult.block_reason |
| sender_id | PropagationSource.source_id |
| receiver_ids | PropagationTarget.target_ids |

### 8.2 PropagationFailed

触发：

```text
传播被阻断、失败、拒绝、过期、条件不匹配或规则拒绝。
```

### 8.3 相关后续事件

传播系统可以输出候选，但不直接替代后续系统事件：

| 后续事件 | 归属 |
|---|---|
| MemoryCreated | 记忆系统 |
| KnowledgeCreated | 知识系统 |
| KnowledgeUpdated | 知识系统 |
| KnowledgeContradicted | 知识系统 |
| TribeBehaviorChanged | 族群状态系统 |
| CapabilityUnlocked | 文明能力系统 |

### 8.4 事件可见性建议

| 场景 | EventVisibility | EventImportance |
|---|---|---|
| 普通教学成功 | PlayerVisible | Normal |
| 教会危险知识 | PlayerVisible / TribeVisible | Important |
| 传播失败 | PlayerVisible | Normal / Important |
| 有害扩散 | PlayerVisible / TribeVisible | Critical |
| 被动扩散 | Hidden / TribeVisible | Trace / Normal |
| 调试渠道评估 | DebugOnly | Trace |

```text
直接复制 KnowledgeClaim 到接收者。
直接创建最终 Receiver KnowledgeClaim。
直接修改 TribeState 或 CivilizationCapability。
忽略 condition_summary 和 risk_summary。
使用自由脚本。
调用随机数。
前端自行决定传播结果。
```

## 10. 示例用法

### 10.1 直接传授红浆果知识

输入：

```text
KnowledgeClaim:
red_berry + Eat + restore_hunger + state:fresh
status = Teachable

Channel = DirectTeaching
Target = tribe_member_001
```

输出：

```text
PropagationResult = Accepted / Understood
ReceiverCognitionCandidate = red_berry + Eat + restore_hunger + state:fresh
ReceiverMemoryCandidate = Teaching / Social
EventDraft = KnowledgeShared
```

### 10.2 演示采集更可靠

```text
Channel = Demonstration
available_object_ids 包含 red_berry
接收者共同观察食用效果
Result = Understood / Memorized
condition_loss_risk 低于 DirectTeaching
```

### 10.3 条件丢失导致误解

```text
原载荷：新鲜红浆果可食用，腐烂有毒。
接收者保留：红色果实可食用。
lost_fields = condition_summary / risk_summary
result = Misunderstood
misconception_hint = true
```

### 10.4 危险标记

```text
Channel = WarningSignal / EnvironmentalMark
Knowledge = 毒蘑菇食用有风险
Result = Understood
MemoryCandidate = Warning / Location
TribeInfluenceCandidate = 避免危险区域
```

### 10.5 知识不可传授

```text
KnowledgeStatus = Hypothesis
PropagationBlockReason = KnowledgeNotTeachable
Result = Failed
EventDraft = PropagationFailed
```

## 11. 校验规则

### 11.1 PropagationAttempt 校验

| 规则 | 错误级别 |
|---|---|
| attempt_id 为空 | RuleError |
| source 为空 | RuleError |
| target 为空且 require_receiver=true | RuleError |
| payload.knowledge_id 为空 | RuleError |
| payload.subject_id 为空 | RuleError |
| payload.condition_summary 为空 | RuleError |
| payload.risk_summary 缺失且知识有风险 | RuleError |
| channel_types 为空 | RuleError |
| KnowledgeStatus 未达 Teachable | RuleError / Blocked |

### 11.2 传播载荷校验

| 规则 | 错误级别 |
|---|---|
| 只传播 ObjectId 不传播行为和效果 | RuleError |
| 缺少条件字段 | RuleError |
| 缺少风险字段但原知识有风险 | RuleError |
| 置信度低于策略阈值 | Blocked |
| scope 超出来源知识范围 | RuleError |

### 11.3 结果校验

| 规则 | 错误级别 |
|---|---|
| result_type=Success 类但 block_reason 非 None | RuleError |

## 14. 设计注意事项

研发实现时必须遵守：

```text
不要把传播实现成复制 KnowledgeClaim。
不要传播裸 ObjectId。
不要丢失 condition_summary。
不要丢失 risk_summary。
不要让前端计算传播成功或失败。
不要让传播系统直接创建最终 KnowledgeClaim。
不要让传播系统直接修改 TribeState 或 CivilizationCapability。
不要忽略 Misunderstood 和 HarmfulSpread。
不要使用随机数决定传播结果，第一版保持可复盘。
```

### 14.1 条件保护原则

传播载荷中必须重点保护：

```text
条件。
风险。
范围。
证据摘要。
置信度。
```

### 14.2 误解可追溯原则

当传播产生误解时，必须记录：

```text
lost_fields。
distorted_fields。
receiver_id。
source_knowledge_id。
传播渠道。
传播上下文。
```

### 14.3 可复盘原则

同一输入必须得到同一输出：

```text
同一个 PropagationAttempt
+ 同一个 PropagationPolicy
+ 同一个 source KnowledgeProjection
+ 同一个 receiver context
= 同一个 PropagationResolveResult
```

传播系统第一版不使用随机数。

## 16. 阶段结论

传播系统是《先驱者》从个人先驱者走向种群文明的关键桥梁。

当前设计支持：

- 传播不是复制。
- 直接教学、演示、模仿、符号记录、环境标记等渠道。
- 成功、失败、部分接受、误解、有害扩散、反证等结果。
- 条件、风险、证据摘要保护。
- 接收者认知、记忆、知识候选输出。
- 族群影响候选输出。
- 传播历史和知识差距投影。
- AI / 强化学习教学策略预留。
