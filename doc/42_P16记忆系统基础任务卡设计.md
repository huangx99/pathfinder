# P16 记忆系统基础任务卡设计

## 1. 阶段定位

P16 是《先驱者》认知、记忆、知识、传播学习主线的第二个核心玩法阶段。

P15 解决：

```text
一个主体当前相信什么。
这条认知来自什么反馈。
这条认知的置信度、风险、效用、可传授程度是多少。
```

P16 解决：

```text
这次经历是否会被记住。
它是短期记忆、中期记忆还是长期记忆。
重复经历如何增强记忆。
普通记忆如何随时间衰减或遗忘。
关键记忆如何被保护并晋升。
```

P16 不解决：

```text
大量记忆如何压缩为摘要。
Agent 如何按复杂条件检索长期记忆。
记忆如何变成正式知识。
知识如何传播给族群。
H5 如何展示完整记忆列表。
```

这些留给后续阶段：

- P17：记忆压缩与检索。
- P18：知识系统。
- P19：知识冲突与修正。
- P20：传播系统。
- P21：学习闭环。

---

## 2. 本阶段目标

P16 的目标是建立一个可扩展、可审计、可复盘、对 AI 友好的基础记忆层。

它必须做到：

1. 能把 P15 的 `CognitionUpdateResult` / `CognitionEvidenceRecord` 转换为 `MemoryCandidate`。
2. 能把 `MemoryCandidate` 写成 `MemoryRecord`。
3. 能区分 `ShortTerm`、`MidTerm`、`LongTerm`。
4. 能根据重复经历增强记忆强度。
5. 能根据时间推进衰减普通短期记忆。
6. 能保护高风险、首次发现、强冲突、可教学候选等关键记忆。
7. 能输出记忆事件草稿和状态变更草稿，但不直接绕过规则管线改世界状态。
8. 能给 P17/P18 留出压缩、检索、知识证据接口，不需要后续推倒重来。

一句话：

```text
P16 让“我刚刚知道了什么”变成“我会不会记住这件事，以及记多久”。
```

---

## 3. 玩家体验映射

P16 对玩家体验的意义：

```text
玩家第一次吃未知果实：主体形成一条短期记忆。
玩家多次吃后都恢复需求：短期记忆被增强，并可能晋升中期记忆。
玩家吃后中毒：高风险记忆被保护，不会轻易遗忘。
玩家很久没再遇到普通无效对象：低强度短期记忆衰减并遗忘。
玩家带领族群前进：后续知识/传播系统可以从长期记忆中沉淀经验。
```

这不是 UI 功能，而是核心玩法根基：

```text
探索 → 反馈 → 认知 → 记忆 → 知识 → 传播 → 文明
```

P16 只做到第四步。

## 5. 本阶段不做什么

P16 明确禁止：

```text
不做记忆压缩。
不做复杂检索排序。
不做 MemorySummary。
不做 MemoryBundle。
不做 MemoryReconstructionView。
不做 KnowledgeClaim。
不做 KnowledgeRepository。
不做知识传播。
不做族群共享记忆。
不做 H5 接口。
不做 HTTP / WebSocket / JSON。
不做 SaveManager。
不做强化学习 reward/done。
不让 AgentRuntime 或 Policy 直接写 MemoryStore。
不读取 ObjectDefinition hidden truth。
不读取 GameState 原始隐藏状态生成记忆。
```

P16 可以预留接口字段，但不能实现 P17/P18 的核心逻辑。

## 8. 枚举设计

所有枚举必须实现：

```cpp
std::string toString(Enum value);
Result<Enum> enumFromString(const std::string& value);
```

所有 enum fromString 遇到未知字符串必须返回 `Result::fail`，不能静默返回 `Unknown`。

### 8.1 MemoryOwnerKind

中文：记忆拥有者类型。

用途：表示这条记忆属于谁。

```cpp
enum class MemoryOwnerKind {
    Unknown,
    Actor,
    Agent,
    Group,
    Tribe,
    Civilization,
    TestOnly
};
```

中文翻译：

| 枚举 | 中文 | 含义 |
|---|---|---|
| Unknown | 未知 | 非法或未初始化 |
| Actor | 行动者 | 世界中的实际行动主体 |
| Agent | Agent | 决策主体，可对应狼、蜘蛛、人、外星人 |
| Group | 小组 | 临时协作单位 |
| Tribe | 族群 | 族群级记忆预留 |
| Civilization | 文明 | 文明级记忆预留 |
| TestOnly | 测试专用 | 仅测试使用 |

P16 允许实现 `Actor`、`Agent`、`TestOnly`，其他作为预留。

### 8.2 MemorySubjectKind

中文：记忆对象类型。

用途：表示这段记忆关于什么。

```cpp
enum class MemorySubjectKind {
    Unknown,
    ObjectDefinition,
    WorldObject,
    ObjectCategory,
    AgentSpecies,
    EnvironmentFeature,
    ActionOutcome,
    TestOnly
};
```

中文翻译：

| 枚举 | 中文 | 含义 |
|---|---|---|
| Unknown | 未知 | 非法或未初始化 |
| ObjectDefinition | 对象定义 | 关于某类物体的记忆，例如 berry_red |
| WorldObject | 世界对象 | 关于某个具体对象实例的记忆 |
| ObjectCategory | 对象类别 | 关于食物、工具、危险物等类别的记忆 |
| AgentSpecies | 生物种类 | 关于狼、蜘蛛等生物类型的记忆 |
| EnvironmentFeature | 环境特征 | 关于火、水、地形等环境特征 |
| ActionOutcome | 行为结果 | 关于某类行为结果的记忆 |
| TestOnly | 测试专用 | 仅测试使用 |

注意：

```text
ObjectDefinition 在这里是目标 ID 类型，不表示读取 ObjectDefinition hidden truth。
P16 只能记录来自 cognition evidence 的安全目标标识。
```

### 8.3 MemoryKind

中文：记忆类型。

用途：表示记忆的内容性质。

```cpp

    Success,
    Failure,
    Warning,
    Accident,
    Discovery,
    Contradiction,
    TeachingRelated,
    TestOnly
```

中文翻译：

| 枚举 | 中文 | 含义 |
|---|---|---|
| Unknown | 未知 | 非法或未初始化 |
| Experiment | 尝试 | 主体进行了一次尝试 |

中文：记忆强度区间。

```cpp

    Weak,
    Stable,

    Core
```

中文翻译：

| 枚举 | 中文 | 含义 |
|---|---|---|
| Unknown | 未知 | 非法或未初始化 |

## 9. ID 与基础结构

### 9.1 MemoryOwner

中文：记忆拥有者。

```cpp
struct MemoryOwner {
    MemoryOwnerKind kind = MemoryOwnerKind::Unknown;
    pathfinder::foundation::EntityId entity_id;
    pathfinder::foundation::TribeId tribe_id;
    std::string external_key;

    pathfinder::foundation::Result<void> validateBasic() const;
};
```

规则：

```text
Actor / Agent：entity_id 必须非空。
Tribe：tribe_id 必须非空。
TestOnly：external_key 必须非空。
P16 不使用 Civilization 写入真实记忆，仅预留。
```

### 9.2 MemorySubject

中文：记忆对象。

```cpp
struct MemorySubject {
    MemorySubjectKind kind = MemorySubjectKind::Unknown;
    std::string subject_id;
    std::string subject_type_key;
    std::vector<std::string> safe_tags;

    pathfinder::foundation::Result<void> validateBasic() const;
};
```

注意：

```text
subject_id 只能是安全 ID 或安全 key。
不能在 subject_id / subject_type_key / safe_tags 中塞 ObjectDefinition hidden truth。
```

### 9.3 MemoryEvidenceRef

中文：记忆证据引用。

```cpp
struct MemoryEvidenceRef {
    MemorySourceKind source_kind = MemorySourceKind::Unknown;
    pathfinder::cognition::CognitionEvidenceRecordId cognition_evidence_id;
    pathfinder::cognition::CognitionClaimV2Id cognition_claim_id;
    pathfinder::foundation::EventId source_event_id;
    pathfinder::foundation::Tick observed_tick;
    std::vector<std::string> reason_keys;

    pathfinder::foundation::Result<void> validateBasic() const;
};
```

边界：

```text
P16 只保存 cognition evidence/claim 的 ID 和安全 reason key。
P16 不复制 hidden effect 数值，不读取真实 ObjectDefinition。
```

### 9.4 MemoryCandidate

中文：记忆候选。

`MemoryCandidate` 是 P16 的写入输入，不是最终记忆。

```cpp
struct MemoryCandidate {
    std::string candidate_id;
    MemoryOwner owner;
    MemorySubject subject;
    std::vector<MemoryKind> memory_kinds;
    MemorySourceKind source_kind = MemorySourceKind::Unknown;
    MemoryImportance importance = MemoryImportance::Unknown;
    double initial_strength = 0.0;
    bool protect_from_decay = false;
    bool protect_from_forgetting = false;
    bool teaching_candidate = false;
    MemoryEvidenceRef evidence_ref;
    std::vector<std::string> reason_keys;
    std::vector<std::string> warning_keys;

    pathfinder::foundation::Result<void> validateBasic() const;
};
```

候选生成规则：

| 输入 | 候选类型 | 初始层级建议 |
|---|---|---|
| P15 Created + direct feedback | Experiment / Discovery | ShortTerm |
| P15 Reinforced | Success 或 Failure | ShortTerm，可强化已有记忆 |
| P15 Conflicted | Contradiction | MidTerm 候选 |
| P15 High/Critical risk | Warning / Accident | MidTerm 或 LongTerm 候选 |
| P15 Teachable band | TeachingRelated | MidTerm 候选 |

### 9.5 MemoryRecord

中文：记忆记录。

`MemoryRecord` 是 P16 的核心实体。

```cpp
struct MemoryRecord {
    pathfinder::foundation::MemoryId memory_id;
    std::string schema_version = "memory_record.v1";
    MemoryOwner owner;
    MemorySubject subject;
    std::vector<MemoryKind> memory_kinds;
    MemoryScope scope = MemoryScope::Unknown;
    MemoryLifecycle lifecycle = MemoryLifecycle::Unknown;
    MemoryImportance importance = MemoryImportance::Unknown;
    double strength = 0.0;
    MemoryStrengthBand strength_band = MemoryStrengthBand::Unknown;
    size_t reinforcement_count = 0;
    size_t contradiction_count = 0;
    bool protect_from_decay = false;
    bool protect_from_forgetting = false;
    bool teaching_candidate = false;
    pathfinder::foundation::Tick created_tick;
```

## 12. 强度、强化、晋升规则

### 12.1 初始强度

建议默认：

| importance | initial_strength |
|---|---|
| Trivial | 0.15 |
| Normal | 0.35 |
| Important | 0.55 |
| Critical | 0.80 |

如果 `candidate.initial_strength > 0`，优先使用 candidate，但必须 clamp 到 [0, 1] 或校验失败。

### 12.2 强化公式

P16 使用简单、确定性公式：

```text
new_strength = old_strength + (1 - old_strength) * reinforce_gain
```

建议：

| candidate importance | reinforce_gain |
|---|---|
| Trivial | 0.05 |
| Normal | 0.15 |
| Important | 0.25 |
| Critical | 0.40 |

必须保证：

```text
new_strength <= 1.0
reinforcement_count += 1
last_touched_tick 更新
追加 evidence_ref，但不超过 max_evidence_refs_per_record
```

### 12.3 冲突削弱

如果 candidate 包含 `Contradiction`：

```text
contradiction_count += 1
strength 可以小幅下降，但不能低于 0
追加 Contradiction kind
reason_keys 增加 memory.contradiction_observed
```

P16 只记录冲突，不做知识修正。

### 12.4 晋升规则

ShortTerm -> MidTerm：满足任一条件即可：

```text
strength >= short_to_mid_strength_threshold
reinforcement_count >= short_to_mid_reinforcement_count
importance == Critical
memory_kinds 包含 Accident / Warning / Discovery
```

MidTerm -> LongTerm：满足任一条件即可：

```text
strength >= mid_to_long_strength_threshold
reinforcement_count >= mid_to_long_reinforcement_count
importance == Critical 且 protect_from_forgetting=true
memory_kinds 包含 Accident 且 observed_risk 为 High/Critical 的安全映射
```

注意：

```text
P16 可以依据 CognitionEvidenceRecord 的 risk enum 映射重要性。
不能读取真实伤害数值或隐藏效果。
```

## 13. 衰减规则

### 13.1 衰减输入

衰减只需要：

```text
records
current_tick
MemoryDecayOptions
```

不需要 GameState。

### 13.2 衰减公式

```text
elapsed = current_tick - last_touched_tick
new_strength = old_strength - decay_rate * elapsed
```

约束：

```text
new_strength = max(0, new_strength)
Protected 跳过。
LongTerm 默认跳过，除非 decay_long_term=true。
Forgotten 跳过。
```

### 13.3 生命周期转换

```text
new_strength <= forgotten_threshold -> Forgotten
new_strength <= fading_threshold -> Fading
否则 -> Active
```

如果 `protect_from_decay=true`：

```text
lifecycle 可保持 Protected 或 Active。
输出 memory.protected_skipped 事件草稿。
```

## 14. 相似记忆匹配

P16 只做简单确定性匹配，不做向量检索、不做复杂召回。

相同记忆条件：

```text
owner 相同。
subject kind 相同。
subject_id 相同。
memory_kinds 有交集。
source action/context 对应 reason_keys 不冲突。
lifecycle != Forgotten。
```

如果匹配到多条：

```text
选择 last_touched_tick 最大的一条。
如果 tick 相同，选择 memory_id 字典序最小的一条。
```

这样保证复盘确定性。

## 18. 与 Agent / AI 的关系


## 22. 验收标准

P16 完成必须满足：

```text
1. P15 cognition evidence/update 可以生成 MemoryCandidate。
2. MemoryCandidate 可以生成 MemoryRecord。
3. 一次尝试行为形成 ShortTerm 记忆。
4. 重复证据可以增强同一条记忆。
5. 重复证据达到阈值可以晋升 MidTerm。
6. 高风险或关键记忆可以被保护并晋升 LongTerm。
7. 普通 ShortTerm 记忆会随 tick 衰减。
8. 低强度普通记忆会进入 Fading/Forgotten。
9. Protected / LongTerm 默认不衰减。
10. 所有记忆记录都保留 source_event_id 或 cognition evidence 引用。
11. 隐藏真值和原始状态字段被拒绝。
12. P16 不依赖 AgentRuntime、Policy、RulePipeline 执行、GameState 原始读取、SaveManager、HTTP、JSON。
13. P16 定向测试、相关回归、全量测试通过。
```

## 25. 阶段结论

P16 的完成标准不是“记忆系统全做完”，而是：

```text
认知结果已经可以稳定沉淀为短期/中期/长期记忆。
记忆强度、保护、衰减、遗忘都有可测试的基础规则。
后续 P17 可以在不推翻 P16 的前提下增加压缩、检索和摘要。
后续 P18 可以在不推翻 P16 的前提下从长期记忆形成知识。
```

P16 完成后，游戏主线将具备：

```text
探索行为产生反馈。
反馈更新认知。
认知沉淀记忆。
记忆会强化、晋升、衰减、遗忘。
```

```text
P17：记忆压缩与检索。
```
