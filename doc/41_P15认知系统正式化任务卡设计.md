# P15 认知系统正式化任务卡设计

## 1. 阶段定位

P15 是《先驱者》真正进入“认知、记忆、知识、传播学习主线”的第一个阶段。

P15 不是重新设计 P3，也不是跳到记忆或知识系统。

P15 的一句话目标：

```text
把 P3/P6 中最小的 CognitionClaim 扩展为正式认知层，让生物能基于尝试反馈形成“可食用、可使用、效果、风险、效用、置信度”的主观认知。
```

P15 承接关系：

```text
P3：已有最小 CognitionEvidence / CognitionClaim / CognitionState / CognitionUpdater。
P6：Agent Observation 可读取最小 cognition claim。
P13：本地调试 CLI 已能导出 Agent 调试结果。
P14：正在开发安全调试输入适配，后续可接入 P15 的认知调试摘要。
P15：正式化认知层，但不进入记忆、知识、传播。
```

P15 完成后，系统仍然只解决“个体如何形成认知”，不解决“记忆如何长期保存”和“知识如何传播”。

## 2. P15 要解决什么问题

P3 的认知模型是最小闭环：

```text
subject + object_definition + action + effect_kind -> confidence
```

它能证明“吃未知果实后产生认知”，但还不足以支撑游戏核心玩法。

玩家真正关心的是：

```text
这个东西能吃吗？
吃了之后对我有什么影响？
这个东西能用吗？
用一下有什么作用？
它有风险吗？
我有多确定？
这个认知是直接尝试来的，还是别人告诉我的？
这个认知是否足够可靠，可以行动或传授？
```

P15 要把这些问题变成可测试、可查询、可调试的工程结构。

## 3. P15 设计原则

### 3.1 认知是主观层，不是真相层

认知不是 `ObjectDefinition`。

认知只能表达主体当前相信什么，不能表达物体真实隐藏属性。

正确：

```text
actor_001 认为 unknown_fruit 通过 eat 可能可食用，置信度 0.35。
actor_001 认为 unknown_fruit 通过 eat 有低风险，置信度 0.25。
```

错误：

```text
unknown_fruit.edible_profile.hunger_delta = -20。
unknown_fruit.effect_kind = edible。
```

### 3.2 认知只来自反馈和事件

P15 不能读取：

```text
ObjectDefinition hidden truth。
edible_profile。
hunger_delta / health_delta 作为真相字段。
effect_kind 作为物体真实效果。
GameState 完整内部结构。
SaveGame。
```

P15 可以读取：

```text
FeedbackGenerated 事件。
ActionResolved 事件。
StateChange 中可观察的 before/after 结果。
P3 legacy CognitionEvidence 中已经作为反馈产物存在的 observed 字段。
测试构造的安全反馈信号。
```

注意：P15 可以做 P3 legacy 兼容适配，但 V2 新 DTO 不再暴露 `hunger_delta` / `health_delta` 这种隐藏真相式字段名。

### 3.3 不关注形态颜色

P15 不设计：

```text
ObjectRevealLevel。
形状。
颜色。
外观。
视觉揭示等级。
```

原因：用户已确定本游戏关注的是认知用途，而不是视觉形态。

P15 只关注：

```text
能否食用。
能否使用。
食用后的可观察影响。
使用后的可观察作用。
风险。
效用。
证据。
置信度。
```

### 3.4 V2 与 P3 兼容，不破坏旧测试

P15 不删除 P3 的：

```text
CognitionEffectKind。
CognitionKey。
CognitionEvidence。
CognitionClaim。
CognitionState。
CognitionUpdater。
```

P15 新增 V2 层：

```text
CognitionClaimV2。
CognitionEvidenceRecord。
CognitionStateV2。
CognitionConfidenceModel。
CognitionQueryService。
```

P3 legacy 认知继续存在，P15 先做并行扩展。

建议 P15 在 `GameState` 中保留原字段：

```cpp
pathfinder::cognition::CognitionState cognition_state;
```

并新增：

```cpp
pathfinder::cognition::CognitionStateV2 cognition_state_v2;
```

规则管线可以在应用 P3 evidence 后，同步构建并应用 V2 evidence，确保旧逻辑不被破坏。

## 4. 本阶段不做什么

P15 明确不做：

```text
不做记忆系统。
不做短期/中期/长期记忆。
不做记忆压缩。
不做知识系统。
不做知识传播。
不做族群教学。
不做文明系统。
不做强化学习训练。
不做 LLM。
不做 H5。
不做 HTTP/WebSocket。
不做 SaveManager。
不做完整 use_object 规则。
不做对象反应系统。
不做火 + 木棍 = 火把。
```

P15 可以支持“Use 行为反馈形成认知”的安全输入结构，但不实现完整使用规则。

完整对象反应和组合留给后续对象反应阶段。

## 7. 枚举设计

### 7.1 CognitionSubjectKind

中文：认知主体类型。

用途：表示“谁拥有这条认知”。

```cpp
enum class CognitionSubjectKind {
    Unknown,
    Actor,
    Agent,
    Group,
    Tribe,
    Civilization,
    TestOnly
};
```

取值说明：

```text
Unknown：未知主体类型，校验失败。
Actor：当前世界中的行动者，P15 主要使用。
Agent：抽象 Agent，后续可用于狼、蜘蛛、人类、外星人等不同层级 Agent。
Group：小群体，P20/P22 后使用。
Tribe：族群，P20/P22 后使用。
Civilization：文明级主体，后续文明阶段使用。
TestOnly：测试专用主体。
```

P15 默认只正式支持 `Actor` 和 `TestOnly`。

### 7.2 CognitionTargetKind

中文：认知目标类型。

用途：表示“主体正在认知什么”。

```cpp
enum class CognitionTargetKind {
    Unknown,
    ObjectDefinition,
    WorldObject,
    ObjectCategory,
    AgentSpecies,
    EnvironmentFeature,
    TestOnly
};
```

取值说明：

```text
ObjectDefinition：对象定义级目标，例如 unknown_fruit，P15 主路径。
WorldObject：对象实例级目标，例如 obj_unknown_fruit_001，可用于短期个体经验。
ObjectCategory：对象类别，例如 fruit_like，P17/P18 后更有意义。
AgentSpecies：生物种类，例如 wolf/spider，后续危险/族群阶段使用。
EnvironmentFeature：环境特征，例如 fire_area/water_source，后续环境阶段使用。
TestOnly：测试专用目标。
```

P15 默认使用 `ObjectDefinition`，允许测试使用 `WorldObject` / `TestOnly`。

### 7.3 CognitionAspect

中文：认知方面。

用途：表示主体关心目标的哪一种用途或结果。

```cpp
enum class CognitionAspect {
    Unknown,
    Edibility,
    Usability,
    ConsumptionEffect,
    UseEffect,
    Risk,
    Utility
};
```

取值说明：

```text
Edibility：能否食用。
Usability：能否使用。
ConsumptionEffect：食用后的可观察影响。
UseEffect：使用后的可观察作用。
Risk：危险和负面可能性。
Utility：综合效用，后续供 Agent 行动选择使用。
```

注意：不允许增加 Color / Shape / VisualReveal 相关 aspect。

### 7.4 CognitionBeliefPolarity

中文：认知倾向。

用途：表示 claim 当前倾向支持还是反对某个认知。

```cpp
enum class CognitionBeliefPolarity {
    Unknown,
    Positive,
    Negative,
    Mixed
};
```

例子：

```text
Edibility + Positive：认为可食用。
Edibility + Negative：认为不可食用。
Risk + Positive：认为有风险。
Risk + Negative：认为风险低或暂未发现风险。
Usability + Positive：认为可使用。
UseEffect + Positive：认为使用会产生某种可观察效果。
Mixed：存在明显冲突证据。
```

### 7.5 CognitionEvidenceSupport

中文：证据支持方向。

用途：表示一条证据是在支持、反驳还是制造冲突。

```cpp
enum class CognitionEvidenceSupport {
    Unknown,
    Supports,
    Refutes,
    Neutral,
    Conflicts
};
```

### 7.6 CognitionEvidenceSourceKind

中文：认知证据来源类型。

用途：表示证据从哪里来。

```cpp
enum class CognitionEvidenceSourceKind {
    Unknown,
    DirectActionFeedback,
    ObservedEvent,
    StateChangeObservation,
    TaughtByOther,
    ImportedSafeProjection,
    TestOnly
};
```

P15 只正式实现：

```text
DirectActionFeedback。
ObservedEvent。
StateChangeObservation。
TestOnly。
```

`TaughtByOther` 只预留给 P20，P15 不实现传播。

### 7.7 CognitionActionContextKind

中文：认知行为上下文。

用途：表示这条认知来自哪种尝试行为。

```cpp
enum class CognitionActionContextKind {
    Unknown,
    Eat,
    Use,
    Touch,
    Combine,
    Observe,
    TestOnly
};
```

P15 只需要：

```text
Eat：食用反馈。
Use：使用反馈。
Observe：观察到事件。
TestOnly：测试构造。
```

`Combine` 只是为后续对象反应预留，P15 不实现组合规则。

### 7.8 CognitionOutcomeSignal

中文：认知结果信号。

用途：用安全、主观、可观察的方式描述反馈结果。

```cpp
enum class CognitionOutcomeSignal {
    Unknown,
    NeedImproved,
    NeedWorsened,
    HealthImproved,
    HealthWorsened,
    DamageTaken,
    ObjectConsumed,
    ToolActivated,
    NewObjectProduced,
    NoVisibleEffect,
    DangerObserved,
    TestOnly
};
```

注意：这是“观察信号”，不是隐藏真相。

正确：

```text
NeedImproved。
HealthWorsened。
ToolActivated。
```

错误：

```text
hunger_delta = -20。
health_delta = -5。
effect_kind = poison。
```

### 7.9 CognitionRiskLevel

中文：认知风险等级。

```cpp
enum class CognitionRiskLevel {
    Unknown,
    None,
    Low,
    Medium,
    High,
    Critical
};
```

P15 中风险来自可观察负面信号，不来自 `ObjectDefinition.risk_level`。

### 7.10 CognitionConfidenceBand

中文：认知置信区间。

用途：把数字置信度转换成可行动、可传授的阶段。

```cpp
enum class CognitionConfidenceBand {
    Unknown,
    Untrusted,
    Hypothesis,
    Actionable,
    Reliable,
    Teachable
};
```

默认阈值：

```text
Untrusted：0.00 - 0.19。
Hypothesis：0.20 - 0.49。
Actionable：0.50 - 0.69。
Reliable：0.70 - 0.84。
Teachable：0.85 - 1.00。
```

设计意图：

```text
刚吃一口：通常只是 Hypothesis。
多次尝试一致：逐渐 Actionable / Reliable。
足够可靠：未来 P20 可作为可传授知识输入。
```

P15 只计算 teachable band，不实现传播。

### 7.11 CognitionUpdateDecision

中文：认知更新决策。

```cpp
enum class CognitionUpdateDecision {
    Unknown,
    Created,
    Reinforced,
    Weakened,
    Conflicted,
    Ignored
};
```

### 7.12 CognitionQueryMode

中文：认知查询模式。

```cpp
enum class CognitionQueryMode {
    Unknown,
    ExactTarget,
    IncludeTargetKindFallback,
    BestActionable,
    TeachableCandidates
};
```

## 8. 数据结构设计

### 8.1 CognitionSubject

中文：认知主体。

```cpp
struct CognitionSubject {
    CognitionSubjectKind kind = CognitionSubjectKind::Unknown;
    pathfinder::foundation::EntityId subject_id;
    std::optional<pathfinder::foundation::EntityId> owner_group_id;

    pathfinder::foundation::Result<void> validateBasic() const;
};
```

规则：

```text
kind 不能是 Unknown。
subject_id 必须合法。
P15 中 kind=Actor 时 subject_id 对应 actor_id。
owner_group_id 预留给后续族群/传播。
```

### 8.2 CognitionTarget

中文：认知目标。

```cpp
struct CognitionTarget {
    CognitionTargetKind kind = CognitionTargetKind::Unknown;
    std::string target_id;
    std::optional<std::string> category_id;
    std::string public_label_key;

    pathfinder::foundation::Result<void> validateBasic() const;
};
```

规则：

```text
kind 不能是 Unknown。
target_id 必须是安全 ID。
public_label_key 只能是展示/调试 key，不能放 hidden truth。
category_id 只是主观或公开类别，不等同 ObjectDefinition hidden category。
```

### 8.3 CognitionClaimKeyV2

中文：认知声明键。

```cpp
struct CognitionClaimKeyV2 {
    CognitionSubject subject;
    CognitionTarget target;
    pathfinder::foundation::ActionId action_id;
    CognitionActionContextKind action_context = CognitionActionContextKind::Unknown;
    CognitionAspect aspect = CognitionAspect::Unknown;

    bool operator==(const CognitionClaimKeyV2& other) const;
    pathfinder::foundation::Result<void> validateBasic() const;
};
```

语义：

```text
某主体 + 对某目标 + 在某行为上下文下 + 关于某方面 的认知。
```

例子：

```text
actor_001 + unknown_fruit + eat + Edibility。
actor_001 + unknown_fruit + eat + ConsumptionEffect。
actor_001 + unknown_tool + use + Usability。
actor_001 + unknown_tool + use + UseEffect。
```

### 8.4 CognitionEvidenceId / CognitionClaimV2Id

```cpp
struct CognitionEvidenceRecordIdTag {};
using CognitionEvidenceRecordId = pathfinder::foundation::StrongId<CognitionEvidenceRecordIdTag>;

struct CognitionClaimV2IdTag {};
using CognitionClaimV2Id = pathfinder::foundation::StrongId<CognitionClaimV2IdTag>;
```

ID 生成规则：

```text
evidence：cognition_evidence_<source_event_id>_<aspect>_<index>
claim：cognition_claim_<subject_id>_<target_id>_<action_id>_<aspect>
```

注意事项：

```text
不要手写随机 ID。
不要把真实存档路径放入 ID。
不要把隐藏属性值放入 ID。
ID 必须稳定、可测试、可复盘。
```

### 8.5 CognitionEvidenceRecord

中文：认知证据记录。

```cpp
struct CognitionEvidenceRecord {
    CognitionEvidenceRecordId evidence_id;
    CognitionClaimKeyV2 key;
    CognitionEvidenceSourceKind source_kind = CognitionEvidenceSourceKind::Unknown;
    CognitionEvidenceSupport support = CognitionEvidenceSupport::Unknown;
    CognitionOutcomeSignal outcome_signal = CognitionOutcomeSignal::Unknown;
    CognitionRiskLevel observed_risk = CognitionRiskLevel::Unknown;
    double confidence_weight = 0.0;
    double utility_signal = 0.0;
    pathfinder::foundation::EventId source_event_id;
    pathfinder::foundation::Tick observed_tick;
    std::vector<std::string> reason_keys;
    std::vector<std::string> warning_keys;

    pathfinder::foundation::Result<void> validateBasic() const;
};
```

字段说明：

```text
evidence_id：稳定证据 ID。
key：证据对应的认知键。
source_kind：证据来源。
support：支持或反驳。
outcome_signal：可观察结果信号。
observed_risk：观察到的风险等级。
confidence_weight：证据权重，范围 0.0 - 1.0。
utility_signal：效用信号，范围 -1.0 - 1.0。
source_event_id：来源事件。
observed_tick：观察时间。
reason_keys：解释 key。
warning_keys：警告 key。
```

校验规则：

```text
confidence_weight 必须 0.0 - 1.0。
utility_signal 必须 -1.0 - 1.0。
source_kind 不能 Unknown。
support 不能 Unknown。
aspect 不能 Unknown。
不能包含 hidden truth key。
```

### 8.6 CognitionClaimV2

中文：认知声明 V2。

```cpp
struct CognitionClaimV2 {
    CognitionClaimV2Id claim_id;
    CognitionClaimKeyV2 key;
    CognitionBeliefPolarity polarity = CognitionBeliefPolarity::Unknown;
    double confidence = 0.0;
    CognitionConfidenceBand confidence_band = CognitionConfidenceBand::Unknown;
    CognitionRiskLevel risk_level = CognitionRiskLevel::Unknown;
    double utility_score = 0.0;
    size_t evidence_count = 0;
    size_t supporting_evidence_count = 0;
    size_t refuting_evidence_count = 0;
    bool conflicted = false;
    std::optional<CognitionEvidenceRecordId> first_evidence_id;
    std::optional<CognitionEvidenceRecordId> last_evidence_id;
    std::vector<std::string> reason_keys;
    std::vector<std::string> warning_keys;

    pathfinder::foundation::Result<void> validateBasic() const;
};
```

规则：

```text
confidence 必须 0.0 - 1.0。
utility_score 必须 -1.0 - 1.0。
evidence_count = supporting + refuting 或大于等于二者之和。
conflicted=true 时 polarity 应为 Mixed。
confidence_band 必须由 CognitionConfidenceModel 计算，不要手写。
```

### 8.7 CognitionStateV2

中文：认知状态 V2。

```cpp
class CognitionStateV2 {
public:
    const std::vector<CognitionClaimV2>& claims() const;
    const std::vector<CognitionEvidenceRecord>& evidence_records() const;

    const CognitionClaimV2* findClaim(const CognitionClaimKeyV2& key) const;
    const CognitionClaimV2* findById(const CognitionClaimV2Id& claim_id) const;

    void upsertClaim(CognitionClaimV2 claim);
    pathfinder::foundation::Result<void> appendEvidence(CognitionEvidenceRecord evidence);

    pathfinder::foundation::Result<void> validateBasic() const;

private:
    std::vector<CognitionClaimV2> claims_;
    std::vector<CognitionEvidenceRecord> evidence_records_;
};
```

规则：

```text
appendEvidence 只保存安全证据记录，不保存 ObjectDefinition hidden truth。
upsertClaim 按 key 替换。
findClaim 不做模糊查询，模糊/候选查询交给 CognitionQueryService。
```

### 8.8 CognitionConfidenceModelOptions

中文：认知置信模型选项。

```cpp
struct CognitionConfidenceModelOptions {
    double initial_direct_feedback_confidence = 0.35;
    double reinforce_multiplier = 0.65;
    double refute_multiplier = 0.75;
    double conflict_penalty = 0.20;
    double min_confidence = 0.0;
    double max_confidence = 1.0;
    double actionable_threshold = 0.50;
    double reliable_threshold = 0.70;
    double teachable_threshold = 0.85;

    pathfinder::foundation::Result<void> validateBasic() const;
};
```

设计目的：

```text
第一次尝试形成假设。
重复一致证据增强置信。
反向证据降低置信。
冲突证据标记 conflicted。
置信区间支持后续行为和传播，但 P15 不实现传播。
```

### 8.9 CognitionUpdateRequest / CognitionUpdateResult

```cpp
struct CognitionUpdateRequest {
    std::optional<CognitionClaimV2> existing_claim;
    CognitionEvidenceRecord evidence;
    CognitionConfidenceModelOptions options;

    pathfinder::foundation::Result<void> validateBasic() const;
};

struct CognitionUpdateResult {
    CognitionUpdateDecision decision = CognitionUpdateDecision::Unknown;
    std::optional<CognitionClaimV2> before_claim;
    CognitionClaimV2 after_claim;
    std::vector<std::string> reason_keys;
    std::vector<std::string> warning_keys;
```

## 9. 服务接口设计

### 9.1 CognitionConfidenceModel

中文：认知置信模型。

职责：根据旧 claim 和新 evidence 计算新 claim。

```cpp
class CognitionConfidenceModel {
public:
    CognitionConfidenceBand bandFor(double confidence, const CognitionConfidenceModelOptions& options = {}) const;

    pathfinder::foundation::Result<CognitionUpdateResult> applyEvidence(
        const CognitionUpdateRequest& request) const;
};
```

规则：

```text
没有 existing_claim：创建新 claim。
Supports：提升或创建正向 claim。
Refutes：降低旧 claim 或创建负向 claim。
Conflicts：标记 conflicted，polarity=Mixed。
Neutral：只记录 evidence，不应显著改变 confidence。
```

### 9.2 CognitionUpdaterV2

中文：认知更新器 V2。

职责：把 evidence 应用到 CognitionStateV2。

```cpp
class CognitionUpdaterV2 {
public:
    pathfinder::foundation::Result<CognitionUpdateResult> applyEvidence(
        CognitionStateV2& state,
        const CognitionEvidenceRecord& evidence,
        const CognitionConfidenceModelOptions& options = {}) const;
};
```

规则：

```text
先 validate evidence。
再 appendEvidence。
再读取 existing claim。
再调用 CognitionConfidenceModel。
最后 upsert claim。
失败时不要写入半成品 claim。
```

建议实现顺序：先计算 claim，再 append evidence + upsert claim，避免部分写入。

### 9.3 CognitionEvidenceBuilder

中文：认知证据构造器。

职责：从安全反馈/事件生成 V2 evidence。

```cpp
struct CognitionFeedbackSignal {
    CognitionSubject subject;
    CognitionTarget target;
    pathfinder::foundation::ActionId action_id;
    CognitionActionContextKind action_context = CognitionActionContextKind::Unknown;
    CognitionOutcomeSignal outcome_signal = CognitionOutcomeSignal::Unknown;
    CognitionRiskLevel risk_level = CognitionRiskLevel::Unknown;
    double utility_signal = 0.0;
    pathfinder::foundation::EventId source_event_id;
    pathfinder::foundation::Tick observed_tick;
    std::vector<std::string> reason_keys;

    pathfinder::foundation::Result<void> validateBasic() const;
};

class CognitionEvidenceBuilder {
public:
    pathfinder::foundation::Result<std::vector<CognitionEvidenceRecord>> fromFeedbackSignal(
        const CognitionFeedbackSignal& signal) const;

    pathfinder::foundation::Result<std::vector<CognitionEvidenceRecord>> fromLegacyEvidence(
        const CognitionEvidence& legacy_evidence) const;
};
```

`fromFeedbackSignal` 映射规则：

```text
Eat + NeedImproved：生成 Edibility Positive、ConsumptionEffect Positive、Risk Negative/None、Utility Positive。
Eat + HealthWorsened/DamageTaken：生成 Edibility Negative 或 Risk Positive、ConsumptionEffect Negative、Utility Negative。
Use + ToolActivated：生成 Usability Positive、UseEffect Positive、Utility Positive。
Use + NewObjectProduced：生成 Usability Positive、UseEffect Positive。
Use + NoVisibleEffect：生成 Usability Negative 或低置信 Neutral。
DangerObserved：生成 Risk Positive。
```

`fromLegacyEvidence` 规则：

```text
只用于兼容 P3。
可以读取 legacy evidence 中已经作为反馈结果存在的 observed 字段。
必须转换为 CognitionOutcomeSignal，不把 legacy 字段名写入 V2 claim/evidence 的 reason_keys。
不能读取 ObjectDefinition。
不能读取 edible_profile。
```

### 9.4 CognitionQueryService

中文：认知查询服务。

职责：给 Agent、调试、后续记忆系统查询认知。

```cpp
class CognitionQueryService {
public:
    pathfinder::foundation::Result<CognitionQueryResult> query(
        const CognitionStateV2& state,
        const CognitionQuery& query) const;

    pathfinder::foundation::Result<std::optional<CognitionClaimV2>> findBestClaim(
        const CognitionStateV2& state,
        const CognitionSubject& subject,
        const CognitionTarget& target,
        CognitionAspect aspect) const;

    pathfinder::foundation::Result<std::optional<CognitionClaimV2>> findEdibility(
        const CognitionStateV2& state,
        const CognitionSubject& subject,
        const CognitionTarget& target) const;

    pathfinder::foundation::Result<std::optional<CognitionClaimV2>> findUsability(
        const CognitionStateV2& state,
        const CognitionSubject& subject,
        const CognitionTarget& target) const;

    pathfinder::foundation::Result<std::optional<CognitionClaimV2>> findRisk(
        const CognitionStateV2& state,
        const CognitionSubject& subject,
        const CognitionTarget& target) const;
};
```

规则：

```text
QueryService 只读 CognitionStateV2。
QueryService 不读取 ObjectDefinition。
QueryService 不推导隐藏真相。
QueryService 可以按 confidence 排序。
TeachableCandidates 只返回 band=Teachable 的 claim，不执行传播。
```

## 10. P3/P15 兼容管线

P15 推荐采用并行兼容管线：

```text
RuleResolver 生成 P3 legacy CognitionEvidence。
RulePipeline 应用 P3 CognitionUpdater，保持旧测试通过。
RulePipeline 调用 CognitionEvidenceBuilder::fromLegacyEvidence。
RulePipeline 应用 CognitionUpdaterV2，生成 CognitionClaimV2。
```

管线示意：

```text
CommandEnvelope
-> RulePipeline
-> EatObjectResolver
-> StateChange + Event + legacy CognitionEvidence
-> CognitionUpdater.applyEvidence      // P3 legacy
-> CognitionEvidenceBuilder.fromLegacyEvidence
-> CognitionUpdaterV2.applyEvidence    // P15 V2
-> EventStream 可继续使用 CognitionUpdated
```

注意：

```text
P15 不要求 EatObjectResolver 直接生成 V2 evidence。
P15 不要求新增完整 UseObjectResolver。
P15 的 Use 认知可以先通过 CognitionFeedbackSignal 单元/集成测试验证。
```

## 11. 玩法映射

P15 对玩家体验的意义：

```text
玩家第一次吃未知果实后，不再只是“知道 edible”。
系统能表达：我认为它可食用、它改善/恶化了某种状态、风险是多少、我有多确定。
玩家多次尝试后，置信度会提升。
出现反向反馈后，认知会变成冲突或降低置信。
未来传播时，传授的是 claim，而不是真实 ObjectDefinition。
```

玩家看到的不是隐藏数值，而是认知结论：

```text
“可能能吃”。
“吃了似乎有帮助”。
“暂未发现明显风险”。
“这个结论还不够可靠”。
```

## 15. 实现拆分建议

为了降低 AI 上下文压力，P15 建议拆成 8 个小任务：

```text
TASK-P15-001：实现 cognition_v2_types.h/cpp，含 enum、Subject、Target、Key、EvidenceRecord、ClaimV2。
TASK-P15-002：实现 CognitionStateV2，支持 find/upsert/append/validate。
TASK-P15-003：实现 CognitionConfidenceModel，支持创建、增强、反驳、冲突和 band。
TASK-P15-004：实现 CognitionUpdaterV2，组合 state + confidence model。
TASK-P15-005：实现 CognitionEvidenceBuilder，支持 FeedbackSignal 和 legacy evidence 适配。
TASK-P15-006：实现 CognitionQueryService，支持 exact/best/teachable 查询。
TASK-P15-007：接入 GameState / RulePipeline，让 P3 eat flow 同步生成 V2 claim。
TASK-P15-008：补齐 P15 单元/集成/边界测试和 dev_notes。
```

实现顺序必须从底层到上层：

```text
types -> state -> confidence -> updater -> evidence builder -> query -> pipeline integration -> tests。
```

不要一开始就改 Pipeline。

## 16. 风险与规避

### 16.1 风险：把隐藏真相当认知

规避：

```text
V2 DTO 只保存 outcome_signal、risk_level、utility_signal。
不保存 edible_profile / hunger_delta / health_delta / effect_kind。
```

### 16.2 风险：P15 过早变成记忆系统

规避：

```text
P15 只保存当前认知状态和证据记录。
不实现 ShortTerm/MidTerm/LongTerm。
不实现 decay。
不实现 compression。
```

### 16.3 风险：Use 规则范围膨胀

规避：

```text
P15 只接受 Use 反馈信号生成认知。
不实现 UseObjectResolver。
不实现对象反应和组合。
```

### 16.4 风险：破坏 P3/P7 旧闭环

规避：

```text
保留 legacy CognitionState。
新增 CognitionStateV2 并行更新。
旧测试必须继续通过。
```

## 17. P15 与 P16 的关系

P15 输出的是“当前认知”：

```text
我认为这个东西可食用。
我认为这个东西可使用。
我认为这个行为有风险。
我对这件事有多少置信度。
```

P16 才处理“记忆”：

```text
这条认知是什么时候形成的。
它是短期、中期还是长期。
它是否会遗忘。
重复经验如何增强记忆。
关键事件如何晋升长期记忆。
```

P15 不要实现 P16 的记忆衰减、记忆强度和存储周期。
