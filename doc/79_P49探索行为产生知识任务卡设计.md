# P49 探索行为产生知识任务卡设计

更新时间：2026-05-21
阶段：P49
依赖：P42、P43、P44、P45、P46、P47、P48、P15-P21、P27、P28.5
状态：待实现

## 1. 阶段定位

P49 是 V2 开放世界网格路线中“世界行为接回认知、记忆、知识”的桥接阶段。

前置阶段已经具备：

```text
P42：JSON 内容包、ContentRegistry、knowledge template、reaction、effect、object。
P43：统一 WorldCommand 管线，所有世界行为走 Command。
P44：世界网格 runtime、Actor 位置、探索投影。
P45：背包、物品实例、OnMap / InInventory 归属权威接口。
P46：世界生成器与资源节点。
P47：采集、砍伐、挖掘产生真实世界材料和 WorldExperienceDto。
P48：制作 / 反应消耗材料、生成产物，并输出 WorldExperienceDto。
P15-P21：旧认知、记忆、知识、学习闭环已经存在。
```

P49 要把这些能力连起来：

```text
玩家 / NPC 执行世界 Command
-> Command 成功产生 WorldExperienceDto
-> P49 读取 experience + ContentRegistry knowledge template
-> 转成 LearningSafeFeedbackInput
-> 调用 LearningLoopService
-> 写入 / 更新 actor 的 KnowledgeRepository
-> 返回 knowledge deltas / events / projection patch
```

P49 的本质不是“新写一套知识系统”，而是把 V2 世界经验接入旧的认知学习闭环。

## 2. 为什么现在做

P48 完成后，系统已经可以真实制作物品，但还没有把制作结果变成知识。

如果不做 P49，会出现下面问题：

```text
玩家砍树得到木头，但不会学会“砍树可得木头”。
玩家制作火把成功，但不会学会“木头和火源可以制作火把”。
NPC 即使看到了经验，也没有可查询的知识来源。
P50/P51 的无知 NPC 和目标执行无法从知识推导行为。
前端只能看到一次事件，无法看到长期认知成长。
```

P49 必须在 P50 之前完成，因为 P50 要做“教学与无知 NPC 基础行动”。教学必须有可传播的知识。

## 3. 阶段目标

P49 完成后，系统必须具备以下能力：

```text
1. 能从 WorldCommandExecutionResult.experiences 中提取安全世界经验。
2. 能根据 experience.reason_keys / effect_key / action_key / object_key 找到 JSON knowledge template。
3. 能把世界经验转成旧 LearningLoopInput / LearningSafeFeedbackInput。
4. 能调用 LearningLoopService 形成认知、记忆、知识。
5. 能把形成的 KnowledgeClaim 写入 actor 的 KnowledgeRepository。
6. 能返回知识变化事件、状态 delta、projection patch。
7. 能支持采集、砍伐、制作、使用、食用等多类经验。
8. 失败命令不能形成正向知识，只能形成可选的失败记忆或跳过。
9. 不能泄露隐藏真实属性，只能使用安全经验和内容模板。
10. 必须给 P50/P51 提供 NPC 可查询的知识结果。
```

## 4. 本阶段不解决的问题

P49 不做以下内容：

```text
不做 NPC 自主规划：P50/P51 做。
不做教学传播流程：P50 做。
不做野兽学习和生态：P52 做。
不做 H5 V2 地图界面：P53 做。
不做长期存档：P54 做。
不重写 P15-P21 认知、记忆、知识系统。
不在 P49 内实现复杂因果归因完整重构。
不让前端参与知识判断。
不让 JSON 直接写 runtime 状态。
```

P49 只做一件事：世界经验安全转化为主体知识。

## 5. 核心设计原则

### 5.1 世界经验是输入，不是结论

`WorldExperienceDto` 只代表“某主体经历了某次结果”。

它不能直接等于稳定知识。

必须经过：

```text
experience -> safe feedback -> cognition -> memory -> summary -> knowledge claim
```

禁止实现：

```text
看到 experience 就直接 new KnowledgeClaim 并标记 Active。
```

### 5.2 ContentRegistry 是模板权威

P49 只能用 `ContentRegistry::findKnowledgeTemplate()` 查知识模板。

知识模板来自 JSON，不允许在 P49 写死：

```text
red_berry + eat -> restore_hunger
wood + craft -> torch
fire + use -> repel_beast
```

如果没有模板，P49 可以跳过或输出 `TemplateMissing`，不能猜。

### 5.3 学习系统是知识形成权威

P49 不能重写知识形成规则。

P49 的职责是适配：

```text
WorldExperienceDto -> LearningLoopInput
LearningLoopResult -> WorldKnowledgeLearningResult
```

真正的认知、记忆、知识状态变化仍由 P15-P21 的服务决定。

### 5.4 知识归属必须明确

每条知识必须属于某个 owner：

```text
玩家 actor。
NPC agent。
族群 / 群体。
后续文明。
```

P49 MVP 只要求 Actor / Agent owner。

禁止把知识写到全局静态状态。

### 5.5 成功经验和失败经验分开

成功命令可以形成正向知识。

失败命令默认不形成正向知识，只能形成：

```text
Blocked 记忆：这个动作目前做不了。
MissingInput 记忆：缺材料。
StateMismatch 记忆：材料状态不对。
Risk / Warning 记忆：尝试有危险。
```

P49 MVP 可以先跳过失败经验，但必须保留枚举和测试边界。

### 5.6 不泄露隐藏真相

P49 只能消费：

```text
WorldExperienceDto 中的 subject_entity_key / target_entity_key / effect_key / condition_keys / reason_keys。
WorldEventDto 中的安全文案和事件 key。
KnowledgeTemplateContent 中公开模板字段。
EffectDefinitionContent.learning 中公开学习参数。
```

禁止消费：

```text
未投影给 actor 的隐藏 trait。
object 内部真实状态。
开发调试 key。
前端自己传的知识结论。
```

### 5.7 一次命令可产生多条知识

例如制作火把可能同时带来：

```text
木头参与制作火把。
火源是催化条件。
火把是反应产物。
制作成功本身可重复。
```

P49 必须支持一个 `WorldCommandExecutionResult` 内多个 `WorldExperienceDto`，以及一个 experience 对应多个 knowledge template。

### 5.8 可被 NPC 使用

P49 形成的知识必须保留：

```text
teachable
usable_by_ai
usable_for_action
confidence/status
conditions
```

否则 P50/P51 无法让 NPC 根据知识推导行为。

## 6. 模块划分

| 模块 | 中文含义 | 职责 | 输入 | 输出 | 禁止事项 |
|---|---|---|---|---|---|
| WorldLearningTypes | 世界学习类型 | 定义 P49 请求、计划、结果、失败码、知识 delta | command result / registry | DTO | 不读写 runtime |
| WorldExperienceExtractor | 世界经验提取器 | 从 WorldCommandExecutionResult 提取可学习经验 | command result | experience list | 不判断知识真假 |
| WorldKnowledgeTemplateResolver | 知识模板解析器 | 从 reason_keys/effect/action/object 找模板 | experience / registry | template refs | 不硬编码模板 |
| WorldSafeFeedbackBuilder | 安全反馈构建器 | 把 experience + template 转成 LearningSafeFeedbackInput | experience/template | feedback input | 不泄露隐藏状态 |
| WorldLearningLoopBridge | 学习闭环桥 | 调用 LearningLoopService | feedback input | LearningLoopResult | 不重写学习算法 |
| WorldKnowledgeStorePort | 知识存储端口 | 查询/写入 actor 当前知识 | owner / claims | repository update | 不使用全局静态 |
| WorldKnowledgeLearningService | P49 编排服务 | extract -> resolve -> build -> learn -> apply | request | result | 不直接改世界对象 |
| WorldKnowledgeProjectionBridge | 投影桥 | 把知识变化转成 patch / events / deltas | result | frontend-safe patch | 不让前端推导知识 |

## 7. 核心枚举

### 7.1 WorldLearningSourceKind

中文：世界学习来源类型。

| 枚举值 | 中文 | 使用场景 |
|---|---|---|
| Unknown | 未知 | 非法默认值 |
| DirectExperience | 亲身经历 | 自己吃、用、采集、制作 |
| Observation | 观察学习 | 看见别人行为，P50 后续扩展 |
| Taught | 被教学 | NPC/玩家教学，P50 接入 |
| SystemGranted | 系统授予 | 调试或剧情，默认禁用 |
| TestOnly | 测试专用 | 只能测试使用 |

P49 MVP 必须支持 `DirectExperience`。

### 7.2 WorldLearningExperienceKind

中文：世界经验类型。

| 枚举值 | 中文 | 说明 |
|---|---|---|
| Unknown | 未知 | 非法默认值 |
| ObjectAction | 对对象行动 | eat/use/inspect/pickup/drop |
| ResourceHarvest | 资源采集 | gather/chop/mine/dig |
| ReactionCraft | 制作反应 | craft/combine/use reaction |
| EnvironmentEvent | 环境事件 | 区域事件、天气，后续 |
| ThreatEncounter | 危险遭遇 | 野兽/陷阱，P52 后续 |
| Teaching | 教学经验 | P50 后续 |
| TestOnly | 测试专用 | 只能测试使用 |

### 7.3 WorldLearningFailureKind

中文：世界学习失败类型。

| 枚举值 | 中文 | 说明 |
|---|---|---|
| None | 无失败 | 成功 |
| InvalidRequest | 请求非法 | 缺 actor / command id |
| NoExperience | 没有经验 | 命令无可学习输出 |
| ActorMissing | 主体不存在 | owner 无法解析 |
| TemplateMissing | 模板缺失 | 找不到 knowledge template |
| TemplateInvalid | 模板非法 | 模板字段不完整 |
| UnsafeExperience | 不安全经验 | 含隐藏或禁用 key |
| FeedbackBuildFailed | 反馈构建失败 | 无法转 LearningSafeFeedbackInput |
| LearningLoopFailed | 学习闭环失败 | LearningLoopService 返回失败 |
| StoreFailed | 存储失败 | KnowledgeRepository put 失败 |
| ProjectionFailed | 投影失败 | patch/event 构建失败 |
| PartialFailed | 部分失败 | 多经验部分成功部分失败 |

### 7.4 WorldKnowledgeLearningDecision

中文：世界知识学习决策。

| 枚举值 | 中文 | 说明 |
|---|---|---|
| Unknown | 未知 | 非法默认值 |
| Learned | 学到知识 | 新增或强化知识 |
| Reinforced | 强化已有知识 | 同类经验重复出现 |
| Revised | 修正知识 | 与旧知识冲突，走 revision |
| SkippedNoTemplate | 无模板跳过 | 没有配置知识模板 |
| SkippedUnsafe | 不安全跳过 | 防隐藏真相 |
| Failed | 失败 | 不能完成流程 |
| TestOnly | 测试专用 | 只能测试 |

## 8. 核心 DTO

### 8.1 WorldKnowledgeLearningRequest

```cpp
struct WorldKnowledgeLearningRequest {
    std::string request_id;
    world_command::WorldCommandDto source_command;
    world_command::WorldCommandExecutionResult command_result;
    WorldLearningSourceKind source_kind;
    std::string actor_key;
    std::vector<knowledge::KnowledgeClaim> existing_actor_claims;
    std::vector<memory::MemoryRecord> existing_memories;
    std::vector<memory::MemorySummary> existing_summaries;
    uint64_t tick;
    std::map<std::string, std::string> parameters;
};
```

要求：

```text
request_id 必须稳定。
actor_key 必须和 command actor 对齐。
command_result.result_kind 必须成功才默认形成正向学习。
existing_actor_claims 由调用方传入或由 store port 查询。
```

### 8.2 WorldExperienceLearningDraft

```cpp
struct WorldExperienceLearningDraft {
    std::string draft_id;
    world_command::WorldExperienceDto experience;
    WorldLearningExperienceKind experience_kind;
    std::vector<std::string> candidate_template_keys;
    std::vector<content::KnowledgeTemplateContent> templates;
    std::vector<learning::LearningLoopInput> learning_inputs;
    std::vector<std::string> warning_keys;
};
```

### 8.3 WorldKnowledgeDelta

```cpp
struct WorldKnowledgeDelta {
    std::string delta_id;
    foundation::KnowledgeId knowledge_id;
    std::string owner_key;
    std::string subject_object_key;
    std::string action_key;
    std::string effect_key;
    std::string target_object_key;
    std::string status;
    std::string decision;
    std::vector<std::string> reason_keys;
};
```

### 8.4 WorldKnowledgeLearningResult

```cpp
struct WorldKnowledgeLearningResult {
    bool ok;
    WorldKnowledgeLearningDecision decision;
    WorldLearningFailureKind failure_kind;
    std::vector<WorldExperienceLearningDraft> drafts;
    std::vector<learning::LearningLoopResult> learning_results;
    std::vector<knowledge::KnowledgeClaim> learned_claims;
    std::vector<WorldKnowledgeDelta> knowledge_deltas;
    std::vector<world_command::WorldEventDto> events;
    std::vector<world_command::WorldStateDeltaDto> state_deltas;
    std::optional<world_command::WorldProjectionPatchDto> projection_patch;
    std::vector<std::string> warning_keys;
};
```

## 9. 模板解析规则

P49 查找知识模板的顺序必须稳定：

```text
1. experience.reason_keys 中以 knowledge template key 命中的项。
2. reaction.knowledge_templates 由 P48 写入 reason_keys 的项。
3. feedback/effect 映射：subject_entity_key + command_key + effect_key。
4. fallback：subject_entity_key + action_key + effect_key 在全部 template 中匹配。
```

匹配字段：

```text
KnowledgeTemplateContent.subject_object_key == experience.subject_entity_key 或 target_entity_key。
KnowledgeTemplateContent.action_key == experience.command_key。
KnowledgeTemplateContent.effect_key == experience.effect_key。
target_object_key 为空或与 experience.target_entity_key 匹配。
```

禁止：

```text
按中文文案猜模板。
按 object description 猜效果。
找不到模板时自动生成新模板。
```

## 10. 安全反馈构建规则

`WorldSafeFeedbackBuilder` 负责把经验变成 `LearningSafeFeedbackInput`。

映射规则：

```text
feedback_key = experience.experience_id + ":feedback"
action_key = experience.command_key
effect_key = experience.effect_key
observed_tick = experience.tick 或 request.tick
source_event_ids = command_result.events[*].event_id
condition_keys = experience.condition_keys + template 条件摘要
reason_keys = experience.reason_keys + template.knowledge_key
```

主体映射：

```text
cognition_target.target_id = template.subject_object_key 或 experience.subject_entity_key
memory_subject.subject_id = 同上
knowledge_subject.subject_id = 同上
```

结果信号映射：

```text
beneficial effect -> PositiveUtility / Success
risk_delta > 0 -> Risk / Warning
utility_delta < 0 -> NegativeUtility
craft success -> Success
harvest success -> Discovery + Success
```

P49 MVP 可以先用 effect learning 字段：

```text
EffectDefinitionContent.learning.confidence_delta
EffectDefinitionContent.learning.teachable
```

如果 effect 缺 learning 字段：

```text
使用安全默认值：confidence_delta = 0.35，teachable = template.teachable。
```

## 11. 知识状态规则

P49 不直接决定最终知识状态，但必须把模板状态传入学习链。

模板字段：

```text
default_status
teachable
usable_by_ai
summary_key
```

默认策略：

```text
首次亲身成功经验：Hypothesis 或 Candidate。
重复经验：通过 LearningLoopService 强化。
超过旧知识阈值：由 KnowledgeFormationService 转 Active / Teachable。
模板 default_status 只能作为上限提示，不可强行越权。
```

必须避免：

```text
一次制作火把就直接让全族都知道。
一次观察就等同亲身经历。
失败缺材料变成“这个反应不存在”。
```

## 12. Command 管线接入方式

P49 建议先实现为独立服务，不立即强行嵌入所有 command handler。

P49 MVP 入口：

```cpp
WorldKnowledgeLearningService::learnFromCommandResult(request)
```

后续 P50/P51 再把它挂到 `WorldCommandPipeline` 的 post-command hook：

```text
command execute success
-> collect experiences
-> P49 learn
-> merge learning events / deltas / patch
-> return to frontend / NPC runtime
```

本阶段可以添加一个轻量 helper：

```cpp
WorldCommandLearningBridge::appendLearningResult(command_result, learning_result)
```

但不要破坏 P43 Command 模块边界。

## 13. 与 P47/P48 的关系

P47 采集经验示例：

```text
subject_entity_key = berry_bush
action_key = gather
effect_key = gather_red_berry
reason_keys = [resource_node_key, output_object_key]
```

P49 应形成：

```text
berry_bush + gather -> red_berry / harvest_success
```

P48 制作经验示例：

```text
subject_entity_key = torch
target_entity_key = wood_plus_fire_make_torch
action_key = craft
effect_key = make_torch
reason_keys = [reaction_key, knowledge_make_torch]
```

P49 应形成：

```text
wood/fire + craft -> torch / make_torch
```

如果模板只描述产物，P49 先形成产物知识；复杂输入知识可在后续模板扩展。

## 14. 与 P50/P51 的关系

P50 教学需要：

```text
KnowledgeClaim.teaching_profile.teachable = true
projection_flags.usable_for_teaching = true
```

P51 目标执行需要：

```text
projection_flags.usable_by_ai = true
projection_flags.usable_for_action = true
predicate.action_key / effect_key 可查询
conditions 可用于条件表达式
```

所以 P49 生成的知识必须保留这些字段，不允许只生成文本摘要。

## 15. 错误处理

失败不能静默吞掉。

最低要求：

```text
NoExperience：返回 ok=true decision=SkippedNoTemplate 或 Skipped，无事件。
TemplateMissing：返回 warning，不阻断原 command。
UnsafeExperience：返回 warning，并生成审计 trace。
LearningLoopFailed：P49 result ok=false，但不能回滚原世界 command。
StoreFailed：P49 result ok=false，不能伪装学会。
```

P49 是 post-command 学习阶段，不能反向撤销 P47/P48 的世界状态变化。

## 18. 最小验收故事

### 18.1 制作火把学习

```text
给 actor 背包木头和火源。
执行 Craft reaction。
P48 输出 make_torch experience。
P49 读取 knowledge_make_torch template。
P49 形成 actor 的火把制作知识。
知识可 teachable / usable_by_ai。
```

### 18.2 采集学习

```text
地图生成 berry_bush resource node。
执行 Gather。
P47 输出 harvest experience。
P49 形成“这个资源可采集得到某物”的知识。
```

### 18.3 缺模板跳过

```text
命令成功但没有 knowledge template。
P49 返回 SkippedNoTemplate。
世界状态不变。
不会生成假知识。
```

## 19. 后续扩展预留

P49 后续可扩展：

```text
观察学习：NPC 看见玩家行为形成低置信知识。
教学学习：P50 教学直接传递知识。
族群知识：多个成员重复经验汇总为 tribe claim。
因果归因增强：多因素经验区分“吃多了”和“腐烂导致”。
失败知识：缺材料、工具不锋利、环境不满足也可成为可行动知识。
高级文明知识：科技、魔法、机械危机都作为 action/effect/template 链路表达。
```

这些扩展不能改变 P49 的主链路，只增加新的 source_kind、experience_kind 或 template 解析策略。
