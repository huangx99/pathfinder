# P46 世界生成器后续扩展风险头脑风暴

## 1. 文档目的

本文档是 P46 主设计的扩展风险补充，不替代 `doc/76_P46世界生成器与资源分布任务卡设计.md`。

它回答一个问题：

```text
P46 世界生成器做完以后，随着 P47 采集、P48 制作、P49 认知、P51 Agent、P52 野兽生态、P54 存档复盘逐步接入，哪些问题最容易把架构搞乱？
```

本文档的目标不是提前实现所有能力，而是提前标出风险、边界和测试门禁，避免后续扩展时出现“世界看起来能玩，但底层状态是假的”。

## 2. 总结论

P46 最大风险不是“资源生成得少”，而是生成层绕过已有权威系统。

必须长期坚持三条硬规则：

```text
1. 世界生成只产生世界状态草稿，不直接产生玩法结算结果。
2. 物品落地必须经过 P45 归属接口建立 OnMap 或 InInventory 状态。
3. 后续采集、制作、NPC 行动必须通过 Command/规则链路改变世界，不能回到临时数组操作。
```

特别注意：P46/P47 生成或采集物品时，必须通过 P45 的归属接口建立 `OnMap` 或 `InInventory` 状态，不能绕过 runtime 直接改数组。

## 3. 风险分级

| 等级 | 含义 | 处理方式 |
|---|---|---|
| P0 | 不修会破坏架构权威或导致状态不可相信 | 进入开发前必须写测试或门禁 |
| P1 | 会影响后续扩展，但短期可通过边界约束控制 | P46 完成前写入注意事项和回归测试 |
| P2 | 后续内容量上来后会暴露 | 进入后续阶段前补强 |

## 4. P0 风险

### 4.1 生成器绕过 P45 归属接口

问题：

```text
P46 为了方便，直接往 world entity 数组或 inventory entries 里塞数据。
```

后果：

```text
地面物品可能没有 item_locations_。
背包物品可能没有 InInventory 归属。
同一个 entity_id 可能同时属于地图和背包。
P47 采集、P48 制作、NPC 拾取都会基于假状态继续扩散错误。
```

门禁：

```text
生成地面物品必须通过统一 WorldGenerationApplier，再由 applier 调用 runtime/归属接口建立 OnMap。
采集产出必须进入 P47，通过 P45 接口建立 InInventory 或 OnMap。
禁止任何测试直接验证“数组里有物品”就算通过，必须验证归属状态。
```

### 4.2 把资源节点当成可拾取物品

问题：

```text
树、浆果丛、矿脉、狼穴被生成成普通可 pickup 物品。
```

后果：

```text
玩家可以直接捡起树。
NPC 无法判断这是采集目标还是物品。
采集、工具耐久、资源枯竭、再生都没有落点。
```

门禁：

```text
ResourceNodeRuntime 和 GroundItem 必须是不同类型。
资源节点必须有 required_action_key、remaining_charges、output_entity_keys。
地面物品才允许 pickup。
```

### 4.3 确定性被破坏

问题：

```text
使用 random_device、time、全局 rand、遍历 unordered_map 的非稳定顺序生成世界。
```

后果：

```text
同 seed 不能复盘。
玩家存档无法重建地图。
AI 测试同一个 case 每次结果不同。
```

门禁：

```text
所有随机只允许来自 deterministic RNG。
所有候选列表进入 RNG 前必须排序。
manifest 必须记录 seed、profile、content_version、generator_version。
```

### 4.4 生成结果没有 layer_key

问题：

```text
只用 x/y 生成，没有 surface/underground/air/water/structure 层。
```

后果：

```text
地下矿脉、空中单位、水面单位、建筑内部无法扩展。
同坐标不同层对象冲突。
```

门禁：

```text
所有 cell/entity/node/spawn/danger draft 必须携带 layer_key。
测试必须覆盖同 x/y 不同 layer 可以共存。
```

### 4.5 区域重复生成覆盖玩家改动

问题：

```text
玩家砍掉树、采集浆果后，再次 GenerateRegion 把树和浆果刷回来。
```

后果：

```text
资源无限复制。
存档状态被生成器覆盖。
玩家行为失去意义。
```

门禁：

```text
GenerateRegion 默认 idempotent：已生成区域不能重复 apply。
如果 future 支持 regenerate，必须只生成缺失的未修改层，且需要明确 force reason。
```

### 4.6 出生安全靠写死坐标

问题：

```text
if x==0 y==1 spawn red_berry。
```

后果：

```text
换 profile 就坏。
换地图尺寸就坏。
内容配置无法扩展。
```

门禁：

```text
出生保障必须通过 guaranteed_resource_tags 和 SpawnSafetyPolicy 选候选。
测试不能只测固定坐标，必须测标签保障成立。
```

## 5. P1 风险

### 5.1 区域边界出现断崖

问题：相邻 region 独立生成，边界地形、河流、道路、生态带断裂。

处理：P46 最小版可以只做单区域，但 DTO 必须保留 region_coord 和 neighbor sampling 位置；测试至少确认相邻 region stable id 不冲突。

### 5.2 内容配置引用失效

问题：worldgen JSON 引用了不存在的 terrain/resource/entity/action/tool key。

处理：加载阶段必须有 content validation。P46 不能静默跳过错误配置。

### 5.3 密度失控导致世界不可玩

问题：资源太少导致无法生存，或资源太多导致探索无意义。

处理：测试同时覆盖 minimum guarantee 和 max density cap。

### 5.4 危险分布没有生态语义

问题：野兽、毒雾、巢穴随机贴脸生成。

处理：P46 只生成 danger zone / lair marker，不生成主动攻击；危险必须受 safe_radius、terrain、biome tag 限制。

### 5.5 资源耗尽和再生没有承接点

问题：P47 采集后节点剩余次数怎么变、是否再生没有字段。

处理：P46 ResourceNodeRuntime 必须预留 remaining_charges、max_charges、regen_policy_key、depleted_state_key。

### 5.6 Stable id 冲突

问题：不同 region/layer/profile 生成同一个 entity_id。

处理：stable id 必须包含 world_id、region_coord、layer_key、object_kind、local_index、content key。

### 5.7 投影缺少生成事件

问题：前端只看到地图变了，不知道为什么变。

处理：GenerateWorld/GenerateRegion 必须返回 events、trace、changed_cells、changed_entities。

### 5.8 JSON 直接驱动前端

问题：前端为了快，直接读 worldgen 配置画资源。

处理：前端只能读 projection，不能读 content JSON 推导真实世界状态。

### 5.9 NPC 出生点和玩家出生点混淆

问题：生成 NPC spawn marker 后，误以为已经生成了 NPC 行动主体。

处理：P46 可以生成 spawn point，但真正 Agent/NPC 创建和行为由后续阶段接入。

### 5.10 采集产出缺少归属策略

问题：树砍出木头时，到底进背包、掉地上、进营地仓库不清楚。

处理：P47 必须显式指定 output_location_policy，并通过 P45 建立 `OnMap` 或 `InInventory`。

## 6. P2 风险

### 6.1 大世界流式生成性能

后续问题：玩家移动到新区域时生成卡顿。

预留：P46 不做异步，但 manifest 和 region cache 接口要可扩展。

### 6.2 多生态层级

后续问题：高级文明、魔法区域、机械危机需要完全不同生成逻辑。

预留：worldgen_profile 不能只服务森林开局，必须支持 profile composition 和 tags。

### 6.3 事件型生成

后续问题：流星坠落、魔法污染、虫潮迁徙会改变地图。

预留：这类不是 P46 初始生成，而是后续 WorldEvent/Command 产生的 world mutation。

### 6.4 RL/复盘训练数据

后续问题：AI 训练需要知道世界从何而来。

预留：manifest/trace 必须可序列化，不能只在日志里打印。

### 6.5 地图美术与规则对象分离

后续问题：一个大树贴图可能代表阻挡、木材资源、鸟巢、遮蔽等多个规则对象。

预留：visual_key 不能等于 entity_key/resource_key，前端表现不能反推规则。

## 7. 扩展场景压力想象

### 7.1 星露谷式二维生活地图

需要：田地、树、石头、水边、建筑入口、NPC 路径。  
P46 能支撑：地形、资源节点、出生点、区域标签。  
后续补强：季节、作物生长、建筑内部层。

### 7.2 我的世界式材料探索

需要：木、石、矿、洞穴、地下层。  
P46 能支撑：surface/underground layer、资源节点、stable region。  
后续补强：方块破坏、堆叠地形、多层生成。

### 7.3 荒野生存

需要：食物、水源、火源、寒冷、野兽。  
P46 能支撑：安全出生区、危险源、基础资源标签。  
后续补强：天气、温度、动物行动。

### 7.4 魔法世界

需要：魔力区域、元素矿、咒语遗迹。  
P46 能支撑：danger_zone/resource_node/tag。  
后续补强：区域魔法事件、表达式条件。

### 7.5 机械危机

需要：废墟、机械残骸、巡逻机器人。  
P46 能支撑：结构标记、资源节点、spawn point。  
后续补强：机械 Agent 行为、维修/拆解规则。

### 7.6 高级文明城市

需要：道路、建筑、容器、交易点、权限区。  
P46 能支撑：区域标签、spawn point、structure marker。  
后续补强：结构内部、社会权限、NPC 日程。

### 7.7 部落扩张

需要：营地、仓库、采集区、危险边界。  
P46 能支撑：初始区域和资源带。  
后续补强：营地归属、共享库存、自动任务。

### 7.8 地下洞穴

需要：地下层、矿脉、暗区、洞穴入口。  
P46 能支撑：layer_key 和 cave marker。  
后续补强：入口连接、光照、坍塌。

### 7.9 空中单位

需要：飞行层、空中危险、地面不可达目标。  
P46 能支撑：air layer。  
后续补强：移动规则和攻击范围。

### 7.10 灾变事件

需要：火灾、洪水、污染扩散。  
P46 能支撑：初始 danger zone。  
后续补强：事件传播系统，不能由 P46 初始生成硬做。

## 8. 进入开发前必须回答的问题

```text
1. 哪些对象是 ResourceNode，哪些是 GroundItem？
2. GroundItem 通过哪个接口建立 OnMap 归属？
3. GenerateWorld 重复执行时是否拒绝、幂等返回还是重建？
4. 同 seed 同 profile 的 snapshot 如何比较？
5. content key 缺失时是失败还是降级？
6. 出生区保障是标签驱动还是写死物品？
7. 每个生成 draft 是否都有 layer_key？
8. P46 是否完全不创建玩家背包物品？
9. P47 采集产出如何通过 P45 建立归属？
10. 前端是否只能通过 projection 看到结果？
```

## 9. 给后续阶段的结论

P46 可以先小步实现，但不能偷懒绕过权威边界。

真正允许简化的是：

```text
地形种类少一点。
资源节点少一点。
危险源只是 marker。
区域只生成出生周边。
```

绝对不能简化的是：

```text
确定性。
layer_key。
P45 归属。
ResourceNode 和 GroundItem 分离。
manifest/trace。
Command/projection 接入边界。
```
