# P46 世界生成器测试文档

## 1. 文档目的

本文档定义 P46 的测试矩阵、测试命名、验收门禁和后续 P47 承接测试要求。

P46 不是简单测试“生成了几个对象”，而是要证明：

```text
同 seed 可复盘。
对象类型正确。
地形和资源分布符合配置。
生成结果不会绕过 P45 归属系统。
Command/projection 能把生成变化交给前端。
```

特别门禁：P46/P47 生成或采集物品时，必须通过 P45 的归属接口建立 `OnMap` 或 `InInventory` 状态，不能绕过 runtime 直接改数组。

## 2. 测试分层

| 层级 | 目标 | 示例 |
|---|---|---|
| 单元测试 | 验证生成算法、DTO、stable id、配置校验 | world_generation_same_seed_same_region_same_result |
| 集成测试 | 验证 GenerateWorld/GenerateRegion 通过 command 接入 runtime | world_command_generate_world_uses_world_generation_service |
| 架构扫描 | 验证没有绕过 P45/Command 边界 | architecture_world_generation_no_direct_inventory_mutation |
| 压力测试 | 验证多 seed、多 region、多 layer 稳定 | world_generation_many_regions_no_id_collision |
| 回归测试 | 验证 P44/P45/P46 组合不破坏 | world_generation_ground_items_are_on_map_only |

## 3. 测试命名规范

建议统一使用：

```text
world_generation_<能力>_<预期>
world_command_generate_<能力>_<预期>
architecture_world_generation_<边界>_<预期>
```

不要使用模糊命名：

```text
test_generator_ok
test_world_basic
test_spawn_item
```

## 4. 确定性测试

### 4.1 same seed same result

测试名：

```text
world_generation_same_seed_same_region_same_result
```

断言：

```text
同 world_seed、profile、content_version、generator_version、region_coord。
生成 cell_drafts 完全一致。
生成 resource_node_drafts 完全一致。
生成 ground item drafts 完全一致。
stable ids 完全一致。
manifest 完全一致。
```

### 4.2 different seed different distribution

测试名：

```text
world_generation_different_seed_different_distribution
```

断言：

```text
不同 seed 可以有不同资源分布。
不同不是必须每个对象都不同，但至少 terrain/resource snapshot hash 不应完全相同。
出生安全保障仍然成立。
```

### 4.3 manifest contains required inputs

测试名：

```text
world_generation_manifest_contains_seed_profile_version
```

断言：

```text
manifest.world_id 非空。
manifest.world_seed 等于 request。
manifest.generator_version 非空。
manifest.content_version 非空。
manifest.worldgen_profile_key 非空。
manifest.region_coord 正确。
manifest.region_size 正确。
```

## 5. 配置校验测试

### 5.1 profile missing fails

测试名：

```text
world_generation_profile_missing_fails
```

断言：

```text
未知 worldgen_profile_key 返回 ProfileNotFound。
不能 fallback 到默认 profile。
不能生成半成品世界。
```

### 5.2 referenced content missing fails

测试名：

```text
world_generation_referenced_content_missing_fails
```

断言：

```text
resource_rule 引用不存在 terrain_tag/action/tool/output_entity_key 时失败。
失败结果包含 content key 和引用来源。
```

### 5.3 invalid density rejected

测试名：

```text
world_generation_invalid_density_rejected
```

断言：

```text
density < 0 失败。
density > 1 或超过配置上限失败。
min_distance > max_distance 失败。
```

## 6. 地形与坐标测试

### 6.1 cells have layer key

测试名：

```text
world_generation_creates_cells_with_layer_key
```

断言：

```text
所有 cell draft 都有 layer_key。
所有 entity/resource/spawn/danger draft 都有 coord.layer_key。
```

### 6.2 valid terrain only

测试名：

```text
world_generation_cells_have_valid_terrain
```

断言：

```text
terrain_key 必须来自配置。
terrain tags 必须来自内容表。
不存在空 terrain_key。
```

### 6.3 spawn cell walkable

测试名：

```text
world_generation_spawn_area_not_blocked
```

断言：

```text
玩家出生 cell 可通行。
出生安全半径内不出现 blocked 全包围。
至少有一条可移动邻接路径。
```

### 6.4 same x/y different layer coexist

测试名：

```text
world_generation_same_xy_different_layer_coexist
```

断言：

```text
surface:0:0 和 underground:0:0 id 不同。
同 x/y 不同 layer 的 cell 不互相覆盖。
```

## 7. 资源节点测试

### 7.1 allowed terrain only

测试名：

```text
world_generation_places_resource_nodes_on_allowed_terrain
```

断言：

```text
berry_bush 不生成在 water 或 blocked。
ore_node 不生成在 plain，除非配置允许。
每个 node 的 terrain tag 与 resource rule 匹配。
```

### 7.2 node has action and outputs

测试名：

```text
world_generation_resource_node_has_action_and_output_keys
```

断言：

```text
required_action_key 非空。
output_entity_keys 非空。
remaining_charges > 0。
max_charges >= remaining_charges。
```

### 7.3 resource node is not pickup item

测试名：

```text
world_generation_resource_node_not_ground_item
```

断言：

```text
ResourceNodeDraft 不出现在 ground item/entity item 列表里。
ResourceNodeDraft 不设置 InventoryLocationKind。
ResourceNodeDraft 不允许 pickup command 直接拾取。
```

## 8. 出生安全测试

### 8.1 basic food guarantee

测试名：

```text
world_generation_spawn_safety_has_basic_food
```

断言：

```text
safe_radius 或 configured_radius 内至少有 food_basic resource tag。
不是固定 red_berry，而是按 tag 满足。
```

### 8.2 basic material guarantee

测试名：

```text
world_generation_spawn_safety_has_basic_materials
```

断言：

```text
出生附近至少有 wood_basic。
出生附近至少有 stone_basic。
如果 profile 要求 fire_basic，也必须满足。
```

### 8.3 no immediate active threat

测试名：

```text
world_generation_spawn_safety_blocks_immediate_active_threat
```

断言：

```text
safe_radius 内不能有 active_predator。
可以有 tracks/hint/danger_marker，但不能是立即攻击源。
```

## 9. P45 归属测试

### 9.1 generated ground item is OnMap only

测试名：

```text
world_generation_ground_items_are_on_map_only
```

断言：

```text
所有 P46 生成的地面物品都是 WorldEntityLocationKind::OnMap。
对应 inventory location 是 InventoryLocationKind::OnMap 或可由 P45 查询到 OnMap。
不在任何 actor inventory entries 中。
```

### 9.2 generated item has stack data

测试名：

```text
world_generation_ground_items_have_quantity_stack_key
```

断言：

```text
quantity > 0。
stack_key 非空或可由 entity_key/state 生成。
state_keys/numeric_states 参与 stack_key。
```

### 9.3 no initial inventory injection

测试名：

```text
world_generation_does_not_create_player_inventory_items
```

断言：

```text
GenerateWorld 不给玩家背包塞物品。
玩家背包即使存在，也不会因为 P46 生成而新增 entries。
```

### 9.4 no direct runtime array mutation

测试名：

```text
architecture_world_generation_no_direct_inventory_mutation
```

断言：

```text
world_generation 模块不能直接操作 InventoryRuntimeState.entries。
不能直接写 item_locations_。
需要通过 P45 暴露的接口或 P46 applier 统一门禁。
```

说明：该测试可以先用 rg/架构扫描实现，后续再升级成编译期边界测试。

## 10. Command 集成测试

### 10.1 generate world command uses service

测试名：

```text
world_command_generate_world_uses_world_generation_service
```

断言：

```text
GenerateWorld command 调用 WorldGenerationService。
返回 CommandResult.ok。
返回 events。
返回 changed_cells 或 projection patch。
trace 包含 generation manifest key。
```

### 10.2 repeated region generation blocked

测试名：

```text
world_command_generate_region_repeated_is_idempotent_or_blocked
```

断言：

```text
同 region 第二次生成不能覆盖玩家改动。
P46 默认建议返回 RegionAlreadyGenerated。
如果实现 idempotent success，也必须 changed_cells 为空且不重复生成对象。
```

### 10.3 projection patch safe for frontend

测试名：

```text
world_command_generate_region_returns_projection_patch
```

断言：

```text
前端可通过 projection 看到地图变化。
projection 不暴露 worldgen JSON 原始配置。
projection 不要求前端推导隐藏资源。
```

## 11. 压力测试

### 11.1 many seeds stable

测试名：

```text
world_generation_many_seeds_spawn_safety_stable
```

断言：

```text
至少 50 个 seed。
每个 seed 出生保障成立。
没有崩溃。
没有空 region。
```

### 11.2 many regions no id collision

测试名：

```text
world_generation_many_regions_no_id_collision
```

断言：

```text
生成 5x5 regions。
所有 cell_id/entity_id/node_id 不冲突。
同 world 不同 layer 不冲突。
```

### 11.3 deterministic snapshot hash

测试名：

```text
world_generation_snapshot_hash_stable
```

断言：

```text
排序后的 snapshot hash 在同输入下稳定。
hash 不依赖容器遍历顺序。
```

## 12. 禁止通过的假测试

以下测试不算有效：

```text
只检查生成数量 > 0。
只检查某个固定坐标有红果。
只检查 vector size，不检查 layer_key。
只检查背包有东西，不检查 P45 归属接口。
只检查 command ok，不检查 projection/trace/events。
只跑一个 seed。
```

## 14. P47 承接测试预留

P46 完成后，P47 开发前必须补以下承接测试：

```text
world_gather_resource_node_consumes_charge
world_gather_output_item_created_through_p45_inventory_interface
world_gather_output_item_created_on_map_when_inventory_full
world_gather_cannot_gather_ground_item_as_resource_node
world_gather_repeated_after_depleted_fails
```

这组测试的核心不是 P46 实现采集，而是确保 P46 生成出来的 ResourceNode 能被 P47 正确消费，并且采集产物继续遵守 P45 归属规则。
