# P54 前端开发交付说明

这份文档给前端同学使用。前端不需要了解后端实现，只需要按本文实现静态文件，完成后复制到指定目录即可运行。

## 1. 交付目标

做一个玩家视角的地图客户端，用来消费 P53 客户端协议：

```text
bootstrap -> full_projection + available_commands
command(option_id) -> result + projection_patch + available_commands
refresh -> full_projection
```

前端只负责展示和提交玩家选择，不实现任何玩法规则。

## 2. 交付目录

最终只交付这 3 个文件：

```text
frontend/h5_playable/index.html
frontend/h5_playable/style.css
frontend/h5_playable/app.js
```

要求：

```text
1. 不使用构建工具。
2. 不使用 npm / vite / webpack。
3. 不使用 ES module import。
4. 不依赖外部 CDN。
5. 不新增图片、字体、音频等额外资源；如必须使用图标，用 emoji / CSS / inline SVG。
6. 所有逻辑写在 app.js，所有样式写在 style.css。
```

原因：当前静态服务器只保证服务：

```text
/
/style.css
/app.js
```

## 3. 本地运行方式

前端完成后，把文件复制到：

```text
frontend/h5_playable/
```

然后运行：

```bash
./scripts/restart_h5_1999.sh
```

打开：

```text
http://127.0.0.1:1999/
```

如果手机通过内网 / frp 访问，以后端同学给的实际地址为准。

## 4. 页面必须包含的区域

建议横屏优先，但竖屏也不能崩。

页面分 5 个区域：

```text
1. 顶部状态栏
2. 中央地图区
3. 右侧操作区
4. 下方事件日志
5. 调试信息折叠区
```

### 4.0 推荐整体布局

整体布局可以借鉴“全屏 Canvas 背景 + 浮层 HUD”的方式：地图始终铺满屏幕，状态、任务、操作、日志以半透明卡片浮在上面。不要把地图压缩成小区域。

推荐 DOM 结构：

```text
<body>
  <div id="app" class="game-shell">
    <canvas id="gameCanvas"></canvas>

    <header class="top-hud">
      <section class="status-panel"></section>
      <section class="quest-panel"></section>
    </header>

    <aside class="command-panel"></aside>
    <section class="event-log"></section>
    <details class="debug-panel"></details>

    <div id="toast-container"></div>
    <canvas id="minimapCanvas"></canvas>
  </div>
</body>
```

布局要求：

```text
1. game-shell 固定占满 100vw x 100vh，overflow hidden。
2. gameCanvas 绝对定位铺满全屏，作为最底层。
3. top-hud 放在左上到右上，使用 pointer-events: none；内部卡片再打开 pointer-events。
4. 左上 status-panel 显示玩家/同步状态，右上 quest-panel 显示任务或目标摘要。
5. 右侧 command-panel 显示 available_commands，宽度建议 260-340px。
6. 底部 event-log 显示最近事件，建议半透明悬浮，不要遮住主要视野。
7. 右下角 minimapCanvas 显示小地图，移动端可隐藏。
8. toast-container 居中靠下，用于成功、失败、警告等短提示。
```

视觉风格建议：

```text
1. 面板使用深色半透明背景，如 rgba(15, 23, 42, 0.82)。
2. 使用 border、box-shadow、backdrop-filter: blur() 做玻璃质感。
3. 按钮使用大圆角、明显 hover / active / disabled 状态。
4. HP/XP/同步进度可以用 CSS 渐变条表现。
5. 字体优先系统字体，不依赖 Google Fonts；像素标题可用 monospace + letter-spacing 模拟。
6. 颜色优先暗底高对比：黄=重要，红=危险，蓝/青=同步，绿=成功，灰=禁用。
```

移动端要求：

```text
1. 宽度小于 768px 时，右侧 command-panel 改为底部抽屉或底部横向按钮栏。
2. quest-panel 和 debug-panel 默认隐藏或折叠。
3. event-log 只显示最近 2-3 条。
4. 可预留左下虚拟摇杆、右下主操作按钮的位置，但只能作为提交 available_commands 的 UI，不做本地移动/攻击规则。
5. 所有按钮触控尺寸不小于 44px。
```

严禁：

```text
1. 不使用 Tailwind CDN。
2. 不引入 Google Fonts。
3. 不为了布局引入第三方 UI 库。
4. 不把 HUD 数值作为真实玩法状态自行结算；只能展示 projection / event / hint 给出的安全字段。
```

### 4.1 顶部状态栏

显示：

```text
session_id
projection_version
actor_key
active_layer_key
同步状态：同步中 / 已同步 / 需要刷新 / 出错
```

### 4.2 中央地图区

从 `full_projection.visible_cells` 和 `full_projection.visible_entities` 渲染。

最低要求：

```text
1. visible_cells 渲染成网格。
2. visible_entities 渲染到对应格子或列表。
3. 不认识的字段也不能报错。
4. 空 projection 时显示“世界投影为空 / 等待后端数据”。
```

不要根据 `entity_key` / `object_key` 自己生成按钮。

### 4.2.1 Canvas 像素渲染要求

中央地图区建议使用 `HTML5 Canvas` 渲染，不要用大量 DOM div 拼地图。

目标风格：精美像素风 / 俯视角 RPG 地图。可以借鉴以下做法，但只用于渲染表现，不用于实现玩法规则。

基础要求：

```text
1. 使用 <canvas id="gameCanvas"> 作为地图主渲染层。
2. CSS 设置 image-rendering: pixelated。
3. JS 设置 ctx.imageSmoothingEnabled = false。
4. visible_cells 作为 tile 数据源，渲染成固定尺寸网格。
5. visible_entities 作为实体数据源，绘制到对应世界坐标或格子中心。
6. 不使用外部图片素材，不依赖 CDN，不引入外部字体。
```

推荐渲染技术：

```text
1. 地图块使用 ctx.fillRect() 绘制基础色块。
2. 草地、沙地、水面等 tile 可额外画少量像素点缀。
3. 小人、怪物、树、石头、金币等实体使用 fillRect / arc / ellipse 拼出像素风外观。
4. 实体脚下绘制半透明 ellipse 阴影，增强落地感。
5. 实体按照 y 坐标排序后绘制，形成前后遮挡效果。
6. 选中、可交互、受击、成功反馈可以用粒子、飘字、描边或轻微闪烁表现。
7. 相机可以跟随 actor_key 对应实体，使用轻微缓动提升观感。
8. 小地图可用第二个 canvas 绘制简化版 visible_cells。
```

地图块建议：

```text
1. 每个 cell 映射为一个 tile，建议 TILE_SIZE 32 或 48。
2. 根据后端给出的 terrain_key / safe_summary_keys / display 字段选择颜色。
3. 未识别地形使用默认暗色格，不要报错。
4. 水面可以画浅色短线，草地可以画 2-3 个深浅色小点，沙地可以画浅棕色小方块。
5. tile 之间避免明显缝隙，fillRect 宽高可以多加 0.5 像素。
```

小人和实体建议：

```text
1. 玩家小人用矩形拼身体、头部、眼睛、披风、手持物。
2. 眼睛可根据朝向偏移 1-3 像素，表现面朝方向。
3. 行走可以用 Math.sin(Date.now() * speed) 做脚步上下摆动。
4. 攻击、采集等表现只能来自后端返回的 event_feed / frontend_hints，不能本地判定结果。
5. 怪物或 NPC 可用 ellipse / arc 拼身体，并加阴影和轻微呼吸缩放。
6. 金币可用 scale + arc 做旋转感；树木可用树干矩形 + 多个圆形树冠。
```

可以做的表现层动画：

```text
1. projection 更新后，对变化的 cell/entity 做短暂高亮。
2. event_feed 有成功事件时，显示飘字或 toast。
3. warning_keys / failure_reason_keys 显示红色提示或轻微抖动。
4. command 提交中让按钮进入 loading 状态。
5. 网络错误显示可读错误，不阻塞页面。
```

严禁把示例里的本地游戏逻辑搬进来：

```text
1. 不在前端生成无限世界。
2. 不在前端生成怪物、金币、树、石头。
3. 不在前端做碰撞、攻击、掉落、升级、AI、血量结算。
4. 不根据 entity_key 自己决定是否可采集 / 可攻击 / 可移动。
5. 不根据动画结果修改 projection。
```

一句话：渲染可以精美，规则必须全由后端 projection / patch / available_commands 驱动。

### 4.3 右侧操作区

只显示后端返回的：

```text
available_commands
```

每个 command 渲染为按钮：

```text
label_text
```

按钮行为：

```text
enabled=true：可点击，提交 option_id。
enabled=false：禁用，显示 disabled_reason_text。
```

禁止：

```text
1. 前端自己根据地图对象生成“采集 / 攻击 / 制作 / 移动”按钮。
2. 前端提交 WorldCommandDto。
3. 前端提交 parameters。
4. 前端修改背包、位置、知识或实体状态。
```

### 4.4 下方事件日志

显示：

```text
event_feed
frontend_hints
result.failure_reason_keys
warning_keys
```

只做解释，不做规则判断。

错误示例：

```text
看到 event 写“采集成功”，前端自己给背包 +1。
```

这是禁止的。

### 4.5 调试信息折叠区

默认折叠。用于展示原始 JSON：

```text
last bootstrap response
last command response
last refresh response
```

用于联调，不要作为玩家主界面。

## 5. 前端状态模型

`app.js` 内建议只维护这些状态：

```js
const state = {
  clientId: '',
  sessionId: '',
  clientSequence: 0,
  projectionVersion: 0,
  projection: null,
  availableCommands: [],
  eventFeed: [],
  frontendHints: [],
  warningKeys: [],
  syncState: 'bootstrapping',
  lastResponse: null,
};
```

### 5.1 client_id

首次打开页面生成并保存：

```js
localStorage.getItem('pathfinder_client_id')
```

没有就生成：

```text
web_<随机字符串>
```

### 5.2 session_id

首次打开页面生成并保存：

```js
localStorage.getItem('pathfinder_session_id')
```

没有就生成：

```text
session_<随机字符串>
```

页面提供“重开世界”按钮时，可以清除 session 或调用 reset。

## 6. HTTP API 约定

前端只调用这些接口。

```text
POST /api/client/bootstrap
POST /api/client/command
POST /api/client/refresh
POST /api/client/reset
```

请求头：

```http
Content-Type: application/json
Accept: application/json
```

注意：如果当前后端还没接好 `/api/client/*`，前端应提供 Mock 模式，见第 11 节。这样复制到目录后页面仍能打开和演示。

## 7. Bootstrap

页面加载后立即调用。

### 7.1 Request

```json
{
  "client_id": "web_xxx",
  "session_id": "session_xxx",
  "requested_actor_key": "player",
  "requested_layer_key": "surface",
  "client_protocol_version": 1,
  "create_if_missing": true,
  "dev_reset_if_allowed": false
}
```

### 7.2 Response

```json
{
  "session_id": "session_xxx",
  "server_protocol_version": 1,
  "projection_version": 1,
  "full_projection": {
    "projection_version": 1,
    "actor_key": "player",
    "active_layer_key": "surface",
    "visible_cells": [],
    "visible_entities": [],
    "inventories": [],
    "knowledge": [],
    "area_effects": [],
    "safe_summary_keys": []
  },
  "available_commands": [],
  "event_feed": [],
  "frontend_hints": [],
  "warning_keys": []
}
```

### 7.3 前端处理

```text
1. 保存 session_id。
2. 保存 projection_version。
3. 用 full_projection 整体替换本地 projection。
4. 用 available_commands 整体替换按钮列表。
5. 渲染地图和按钮。
```

## 8. Command

用户点击 `available_commands` 中的按钮时调用。

前端只提交 `option_id`，不要提交 `WorldCommandDto`。

### 8.1 Request

```json
{
  "session_id": "session_xxx",
  "client_id": "web_xxx",
  "client_sequence": 1,
  "known_projection_version": 1,
  "submit_mode": "option_id",
  "option_id": "opt_wait_player",
  "selection_context": {
    "target": {
      "target_kind": "none",
      "target_coord": null,
      "target_entity_id": "",
      "target_actor_key": "",
      "target_item_key": "",
      "target_inventory_id": "",
      "target_area_id": "",
      "knowledge_key": "",
      "recipe_key": ""
    },
    "selected_actor_key": "",
    "selected_entity_id": "",
    "selected_inventory_id": "",
    "selected_area_id": ""
  }
}
```

### 8.2 Response

```json
{
  "session_id": "session_xxx",
  "accepted_client_sequence": 1,
  "base_projection_version": 1,
  "new_projection_version": 2,
  "requires_full_refresh": false,
  "result": {
    "command_id": "cmd_xxx",
    "command_key": "wait",
    "result_kind": "succeeded",
    "actor_key": "player",
    "target_ref": "",
    "failure_reason_keys": [],
    "warning_keys": []
  },
  "projection_patch": {
    "base_projection_version": 1,
    "new_projection_version": 2,
    "requires_full_refresh": false,
    "changed_cells": [],
    "changed_entities": [],
    "changed_inventories": [],
    "changed_knowledge": [],
    "changed_area_effects": []
  },
  "available_commands": [],
  "event_feed": [],
  "experiences": [],
  "frontend_hints": [],
  "warning_keys": []
}
```

### 8.3 前端处理

收到 response 后：

```text
1. 如果 requires_full_refresh=true：不要应用 patch，直接调用 refresh。
2. 如果 projection_patch.base_projection_version !== state.projectionVersion：不要应用 patch，直接调用 refresh。
3. 否则应用 projection_patch。
4. state.projectionVersion = new_projection_version。
5. 用 response.available_commands 整体替换按钮列表。
6. 追加 event_feed / frontend_hints / warning_keys 到日志。
```

## 9. Refresh

用于全量刷新。

触发时机：

```text
1. command response.requires_full_refresh=true。
2. patch base version 与本地 projectionVersion 不一致。
3. 用户点击“刷新”。
4. 网络错误后重试。
```

### 9.1 Request

```json
{
  "session_id": "session_xxx",
  "client_id": "web_xxx",
  "known_projection_version": 1,
  "requested_scopes": ["full_safe_world"],

  "viewport_center_x": 0,
  "viewport_center_y": 0
}
```

### 9.2 Response

```json
{
  "session_id": "session_xxx",
  "projection_version": 2,
  "full_projection": {
    "projection_version": 2,
    "actor_key": "player",
    "active_layer_key": "surface",
    "visible_cells": [],
    "visible_entities": [],
    "inventories": [],
    "knowledge": [],
    "area_effects": [],
    "safe_summary_keys": []
  },
  "available_commands": [],
  "event_feed": [],
  "frontend_hints": [],

```

### 9.3 前端处理


## 11. Mock 模式要求

为了让前端开发完“复制进目录就能打开”，必须内置 Mock 模式。

触发条件：

```text
1. URL 带 ?mock=1。
2. /api/client/bootstrap 返回 404 / network error。
```

Mock 模式至少返回：

```text
1. 5x5 网格。
2. 一个 player entity。
3. 两个按钮：等待、观察。
4. 点击按钮后 projection_version +1，并追加一条日志。
```

Mock 模式只用于前端视觉和交互自测，不代表真实规则。

页面上要明显显示：

```text
MOCK MODE
```

## 12. Patch 应用规则

前端只做“显示层合并”。

### 12.1 changed_cells

根据 `cell_id` 合并：

```text
op=add/update/replace：写入或替换。
op=remove：删除。
op=clear：清空 cells。
```

### 12.2 changed_entities

根据 `entity_id` 合并。

### 12.3 inventories

根据 `inventory_id` 合并。

### 12.4 knowledge

根据 `actor_key` 合并。

### 12.5 area_effects

根据 `area_effect_id` 合并。

如果不确定怎么合并，直接调用 refresh，不要猜。

## 13. 字段容错要求

前端必须容错：

```text
1. 数组字段缺失时按 [] 处理。
2. 字符串字段缺失时按 "" 处理。
3. 布尔字段缺失时按 false 处理。
4. 数字字段缺失时按 0 处理。
5. 遇到未知字段不要报错。
6. 遇到未知 command_kind / event_kind 仍要能显示。
```

## 14. 严禁事项

前端绝对不要做这些事：

```text
1. 不读 content/core JSON。
2. 不根据 entity_key / object_key / terrain_key 生成玩法按钮。
3. 不提交 WorldCommandDto。
4. 不提交 command.parameters。
5. 不修改 actor 位置。
6. 不修改背包数量。
7. 不修改知识列表。
8. 不根据 event / hint 反推规则。
9. 不写死 red_berry / torch / fire / wolf / wood / companion 等内容 key。
10. 不把 debug JSON 作为玩家主界面。
```

只允许：

```text
点击后端给的 available_commands.option_id。
```

## 15. UI 验收标准

交付时至少满足：

```text
1. 页面可打开，无控制台报错。
2. bootstrap 成功或自动进入 Mock 模式。
3. 地图区能显示 projection。
4. 操作区只显示 available_commands。
5. 点击 enabled command 会提交 option_id。
6. requires_full_refresh 时会自动 refresh。
7. event_feed / warning_keys 能显示到日志。
8. 重开按钮可调用 reset 或清 session 后 bootstrap。
9. 手机横屏可用。
10. 断网 / 接口 500 时有可读错误提示。
```

## 17. API adapter 推荐伪代码

```js
async function postJson(path, body) {
  const response = await fetch(path, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Accept': 'application/json',
    },
    body: JSON.stringify(body),
  });
  if (!response.ok) {
    throw new Error(`${path} failed: ${response.status}`);
  }
  return response.json();
}

async function bootstrap() {
  return postJson('/api/client/bootstrap', {
    client_id: state.clientId,
    session_id: state.sessionId,
    requested_actor_key: 'player',
    requested_layer_key: 'surface',
    client_protocol_version: 1,
    create_if_missing: true,
    dev_reset_if_allowed: false,
  });
}

async function submitOption(optionId) {
  state.clientSequence += 1;
  return postJson('/api/client/command', {
    session_id: state.sessionId,
    client_id: state.clientId,
    client_sequence: state.clientSequence,
    known_projection_version: state.projectionVersion,

  });
}
```
