# P54 前端开发交付说明

这份文档给前端同学使用。前端不需要了解后端实现，只需要按本文实现静态文件，完成后复制到指定目录即可运行。

## 1. 交付目标

做一个玩家视角的地图客户端，用来消费 P53 客户端协议：

```text
bootstrap -> full_projection + available_commands
command(option_id) -> result + projection_patch + available_commands
refresh -> full_projection
```

前端只负责展示和提交玩家选择，不实现任何玩法规则。

## 2. 交付目录

最终只交付这 3 个文件：

```text
frontend/h5_playable/index.html
frontend/h5_playable/style.css
frontend/h5_playable/app.js
```

要求：

```text
1. 不使用构建工具。
2. 不使用 npm / vite / webpack。
3. 不使用 ES module import。
4. 不依赖外部 CDN。
5. 不新增图片、字体、音频等额外资源；如必须使用图标，用 emoji / CSS / inline SVG。
6. 所有逻辑写在 app.js，所有样式写在 style.css。
```

原因：当前静态服务器只保证服务：

```text
/
/style.css
/app.js
```

## 3. 本地运行方式

前端完成后，把文件复制到：

```text
frontend/h5_playable/
```

然后运行：

```bash
./scripts/restart_h5_1999.sh
```

打开：

```text
http://127.0.0.1:1999/
```

如果手机通过内网 / frp 访问，以后端同学给的实际地址为准。

## 4. 页面必须包含的区域

建议横屏优先，但竖屏也不能崩。

页面分 5 个区域：

```text
1. 顶部状态栏
2. 中央地图区
3. 右侧操作区
4. 下方事件日志
5. 调试信息折叠区
```

### 4.1 顶部状态栏

显示：

```text
session_id
projection_version
actor_key
active_layer_key
同步状态：同步中 / 已同步 / 需要刷新 / 出错
```

### 4.2 中央地图区

从 `full_projection.visible_cells` 和 `full_projection.visible_entities` 渲染。

最低要求：

```text
1. visible_cells 渲染成网格。
2. visible_entities 渲染到对应格子或列表。
3. 不认识的字段也不能报错。
4. 空 projection 时显示“世界投影为空 / 等待后端数据”。
```

不要根据 `entity_key` / `object_key` 自己生成按钮。

### 4.3 右侧操作区

只显示后端返回的：

```text
available_commands
```

每个 command 渲染为按钮：

```text
label_text
```

按钮行为：

```text
enabled=true：可点击，提交 option_id。
enabled=false：禁用，显示 disabled_reason_text。
```

禁止：

```text
1. 前端自己根据地图对象生成“采集 / 攻击 / 制作 / 移动”按钮。
2. 前端提交 WorldCommandDto。
3. 前端提交 parameters。
4. 前端修改背包、位置、知识或实体状态。
```

### 4.4 下方事件日志

显示：

```text
event_feed
frontend_hints
result.failure_reason_keys
warning_keys
```

只做解释，不做规则判断。

错误示例：

```text
看到 event 写“采集成功”，前端自己给背包 +1。
```

这是禁止的。

### 4.5 调试信息折叠区

默认折叠。用于展示原始 JSON：

```text
last bootstrap response
last command response
last refresh response
```

用于联调，不要作为玩家主界面。

## 5. 前端状态模型

`app.js` 内建议只维护这些状态：

```js
const state = {
  clientId: '',
  sessionId: '',
  clientSequence: 0,
  projectionVersion: 0,
  projection: null,
  availableCommands: [],
  eventFeed: [],
  frontendHints: [],
  warningKeys: [],
  syncState: 'bootstrapping',
  lastResponse: null,
};
```

### 5.1 client_id

首次打开页面生成并保存：

```js
localStorage.getItem('pathfinder_client_id')
```

没有就生成：

```text
web_<随机字符串>
```

### 5.2 session_id

首次打开页面生成并保存：

```js
localStorage.getItem('pathfinder_session_id')
```

没有就生成：

```text
session_<随机字符串>
```

页面提供“重开世界”按钮时，可以清除 session 或调用 reset。

## 6. HTTP API 约定

前端只调用这些接口。

```text
POST /api/client/bootstrap
POST /api/client/command
POST /api/client/refresh
POST /api/client/reset
```

请求头：

```http
Content-Type: application/json
Accept: application/json
```

注意：如果当前后端还没接好 `/api/client/*`，前端应提供 Mock 模式，见第 11 节。这样复制到目录后页面仍能打开和演示。

## 7. Bootstrap

页面加载后立即调用。

### 7.1 Request

```json
{
  "client_id": "web_xxx",
  "session_id": "session_xxx",
  "requested_actor_key": "player",
  "requested_layer_key": "surface",
  "client_protocol_version": 1,
  "create_if_missing": true,
  "dev_reset_if_allowed": false
}
```

### 7.2 Response

```json
{
  "session_id": "session_xxx",
  "server_protocol_version": 1,
  "projection_version": 1,
  "full_projection": {
    "projection_version": 1,
    "actor_key": "player",
    "active_layer_key": "surface",
    "visible_cells": [],
    "visible_entities": [],
    "inventories": [],
    "knowledge": [],
    "area_effects": [],
    "safe_summary_keys": []
  },
  "available_commands": [],
  "event_feed": [],
  "frontend_hints": [],
  "warning_keys": []
}
```

### 7.3 前端处理

```text
1. 保存 session_id。
2. 保存 projection_version。
3. 用 full_projection 整体替换本地 projection。
4. 用 available_commands 整体替换按钮列表。
5. 渲染地图和按钮。
```

## 8. Command

用户点击 `available_commands` 中的按钮时调用。

前端只提交 `option_id`，不要提交 `WorldCommandDto`。

### 8.1 Request

```json
{
  "session_id": "session_xxx",
  "client_id": "web_xxx",
  "client_sequence": 1,
  "known_projection_version": 1,
  "submit_mode": "option_id",
  "option_id": "opt_wait_player",
  "selection_context": {
    "target": {
      "target_kind": "none",
      "target_coord": null,
      "target_entity_id": "",
      "target_actor_key": "",
      "target_item_key": "",
      "target_inventory_id": "",
      "target_area_id": "",
      "knowledge_key": "",
      "recipe_key": ""
    },
    "selected_actor_key": "",
    "selected_entity_id": "",
    "selected_inventory_id": "",
    "selected_area_id": ""
  }
}
```

### 8.2 Response

```json
{
  "session_id": "session_xxx",
  "accepted_client_sequence": 1,
  "base_projection_version": 1,
  "new_projection_version": 2,
  "requires_full_refresh": false,
  "result": {
    "command_id": "cmd_xxx",
    "command_key": "wait",
    "result_kind": "succeeded",
    "actor_key": "player",
    "target_ref": "",
    "failure_reason_keys": [],
    "warning_keys": []
  },
  "projection_patch": {
    "base_projection_version": 1,
    "new_projection_version": 2,
    "requires_full_refresh": false,
    "changed_cells": [],
    "changed_entities": [],
    "changed_inventories": [],
    "changed_knowledge": [],
    "changed_area_effects": []
  },
  "available_commands": [],
  "event_feed": [],
  "experiences": [],
  "frontend_hints": [],
  "warning_keys": []
}
```

### 8.3 前端处理

收到 response 后：

```text
1. 如果 requires_full_refresh=true：不要应用 patch，直接调用 refresh。
2. 如果 projection_patch.base_projection_version !== state.projectionVersion：不要应用 patch，直接调用 refresh。
3. 否则应用 projection_patch。
4. state.projectionVersion = new_projection_version。
5. 用 response.available_commands 整体替换按钮列表。
6. 追加 event_feed / frontend_hints / warning_keys 到日志。
```

## 9. Refresh

用于全量刷新。

触发时机：

```text
1. command response.requires_full_refresh=true。
2. patch base version 与本地 projectionVersion 不一致。
3. 用户点击“刷新”。
4. 网络错误后重试。
```

### 9.1 Request

```json
{
  "session_id": "session_xxx",
  "client_id": "web_xxx",
  "known_projection_version": 1,
  "requested_scopes": ["full_safe_world"],
  "requested_layer_key": "surface",
  "viewport_center_x": 0,
  "viewport_center_y": 0
}
```

### 9.2 Response

```json
{
  "session_id": "session_xxx",
  "projection_version": 2,
  "full_projection": {
    "projection_version": 2,
    "actor_key": "player",
    "active_layer_key": "surface",
    "visible_cells": [],
    "visible_entities": [],
    "inventories": [],
    "knowledge": [],
    "area_effects": [],
    "safe_summary_keys": []
  },
  "available_commands": [],
  "event_feed": [],
  "frontend_hints": [],
  "warning_keys": []
}
```

### 9.3 前端处理

```text
1. 用 full_projection 整体替换本地 projection。
2. state.projectionVersion = response.projection_version。
3. 用 available_commands 整体替换按钮列表。
```

## 10. Reset

用于“重开世界”。

### 10.1 Request

```json
{
  "session_id": "session_xxx",
  "client_id": "web_xxx",
  "confirmed": true
}
```

### 10.2 Response

```json
{
  "session_id": "session_xxx",
  "bootstrap": {
    "session_id": "session_xxx",
    "server_protocol_version": 1,
    "projection_version": 1,
    "full_projection": {},
    "available_commands": [],
    "event_feed": [],
    "frontend_hints": [],
    "warning_keys": []
  }
}
```

### 10.3 前端处理

按 bootstrap response 处理。

## 11. Mock 模式要求

为了让前端开发完“复制进目录就能打开”，必须内置 Mock 模式。

触发条件：

```text
1. URL 带 ?mock=1。
2. /api/client/bootstrap 返回 404 / network error。
```

Mock 模式至少返回：

```text
1. 5x5 网格。
2. 一个 player entity。
3. 两个按钮：等待、观察。
4. 点击按钮后 projection_version +1，并追加一条日志。
```

Mock 模式只用于前端视觉和交互自测，不代表真实规则。

页面上要明显显示：

```text
MOCK MODE
```

## 12. Patch 应用规则

前端只做“显示层合并”。

### 12.1 changed_cells

根据 `cell_id` 合并：

```text
op=add/update/replace：写入或替换。
op=remove：删除。
op=clear：清空 cells。
```

### 12.2 changed_entities

根据 `entity_id` 合并。

### 12.3 inventories

根据 `inventory_id` 合并。

### 12.4 knowledge

根据 `actor_key` 合并。

### 12.5 area_effects

根据 `area_effect_id` 合并。

如果不确定怎么合并，直接调用 refresh，不要猜。

## 13. 字段容错要求

前端必须容错：

```text
1. 数组字段缺失时按 [] 处理。
2. 字符串字段缺失时按 "" 处理。
3. 布尔字段缺失时按 false 处理。
4. 数字字段缺失时按 0 处理。
5. 遇到未知字段不要报错。
6. 遇到未知 command_kind / event_kind 仍要能显示。
```

## 14. 严禁事项

前端绝对不要做这些事：

```text
1. 不读 content/core JSON。
2. 不根据 entity_key / object_key / terrain_key 生成玩法按钮。
3. 不提交 WorldCommandDto。
4. 不提交 command.parameters。
5. 不修改 actor 位置。
6. 不修改背包数量。
7. 不修改知识列表。
8. 不根据 event / hint 反推规则。
9. 不写死 red_berry / torch / fire / wolf / wood / companion 等内容 key。
10. 不把 debug JSON 作为玩家主界面。
```

只允许：

```text
点击后端给的 available_commands.option_id。
```

## 15. UI 验收标准

交付时至少满足：

```text
1. 页面可打开，无控制台报错。
2. bootstrap 成功或自动进入 Mock 模式。
3. 地图区能显示 projection。
4. 操作区只显示 available_commands。
5. 点击 enabled command 会提交 option_id。
6. requires_full_refresh 时会自动 refresh。
7. event_feed / warning_keys 能显示到日志。
8. 重开按钮可调用 reset 或清 session 后 bootstrap。
9. 手机横屏可用。
10. 断网 / 接口 500 时有可读错误提示。
```

## 17. API adapter 推荐伪代码

```js
async function postJson(path, body) {
  const response = await fetch(path, {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
      'Accept': 'application/json',
    },
    body: JSON.stringify(body),
  });
  if (!response.ok) {
    throw new Error(`${path} failed: ${response.status}`);
  }
  return response.json();
}

async function bootstrap() {
  return postJson('/api/client/bootstrap', {
    client_id: state.clientId,
    session_id: state.sessionId,
    requested_actor_key: 'player',
    requested_layer_key: 'surface',
    client_protocol_version: 1,
    create_if_missing: true,
    dev_reset_if_allowed: false,
  });
}

async function submitOption(optionId) {
  state.clientSequence += 1;
  return postJson('/api/client/command', {
    session_id: state.sessionId,
    client_id: state.clientId,
    client_sequence: state.clientSequence,
    known_projection_version: state.projectionVersion,
    submit_mode: 'option_id',
    option_id: optionId,
    selection_context: emptySelectionContext(),
  });
}
```

```js
      recipe_key: '',
    },
    selected_actor_key: '',
    selected_entity_id: '',
    selected_inventory_id: '',

  };
}
```
