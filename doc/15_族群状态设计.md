# 先驱者：族群状态设计

## 1. 设计目标

族群状态系统负责把知识、传播、资源、风险、组织、战斗和族群关系整理成族群层面的稳定状态。

它回答以下问题：

```text
族群现在有多少人？
族群当前缺什么？
族群掌握了哪些知识？
哪些知识只停留在“知道”，哪些已经能稳定执行？
族群是否存在误解或错误知识扩散？
族群是否能共同作战？
族群是否存在分裂风险？
族群和其他族群是什么关系？
哪些状态已经接近文明能力解锁条件？
AI 如何观察并带领族群？
```

核心链路：

```text
KnowledgeClaim / KnowledgeProjection
+ PropagationResult / TribeInfluenceCandidate / TribeKnowledgeGapProjection
+ Resource / Entity / Region State
  ↓
TribeStateResolver
  ↓
TribeState
  ↓
TribeProjection / CivilizationReadinessCandidate
  ↓
CivilizationCapability
```

本系统必须保证：

- 族群状态不是单一等级或单一资源数值。
- 族群知道某知识，不等于族群能稳定执行。
- 族群状态必须表达知识、资源、行为、风险、组织、战斗、凝聚和对外关系。
- 族群状态可以被知识传播、错误传播、资源压力、伤亡和外部威胁影响。
- 族群状态为文明能力提供候选条件，但不直接解锁文明能力。
- 具体战斗伤害、命中、阵型结算不在本章实现；本章只设计族群层面的战斗准备、关系和结果影响。
- 族群状态变化必须可复盘、可解释、可测试。

## 2. 不解决的问题

| 问题 | 原因 | 归属文档 |
|---|---|---|
| 具体战斗结算 | 本章只设计族群战斗能力和战争状态 | 后续战斗 / 冲突系统 |
| 文明能力最终解锁 | 本章只输出 CivilizationReadinessCandidate | `16_文明能力设计.md` |
| 单个 NPC 完整 AI | 本章只处理群体画像，不展开个体行为树 | 后续 AI 系统 |
| 资源物理生成和消耗细节 | 本章只汇总资源画像 | 世界对象 / 规则管线 |
| 知识真伪判断 | 已由知识系统负责 | `13_知识系统设计.md` |
| 知识传播结算 | 已由传播系统负责 | `14_传播系统设计.md` |
| 前端推导族群状态 | 族群状态必须由后端规则输出 | `20_前后端协议设计.md` |

关键边界：

```text
KnowledgeClaim：族群知道什么。
PropagationResult：知识是否传开。
TribeState：族群整体状态。
TribeBehaviorProfile：族群会做什么。
TribeCombatProfile：族群能否共同作战。
TribeCohesionProfile：族群是否稳定或分裂。
InterTribeRelationProfile：族群与其他族群关系。
CivilizationReadinessCandidate：文明能力前置候选。
```

### 3.1 知道不等于会做

示例：

```text
族群知道火能驱狼。
但没有稳定火源。
没有守火分工。
没有夜间预警。
没有共同作战意识。
所以不能说族群已具备夜间防御能力。
```

因此族群行为必须有采纳状态：

```text
Known → Adopted → Practiced → Operational → Institutionalized
```

### 3.2 族群状态不是单个等级

错误：

```text
tribe_level = 3
```

正确：

```text
TribeProfile
TribeKnowledgeProfile
TribeBehaviorProfile
TribeNeedProfile
TribeRiskProfile
TribeCombatProfile
TribeCohesionProfile
InterTribeRelationProfile
```

### 3.3 分裂风险是确定性评分

第一版不使用随机数决定分裂。

```text
fragmentation_risk = f(resource_pressure, knowledge_conflict, trust, casualties, teaching_inequality, external_threat)
```

如果未来要触发随机事件，应由后续事件/模拟系统基于该评分处理。

## 7. 族群状态生命周期

### 7.1 知识进入族群

```text
1. 知识系统产生 Tribe scope 或 Teachable 知识。
2. 传播系统产生 TribeInfluenceCandidate。
3. TribeKnowledgeProfile 更新 known/shared/disputed/misunderstood 列表。
4. 如果传播足够稳定，知识进入 operational_knowledge_ids。
```

### 7.2 知识转为行为

```text
Known → Adopted → Practiced → Operational → Institutionalized
```

示例：

```text
族群知道红浆果可食用。
开始采集红浆果。
多次采集成功。
形成稳定采集行为。
最终成为安全采集制度或文明能力前置。
```

### 7.3 风险上升

```text
错误知识扩散。
资源压力上升。
中毒或伤亡增加。
外部族群威胁增加。
```

结果：

```text
TribeRiskProfile 上升。
TribeCohesionProfile 下降。
可能生成纠正知识、教学、迁移或防御需求。
```

### 7.4 共同作战能力形成

```text
相关知识掌握：火能驱狼、工具可作武器、危险预警。
传播成功：足够成员理解并记住。
行为稳定：守火、警戒、协作、撤退路线。
战斗画像提升：Defensive / Mobilized。
文明候选：夜间防御、协同狩猎、守卫制度。
```

### 7.5 分裂风险变化

```text
资源压力 + 知识冲突 + 教学不均 + 伤亡 + 领导信任下降
  ↓
fragmentation_risk 上升
  ↓
Stable → Tense → Fracturing → Splintered
```

反向恢复：

```text
食物改善 + 教学成功 + 风险降低 + 信任恢复
  ↓
Recovering → Stable
```

### 7.6 族群间关系变化

```text
资源竞争、冲突事件、交易、共同威胁、知识差距
  ↓
InterTribeRelationProfile 更新
  ↓
Neutral / Competitive / Hostile / War / Alliance
```

## 11. 校验规则

### 11.1 TribeState 校验

|---|---|

| combat_readiness 与 combat_profile 分数严重不符 | RuleError |
| Operational 行为没有相关 KnowledgeId | RuleError |
| CivilizationReadinessCandidate 无相关知识或行为 | RuleError |

| 规则 | 错误级别 |
|---|---|

|---|---|

## 14. 设计注意事项

研发实现时必须遵守：

```text
不要把 TribeState 简化成 tribe_level。
不要把族群知道某知识等同于族群会执行。
不要在本章实现具体战斗伤害结算。
不要使用随机数直接决定分裂或战争。
不要让前端自行推导族群状态。
不要直接解锁文明能力，只输出候选。
不要忽略错误知识扩散对族群状态的影响。
不要忽略共同作战能力和对外族群关系。
```

### 14.1 行为采纳原则

```text
Known：知道。
Adopted：开始采用。
Practiced：多次练习。
Operational：稳定执行。
Institutionalized：制度化。
```

### 14.2 战斗边界原则

本章设计：

```text
战斗准备。
协同能力。
士气。
指挥信任。
战争状态。
冲突结果对族群状态的影响。
```

本章不设计：

```text
伤害公式。
命中率。
详细阵型格子。
武器攻击动画。
回合制战斗流程。
```

### 14.3 可复盘原则

同一输入必须得到同一输出：

```text
同一个 TribeState
+ 同一组 KnowledgeClaim
+ 同一组 TribeInfluenceCandidate
+ 同一组资源/实体/区域摘要
+ 同一个 TribeStatePolicy
= 同一个 TribeStateResolveResult
```

族群状态系统不使用随机数。

## 16. 阶段结论

族群状态系统是判断“知识是否真正改变种群”的核心层。

当前设计支持：

- 族群基础、生存、资源、需求、风险画像。
- 族群知识掌握、误解、争议和缺口。
- 行为从知道到制度化的采纳过程。
- 共同作战能力、战斗准备、士气和指挥信任。
- 族群凝聚力、分裂风险、派系摘要。
- 不同族群关系、战争状态和冲突影响。
- 文明能力就绪候选。
- 前端、AI、强化学习族群投影。
