# P56 客户端世界投影与命令桥接任务卡设计

## 1. 阶段定位

P56 是 V2.0 客户端协议进入真实可玩世界之前的桥接阶段。

它不是前端 UI 阶段，也不是重新设计世界网格阶段。

它的职责是把已经存在的后端世界能力：

```text
P43 WorldCommand 管线
P44 WorldGridRuntime / 探索投影
P45 背包归属接口
P46 世界生成器
P47 采集砍伐挖掘
P48 制作与反应
P49 世界经验学习
P50 教学与无知 NPC
P51 Agent 网格目标执行
P52 野生 Agent 与野兽生态
P53 Client Protocol
P54 Client HTTP Gateway
P55 客户端真实协议接入
```

连接成一条客户端可以使用的真实链路：

```text
真实 world runtime
  -> 安全 client projection
  -> 后端生成 available_commands
  -> 前端只提交 option_id
  -> WorldCommandPipeline 执行真实命令
  -> projection patch / full refresh 回到前端
```

当前阻塞点非常明确：

```text
client_projection_adapter.cpp 当前还是 stub，返回空 visible_cells / visible_entities。
client_available_command_adapter.cpp 当前还是 stub，只返回 Wait / Inspect。
client_server_runtime_factory.cpp 当前注册的是最小 stub handler，没有把 WorldGridRuntime 作为真实运行时接进客户端服务。
```

P56 的目标就是移除这些生产路径上的 stub，让客户端第一次真正看到地图、实体和移动命令。

## 2. 为什么现在做

P55 已经让前端可以访问：

```text
POST /api/client/bootstrap
POST /api/client/command
POST /api/client/refresh
POST /api/client/reset
```

但是 P55 只能证明协议能通，不能证明游戏能玩。

如果现在继续做前端界面，会出现三个严重问题：

1. 前端没有真实地图，只能自己 mock 地图。
2. 前端没有真实可用命令，只能自己拼 Move / Gather 等命令。
3. 低上下文 AI 会误以为“画出来就是游戏”，再次把规则写到前端。

因此 P56 必须先做。

P56 完成后，前端研发只需要做一件事：

```text
把后端返回的 projection 画出来，把后端返回的 available_commands 渲染成按钮或摇杆。
```

前端不需要知道世界如何生成、移动如何校验、NPC 如何行动、知识如何形成。

## 3. 阶段目标

P56 完成后必须具备以下能力：

1. `bootstrap` 返回非空 `visible_cells`。
2. `bootstrap` 返回玩家实体，以及玩家视野内可见实体。
3. `bootstrap` 返回基于真实 actor 位置生成的 `available_commands`。
4. 至少包含 `Wait`、`Inspect`、四方向 `Move` 命令。
5. 点击 Move 的 `option_id` 后，后端通过 `WorldCommandPipeline` 执行真实移动。
6. 移动后返回 projection patch 或要求 full refresh，前端能看到玩家位置变化。
7. 不能移动到阻挡格、水、山、不可达层或非相邻格。
8. 客户端不能伪造 actor、不能伪造 option、不能绕过 projection version。
9. 生产路径不再依赖空 projection stub。
10. 生产路径不再只返回 Wait / Inspect stub。

本阶段最小可玩效果：

```text
打开客户端 -> 后端生成/加载世界 -> 前端看到玩家周围地图 -> 玩家可以移动 -> 地图随探索更新。
```

## 4. 本阶段不解决的问题

P56 不做正式 UI。

```text
不设计横屏美术。
不设计正式 HUD。
不设计手机触控手感。
不设计小地图。
不设计动画。
```

这些属于后续客户端表现阶段。

P56 不重新发明地图。

```text
不新增第二套 WorldGrid。
不新增前端地图数据源。
不新增和 WorldGridRuntime 平行的 runtime。
```

P56 不完整重做采集、制作、教学、野兽生态。

```text
已有 handler 可接就接。
没有稳定 runtime 上下文的 provider 先预留接口。
本阶段硬验收聚焦：地图可见、实体可见、移动可执行。
```

P56 不让前端直接读取 JSON。

```text
content/core 仍然只能由后端加载与解释。
前端不能 fetch JSON 配置。
```

P56 不解决正式存档。

```text
本地存档、读档、世界持久化放到后续阶段。
P56 可以使用内存 runtime。
```

## 5. 核心设计原则

### 5.1 后端是唯一世界权威

地图、实体、背包、知识、区域效果、可用命令都来自后端。

禁止：

```text
前端生成地图。
前端自己判断能不能移动。
前端根据 terrain_key 自己决定 blocks_movement。
前端自己拼 WorldCommandDto 作为正常路径。
```

### 5.2 projection 是安全视图，不是运行时状态

`ClientWorldProjection` 只能暴露玩家当前可以知道的内容。

禁止暴露：

```text
未探索格子的真实地形。
不可见实体。
隐藏资源点。
野兽内部仇恨值。
NPC 内部目标链完整细节。
真实随机种子。
hidden_truth / raw_state / debug_only 字段。
```

### 5.3 available_commands 必须由后端生成

前端只能提交当前 session 快照里的 `option_id`。

禁止：

```text
前端根据 WASD 直接拼 Move command。
前端根据按钮文字推导 command_kind。
前端提交旧 projection_version 还继续执行。
前端提交不在 snapshot 中的 option_id。
```

### 5.4 option 是授权快照，不是 UI 装饰

`WorldCommandOptionDto` 代表“当前这一刻后端允许玩家尝试的动作”。

所以 option 必须包含：

```text
command_kind
command_key
label_text
target
enabled
disabled_reason_text
priority
```

如果 option 是禁用状态，前端可以显示，但提交后仍必须被后端校验挡住。

### 5.5 runtime bridge 只读投影，写入必须走 command

P56 的 projection bridge 只能读取 runtime，不能修改 runtime。

所有写入必须走：

```text
ClientCommandGateway
  -> WorldCommandPipeline
  -> WorldCommandHandler
  -> runtime / inventory / learning / agent 系统
```

禁止 projection bridge 为了“补画面”偷偷生成实体、移动玩家或改背包。

### 5.6 生产路径不能静默退回 stub

测试可以保留 stub，但正式 `pathfinder_client_server_exe` 必须连接真实 runtime bridge。

如果真实 bridge 没有注入，生产启动必须失败或测试明确发现。

禁止：

```text
生产环境 projection 为空但返回成功。
生产环境 available_commands 只有 Wait / Inspect 仍算通过。
```

### 5.7 先接最小真实闭环，再扩展复杂互动

本阶段必须先保证：

```text
生成世界 -> 看见地图 -> 移动 -> 探索更新
```

然后再逐步扩展：

```text
采集 -> 背包 -> 制作 -> 知识 -> 教学 -> NPC 自主行动 -> 野兽生态
```

不能在 P56 一次性把所有玩法都混进同一个巨大函数。

## 6. 头脑风暴：后续扩展可能遇到的问题

### 6.1 大地图分页与视野裁剪

未来地图会变大，不能每次 full projection 返回整个世界。

P56 必须预留 `ClientProjectionScope::Viewport`，但本阶段可以先按 actor 视野输出。

### 6.2 二维地图加虚拟 Z 轴

游戏是二维的，但会有地下、地表、空中。

P56 必须保留 `layer_key`，并禁止把坐标写成只有 x/y。

### 6.3 可见与已探索的区别

玩家当前看得见的格子是 `Visible`。

玩家以前见过但现在看不见的是 `Discovered`。

前端可以画旧记忆，但不能显示旧格子上的当前实体。

### 6.4 野兽在视野外行动

野兽离开视野后仍在世界中行动，但 projection 不能泄露它的位置。

P56 只能暴露可见野兽或安全的区域提示，例如：

```text
area_effect: nearby_danger_hint
```

不能暴露完整野兽路径。

### 6.5 NPC 自主行动与玩家行动同时改变世界

玩家等待或移动后，NPC/野兽可能也执行命令。

P56 的 projection version 必须以 runtime state version 为准，不能只按玩家命令自增。

### 6.6 前端刷新页面

刷新后 session 还在，客户端应该能重新 `bootstrap` 或 `refresh` 拿到当前真实 projection。

不能因为前端丢失本地状态就重置世界。

### 6.7 option 过期

玩家看到“向东移动”，但世界下一 tick 东边被野兽或障碍占了。

旧 option 必须失效，或执行时返回 blocked。

P56 继续沿用 P53/P54 的 snapshot authority。

### 6.8 未来多客户端或服务器部署

虽然当前是单机游戏，但 HTTP Gateway 可被远程访问。

P56 不能把 session、actor、runtime 写成全局不可区分的静态变量。

本阶段可以只有一个玩家，但接口要保留 session/actor/layer。

### 6.9 内容配置逐渐接入

后续 terrain、entity、resource 的显示名和图标会来自 JSON。

P56 不让前端读 JSON，而是在 projection fields 里给安全 key：

```text
terrain_key
entity_key
display_name_key
tag_keys
```

前端可以根据 key 做表现映射，但不能根据 key 推导规则结果。

### 6.10 前端摇杆与后端 option 的映射

摇杆输入不是直接 command。

摇杆只能选择当前 `available_commands` 中匹配方向的 Move option。

如果没有匹配 option，前端应该显示“当前不能移动”，不能伪造。

## 7. 总体架构

P56 后的链路：

```text
Client HTTP
  -> ClientHttpGateway
  -> ClientSessionGateway / ClientCommandGateway
  -> ClientRuntimeBridgePort
  -> WorldGridRuntime / InventoryRuntime / KnowledgeBridge / AreaEffectBridge
  -> ClientWorldProjection + WorldCommandOptionDto[]

Client command submit
  -> option_id snapshot 校验
  -> materialize WorldCommandDto
  -> WorldCommandPipeline
  -> runtime-aware handlers
  -> projection patch / event_feed / available_commands
```

生产服务必须由 `ClientServerRuntimeFactory` 组装：

```text
WorldGridRuntime
WorldInventoryRuntime（可选，本阶段如果未完整接入必须有接口占位）
WorldProjectionAdapter
ClientRuntimeProjectionBridge
ClientRuntimeCommandOptionBridge
ClientProjectionAdapter
ClientAvailableCommandAdapter
ClientSessionGateway
ClientCommandGateway
WorldCommandPipeline
ClientHttpGateway
```

## 8. 模块划分

| 模块 | 中文含义 | 职责 | 输入 | 输出 | 禁止事项 |
|---|---|---|---|---|---|
| `ClientRuntimeBridgePort` | 客户端运行时桥接口 | 为 client_protocol 提供只读投影和命令 option 来源 | actor_key、layer_key、projection scope | projection parts、options、version | 禁止修改 runtime |
| `ClientRuntimeProjectionBridge` | 客户端投影桥 | 从 world runtime 构建安全 ClientWorldProjection | IWorldRuntime、可选 inventory/knowledge/area bridge | visible_cells、visible_entities、inventories、knowledge、area_effects | 禁止读取 hidden truth |
| `ClientRuntimeCommandOptionBridge` | 客户端命令桥 | 根据当前真实上下文生成 available_commands | actor、coord、visible cells、handler registry | WorldCommandOptionDto[] | 禁止前端规则、禁止无 handler option |
| `MovementCommandOptionProvider` | 移动命令提供器 | 生成四方向 Move option | actor 坐标、邻格状态 | enabled/disabled Move options | 禁止绕过 Move handler 校验 |
| `InspectCommandOptionProvider` | 观察命令提供器 | 生成当前格、邻格、实体 Inspect option | 可见 cells/entities | Inspect options | 禁止观察不可见对象 |
| `WaitCommandOptionProvider` | 等待命令提供器 | 生成 Wait option | actor/session | Wait option | 禁止改时间 |
| `RuntimeHandlerRegistrar` | runtime handler 注册器 | 将 Move/Inspect/Wait/GenerateWorld 注册为 runtime-aware handler | IWorldRuntime、registry | handler registry | 禁止注册 stub handler 到生产 server |
| `ClientServerRuntimeFactory` | 客户端服务运行时工厂 | 组装真实 runtime 与 client HTTP 服务 | config | server dependencies | 禁止只组装 stub adapter |
| `ClientProjectionAdapter` | 协议投影适配器 | 保留 mergePatch 和 provider 调用入口 | bridge 输出 | ClientWorldProjection | 禁止自己造数据 |
| `ClientAvailableCommandAdapter` | 协议命令适配器 | 保留 snapshot materialize，并调用 option bridge | bridge options、snapshot | WorldCommandDto | 禁止无 snapshot 正常执行 |

## 9. 核心枚举设计

### 9.1 ClientRuntimeBridgeMode

中文：客户端运行时桥模式。

用途：区分生产真实桥、测试 stub、禁用状态，避免生产静默跑 stub。

| 枚举值 | 中文 | 使用场景 |
|---|---|---|
| `Unknown` | 未知 | 默认值，validate 必须拒绝 |
| `RuntimeBacked` | 真实运行时 | 生产 `client_server` 必须使用 |
| `StubForProtocolTest` | 协议测试占位 | 仅 P53/P54 单元测试可使用 |
| `Disabled` | 禁用 | 构造失败或测试故意禁用 |
| `TestOnly` | 测试专用 | 枚举 roundtrip 与异常路径测试 |

规则：

```text
toString 使用 snake_case。
fromString 非法值返回结构化错误。
生产 factory 如果不是 RuntimeBacked 必须测试失败。
```

### 9.2 ClientCommandOptionProviderKind

中文：客户端命令提供器类型。

| 枚举值 | 中文 | 使用场景 |
|---|---|---|
| `Unknown` | 未知 | 默认拒绝 |
| `Wait` | 等待 | 生成 Wait option |
| `Move` | 移动 | 生成四方向 Move option |
| `Inspect` | 观察 | 生成可见目标 Inspect option |
| `Inventory` | 背包 | 后续生成 Pickup/Drop/Eat/Use option |
| `Harvest` | 采集 | 后续生成 Gather/Chop/Mine/Dig option |
| `Craft` | 制作 | 后续生成 Craft option |
| `Teach` | 教学 | 后续生成 Teach option |
| `Combat` | 战斗 | 后续生成 Attack/Flee option |
| `AreaEffect` | 区域效果 | 后续生成区域魔法/环境命令 |
| `TestOnly` | 测试专用 | 单测 |

P56 硬要求实现：

```text
Wait
Move
Inspect
```

其他 provider 可以只预留接口，不能伪造玩法。

### 9.3 ClientMoveDirection

中文：客户端移动方向。

| 枚举值 | 中文 | dx | dy | 说明 |
|---|---|---:|---:|---|
| `Unknown` | 未知 | 0 | 0 | 默认拒绝 |
| `North` | 北/上 | 0 | -1 | 二维地图向上 |
| `South` | 南/下 | 0 | 1 | 二维地图向下 |
| `West` | 西/左 | -1 | 0 | 二维地图向左 |
| `East` | 东/右 | 1 | 0 | 二维地图向右 |
| `TestOnly` | 测试专用 | 0 | 0 | 单测 |

规则：

```text
P56 只允许四方向移动。
斜向、跳跃、传送、跨 layer 移动后续单独设计。
```

### 9.4 ClientProjectionBuildReason

中文：客户端投影构建原因。

| 枚举值 | 中文 | 使用场景 |
|---|---|---|
| `Unknown` | 未知 | 默认拒绝 |
| `Bootstrap` | 启动 | 首次建立 session |
| `Refresh` | 刷新 | 前端主动刷新 |
| `CommandResult` | 命令结果 | command 后补 full refresh 或 patch 对齐 |
| `Reset` | 重置 | reset 后返回 bootstrap projection |
| `TestOnly` | 测试专用 | 单测 |

用途：trace 和测试诊断，不改变规则结果。

## 10. 核心 DTO / 数据结构设计

### 10.1 ClientRuntimeViewRequest

中文：客户端运行时视图请求。

| 字段 | 类型 | 中文含义 | 必填 | 谁写入 | 谁读取 | 校验规则 | 禁止内容 |
|---|---|---|---|---|---|---|---|
| `actor_key` | string | 当前会话权威 actor | 是 | ClientSessionGateway | runtime bridge | 非空，来自 session | 禁止来自 selection_context |
| `layer_key` | string | 当前层 | 是 | session / request | runtime bridge | 空则 surface | 禁止前端创建未知层 |
| `projection_version_hint` | uint64 | 客户端已知版本提示 | 否 | session | bridge | 只用于诊断 | 禁止作为 runtime 权威版本 |
| `scopes` | vector<ClientProjectionScope> | 请求范围 | 否 | refresh request | bridge | Unknown 拒绝 | 禁止请求 hidden scope |
| `viewport_center_x` | int | 视口中心 x | 否 | refresh request | bridge | 本阶段可忽略 | 禁止作为玩家位置 |
| `viewport_center_y` | int | 视口中心 y | 否 | refresh request | bridge | 本阶段可忽略 | 禁止作为玩家位置 |
| `reason` | ClientProjectionBuildReason | 构建原因 | 是 | gateway | bridge | Unknown 拒绝 | 禁止影响规则 |

ID 规则：

```text
actor_key 只能引用 session authority。
前端不能创建 actor_key。
```

### 10.2 ClientRuntimeView

中文：客户端运行时安全视图。

| 字段 | 类型 | 中文含义 | 必填 | 谁写入 | 谁读取 | 校验规则 | 禁止内容 |
|---|---|---|---|---|---|---|---|
| `projection_version` | uint64 | 投影版本 | 是 | runtime bridge | session/client | 必须等于 runtime stateVersion 或桥接版本 | 禁止本地随意自增 |
| `actor_key` | string | 当前 actor | 是 | runtime bridge | client | 非空 | 禁止泄露其他 actor 私密状态 |
| `active_layer_key` | string | 当前层 | 是 | runtime bridge | client | 非空 | 禁止丢 layer |
| `visible_cells` | vector<WorldCellPatchDto> | 可见/已探索格子 | 是 | projection bridge | client | cell_id 唯一 | 禁止 unknown cell 真相 |
| `visible_entities` | vector<WorldEntityPatchDto> | 当前可见实体 | 是 | projection bridge | client | entity_id 唯一 | 禁止不可见实体 |
| `inventories` | vector<InventoryPatchDto> | 安全背包摘要 | 否 | inventory bridge | client | owner 可见 | 禁止别人的隐藏库存 |
| `knowledge` | vector<KnowledgePatchDto> | 安全知识摘要 | 否 | knowledge bridge | client | actor 可见 | 禁止 hidden truth |
| `area_effects` | vector<AreaEffectPatchDto> | 安全区域效果 | 否 | area bridge | client | area 可见 | 禁止泄露未来事件 |
| `safe_summary_keys` | vector<string> | 安全摘要 key | 否 | bridge | client | 只允许安全 key | 禁止 debug/raw |
| `warning_keys` | vector<string> | 桥接警告 | 否 | bridge | gateway/test | 可为空 | 禁止吞掉生产缺桥 |

### 10.3 ClientRuntimeCommandOptionRequest

中文：客户端运行时命令选项请求。

| 字段 | 类型 | 中文含义 | 必填 | 谁写入 | 谁读取 | 校验规则 | 禁止内容 |
|---|---|---|---|---|---|---|---|
| `actor_key` | string | 会话 actor | 是 | session | option bridge | 非空 | 禁止前端覆盖 |
| `layer_key` | string | 当前层 | 是 | session | option bridge | 非空 | 禁止未知层 |
| `projection_version` | uint64 | 当前版本 | 是 | session/runtime | option bridge | 与 runtime 对齐 | 禁止旧版本生成新 option |
| `include_disabled` | bool | 是否输出禁用 option | 否 | gateway config | provider | 默认 true | 禁止禁用 option 可执行 |
| `provider_kinds` | vector<ClientCommandOptionProviderKind> | 启用 provider | 否 | factory config | option bridge | Unknown 拒绝 | 禁止前端传入 |

### 10.4 ClientRuntimeBridgeDiagnostics

中文：客户端运行时桥诊断。

| 字段 | 类型 | 中文含义 | 必填 | 谁写入 | 谁读取 | 校验规则 | 禁止内容 |
|---|---|---|---|---|---|---|---|
| `bridge_mode` | ClientRuntimeBridgeMode | 桥模式 | 是 | factory/bridge | tests | 生产必须 RuntimeBacked | 禁止生产 Stub |
| `runtime_state_version` | uint64 | runtime 版本 | 是 | bridge | tests | 非 0 | 禁止和 projection 脱节 |
| `visible_cell_count` | int | 可见格数量 | 是 | bridge | tests | bootstrap > 0 | 禁止空世界成功 |
| `visible_entity_count` | int | 可见实体数量 | 是 | bridge | tests | 至少含 player | 禁止玩家不可见 |
| `option_count` | int | option 数量 | 是 | bridge | tests | bootstrap > 0 | 禁止只有空列表 |
| `warning_keys` | vector<string> | 警告 | 否 | bridge | tests | 可为空 | 禁止隐藏生产错误 |

诊断 DTO 不需要暴露给正式前端，可以写入测试或 debug trace。

## 11. 核心服务接口设计

### 11.1 IClientRuntimeBridgePort

中文职责：client_protocol 到真实 runtime 的只读桥接口。

建议位置：

```text
backend/include/pathfinder/client_runtime_bridge/client_runtime_bridge_port.h
```

接口：

```cpp
class IClientRuntimeBridgePort {
public:
    virtual ~IClientRuntimeBridgePort() = default;

    virtual foundation::Result<ClientRuntimeView> buildRuntimeView(
        const ClientRuntimeViewRequest& request) const = 0;

    virtual foundation::Result<std::vector<world_command::WorldCommandOptionDto>> buildRuntimeOptions(
        const ClientRuntimeCommandOptionRequest& request) const = 0;

    virtual foundation::Result<ClientRuntimeBridgeDiagnostics> diagnostics(
        const std::string& actor_key,
        const std::string& layer_key) const = 0;
};
```

状态规则：

```text
只读 runtime。
不修改世界。
不执行 command。
可重复调用。
不依赖随机数。
失败返回结构化错误。
```

### 11.2 ClientRuntimeProjectionBridge

中文职责：从 `IWorldRuntime` 构建 `ClientRuntimeView`。

输入：

```text
IWorldRuntime
WorldProjectionAdapter
可选 InventoryProjectionAdapter
可选 WorldKnowledgeProjectionBridge
可选 AreaEffectProjectionBridge
ClientRuntimeViewRequest
```

输出：

```text
ClientRuntimeView
```

失败情况：

```text
actor_not_found
runtime_not_initialized
projection_scope_invalid
layer_not_available
bridge_mode_invalid
```

核心规则：

1. 先通过 runtime 查 actor。
2. 以 actor 的真实坐标和 vision/exploration 为准。
3. 生成可见/已探索 cell patch。
4. 只对 Visible cell 输出当前 entity。
5. Discovered cell 不输出当前 entity 列表。
6. projection_version 使用 runtime stateVersion。
7. 不使用 `snapshotForDebug()` 作为生产主路径；如果为了兼容过渡使用，必须写 TODO 并加测试保护，后续提升为正式 query port。

### 11.3 ClientRuntimeCommandOptionBridge

中文职责：从真实上下文生成可用命令。

输入：

```text
IWorldRuntime
WorldCommandHandlerRegistry
provider list
ClientRuntimeCommandOptionRequest
```

输出：

```text
vector<WorldCommandOptionDto>
```

核心规则：

1. 如果 registry 没有对应 handler，不生成该 command option。
2. option target 必须来自真实 runtime 可见或 actor 自身可达上下文。
3. option_id 由后端生成，不能由前端传入。
4. option_id 必须能在 snapshot 中被精确匹配。
5. disabled option 也必须有明确 disabled_reason_text。
6. provider 不能修改 runtime。

### 11.4 MovementCommandOptionProvider

中文职责：生成四方向移动 option。

输入：

```text
actor coord
actor layer
邻接 cell
WorldCommandHandlerRegistry
```

输出：

```text
North/South/West/East Move options
```

option 规则：

```text
command_kind = Move
command_key = move_north / move_south / move_west / move_east
target.target_kind = Coordinate
target.target_coord = 邻格坐标
label_text = 向上 / 向下 / 向左 / 向右 或中文方向
enabled = 邻格存在且不阻挡且同 layer 且 handler 存在
disabled_reason_text = out_of_bounds / cell_blocked / unknown_layer / handler_missing
```

注意：

```text
即使 provider 认为 enabled，Move handler 执行时仍必须再次校验。
provider 只是生成“当前看起来可尝试”的 option，不是最终裁决。
```

### 11.5 InspectCommandOptionProvider

中文职责：生成观察 option。

输入：

```text
当前格
可见邻格
可见实体
```

输出：

```text
inspect_current_cell
inspect_visible_entity_xxx
inspect_adjacent_cell_xxx
```

规则：

```text
只能 inspect Unknown 之外的 cell。
只能 inspect 可见实体。
不能输出不可见实体 option。
```

### 11.6 WaitCommandOptionProvider

中文职责：生成等待 option。

规则：

```text
command_kind = Wait
command_key = wait
target = None
enabled = handler 存在
parameters tick_delta 默认由 materialize 或 handler 使用 1
```

### 11.7 RuntimeHandlerRegistrar

中文职责：生产 client server 注册 runtime-aware handler。

必须注册：

```text
GenerateWorld -> runtime-aware
Move -> runtime-aware
Inspect -> runtime-aware
Wait -> runtime-aware
Noop -> 可保留 stub
SystemTick -> 可保留当前 handler 或后续 runtime-aware
```

禁止：

```text
生产 ClientServerRuntimeFactory 继续注册 world_command/handlers 下的 GenerateWorld stub 作为真实世界生成。
```

### 11.8 ClientWorldBootstrapService

中文职责：保证 client server 启动后有一个可投影的世界。

推荐策略：

```text
ClientServerRuntimeFactory 构造时创建 WorldGridRuntime。
通过内部 GenerateWorld command 或 runtime generation service 建立初始世界。
生成过程必须可 trace。
bootstrap 时如果 runtime 未 ready，返回错误，不返回空 projection 成功。
```

如果第一版为了降低改动直接调用 `runtime.initialize + generateInitialWorld`，必须：

```text
写入 P56 自检报告说明这是临时 bootstrap 策略。
保留后续改为 GenerateWorld command 的 TODO。
测试必须证明不是前端 mock 地图。
```

## 12. 标准流程

### 12.1 服务启动流程

1. `client_server_main` 创建 `ClientServerRuntimeFactory`。
2. Factory 创建 `WorldGridRuntime`。
3. Factory 用固定 config 或配置文件生成初始世界。
4. Factory 注册 runtime-aware handlers。
5. Factory 创建 runtime projection bridge 与 command option bridge。
6. Factory 注入 `ClientProjectionAdapter` 和 `ClientAvailableCommandAdapter`。
7. Factory 创建 session/command/http gateway。
8. 启动 HTTP 服务。

失败处理：

```text
世界生成失败 -> server 启动失败或 bootstrap 返回 runtime_not_ready。
runtime bridge 未注入 -> architecture test 失败。
handler 缺失 -> option provider 不输出对应 option，并在诊断中记录 handler_missing。
```

### 12.2 Bootstrap 投影流程

1. 前端请求 `/api/client/bootstrap`。
2. `ClientSessionGateway` 取得或创建 session。
3. session 决定权威 actor_key，不信任前端后续 selection_context。
4. projection adapter 调用 runtime bridge。
5. runtime bridge 查 actor 和当前 layer。
6. runtime bridge 生成 full safe projection。
7. option adapter 调用 runtime option bridge。
8. session 保存 latest projection_version 和 available_commands snapshot。
9. 返回 `ClientBootstrapResponse`。

验收重点：

```text
full_projection.visible_cells.size() > 0
full_projection.visible_entities 至少包含 player
available_commands 至少包含 Wait/Inspect/Move
projection_version == runtime state version
```

### 12.3 Refresh 流程

1. 前端请求 `/api/client/refresh`。
2. 校验 session/client_id。
3. 读取 requested_scopes。
4. runtime bridge 构建当前 full projection。
5. option bridge 重建 options。
6. session 更新 projection_version 和 snapshot。
7. 返回 full projection。

注意：

```text
Refresh 是客户端重新同步，不执行世界动作。
Refresh 不能推进 world tick。
Refresh 不能生成新世界。
```

### 12.4 Move 命令流程

1. 前端摇杆或按钮选择当前 options 中的 Move option。
2. 前端提交 `submit_mode=option_id`。
3. `ClientCommandGateway` 校验 session、client_id、projection_version。
4. `ClientAvailableCommandAdapter` 从 session snapshot materialize option。
5. 生成 `WorldCommandDto`，source 固定 `PlayerInput`。
6. `WorldCommandPipeline` 执行 Move handler。
7. Move handler 再次校验邻接、阻挡、layer、actor 存在。
8. runtime 修改 actor 坐标并更新探索状态。
9. pipeline 返回 projection patch。
10. command gateway 重建 available_commands 并保存 snapshot。
11. 前端应用 patch 或 full refresh。

### 12.5 Inspect 命令流程

1. 前端选择可见格或可见实体对应 Inspect option。
2. 后端 materialize Inspect command。
3. runtime inspect 只允许 Unknown 之外的 cell 或可见实体。
4. 返回 event_feed 描述。
5. 不修改世界状态，除非后续设计“观察形成知识”，那应由 P49 bridge 处理。

### 12.6 Wait 命令流程

1. 前端提交 Wait option。
2. runtime 推进 tick。
3. 后续如果接入 NPC/野兽 tick，则由 SystemTick/AgentDecision 进入管线。
4. P56 最小版只要求 tick 和 projection version 一致。

## 13. 典型工程与玩法场景

### 13.1 启动后看到地图

玩家打开页面后，bootstrap 返回玩家周围格子。

前端可以画出草地、森林、水、山等安全 key。

### 13.2 玩家向东移动

如果东边是可走格，Move option enabled。

提交后玩家坐标变化，新的视野格子出现，旧可见格子变成 discovered。

### 13.3 玩家试图移动到水里

如果东边是 water 且 blocks_movement，Move option 可以 disabled。

如果前端强行提交旧 option 或伪造 command，也必须被 Move handler blocked。

### 13.4 玩家刷新页面

前端本地 projection 丢失，但 session 仍可 refresh。

后端重新输出真实 full projection，不重置世界。

### 13.5 野兽在视野边缘出现

如果野兽实体在 Visible cell，projection 输出该实体。

如果野兽在视野外，projection 不输出实体，只能由 area_effect 输出安全危险提示。

## 14. 与现有系统关系

### 14.1 与 P43 WorldCommand 管线

P56 不新增第二条执行入口。

所有玩家命令仍然必须经过：

```text

```

### 14.2 与 P44 WorldGridRuntime

P56 首先接入 P44。

P56 的地图、坐标、layer、可见性、移动校验都以 P44 runtime 为准。

### 14.3 与 P45 背包归属

P56 可以先只投影空 inventories，但接口必须预留 P45 adapter。

后续接入时，生成或采集物品必须通过 P45 的归属接口建立 `OnMap` 或 `InInventory` 状态，不能绕过 runtime 直接改数组。

### 14.4 与 P46 世界生成器

P56 初始世界必须来自后端生成能力。

不能在前端写死地图。

如果本阶段先用 `WorldGridRuntime::generateInitialWorld`，后续可以替换为 P46 更完整生成器，但 projection 和 command 协议不应改变。

P56 的 provider 架构必须允许后续新增：

```text
GatherCommandOptionProvider
ChopCommandOptionProvider
MineCommandOptionProvider
CraftCommandOptionProvider
```

新增 provider 不能影响 Move/Inspect/Wait。

### 14.6 与 P49 知识学习

P56 不直接形成知识。

但 P56 输出的 event、experience、command trace 必须保留，方便 P49 从世界行为生成知识。

### 14.7 与 P50-P52 Agent/NPC/野兽

P56 必须把 NPC/野兽当作地图实体投影，而不是前端特殊脚本。

可见就显示，不可见就不显示。

Agent 行动仍由后端 command/tick 决定。

### 14.8 与 P53 Client Protocol

P56 复用 P53 DTO，不新建一套前端协议。

必要时只扩展安全字段，不破坏已有 codec roundtrip。

### 14.9 与 P54 Client HTTP Gateway

P56 不新增另一个 HTTP 服务。

继续使用 `/api/client/*`。

### 14.10 与 P55 前端真实协议接入

P55 的前端只需要消费 P56 返回的数据。

前端不因为 P56 而获得任何规则权威。

## 20. 风险与控制

### 20.1 风险：为了快，把地图写死到前端

后果：后续所有玩法都会变成双轨规则。

控制：架构扫描禁止前端地图 mock；验收必须用 HTTP bootstrap 数据。

### 20.2 风险：projection bridge 泄露隐藏真相

后果：探索、认知、野兽生态全部失去意义。

控制：单测验证不可见实体和 unknown cell 不输出。

### 20.3 风险：option provider 和 handler 校验不一致

后果：按钮显示能走，执行却失败，或显示不能走但能伪造执行。

控制：provider 只做可尝试判断，handler 永远做最终校验；集成测试覆盖阻挡格。

### 20.4 风险：生产继续使用 stub handler

后果：前端能点按钮但世界不变。

控制：factory 测试检查 runtime-backed handler 注册；bootstrap 必须非空。

### 20.5 风险：projection version 与 runtime state version 脱节

后果：stale 校验失效，前端 patch 混乱。

控制：projection_version 以 runtime stateVersion 为准；移动后测试版本变化。

### 20.6 风险：一次性接太多玩法导致失控

后果：P56 变成大重构，难以验收。

控制：硬验收只要求地图、实体、移动；采集制作教学用 provider 扩展点保留。

### 20.7 风险：使用 snapshotForDebug 变成生产依赖

后果：debug 接口背负正式语义，后续难维护。

控制：优先增加正式 projection query port；如临时使用必须写 TODO 和后续任务。

## 24. 最终说明

P56 的成功标准不是“前端看起来像游戏”。

P56 的成功标准是：

```text
客户端拿到的每一格地图、每一个实体、每一个按钮，都来自真实后端世界。
```

只有 P56 成立，后续客户端界面才是在做游戏表现，而不是在伪造游戏。
