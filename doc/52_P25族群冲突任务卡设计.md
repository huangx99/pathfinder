# P25 族群冲突任务卡设计

## 1. 阶段定位

P25 是《先驱者》从“同族群共同作战”进入“不同族群关系与冲突”的第一阶段。

P23 已经完成族群状态：

```text
TribeState。
TribeMember。
TribeMemberRole。
TribeMorale。
TribeTrust。
TribeSplitRisk。
TribeProjection。
TribeStateInput。
TribeStateResolver。
```

P24 已经完成共同作战协调：

```text
GroupIntent。
AssistAction。
DefendAction。
PackTactic。
CombatCoordinationResult。
CombatCoordinationStateDraft。
CombatCoordinationProjection。
CombatCoordinationTrace。
```

P25 要解决的是：

```text
当两个或多个族群因为资源、领地、恐惧、误解、报复或生存压力发生对立时，系统能否以确定性、可解释、可复盘的方式表达敌意、威慑、对峙、撤退和关系变化。
```

一句话：

```text
P25 让“族群与族群之间不再只是互不相干”，而是能形成可验证的敌对关系和冲突结果，但仍不进入随机伤害、死亡、掉落或完整战争模拟。
```

P25 是族群冲突规则，不是完整战斗模拟器，不是文明解锁，不是外交大系统。

---

## 2. 本阶段目标

P25 必须在现有 `tribe` 模块内新增族群冲突能力。

它必须做到：

1. 定义族群关系与敌意状态：`TribeRelation`、`HostilityState`。
2. 定义冲突意图、立场、结果和事件：`ConflictIntentKind`、`ConflictStanceKind`、`ConflictOutcomeKind`、`ConflictEventKind`。
3. 定义冲突输入、解析器、结果、投影和 trace：`ConflictResolutionInput`、`GroupCombatResolver`、`ConflictResolutionResult`、`ConflictProjection`、`ConflictTrace`。
4. 只消费 P23/P24 的安全摘要和投影，不读取 raw `GameState`、`SaveGame`、hidden truth。
5. 使用 P24 产出的 `CombatCoordinationResult` 和 `CombatCoordinationStateDraft` 作为族群冲突的协作依据，不能在 P25 内绕过或重做共同作战。
6. 支持确定性的冲突结果：避免、威慑、对峙、强制撤退、关系恶化、停战意向。
7. 输出对 P23 的安全状态影响草稿，例如士气压力、安全压力、损失压力、分裂风险压力，但不能直接修改 `TribeState`。
8. 输出安全投影和可复盘 trace，给 P26/P31/P37 后续使用。
9. 明确禁止随机命中、随机伤害、具体 HP、死亡、击杀、掉落、真实战利品结算。
10. 必须写测试证明：同输入同配置结果完全一致，投影不泄露隐藏信息，P23 重复成员约束没有被破坏。

核心原则：

```text
P25 只做族群冲突摘要结算，不做完整战斗模拟。
P25 只处理族群之间的敌意、对峙、威慑、撤退和关系变化。
P25 不直接改 TribeState，只输出安全状态草稿。
P25 必须建立在 P24 CombatCoordinationResult 之上。
P25 的所有结果必须确定性、可解释、可测试、可复盘。
```

## 3. 玩家体验映射

P25 让玩家第一次感受到：

```text
世界里不只有我的族群。
其他族群会因为资源、领地、恐惧或误解和我们发生冲突。
我们是否团结、是否有共同作战能力，会影响冲突结果。
有时最好的结果不是击败对方，而是威慑、逼退、避免损失或达成暂时停战。
```

### 3.1 玩家未来感受

示例一：资源对峙。

```text
你们发现一片可食用红果。
另一个族群也靠近了这里。
他们人数更多，但士气不稳。
你的族群形成防御队形，守护者站在前面。
双方没有直接开战，但对方后退了一段距离。
你们获得了短暂采集窗口，两个族群的关系变得紧张。
```

示例二：低士气撤退。

```text
陌生族群在水源附近出现。
你的族群刚经历危险，士气很低。
探索者建议不要接近，守护者掩护学习者后撤。
你们避免了冲突，但对水源区域的控制压力上升。
```

这里的重点不是“杀死几个敌人”，而是：

```text
玩家看懂族群关系正在变化。
玩家理解共同作战、士气、信任、分裂风险会影响冲突结果。
玩家能从反馈中知道为什么对方退了、为什么我们退了、为什么关系恶化。
```

### 3.2 P25 需要证明的体验链

P25 的测试必须证明：

```text
敌意状态会影响冲突意图。
共同作战质量会影响威慑和撤退结果。
士气、信任、分裂风险会影响冲突承压能力。
冲突结果会产生关系变化和族群状态影响草稿。
投影只展示玩家可理解摘要，不展示隐藏真相。
```

## 5. 本阶段不做什么

P25 明确禁止：

```text
不做完整战斗模拟器。
不做命中率。
不做随机伤害。
不做 HP 扣减。
不做具体死亡。
不做击杀归属。
不做掉落和战利品。
不做装备、技能、武器数值系统。
不做大型外交系统。
不做联盟、附庸、贸易条约。
不做文明能力解锁，这是 P26。
不做对象反应，这是 P27。
不做危险探索压力，这是 P28。
不做 H5 新界面。
不做 HTTP / WebSocket / JSON API。
不做专家 NPC 教学。
不做 NPC 识别玩家谋害。
不读 raw GameState。
不读 SaveGame。
不读 hidden truth。
不使用 AgentRuntime 或 Policy 作为规则权威。
不让前端推导冲突结果。
```

P25 可以输出给后续阶段使用的安全摘要，但不能提前实现后续玩法。

## 6. 必须写死的注意点

这是 P25 最重要的边界，研发不能自行放宽。

### 6.1 P23 重复成员约束

P23 复审已经确认：

```text
同一个 TribeStateInput 内不能重复 member_id。
```

所以 P25 必须遵守：

```text
同一 tick 内同一成员多个冲突影响必须先聚合，或拆成多个 input tick。
如果 P25 输出 TribeStateInputDraft，member_events 内同一个 member_id 最多出现一次。
第一版优先输出 tribe-level 的 morale_delta、trust_delta、safety_pressure_delta、loss_pressure_delta、split_risk_delta。
除非必要，不要输出逐成员事件。
```

必须有测试证明：

```text
ConflictResolutionResult 生成的 TribeStateInputDraft 中 member_events 不含重复 member_id。
```

### 6.2 P24 共同作战约束

P24 已经是共同作战协调权威，P25 不能绕过它。

P25 必须遵守：

```text
P25 不能绕过 P24 的 CombatCoordinationResult 和 CombatCoordinationStateDraft。
P25 不能自己重新计算 GroupIntent、AssistAction、DefendAction、PackTactic。
P25 只能读取 P24 输出的 coordination_quality、coordination_score、projection、state_draft、trace 摘要。
如果调用方没有提供 CombatCoordinationResult，P25 必须返回 InvalidInput 或 MissingCoordinationResult。
```

正确管线：

```text
P23 TribeState
  -> P24 CombatCoordinationResolver
  -> CombatCoordinationResult
  -> P25 GroupCombatResolver
  -> ConflictResolutionResult
  -> P23 TribeStateInputDraft / P26 Civilization input / P31 Projection
```

错误管线：

```text
P23 TribeState
  -> P25 直接读成员并自己生成共同作战结果
```

### 6.3 安全摘要约束

P25 必须只消费安全摘要。

禁止字段和概念：

```text
raw_state
GameState
SaveGame
hidden_truth
true_trait
real_effect
real_damage
true_hp
actual_hp
actual_damage
damage_roll
random_hit
critical_hit
death
kill
loot
corpse
hidden_weapon_stat
```

如果输入、投影、trace、事件或测试 fixture 中出现这些字段，必须拒绝实现。

### 6.4 伤亡表达约束

P25 第一版不做具体死亡。

允许表达：

```text
loss_pressure_delta
casualty_pressure_delta
retreat_pressure_delta
safety_pressure_delta
morale_pressure_delta
```

禁止表达：

```text
member_dead
enemy_dead
kill_count
hp_delta
body_count
loot_count
```

中文解释：

```text
P25 可以说“这次冲突让族群承受了损失压力”，不能说“某个成员死亡”。
P25 可以说“对方被威慑后撤退”，不能说“击杀了对方三个单位”。
```

## 8. 枚举设计

### 8.1 HostilityState

族群之间的敌意状态。

```cpp
enum class HostilityState {
    Unknown,
    Neutral,
    Wary,
    Threatened,
    Hostile,
    OpenConflict,
    Retreating,
    Truce,
    TestOnly
};
```

中文说明：

| 枚举 | 中文 | 含义 |
|---|---|---|
| Unknown | 未知 | 非法默认值 |
| Neutral | 中立 | 没有明显敌意，也没有信任 |
| Wary | 警惕 | 接触时保持距离，容易被压力推动到冲突 |
| Threatened | 受威胁 | 一方认为另一方会威胁生存或资源 |
| Hostile | 敌对 | 已形成持续敌意，冲突概率高 |
| OpenConflict | 公开冲突 | 当前处于明确对抗、驱逐或争夺状态 |
| Retreating | 撤退中 | 一方正在脱离冲突，不代表关系修复 |
| Truce | 暂时停战 | 暂时停止冲突，但不是友好同盟 |
| TestOnly | 测试专用 | 正式 DTO 禁止 |

### 8.2 ConflictIntentKind

冲突意图。

```cpp
enum class ConflictIntentKind {
    Unknown,
    Avoid,
    Intimidate,
    DefendTerritory,
    ContestResource,
    Raid,
    Pursue,
    Retreat,
    NegotiateTruce,
    TestOnly
};
```

中文说明：

| 枚举 | 中文 | 含义 |
|---|---|---|
| Avoid | 避免冲突 | 尽量不接触，降低损失压力 |
| Intimidate | 威慑 | 展示力量逼对方退让，不做伤害结算 |
| DefendTerritory | 防守领地 | 保护当前区域、营地或安全范围 |
| ContestResource | 争夺资源 | 围绕食物、水源、火源、工具等摘要资源产生冲突 |
| Raid | 袭扰 | 施加压力、试探或短促攻击摘要，不结算死亡/掉落 |
| Pursue | 追赶 | 对撤退方施加追击压力，不结算击杀 |
| Retreat | 撤退 | 主动脱离冲突 |
| NegotiateTruce | 尝试停战 | 尝试降低敌意并形成暂时停战 |
| TestOnly | 测试专用 | 正式 DTO 禁止 |

### 8.3 ConflictStanceKind

冲突姿态。

```cpp
enum class ConflictStanceKind {
    Unknown,
    Passive,
    Defensive,
    Assertive,
    Aggressive,
    Desperate,
    TestOnly
};
```

中文说明：

| 枚举 | 中文 | 含义 |
|---|---|---|
| Passive | 被动 | 不主动推进，只承受局势 |
| Defensive | 防御 | 优先保护成员和资源 |
| Assertive | 强硬 | 保持压力，尝试迫使对方退让 |
| Aggressive | 进攻 | 主动施压，关系恶化风险高 |
| Desperate | 绝望 | 生存压力过高时的高风险姿态 |
| TestOnly | 测试专用 | 正式 DTO 禁止 |

### 8.4 ConflictOutcomeKind

冲突结果类型。

```cpp
enum class ConflictOutcomeKind {
    Unknown,
    NoContact,
    Avoided,
    Intimidated,
    Standoff,
    ForcedRetreat,
    MinorLossPressure,
    MajorLossPressure,
    TruceOffered,
    TestOnly
};
```

中文说明：

| 枚举 | 中文 | 含义 |
|---|---|---|
| NoContact | 未接触 | 没有实际形成冲突 |
| Avoided | 已避开 | 一方或双方主动避免冲突 |
| Intimidated | 威慑成功 | 一方通过协作或压力迫使对方退让 |
| Standoff | 对峙 | 双方都没有退，关系压力上升 |
| ForcedRetreat | 被迫撤退 | 一方因士气、协作或压力劣势后撤 |
| MinorLossPressure | 轻度损失压力 | 产生损失压力，但不记录具体死亡/伤害 |
| MajorLossPressure | 重度损失压力 | 产生高损失压力，仍不记录具体死亡/伤害 |
| TruceOffered | 提出停战 | 一方尝试降级冲突 |
| TestOnly | 测试专用 | 正式 DTO 禁止 |

### 8.5 ConflictEventKind

冲突事件类型。

```cpp
enum class ConflictEventKind {
    Unknown,
    RelationChanged,
    HostilityChanged,
    IntimidationApplied,
    RetreatTriggered,
    ResourceContested,
    ConflictPressureChanged,
    TruceProposed,
    TestOnly
};
```

中文说明：

| 枚举 | 中文 | 含义 |
|---|---|---|
| RelationChanged | 关系变化 | 族群关系评分或摘要变化 |
| HostilityChanged | 敌意变化 | 敌意状态发生迁移 |
| IntimidationApplied | 施加威慑 | 一方向另一方施加威慑压力 |
| RetreatTriggered | 触发撤退 | 一方进入撤退摘要状态 |
| ResourceContested | 资源争夺 | 围绕安全资源摘要发生争夺 |
| ConflictPressureChanged | 冲突压力变化 | 安全压力、损失压力、士气压力变化 |
| TruceProposed | 提出停战 | 出现停战意向或停战事件 |
| TestOnly | 测试专用 | 正式 DTO 禁止 |

## 9. 数据结构设计

### 9.1 TribeRelation

族群关系。

字段建议：

```text
source_tribe_id: TribeId
target_tribe_id: TribeId
hostility_state: HostilityState
trust_score: double 0..1
fear_pressure: double 0..1
grievance_pressure: double 0..1
resource_tension: double 0..1
last_conflict_tick: optional<Tick>
reason_keys: list<string>
```

说明：

```text
TribeRelation 表示 source 对 target 的关系视角。
它可以是有方向的，因为 A 害怕 B 不等于 B 害怕 A。
如果需要无方向关系，由上层同时维护 A->B 和 B->A 两条关系摘要。
```

校验：

```text
source_tribe_id 和 target_tribe_id 必须有效且不能相同。
所有 pressure/score 必须在 0..1。
hostility_state 不能是 Unknown/TestOnly。
reason_keys 只能放安全原因 key，不能放 hidden truth。
```

### 9.2 TribeRelationDraft

族群关系变化草稿。

字段建议：

```text
source_tribe_id: TribeId
target_tribe_id: TribeId
old_hostility_state: HostilityState
new_hostility_state: HostilityState
trust_delta: double -1..1
fear_pressure_delta: double -1..1
grievance_pressure_delta: double -1..1
resource_tension_delta: double -1..1
reason_keys: list<string>
```

说明：

```text
P25 不直接写入长期关系状态。
它输出 TribeRelationDraft，由后续关系仓库或规则管线统一应用。
```

### 9.3 ConflictPressureSummary

冲突场景压力摘要。

字段建议：

```text
conflict_key: string
resource_pressure: double 0..1
territory_pressure: double 0..1
fear_pressure: double 0..1
misunderstanding_pressure: double 0..1
survival_pressure: double 0..1
visibility_pressure: double 0..1
known_public_knowledge_ids: list<KnowledgeId>
reason_keys: list<string>
```

说明：

```text
conflict_key 是安全摘要，例如 contest_red_berry_patch、water_source_pressure、stranger_near_camp。
不能包含真实隐藏属性、真实伤害、真实掉落或真实 AI 意图。
```

### 9.4 ConflictParticipantTribe

参与冲突的族群摘要。

字段建议：

```text
tribe_id: TribeId
role_in_conflict: string
member_count_summary: int
active_population_summary: int
morale_summary: double 0..1
trust_summary: double 0..1
split_risk_summary: double 0..1
safety_pressure_summary: double 0..1
loss_pressure_summary: double 0..1
coordination_result: CombatCoordinationResult
coordination_state_draft: CombatCoordinationStateDraft
public_knowledge_ids: list<KnowledgeId>
reason_keys: list<string>
```

说明：

```text
ConflictParticipantTribe 是 P25 的输入视图，不是完整 TribeState。
coordination_result 必须来自 P24。
coordination_state_draft 必须来自 P24，用于合并 P24/P25 对 P23 的安全影响。
不能塞入 raw GameState、真实成员隐藏属性、真实 HP 或真实装备数值。
```

### 9.5 ConflictIntent

单方冲突意图。

字段建议：

```text
tribe_id: TribeId
intent_kind: ConflictIntentKind
stance_kind: ConflictStanceKind
confidence: double 0..1
pressure_score: double 0..1
reason_keys: list<string>
```

说明：

```text
ConflictIntent 是规则层产生的中间结果，trace 需要记录。
它不是 Agent 自由决策，也不是前端输入。
```

### 9.6 ConflictStateDraft

族群状态影响草稿。

## 11. 确定性规则建议

P25 第一版使用确定性评分，不使用随机数。

### 11.1 冲突承压评分

建议公式：

```text
coordination_component = P24 coordination_score
morale_component = morale_summary * 0.20
trust_component = trust_summary * 0.15
split_penalty = split_risk_summary * 0.20
safety_penalty = safety_pressure_summary * 0.15
loss_penalty = loss_pressure_summary * 0.10

conflict_strength = clamp(
  coordination_component * 0.40
  + morale_component
  + trust_component
  - split_penalty
  - safety_penalty
  - loss_penalty,
  0,
  1
)
```

说明：

```text
P24 协作分数是最大权重，因为 P25 是建立在共同作战上的冲突摘要。
士气和信任让族群更能维持对峙。
分裂风险、安全压力和损失压力削弱冲突承压能力。
```

### 11.2 意图决策

建议规则：

```text
如果 hostility_state 为 Neutral 且总压力低 -> Avoid 或 NoContact。
如果 resource_pressure 高且 hostility_state >= Wary -> ContestResource。
如果 territory_pressure 高且 coordination_score 中高 -> DefendTerritory。
如果 fear_pressure 高且 conflict_strength 低 -> Retreat。
如果 coordination_score 高且 opponent conflict_strength 低 -> Intimidate。
如果 hostility_state 为 OpenConflict 且 survival_pressure 高 -> Raid 或 Pursue 摘要。
如果双方 loss_pressure 都高且 hostility_state 不高于 Hostile -> NegotiateTruce。
```

注意：

```text
Raid/Pursue 只能表示冲突压力，不得结算击杀、死亡、掉落。
```

### 11.3 结果决策

建议规则：

```text
双方都 Avoid -> Avoided。
一方 Retreat，另一方不 Pursue -> ForcedRetreat。
一方 Intimidate 且 strength 差值 >= intimidation_threshold -> Intimidated。
双方 strength 差值 <= standoff_gap_threshold 且敌意较高 -> Standoff。
敌意高、生存压力高、双方都不退 -> MinorLossPressure 或 MajorLossPressure。
一方 NegotiateTruce 且对方压力低 -> TruceOffered。
```

结果必须：

```text
同输入同配置永远相同。
不依赖系统时间。
不依赖容器遍历无序结果。
不依赖随机数。
```

## 12. 状态流转

### 12.1 HostilityState 迁移建议

```text
Neutral -> Wary：首次资源争夺、陌生接近营地、误解压力上升。
Wary -> Threatened：安全压力或恐惧压力持续上升。
Threatened -> Hostile：威慑失败、重复资源争夺、损失压力上升。
Hostile -> OpenConflict：双方都选择强硬/进攻姿态且没有撤退。
OpenConflict -> Retreating：一方冲突强度明显劣势或主动撤退。
Retreating -> Wary/Hostile：脱离冲突后根据 grievance_pressure 决定。
Hostile/OpenConflict -> Truce：满足停战意向和压力降低条件。
Truce -> Wary：停战期后没有继续冲突。
Truce -> Hostile：停战期间再次争夺或威慑。
```

### 12.2 冲突处理顺序

固定顺序：

```text
1. validate input。
2. validate P24 coordination_result。
3. sanitize pressure_summary 和 reason_keys。
4. derive source_intent。
5. derive target_intent。
6. compute deterministic conflict_strength。
7. resolve outcome。
8. build relation drafts。
9. build conflict state drafts。
10. merge/validate P24 + P25 state effects。
11. build events。
12. build projection。
13. build trace。
14. validate result。
```

## 15. 示例用法

### 15.1 资源争夺示例

```text
输入：
- A 族群：士气 0.70，信任 0.80，分裂风险 0.10，P24 协作质量 Strong。
- B 族群：士气 0.45，信任 0.40，分裂风险 0.35，P24 协作质量 Weak。
- 关系：A->B Wary，B->A Wary。
- 压力：resource_pressure 0.80，territory_pressure 0.30。

P25 结果：
- A intent = Intimidate 或 ContestResource。
- B intent = Retreat 或 Avoid。
- outcome = Intimidated / ForcedRetreat。
- relation draft：双方警惕或敌意略上升。
- A state draft：士气小幅上升或安全压力下降。
- B state draft：撤退压力和恐惧压力上升。
- projection：对方退让，但关系紧张。
```

### 15.2 双方对峙示例

```text
输入：
- A/B 都是 Hostile。
- 双方 P24 协作质量接近。
- 资源压力和恐惧压力都高。

P25 结果：
- source_intent = ContestResource。
- target_intent = DefendTerritory。
- outcome = Standoff。
- relation draft：Hostile 可能迁移到 OpenConflict。
- state draft：双方安全压力和损失压力上升。
- projection：双方都没有退让，冲突升级。
```

### 15.3 停战示例

```text
输入：
- 双方 loss_pressure_summary 高。
- resource_pressure 已降低。
- 一方 intent = NegotiateTruce。
- 另一方 conflict_strength 不足以继续施压。

P25 结果：
- outcome = TruceOffered。
- hostility_state 可能向 Truce 迁移。
- projection：双方暂时停止冲突。
```
