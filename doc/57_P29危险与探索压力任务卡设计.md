# P29 危险与探索压力任务卡设计

## 1. 阶段定位

P29 是 P28 / P28.5 之后的探索压力层。

P28 已经解决：

```text
对象之间能不能发生反应。
```

P28.5 已经解决：

```text
物品、反应、成品、知识、NPC 学习能不能配置化并形成流水线。
```

P29 要解决：

```text
探索为什么有风险。
玩家和 NPC 如何感知危险。
危险如何产生安全后果草稿。
危险如何形成知识、认知、恐惧和族群压力。
危险如何推动逃生、谨慎、共同作战、族群分裂和后续文明能力。
```

一句话：

```text
P29 让“无知探索”开始有代价、有预警、有逃生、有学习结果，但不直接写死死亡、HP、掉落或隐藏真相。
```

---

## 2. 体验目标

P29 完成后，游戏体验应该从：

```text
我试一下这个东西有什么用。
```

升级为：

```text
我可以试，但试错可能带来伤害、恐惧、族群压力或资源损失。
我可以通过知识、工具、同伴、火把、观察和逃离降低危险。
我经历危险后，会形成可传播的知识。
```

玩家在一分钟内应该能理解：

```text
1. 当前有什么危险迹象。
2. 我为什么遇到危险。
3. 我能不能避开或降低危险。
4. 危险发生后影响了谁。
5. 这次经历形成了什么新知识。
6. 族群是否因此更紧张、更害怕、更容易分裂。
```

---

## 3. 本阶段不解决的问题

P29 不做完整战斗系统。

P29 不做：

```text
真实血量 HP。
死亡结算。
尸体和掉落。
完整敌人 AI。
完整地图寻路。
完整背包 UI。
完整动画和音效。
随机命中或暴击。
```

P29 只做：

```text
危险评估。
危险触发。
危险后果草稿。
恐惧信号。
安全反馈。
知识/认知/族群压力桥接。
逃离与规避结果。
```

---

## 4. 设计原则

P29 必须遵守：

```text
1. 条件判断必须复用 P27 ConditionExpressionEvaluator。
2. 对象危险必须复用 P28 / P28.5 的 ReactionRuntimeObject / ReactionInputBuilder / ObjectReactionRule。
3. 危险知识必须进入 P18-P21 的知识/传播/学习链路，不能单独写一套知识系统。
4. 认知变化必须能映射到 P15/P21 认知与学习闭环。
5. 族群影响必须通过 P23 TribeStateInput / state draft 表达。
6. 共同作战和族群冲突必须继续复用 P24/P25 的投影与草稿。
7. H5 只能展示 DangerFeedback / projection，不能读取 raw state 或 hidden truth。
8. P29 不得直接修改 WorldObject，只输出 draft / event / trace。
```

## 6. 核心枚举

### 6.1 DangerSourceKind

表示危险来源。

```cpp
enum class DangerSourceKind {
    Unknown,
    Environment,        // 环境危险，例如寒冷、沼泽、黑暗
    ObjectReaction,     // 对象反应危险，例如火、毒果、腐烂食物
    Creature,           // 生物危险，例如狼、蜘蛛
    TribeConflict,      // 族群冲突危险
    ResourcePressure,   // 资源压力危险，例如缺水、缺食物
    KnowledgeConflict,  // 知识冲突危险，例如错误传授导致争执
    TestOnly
};
```

中文解释：

```text
Environment：环境本身危险。
ObjectReaction：物品或反应带来的危险。
Creature：动物、怪物、NPC 这种 agent 造成的危险。
TribeConflict：族群冲突造成的危险。
ResourcePressure：资源不足导致的危险。
KnowledgeConflict：知识冲突或错误知识导致的危险。
```

### 6.2 HazardTriggerKind

表示危险如何触发。

```cpp
enum class HazardTriggerKind {
    Unknown,
    Explore,        // 探索时触发
    Approach,       // 靠近时触发
    Touch,          // 触碰时触发
    Eat,            // 食用时触发
    Use,            // 使用时触发
    Combine,        // 合成/反应时触发
    Rest,           // 停留/休息时触发
    Teach,          // 传播知识时触发
    Conflict,       // 族群对抗时触发
    TestOnly
};
```

中文解释：

```text
Explore：探索区域。
Approach：靠近危险源。
Touch：摸一下。
Eat：吃一口。
Use：使用物品。
Combine：把对象组合反应。
Rest：在危险区域停留。
Teach：教别人错误或冲突知识。
Conflict：族群冲突。
```

### 6.3 DangerSeverity

表示危险严重度，但不等于 HP 或死亡。

```cpp
enum class DangerSeverity {
    Unknown,
    None,       // 没有明显危险
    Notice,     // 轻微警觉
    Strain,     // 压力/不适
    Harm,       // 受伤/中毒/损耗的安全摘要
    Severe,     // 严重危险，需要逃离或援护
    Critical,   // 极端危险，但仍不直接表达死亡
    TestOnly
};
```

中文解释：

```text
Notice：你感觉不对劲。
Strain：你出现压力、不适、轻微消耗。
Harm：你受到了明确负面影响。
Severe：危险很高，需要处理。
Critical：极高危险，但前端仍不能显示死亡结算。
```

### 6.4 DangerDecision

表示危险裁决结果。

```cpp
enum class DangerDecision {
    Unknown,
    Safe,
    Warned,
    Exposed,
    Avoided,
    Escaped,
    BlockedByCondition,
    NoMatchingHazard,
    Rejected,
    TestOnly
};
```

中文解释：

```text
Safe：安全。
Warned：只是预警。
Exposed：危险发生。
Avoided：通过知识/工具/距离避免。
Escaped：触发了危险但逃离成功。
BlockedByCondition：条件不满足，危险未触发。
NoMatchingHazard：没有匹配危险规则。
Rejected：输入非法或被安全策略拒绝。
```

### 6.5 FearSignalKind

表示恐惧信号类型。

```cpp
enum class FearSignalKind {
    Unknown,
    None,
    Caution,
    Alarm,
    Panic,
    GroupAnxiety,
    TestOnly
};
```

中文解释：

```text
Caution：谨慎。
Alarm：警觉。
Panic：恐慌。
GroupAnxiety：族群整体焦虑。
```

### 6.6 CountermeasureKind

表示降低危险的手段。

```cpp
enum class CountermeasureKind {
    Unknown,
    Knowledge,       // 已知危险知识
    Tool,            // 工具，例如火把、矛、石片
    GroupSupport,    // 同伴/共同作战
    Distance,        // 保持距离
    EscapeRoute,     // 逃离路线
    Civilization,    // 文明能力，例如火种、建筑、药物
    TestOnly
};
```

中文解释：

```text
Knowledge：知道这东西危险。
Tool：用工具降低危险。
GroupSupport：同伴帮忙。
Distance：别靠太近。
EscapeRoute：可以逃跑。
Civilization：文明能力带来的保护。
```

## 7. 核心 DTO

### 7.1 ThreatProfile

表示一个可被识别、评估和反馈的危险源。

```cpp
struct ThreatProfile {
    std::string threat_key;
    DangerSourceKind source_kind;
    pathfinder::foundation::EntityId source_entity_id;
    pathfinder::foundation::ObjectId source_object_id;
    pathfinder::foundation::ObjectDefinitionId source_definition_id;
    std::string public_name_key;
    double base_pressure;
    double proximity_pressure;
    double visibility;
    double predictability;
    DangerSeverity base_severity;
    std::vector<std::string> safe_tag_keys;
    std::vector<std::string> state_keys;
    std::vector<std::string> reason_keys;
};
```

字段解释：

```text
threat_key：危险配置 key。
source_kind：危险来源类型。
source_entity_id：如果危险来自生物/agent，用这个。
source_object_id：如果危险来自运行态物品，用这个。
source_definition_id：如果危险来自物品定义，用这个。
public_name_key：前端可显示名称 key。
base_pressure：基础压力 0..1。
proximity_pressure：接近压力 0..1。
visibility：能否被看见/察觉 0..1。
predictability：能否预测 0..1。
base_severity：基础严重度。
safe_tag_keys：安全标签，只能写公开分类。
state_keys：公开状态，例如 hungry、agitated、burning、poisonous_hint。
reason_keys：安全原因 key。
```

注意：

```text
ThreatProfile 不允许包含真实伤害、死亡、掉落、隐藏属性。
```

### 7.2 HazardRule

表示一个危险触发规则。

```cpp
struct HazardRule {
    std::string rule_key;
    HazardTriggerKind trigger_kind;
    DangerSourceKind source_kind;
    std::vector<pathfinder::condition::ConditionExpressionRef> condition_refs;
    std::vector<CountermeasureRequirement> countermeasure_requirements;
    DangerSeverity severity;
    double pressure_delta;
    double fear_delta;
    double morale_delta;
    double trust_delta;
    double split_risk_delta;
    std::string feedback_key;
    std::string knowledge_effect_key;
    std::vector<std::string> safe_tags;
};
```

字段解释：

```text
rule_key：规则 key。
trigger_kind：什么行为触发。
source_kind：危险来自哪里。
condition_refs：P27 条件表达式。
countermeasure_requirements：哪些反制手段可以降低危险。
severity：危险严重度。
pressure_delta：探索压力变化。
fear_delta：恐惧变化。
morale_delta：族群士气变化。
trust_delta：族群信任变化。
split_risk_delta：分裂风险变化。
feedback_key：安全反馈 key。
knowledge_effect_key：形成的知识效果 key。
safe_tags：安全标签。
```

### 7.3 CountermeasureRequirement

表示反制条件。

```cpp
struct CountermeasureRequirement {
    CountermeasureKind kind;
    std::string requirement_key;
    pathfinder::condition::ConditionExpressionRef condition_ref;
    double mitigation_score;
    std::vector<std::string> reason_keys;
};
```

示例：

```text
kind = Tool
requirement_key = has_burning_torch
condition_ref = condition:target_state:eq:burning
mitigation_score = 0.45
```

含义：

```text
有正在燃烧的火把，可以降低某些野兽危险。
```

## 9. 典型流程

### 9.1 腐烂红果危险

```text
玩家吃腐烂红果。
P28.5 先处理对象反应：def_red_berry_rotten + eat -> poison。
P29 读取该反应输出或危险规则。
生成：
- InjuryStateDraft: poisoned_mild / Harm
- FearSignal: Caution
- DangerFeedback: “你感到身体不适，这种食物可能有风险。”
- DangerKnowledgeDraft: def_red_berry_rotten + eat -> poison_risk
- TribeDangerPressureDraft: safety_pressure 小幅上升
```

### 9.2 火把降低狼威胁

```text
玩家探索时附近有狼。
ThreatProfile: creature wolf。
HazardRule: wolf_approach_risk。
CountermeasureRequirement: has_burning_torch。
如果玩家有 burning + fuel>0 的火把：
- decision = Avoided 或 Warned
- severity 降低
- fear_delta 降低
- knowledge_draft: torch helps against wolf_hint
```

### 9.3 错误知识导致族群压力

```text
玩家把“腐烂红果可食用”错误传给同伴。
同伴尝试后产生危险。
P29 生成 knowledge_conflict_pressure。
TribeDangerBridge 转成 TribeStateInput。
P23 更新 split_risk。
如果多次发生，族群进入 Tense / Fracturing。
```

### 9.4 危险触发共同作战

```text
危险源为狼群。
P29 输出 FearSignal + TribeDangerPressureDraft。
P24 读取安全压力摘要，选择 DefendGroup / ScreenRetreat / FocusThreat。
P29 不直接决定战斗结果，只提供危险压力和逃生/援护输入。
```

## 10. 与现有系统关系

### 10.1 与 P27 条件表达式

P29 所有条件必须写成：

```text
ConditionExpressionRef
```

禁止：

```text
if condition_key == "has_torch"
if object_key == "wolf" && has_fire
```

正确：

```text
condition:target_state:eq:burning
condition:source_resource:gte:fuel:1
condition:knowledge:has:def_wolf:avoid_fire
```

如果现有 P27 暂不支持某个 domain，P29 只能新增 domain 支持，不能绕过 evaluator。

### 10.2 与 P28/P28.5 对象反应

危险不是替代对象反应。

正确分工：

```text
P28/P28.5：吃了什么、用了什么、合成了什么。
P29：这个行为是否带来危险、恐惧、压力、逃生和知识。
```

示例：

```text
P28.5: def_red_berry_rotten + eat -> poison
P29: poison 造成 Harm severity、fear signal、tribe safety pressure
```

### 10.3 与 P18-P21 知识/学习

P29 只生成：

```text
DangerKnowledgeDraft
ReactionKnowledgeDeliveryDraft
```

不直接写：

```text
KnowledgeRepository
```

知识状态由 P18-P21 决定。
