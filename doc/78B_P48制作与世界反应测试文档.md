# P48 制作与世界反应测试文档

更新时间：2026-05-21
阶段：P48

## 1. 测试目标

P48 测试必须证明：

```text
JSON reaction 可以驱动 Craft command。
材料从真实 inventory / map 中解析。
材料消耗走 P45 权威接口。
产物生成走 P45 / location port 权威接口。
失败路径不污染状态。
事件、投影、experience 能被后续系统消费。
```

## 2. 建议测试目标

```text
pathfinder_tests_world_reaction
pathfinder_tests_world_reaction_integration
```


## 3. P45 Inventory 补强测试

必须新增或扩展 inventory 测试。

### 3.1 world_inventory_consume_to_nowhere_reduces_stack

准备：

```text
actor inventory 有 wood x3。
ConsumeToNowhere wood x1。
```

断言：

```text
transfer ok。
wood quantity 变为 2。
changed_inventory_ids 包含 actor inventory。
changed_entity_ids 包含 wood entity。
事件存在。
```

### 3.2 world_inventory_consume_to_nowhere_removes_stack_at_zero

准备：

```text
actor inventory 有 wood x1。
ConsumeToNowhere wood x1。
```

断言：

```text
entry 被移除。
WorldEntityInstance 变成 Nowhere 或 destroyed。
item location 清除或 Nowhere。
```

### 3.3 world_inventory_consume_to_nowhere_insufficient_fails

准备：

```text
actor inventory 有 wood x1。
ConsumeToNowhere wood x2。
```

断言：

```text
transfer failed。
failure_kind = QuantityInsufficient。
wood quantity 仍为 1。
```

### 3.4 world_inventory_spawn_to_inventory_adds_item

准备：

```text
actor inventory 存在。
SpawnToInventory torch x1。
```

断言：

```text
transfer ok。
actor inventory 有 torch。
WorldEntityInstance location_kind = InInventory。
item location = InInventory。
```

### 3.5 world_inventory_spawn_to_inventory_merges_stack

准备：

```text
actor inventory 已有 torch stack x1。
SpawnToInventory torch x2 stack_key 相同。
```

断言：

```text
torch stack quantity = 3。
不产生重复 stack，除非 stack_key 不同。
```

## 4. WorldReaction 单元测试

### 4.1 world_reaction_request_validate_basic

覆盖：

```text
有效 request 通过。
空 reaction_command_id 失败。
空 actor_key 失败。
空 reaction_key 失败。
Unknown policy 失败。
未知 action 失败。
```

### 4.2 world_reaction_validate_reaction_missing

准备：

```text
reaction_key 不存在。
```

断言：

```text
validate failed。
failure = ReactionMissing。
状态不变。
```

### 4.3 world_reaction_validate_actor_missing

准备：

```text
actor_key 不存在。
```

断言：

```text
ActorMissing。
```

### 4.4 world_reaction_validate_input_missing

准备：

```text
reaction 需要 wood + camp_fire。
actor 没 wood，地图没 wood。
```

断言：

```text
InputMissing。
不生成 torch。
```

### 4.5 world_reaction_validate_state_mismatch

准备：

```text
reaction 需要 wood state=processed。
actor inventory 有 wood 但无 processed state。
```

断言：

```text
InputStateMismatch 或 InputMissing。
不消耗 wood。
```

### 4.6 world_reaction_validate_quantity_insufficient

准备：

```text
reaction 需要 wood min=2。
actor inventory 有 wood x1。
```

断言：

```text
QuantityInsufficient。
```

### 4.7 world_reaction_validate_catalyst_not_consumed

准备：

```text
reaction inputs: wood + camp_fire。
consume: wood -1。
camp_fire 在当前格 OnMap。
```

断言：

```text
validate 成功。
consume_drafts 只包含 wood，不包含 camp_fire。
```

### 4.8 world_reaction_apply_craft_torch_to_inventory

准备：

```text
actor inventory 有 processed wood x1。
actor 当前格有 camp_fire。
执行 wood_plus_fire_make_torch。
```

断言：

```text
result ok。
wood 被消耗。
camp_fire 仍在 OnMap。
torch 进入 actor inventory。
events 包含 WorldReactionCrafted / CraftSucceeded。
state_deltas 包含 input consumed 和 output created。
experiences 包含 make_torch effect。
```

### 4.9 world_reaction_apply_output_id_deterministic

准备：

```text
同一个 command_id + reaction + output。
```

断言：

```text
output entity_id 稳定。
不使用 state_version / time / random。
```

### 4.10 world_reaction_failure_does_not_pollute_state

准备：

```text
缺 camp_fire。
```

断言：

```text
wood 不减少。
torch 不生成。

```

### 4.11 world_reaction_rejects_unknown_output_policy

准备：

```text
output_location_policy = bad_policy。
```

断言：

```text
失败。
不消耗、不生成。
```

### 4.12 world_reaction_rejects_duplicate_input_object_key

准备：

```text

```

断言：

```text
ReactionInvalid。
```

## 6. P46 -> P47 -> P45 -> P48 端到端测试

### 6.1 world_generation_harvest_pickup_then_craft_torch

流程：

```text
1. 生成世界。
2. 找到或构造 young_tree resource node。
3. P47 chop 生成 wood OnMap。
4. P45 pickup wood。
5. 给 wood 加 processed state，或使用测试 reaction 要求无 state。
6. 在 actor 当前格生成 camp_fire。
7. P48 craft torch。
```

断言：

```text
资源节点 charge 减少。
wood 从 OnMap 进入 inventory。
craft 后 wood 减少。
torch 在 inventory。
```

说明：

```text
如果当前 P47 生成 wood 不带 processed state，集成测试可以构造 processed wood；但必须在测试说明中写清这是 P48 对接点，后续 P49/P50 或材料加工阶段再打通 processed wood 来源。
```

## 7. 禁止的假测试

不允许：

```text
只 new 一个 result 然后 assert ok。
只测试 toString。
手动 push torch 到 inventory 后说 craft 成功。
测试里直接改 inv.entries 模拟消耗。
只检查 events，不检查运行态。
只遍历 outputs 但不 assert 非空。
```



