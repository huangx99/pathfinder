# P38 世界交互结算与可见后果任务卡设计

## 1. 阶段定位

P38 是“世界交互结算与可见后果”阶段。

它接在 P37 最小剧情闭环之后，不是单纯 H5 界面美化，也不是继续添加更多物品。P37 已经证明后端可以把“红果、斧头、木头、火种、火把、野兽”等内容串成一个文本流程，但试玩暴露出核心问题：

```text
玩家做了动作，但世界没有足够变化。
玩家形成了知识，但知识没有足够改变局势。
同伴、野生生物和其他 NPC 学到了或观察到了知识，但还不会基于知识自主行动。
野兽显示在场景里，但没有真正构成事件压力。
```

所以 P38 的重点是把“动作结果”从文字反馈升级为后端权威的世界变化：

```text
吃红果 -> 红果数量减少，饥饿缓解。
砍木头 -> 木头被处理，斧头耐久下降，产出可用材料。
点燃干草 -> 干草消耗，火源出现。
制作火把 -> 材料组合，火把可用。
野兽靠近 -> 威胁状态变化。
火把驱赶野兽 -> 威胁下降或解除。
教同伴 -> 同伴获得可执行的自动协助能力；野生 Agent 观察到火的危险后，也能形成避让倾向。
```

P38 不是推翻 P15-P37，而是在它们之上补齐“世界会变”的运行层。

### 1.1 当前已有基础

P38 必须复用以下前置能力：

```text
doc/21_Agent系统设计.md：Agent 可以代表玩家、普通族人、敌对族群、野生生物、脚本控制者和强化学习训练体；Agent 只产生意图，不直接修改世界。
```


```text
P21：学习闭环，动作反馈进入认知、记忆、知识。
P27：条件表达式，所有条件判断必须走受限表达式或既有 condition key。
P28/P28.5：对象反应与配置化物品反应流水线。
P29：危险与探索压力。
P32：H5 安全协议投影。
P33：H5 可玩壳。
P34：因果归因与时间窗口。
P35：对象状态、工具耐久、维护。
P36：行为目标知识。
P37：最小剧情闭环和第一天生存内容。
```

### 1.2 P38 的一句话定义

P38 要让玩家明确感受到：

```text
我做的动作改变了世界；
我学到的知识改变了选择；
我教会同伴的知识会改变同伴行为；野生 NPC/野兽观察到经验后，也会改变自己的行动倾向；
我是否准备充分会改变第一夜结局。
```

## 2. 为什么现在做

### 2.1 P37 暴露的问题必须先解决

用户试玩 P37 后出现了几个典型问题：

1. `使用靠近的野兽` 被误判为 `用火把驱赶野兽`。
2. 为修复误判，目标匹配被收紧后，一些对象卡按钮变成 `feedback_not_found`。
3. 修复按钮后，玩家仍然感觉“玩法是什么”。
4. 野兽没有真实推进和结算。
5. 同伴被教会知识后不会自主行动，野生 NPC/野兽也不会基于观察和经验行动。
6. 红果、木头、干草、火把等对象没有足够可见的消耗、生成、状态变化。

这些不是单纯 UI 文案问题，而是交互结算层缺失。

### 2.2 如果继续做 UI 会掩盖问题

原计划 P38 是“H5 玩家界面去调试化”。但现在必须调整顺序。

如果先美化 UI，会出现：

- 界面更像游戏，但规则仍像调试器。
- 玩家看到漂亮按钮，但点击后世界不变。
- 同伴界面更好看，但同伴仍不会做事；野生 NPC/野兽也仍只是静态危险。
- 野兽显示更明显，但仍没有压力。

所以 P38 必须先做后端世界交互结算，再让 P39/P40 做界面体验和内容扩展。

### 2.3 这是后续扩展的地基

未来要支持：

```text
毒蘑菇、普通蘑菇、湿木头、雨天、火把燃尽、野兽学习火的危险、野兽驯服、同伴守夜、野生 NPC 自主行动、族群分工、地图探索、文明升级。
```

如果 P38 不先统一“动作如何改变世界”，后续每加一个内容都会变成临时 if/else。

## 3. 阶段目标

P38 完成后，系统必须具备以下能力：

1. 后端存在统一的世界交互结算入口。
2. 每次玩家动作可以产生结构化世界变更。
3. 世界变更包括对象数量、对象状态、对象生成、对象消耗、角色状态、危险状态、剧情目标状态。
4. H5 只能显示后端投影，不能自己判断物品是否消失、火是否生成、野兽是否被驱赶。
5. 对象卡按钮必须全部可执行，不能出现玩家点击合法按钮却得到 `feedback_not_found`。
6. 目标动作必须精确匹配，不允许 `使用野兽` 误触发 `用火把驱赶野兽`。
7. 红果被吃后可见减少或消失。
8. 木头被砍后产生可用材料或状态变化。
9. 干草被点燃后消耗，火源出现。
10. 制作火把后火把成为可用对象。
11. 野兽威胁随时间推进，并能被火把等手段降低或解除。
12. 教会同伴关键知识后，同伴可以在满足条件时自动协助一次；野生 Agent 至少能基于危险经验做一次避让/趋避行动。
13. 成功撑过第一夜有明确结算反馈。
14. 失败或准备不足有明确故事后果。
15. 所有变化进入测试，不依赖人工点击确认。

## 4. 本阶段不解决的问题

### 4.1 不做完整背包系统

P38 只做最小世界对象实例和数量变化，不做：

```text
背包重量
堆叠规则
装备栏
物品栏拖拽
容器嵌套
复杂掉落表
```

后续正式背包可以基于 P38 的对象实例扩展。

### 4.2 不做完整地图系统

P38 仍然是地点式小场景，不做：

```text
坐标
寻路
视野
碰撞
网格
地图雾
```

地点可以作为 `location_key` 存在，但不引入正式地图。

### 4.3 不做复杂战斗系统

野兽仍是危险事件，不是完整战斗 Agent。

不做：

```text
野兽 HP
攻击回合
技能冷却
伤害公式
掉落尸体
战斗动画
```

但 P38 必须让野兽威胁可推进、可反制、可成功/失败结算。

### 4.4 不做完整 Agent AI

P38 只做“知识驱动的最小自主行动”，覆盖同伴、野生生物和其他 NPC 的统一抽象入口。

不做：

```text
自由规划
多目标决策树
复杂情绪
长期任务队列
多 NPC 调度
生态群体模拟
```

P38 的 Agent 只在明确事件中检查：是否观察到、是否知道、是否具备行动能力、是否有对象、是否处于可行动状态。对同伴额外检查信任；对野生生物额外检查本能、恐惧、趋避和敌意。

### 4.5 不做完整经济/文明系统

P38 不做资源经济、生产建筑、科技树升级。

但必须保证世界变更可以被后续文明系统读取。

### 4.6 不做自由文本大模型理解

仍然优先使用按钮和有限中文指令。

自由文本可以保留，但不能作为核心验收基础。

## 5. 核心设计原则

### 5.1 后端结算唯一权威

所有对象变化、威胁变化、Agent 行动和目标状态都必须由后端结算。

禁止：

- 前端点按钮后自己隐藏红果。
- 前端根据文字猜测火把生成。
- 前端自己判断野兽是否退去。
- 前端自己决定同伴是否帮忙。
- 前端自己决定狼、野兽、野生 NPC 是否学习、害怕火、逃跑、攻击或驯服。

### 5.2 动作反馈和世界变更必须分离但关联

P21 学习反馈回答：

```text
这次动作让我学到了什么。
```

P38 世界变更回答：

```text
这次动作改变了世界什么。
```

它们必须有关联 trace，但不能互相替代。

例如：

```text
吃红果
- 学习反馈：red_berry + eat -> restore_hunger
- 世界变更：red_berry.quantity -1，actor.hunger -20
```

### 5.3 先实例，后定义

对象定义说明“红果是什么”。

对象实例说明“当前场景里这颗/这一堆红果还剩多少、状态如何、在哪里”。

P38 必须引入最小对象实例概念，避免把“红果定义”误当成“红果库存”。

### 5.4 所有变化必须可投影

如果后端发生了变化，H5 必须能显示：

- 对象消失或数量减少。
- 新对象出现。
- 对象状态变化。
- 危险变化。
- Agent 行动，包括同伴、野生生物、敌对 NPC。
- 目标进度。

禁止只在 debug key 中变化但玩家看不到。

### 5.5 按规则组合，不按剧情硬编码

P38 可以服务第一天生存，但不能写死“第一天第 7 步必须成功”。

应该通过：

- 条件。
- 对象状态。
- 知识状态。
- 危险状态。
- 时间阶段。
- Agent 能力、知识、观察和行动倾向。

组合出结果。

### 5.6 合法按钮必须可执行

H5 投影出的每个按钮都必须满足：

```text
点击后不会 feedback_not_found。
点击后要么产生有效变化，要么产生自然中文失败原因。
```

失败原因可以是：

- 缺材料。
- 缺目标。
- 工具状态不对。
- 时间不对。
- 危险太高。
- 同伴不懂。
- 野生生物不害怕、没观察到、被激怒或无法行动。

但不能暴露工程错误。

### 5.7 Agent 行动必须由观察、知识和策略驱动

同伴不是因为剧情写死才帮忙，狼也不是因为剧情写死才害怕火。

P38 必须把“能自主行动的实体”统一看成 Agent-controlled Actor：玩家、同伴、野生生物、敌对 NPC 都可以通过同一套观察-意图-命令入口行动。

同伴自动协助至少检查：

- 是否拥有相关知识。
- 知识是否达到可用状态。
- 是否知道目标对象关系。
- 是否有可用对象。
- 是否处于能行动状态。
- 是否信任玩家或愿意合作。

野生生物自主行动至少检查：

- 是否观察到火、火把、伤害、食物、威胁等线索。
- 是否拥有本能知识或经验知识。
- 是否能形成行动意图，例如靠近、逃离、攻击、警戒、跟随。
- 是否能通过规则命令提交行动，而不是直接改世界。

### 5.8 可扩展优先，但本阶段必须小闭环

P38 必须为未来提供扩展点，但实现范围只覆盖第一天生存。

后续扩展不能要求重写 P38 架构。

## 6. 模块划分

| 模块 | 中文含义 | 职责 | 输入 | 输出 | 禁止事项 |
|---|---|---|---|---|---|
| `pathfinder_world` | 世界交互模块 | 管理对象实例、世界变更、角色状态和场景资源 | 动作结果、反应结果、危险结果 | `WorldChangeSet`、`WorldSnapshot` | 不形成知识、不做 UI |
| `WorldObjectInstance` | 世界对象实例 | 表达当前场景中的具体对象/资源 | 对象定义 key、数量、状态 | 可投影对象状态 | 不替代对象定义 |
| `WorldInteractionService` | 交互结算服务 | 把一次动作转换为世界变化 | `WorldInteractionInput` | `WorldInteractionResult` | 不解析自由文本 |
| `WorldChangeApplier` | 世界变更应用器 | 安全应用对象/角色/危险变化 | `WorldChangeSet`、旧状态 | 新状态 | 不绕过校验直接改状态 |
| `WorldProjectionMapper` | 世界投影映射 | 把世界变化转为 H5 可见提示 | `WorldSnapshot`、`WorldChangeSet` | 对象卡、场景提示、变化日志 | 不裁决规则 |
| `AgentAutonomyService` | Agent 自主行动服务 | 判断同伴、野生生物、NPC 是否能基于观察/知识产生行动 | 世界状态、Agent 观察、知识、危险状态 | 行动意图、协助动作或跳过原因 | 不做完整 AI，不直接改世界 |
| `ThreatEventBridge` | 危险事件桥 | 把野兽/寒冷等危险接入世界变化，并为野生 Agent 提供观察事件 | 时间、威胁状态、反制动作、Agent 行动 | 威胁变化、结局变化、观察事件 | 不做战斗系统 |
| `InteractionTestHarness` | 交互测试夹具 | 测试完整动作链 | 场景脚本 | 可断言状态和投影 | 不成为运行时依赖 |

## 7. 核心枚举

### 7.1 `WorldObjectInstanceKind` 世界对象实例类型

| 枚举值 | 中文含义 | 使用场景 |
|---|---|---|
| `Unknown` | 未知 | 非法默认值，运行时不允许 |
| `ResourceStack` | 资源堆 | 一堆红果、一段木头、一束干草 |
| `ToolInstance` | 工具实例 | 斧头、磨石、火把 |
| `ConsumableInstance` | 消耗品实例 | 红果、腐烂红果、草药 |
| `GeneratedInstance` | 生成物实例 | 木材、火堆、火把 |
| `ThreatMarker` | 威胁标记 | 靠近的野兽、寒意、毒雾 |
| `CompanionHeldItem` | 同伴持有物 | 后续同伴拿着火把、工具 |
| `TestOnly` | 测试专用 | 只允许测试 |

规则：

- `Unknown` 不允许进入有效状态。
- `ThreatMarker` 可以显示但不一定可直接采集。
- `GeneratedInstance` 必须有来源 change trace。

### 7.2 `WorldChangeKind` 世界变更类型

| 枚举值 | 中文含义 | 示例 |
|---|---|---|
| `Unknown` | 未知 | 非法默认值 |
| `ObjectQuantityChanged` | 对象数量变化 | 红果 -1 |
| `ObjectConsumed` | 对象被消耗 | 干草消失 |
| `ObjectGenerated` | 对象生成 | 生成火把 |
| `ObjectStateChanged` | 对象状态变化 | 斧头 sharpness -1 |
| `ActorNeedChanged` | 角色需求变化 | 饥饿下降 |
| `ActorConditionChanged` | 角色状态变化 | 中毒、不适、温暖 |
| `ThreatLevelChanged` | 威胁等级变化 | 野兽威胁 -2 |
| `ThreatResolved` | 威胁解除 | 野兽退去 |
| `ObjectiveProgressed` | 目标推进 | 火源准备完成 |
| `AgentActionQueued` | Agent 行动入队 | 同伴准备守火、狼准备后退 |
| `AgentActionExecuted` | Agent 行动执行 | 同伴举火把驱赶野兽、狼避开火光 |
| `StoryOutcomeReached` | 剧情结局达成 | 撑过第一夜 |
| `TestOnly` | 测试专用 | 只允许测试 |

### 7.3 `InteractionFailureKind` 交互失败类型

| 枚举值 | 中文含义 | 玩家文案方向 |
|---|---|---|
| `Unknown` | 未知 | 不允许展示 |
| `MissingObject` | 对象不存在 | 这里已经没有这个东西了 |
| `InsufficientQuantity` | 数量不足 | 红果已经吃完了 |
| `MissingTarget` | 缺目标 | 需要指定对什么使用 |
| `TargetMismatch` | 目标不匹配 | 这样用不上 |
| `ConditionNotMet` | 条件不满足 | 斧头太钝了 |
| `ToolUnavailable` | 工具不可用 | 火把还没有做出来 |
| `KnowledgeNotKnown` | 不知道方法 | 你还不知道该怎么做 |
| `ThreatNotActive` | 危险未激活 | 现在还没有东西靠近 |
| `CompanionCannotAct` | 同伴不能行动 | 同伴还不会或不敢这么做 |
| `ForbiddenByAudience` | 安全禁止 | 不直接展示给玩家 |
| `TestOnly` | 测试专用 | 只允许测试 |

### 7.4 `AgentAutonomyActionKind` Agent 自主行动类型

| 枚举值 | 中文含义 | 示例 |
|---|---|---|
| `Unknown` | 未知 | 非法默认值 |
| `None` | 不行动 | Agent 没有可执行意图或条件不足 |
| `HoldTorch` | 举火把 | 同伴帮忙守住火光 |
| `GatherMaterial` | 收集材料 | 同伴或 NPC 帮忙采集干草 |
| `WarnDanger` | 警告危险 | 同伴听见低吼提醒玩家 |
| `MaintainTool` | 维护工具 | 后续帮忙打磨斧头 |
| `AvoidHazard` | 避开危险 | 狼观察到火后避开火光 |
| `ApproachTarget` | 接近目标 | 野兽靠近食物或猎物 |
| `AttackTarget` | 攻击目标 | 后续战斗系统接入前只作为危险事件意图 |
| `FollowActor` | 跟随角色 | 驯化后跟随玩家或族群 |
| `TeachBack` | 反向教学 | 后续专家 NPC 教玩家 |
| `TestOnly` | 测试专用 | 只允许测试 |

### 7.5 `ThreatEventPhase` 威胁事件阶段

| 枚举值 | 中文含义 | 示例 |
|---|---|---|
| `Unknown` | 未知 | 非法默认值 |
| `Dormant` | 未激活 | 白天无野兽压力 |
| `Foreshadowing` | 前兆 | 远处有低吼 |
| `Approaching` | 靠近 | 草丛晃动 |
| `Confronting` | 对峙 | 野兽靠近营地 |
| `Mitigated` | 已压制 | 被火光逼退 |
| `Resolved` | 已解决 | 野兽离开 |
| `Failed` | 失败 | 被迫逃离 |
| `TestOnly` | 测试专用 | 只允许测试 |

## 8. 核心 DTO / 数据结构

### 8.1 `WorldObjectInstance`

| 字段 | 类型 | 中文含义 | 必填 | 写入者 | 读取者 | 校验规则 | 禁止内容 |
|---|---|---|---|---|---|---|---|
| `instance_id` | `WorldObjectInstanceId` | 实例 ID | 是 | 世界初始化/生成器 | 世界/投影/存档 | 稳定、非空 | 不允许前端生成 |
| `definition_key` | `std::string` | 对象定义 key | 是 | 内容配置 | 世界/规则 | 必须引用已知对象定义 | 不写中文 |

## 9. 核心服务接口

### 9.1 `WorldInteractionService`

职责：把一次已经解析出的动作和现有世界状态结算为结构化变化。

接口：

```cpp
class WorldInteractionService {
public:
    Result<WorldInteractionResult> resolve(
        const WorldSnapshot& snapshot,
        const WorldInteractionInput& input,
        const InteractionResolveOptions& options) const;
};
```

要求：

- 不解析中文。
- 不直接写入最终状态。
- 不形成知识。
- 返回 `WorldChangeSet`。
- 失败时返回自然中文失败摘要，不返回 `feedback_not_found`。

### 9.2 `WorldChangeApplier`

职责：把结算出的变化安全应用到世界状态。

接口：

```cpp
class WorldChangeApplier {
public:
    Result<WorldSnapshot> apply(
        const WorldSnapshot& before,
        const std::vector<WorldChange>& changes) const;
};
```

要求：

- 应用前后都校验。
- 对象数量不能变负。
- 生成对象必须有定义。
- 消耗对象后投影不可见或显示为“已耗尽”。
- 不能修改未声明字段。

### 9.3 `WorldProjectionMapper`

职责：把世界状态和最近变化映射为 H5 可见投影。

接口：

```cpp
class WorldProjectionMapper {
public:
    Result<WorldProjectionPatch> buildPatch(
        const WorldSnapshot& snapshot,
        const std::vector<WorldChange>& recent_changes,
        ProjectionAudience audience) const;
};
```

要求：

- Player audience 不输出 raw 数值。
- 用中文解释变化：`红果少了一些`、`斧头变钝了`。
- 对象卡按钮只输出可执行动作或明确原因。
- 不裁决规则。

### 9.4 `AgentAutonomyService`

职责：在危险、目标或观察事件触发时，判断同伴、野生生物、NPC 能否基于观察、知识、本能和策略产生自主行动。

接口：

```cpp
class AgentAutonomyService {
public:
    Result<AgentAutonomyResult> tryAct(
        const WorldSnapshot& snapshot,
        const AgentAutonomyRequest& request) const;
};
```

最小规则：

```text
同伴协助：
如果野兽处于 Approaching/Confronting；
并且同伴知道 torch + use @ beast_shadow -> repel_beast；
并且火把存在且可用；
并且同伴 trust >= 最小阈值；
则同伴可以执行 HoldTorch 或 AssistRepel。

野生生物避让：
如果狼/野兽观察到 camp_fire 或 torch；
并且本能知识或经验知识判断 fire/threat_counter 有危险；
则产生 AvoidHazard 或 KeepDistance 意图；
该意图必须提交到规则/世界交互结算，不能直接把野兽删除。
```

### 9.5 `ThreatEventBridge`

职责：把 P29 危险状态和 P38 世界变化连接起来。

接口：

```cpp
class ThreatEventBridge {
public:
    Result<std::vector<WorldChange>> progressThreats(
        const WorldSnapshot& snapshot,
        const ThreatProgressInput& input) const;

    Result<std::vector<WorldChange>> applyCountermeasure(
        const WorldSnapshot& snapshot,
        const ThreatCountermeasureInput& input) const;
};
```

要求：

- 等待会推进威胁。
- 火光、火把、同伴协助、野生 Agent 避让行动会降低威胁。
- 威胁失败产生故事后果，不直接做完整战斗。

## 10. 标准流程

### 10.1 玩家动作主流程

```text
1. H5 发送 action/input_text。
2. DialogIntentParser 解析为 object/action/target。
3. DialogTurnService 找到安全反馈模板，生成学习反馈。
4. WorldInteractionService 读取同一个动作语义，生成 WorldChange。
5. WorldChangeApplier 应用变化。
6. AgentAutonomyService 根据新状态让同伴、野生生物、NPC 尝试产生自主行动。
7. ThreatEventBridge 推进或结算危险。
8. H5PlayableTurnService 合并学习结果、世界变化、剧情变化。
9. H5ProjectionComposer 输出安全 projection。
10. 前端只渲染 projection。
```

### 10.2 吃红果流程

```text
输入：吃红果
前置：red_berry.quantity > 0
学习：red_berry + eat -> restore_hunger
世界变化：red_berry.quantity -1，actor.hunger 降低
投影：红果数量减少；身体的饥饿缓解
失败：红果已经没有了
```

### 10.3 砍木头流程

```text
输入：用斧头砍木头
前置：axe.sharpness > 0，wood.quantity > 0
学习：axe + use @ wood -> cut_wood
世界变化：axe.sharpness -1，wood 状态变为 processed 或生成 cut_wood_material
投影：木头被砍开；斧头变钝一点
失败：斧头太钝 / 木头不存在
```

### 10.4 点燃干草流程

```text
输入：用火种点燃干草
前置：fire_seed 可用，dry_grass.quantity > 0
学习：fire_seed + use @ dry_grass -> ignite_fire
世界变化：dry_grass.quantity -1，生成 camp_fire 或 fire_source.active
投影：干草被点燃，火光出现
失败：没有干草 / 干草太湿 / 火种不可用
```

### 10.5 制作火把流程

```text
输入：制作火把
前置：wood 或 cut_wood_material 可用，fire_seed 或 camp_fire 可用
学习：wood + use @ fire_seed -> make_torch
世界变化：木材数量减少，生成 torch.quantity +1
投影：你做出一个简易火把
失败：缺木材 / 缺火源
```

### 10.6 野兽威胁流程

```text
等待推进时间 -> 黄昏 -> 野兽进入 Foreshadowing
继续等待 -> Approaching
没有火光/火把/同伴协助 -> Confronting 或 Failed
使用火把 -> ThreatLevelChanged 或 ThreatResolved
同伴已学会且有火把 -> AgentActionExecuted + ThreatResolved
野兽观察到火光且具备避火经验 -> AgentActionExecuted(AvoidHazard) + ThreatLevelChanged
```

### 10.7 Agent 自主行动流程

```text
输入：教同伴
传播/观察：同伴获得关键知识；野生 Agent 通过观察获得经验或触发本能知识
世界状态：companion.available_action_keys 增加 potential assist
危险触发：ThreatEventBridge 请求 AgentAutonomyService
同伴检查：知识、对象、信任、状态
野生生物检查：观察、知识/本能、恐惧、敌意、可行动状态
成功：同伴举起火把协助驱赶，或狼主动避开火光
失败：同伴不知道/没有火把/不敢行动；野兽没观察到火、被激怒或不害怕
```

## 10.8 野生 Agent 学习与行动流程

```text
1. 野兽/狼作为 Agent-controlled Actor 出现在场景或危险事件中。
2. 它拥有基础本能知识，例如 fire_is_dangerous，或通过观察玩家火把获得经验。
3. 当火把、火堆或燃烧干草进入可观察范围，AgentAutonomyService 生成 AvoidHazard/KeepDistance 意图。
4. 意图通过统一命令/交互入口结算为 WorldChange，例如 threat_level -20、beast.phase = Mitigated。
5. 如果野兽没有观察到火，或处于饥饿/狂暴/被激怒状态，可能不会避让。
6. 后续驯化、护卫、坐骑都从这个 Agent 入口扩展，不再把野兽当成纯静态对象。
```

P38 最小实现不要求狼拥有完整学习系统，但文档和 DTO 必须允许：

```text
野兽观察 -> 形成记忆/经验 -> 更新行动倾向 -> 提交行动意图 -> 规则结算世界变化。
```

| 斧头 | sharpness=3 | 对木头使用 | sharpness -1，木头处理进度 +1 |
| 木头 | quantity=2 | 被斧头处理/制作火把 | 数量减少或生成木材 |
| 磨石 | usable | 对斧头使用 | 斧头 sharpness 恢复 |
| 干草 | quantity=2, dry | 被火种点燃 | 数量 -1，生成火源 |
| 火种 | active | 对干草使用 | 触发火源 |
| 火把 | quantity=0 初始不可用 | 制作后可用 | quantity +1，可驱赶野兽 |
| 靠近的野兽 | Dormant | 被火把反制 | 威胁降低/解除 |

## 13. 后续扩展点

### 13.4.5 野生 Agent 与火危险知识

本项目之前的 Agent 设计已经明确：Agent 可以代表玩家、普通族人、敌对族群、野生生物、脚本控制者和强化学习训练体。

所以 P38 必须承认：

```text
狼是一个小型 Agent。
蜘蛛也是一个小型 Agent。
普通人类是大型 Agent。
高级文明外星人是顶级 Agent。
```

区别不在于是不是 Agent，而在于：

- 观察范围不同。
- 记忆能力不同。
- 知识复杂度不同。
- 行动空间不同。
- 策略能力不同。
- 是否能语言教学不同。

狼能不能学会火有危险？答案是：架构上必须允许。

P38 最小实现可以先用两种知识来源：

1. 本能知识：狼天生对明火有危险倾向，看到火把会保持距离。
2. 经验知识：狼靠近火源后受到负面反馈，后续形成 `fire + approach -> danger` 之类的经验倾向。

但无论哪种，都不能让狼直接改世界。狼只能：

```text
观察 -> 形成/读取知识 -> 产生行动意图 -> 提交命令 -> 规则结算 -> 世界变化。
```

这保证后续驯服、护卫、坐骑、敌对野兽、生态迁徙都能接到同一套 Agent 架构。

### 13.5 野兽扩展

后续野兽可以从危险事件升级为 Agent：

- 驱赶。
- 诱捕。
- 喂食。
- 驯服。
- 成为护卫。
- 成为坐骑。
- 参与战斗。

P38 不做完整 Agent AI，但必须承认野兽/野生 NPC 是可由 Agent 控制的 Actor；`ThreatMarker`、`AgentAutonomyActionKind` 和观察事件不能阻碍后续升级。

### 13.6 地点扩展

后续地点卡可以包括：

- 林地边缘。
- 溪边。
- 洞穴。
- 草地。
- 废墟。

P38 的 `location_key` 必须为 P39/P40 地点扩展预留。

### 13.7 文明升级扩展

后续知识可以推动文明能力：

```text
火源知识 -> 夜晚生存能力
工具知识 -> 采集效率
保存食物知识 -> 饥荒抵抗
驯化知识 -> 护卫/运输
```

P38 的世界变化必须能被 P26 文明能力读取。

## 16. 测试计划

### 16.1 单元测试

- 枚举字符串转换。
- DTO 校验。
- 对象数量不能为负。
- 生成对象必须有定义。
- 非法变更被拒绝。
- 失败类型能映射中文。

### 16.2 交互测试

必须覆盖：

```text
吃红果 -> 数量减少 + 饥饿变化 + 知识形成
吃完红果后继续吃 -> 中文失败，不是 feedback_not_found
砍木头 -> 木头变化 + 斧头变钝
斧头钝后砍木头 -> 失败或 tool_dull
磨石打磨 -> 斧头恢复
点燃干草 -> 干草减少 + 火源出现
制作火把 -> 火把出现
用火把驱赶野兽 -> 威胁降低
```

### 16.3 Agent 自主行动测试

必须覆盖：

```text
没有教同伴 -> 同伴不会自动帮忙
教同伴火把知识 -> 野兽靠近时同伴可协助
同伴没有火把 -> 知道也不能执行
同伴协助后 -> 投影显示同伴行动
狼/野兽拥有本能知识 fire_is_dangerous -> 观察到火后产生避让行动
狼/野兽没有相关知识但多次受火伤害 -> 后续可以形成经验性避让倾向
野兽观察到火 -> 野兽产生避让行动
野兽没有观察到火 -> 野兽不会凭空避让
```

### 16.4 H5 投影测试

必须覆盖：

```text
对象卡按钮全部可执行
对象数量变化能显示
新对象出现能显示
不可执行原因是中文
Player audience 不泄露 raw/debug key
```

### 16.5 端到端第一夜测试

成功路线：

```text
吃红果
用斧头砍木头
用火种点燃干草
制作火把
教同伴
等待到黄昏
同伴或玩家用火把驱赶野兽，或野兽观察到火后主动避让
撑过第一夜
```

失败路线：

```text
一直等待
野兽靠近
没有火源/火把/同伴协助，也没有让野兽形成避火经验
第一夜失败或被迫逃离
```

## 17. 验收标准

P38 通过必须满足：

1. 所有 P38 新增测试通过。
2. P37 剧情测试仍通过。
3. H5 所有显示按钮点击后不出现 `feedback_not_found`。
4. 玩家能看到红果减少、火源出现、火把出现、野兽退去。
5. 同伴被教会关键知识后，至少能在一个危险场景中自动协助。
6. 野生 Agent 至少能在一个场景中基于观察或本能做出自主行动，例如狼观察火后避让。
7. 野兽威胁有前兆、推进、反制、结算。
8. 成功和失败都有自然中文反馈。
9. 前端无规则裁决。
10. Player projection 无 raw_state、hidden_truth、工程错误 key。
11. 文档、context pack、自检报告齐全。

## 19. 风险与控制

### 19.1 风险：系统变成小背包

控制：P38 只做数量和状态，不做完整背包 UI。

### 19.2 风险：野兽变成战斗系统

控制：P38 仍不做完整战斗系统；野兽可以是 Agent-controlled Actor，但行动只输出趋避/靠近/威胁变化等最小意图，不展开 HP、技能和掉落。

### 19.3 风险：Agent AI 范围膨胀

控制：只做条件触发的单步自主行动，不做完整行为树、GOAP、群体生态和战术 AI。

### 19.4 风险：按钮再次不可执行

控制：新增 H5 按钮门禁测试，所有投影按钮必须跑一遍。

### 19.5 风险：世界变化和知识不一致

控制：同一动作 trace 同时连接学习反馈和世界变更。

### 19.6 风险：内容写死

控制：所有新增变化都通过 DTO、枚举、服务和测试表达，为后续配置化预留。
