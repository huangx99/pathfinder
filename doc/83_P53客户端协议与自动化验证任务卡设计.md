# P53 客户端协议与自动化验证任务卡设计

## 1. 阶段定位

P53 是 V2.0 开放世界网格路线中的“客户端协议收口 + 代码级自动化验证”阶段。

它接在以下阶段之后：

```text
P43：统一 WorldCommand 管线。
P44：二维世界网格与探索投影。
P45：背包、物品实例与归属。
P46：世界生成器与资源分布。
P47：采集、砍伐、挖掘命令。
P48：制作与世界反应。
P49：世界经验进入认知 / 记忆 / 知识。
P50：教学与无知 NPC 基础行动。
P51：Agent 网格目标执行。
P52：野生 Agent 与野兽生态接入。
```

P53 的一句话定位：

```text
先建立所有客户端共用的 V2 协议层，并用代码测试验证客户端如何启动世界、提交命令、接收投影、应用 patch、读取 available_commands；本阶段不设计具体界面。
```

P53 不是地图界面阶段，不写 UI 布局，不写按钮样式，不写手机横屏界面。网页、Godot、Unity、移动 App、调试 CLI 未来都应该复用这套协议。真正的玩家界面顺延到 P54。

## 2. 为什么现在做

上一版 P53 把协议和界面混在一起，导致文档像调试台，而不是游戏。正确顺序应该是：

```text
先有稳定客户端协议。
再用自动化测试证明协议能驱动一轮玩法。
最后才基于协议设计玩家界面。
```

如果跳过 P53 协议阶段直接做界面，会出现以下风险：

```text
界面为了能点，自己生成按钮。
界面为了显示结果，自己改本地状态。
界面为了做玩法，读取 JSON 或写死内容 key。
不同客户端各写一套交互协议，后续无法扩展到 Godot / Unity / App。
研发把 projection、patch、events、available_commands 直接摊给玩家，形成调试台体验。
```

所以 P53 必须只解决“客户端和后端怎么正确说话”，不解决“玩家界面长什么样”。

## 3. 阶段目标

P53 完成后，系统必须具备以下能力：

```text
1. 定义客户端通用协议 DTO，不绑定 H5 / 浏览器。
2. 定义 bootstrap / command / refresh / reset 的请求与响应契约。
3. 定义客户端如何提交 option_id 或 WorldCommandDto。
4. 定义客户端如何接收 full projection 和 projection_patch。
5. 定义客户端如何校验 projection_version，必要时触发 full refresh。
6. 定义客户端如何消费 available_commands，但不生成规则按钮。
7. 定义客户端如何消费 event_feed / experiences / frontend_hints，但不把它们当规则源。
8. 建立 fake client / protocol harness，用代码测试一轮最小世界交互。
9. 建立架构扫描，禁止客户端协议层读取内容 JSON、写死内容 key、传知识 claim、直接改 runtime。
10. 为 P54 玩家界面提供干净、稳定、可测试的协议基础。
```

## 4. 本阶段不解决的问题

```text
不设计地图 UI。
不设计手机横屏布局。
不设计按钮位置、面板、日志表现、视觉层级。
不写前端页面、不写 CSS、不写 DOM 交互。
不做正式美术和动效。
不做 Godot / Unity / App 具体接入。
不做完整存档界面，存档复盘仍放到后续阶段。
不做新的玩法规则，玩法仍来自 P43-P52 后端系统。
```

## 5. 核心设计原则

### 5.1 协议先于界面

P53 只定义客户端和后端的通信契约。界面怎么展示、按钮怎么排列、玩家怎么点击，全部放到 P54。

禁止在 P53 文档和代码里出现具体 UI 布局方案。

### 5.2 客户端协议不绑定 H5

DTO、命名、测试目标必须使用 `Client` / `client_protocol`，不能使用 `H5V2` / `h5_v2`。

当前第一个实现可以是网页，但协议名称不能绑死网页。

### 5.3 WorldCommand 是唯一行动入口

客户端只能提交：

```text
ClientCommandRequest(option_id)
```

或：

```text
ClientCommandRequest(command=WorldCommandDto)
```

后端必须校验并执行。客户端不能直接调用采集、制作、教学、移动、野兽或 NPC 系统内部函数。

### 5.4 available_commands 是操作权威来源

客户端不根据 entity_key、object_key、terrain_key、inventory 内容生成玩法操作。

客户端只能显示 / 提交后端给出的：

```text
WorldCommandOptionDto
```

如果 `available_commands` 不足，应该补后端 option builder，而不是在客户端补规则。

### 5.5 Projection / Patch 是显示状态来源

客户端本地状态只来自：

```text
ClientBootstrapResponse.full_projection
ClientCommandResponse.projection_patch
ClientRefreshResponse.full_projection
```

客户端不能自己移动 actor、扣物品数量、刷新资源节点或修改知识列表。

### 5.6 projection_version 必须强校验

所有 command 请求都必须携带：

```text
known_projection_version
client_sequence
```

响应必须携带：

```text
base_projection_version
new_projection_version
requires_full_refresh
```

版本不一致时，客户端必须进入 full refresh，不允许强行合并 patch。

### 5.7 Event / Hint 只做反馈，不做规则

`event_feed`、`experiences`、`frontend_hints` 用于解释发生了什么，不用于客户端反推规则。

禁止：

```text
看到 event 里写“采集成功”，客户端就自己加物品。
看到 hint 里写“危险”，客户端就自己禁用按钮。
```

### 5.8 安全投影不泄露隐藏真相

协议层只传安全字段。任何 raw_state、hidden_truth、内部分数、真实未发现属性都不能进入客户端响应。

## 6. 模块划分

| 模块 | 中文含义 | 职责 | 输入 | 输出 | 禁止事项 |
|---|---|---|---|---|---|
| `client_protocol_types` | 客户端协议类型 | 定义 DTO、枚举、validate | 无 | Client DTO | 禁止 UI 字段 |
| `client_protocol_codec` | 协议编解码 | JSON <-> DTO | JSON / DTO | 解析结果 | 禁止规则判断 |
| `client_session_gateway` | 会话网关 | bootstrap / refresh / reset | session_id | projection response | 禁止直接改 runtime |
| `client_command_gateway` | 命令网关 | 接收客户端 command 请求 | ClientCommandRequest | ClientCommandResponse | 禁止绕过 WorldCommandPipeline |
| `client_projection_adapter` | 投影适配 | 汇总后端安全投影 | runtime projection / bridges | ClientWorldProjection | 禁止 hidden truth |
| `client_patch_contract` | Patch 契约 | 规范版本和 patch 语义 | WorldProjectionPatchDto | apply plan | 禁止本地推导 |
| `client_available_command_adapter` | 可用命令适配 | 根据后端上下文提供 options | world context / selection | WorldCommandOptionDto[] | 禁止内容硬编码 |
| `client_protocol_harness` | 协议测试器 | 模拟客户端执行流程 | fake session / fake world | 测试断言 | 禁止真实 UI 依赖 |

## 7. 核心枚举

### 7.1 `ClientRequestKind`

| 值 | 中文 | 用途 |
|---|---|---|
| `Unknown` | 未知 | 非法默认值 |
| `Bootstrap` | 启动 | 创建或恢复会话 |
| `Command` | 命令 | 提交玩家操作 |
| `Refresh` | 刷新 | 拉取完整投影或恢复版本 |
| `Reset` | 重置 | 开发/确认后重开 |
| `TestOnly` | 测试 | 只允许测试使用 |

### 7.2 `ClientSubmitMode`

| 值 | 中文 | 用途 |
|---|---|---|
| `Unknown` | 未知 | 非法默认值 |
| `OptionId` | 选项 ID | 推荐方式，提交后端生成的 option_id |
| `WorldCommandDto` | 完整命令 | 测试或高级客户端，仍由后端校验 |
| `RefreshOnly` | 只刷新 | 不执行规则，只恢复投影 |
| `TestOnly` | 测试 | 只允许测试使用 |

### 7.3 `ClientSyncState`

| 值 | 中文 | 用途 |
|---|---|---|
| `Unknown` | 未知 | 非法默认值 |
| `InSync` | 已同步 | 客户端版本等于后端版本 |
| `CommandPending` | 命令等待 | 已提交命令，等待响应 |
| `PatchApplying` | 应用补丁 | 正在应用后端 patch |
| `NeedsFullRefresh` | 需要全量刷新 | 版本不匹配或 patch 要求刷新 |
| `OutOfSync` | 已失步 | 客户端不能继续局部更新 |
| `FatalError` | 致命错误 | 协议错误，必须重建会话 |
| `TestOnly` | 测试 | 只允许测试使用 |

### 7.4 `ClientProjectionScope`

| 值 | 中文 | 用途 |
|---|---|---|
| `Unknown` | 未知 | 非法默认值 |
| `Viewport` | 视口 | 当前可见范围 |
| `ActorSelf` | 自身 | 玩家自身状态 |
| `Inventory` | 背包 | 背包安全投影 |
| `Knowledge` | 知识 | 主观知识摘要 |
| `EventFeed` | 事件 | 安全事件流 |
| `AgentSummary` | Agent摘要 | NPC/野兽安全行动摘要 |
| `FullSafeWorld` | 安全全量 | 调试或恢复用，不含 hidden truth |
| `TestOnly` | 测试 | 只允许测试使用 |

## 8. 核心 DTO

### 8.1 `ClientBootstrapRequest`

```cpp
struct ClientBootstrapRequest {
    std::string client_id;
    std::string session_id;
    std::string requested_actor_key;
    std::string requested_layer_key;
    uint64_t client_protocol_version = 1;
    bool create_if_missing = true;
    bool dev_reset_if_allowed = false;
};
```

### 8.2 `ClientBootstrapResponse`

```cpp
struct ClientBootstrapResponse {
    std::string session_id;
    uint64_t server_protocol_version = 1;
    uint64_t projection_version = 0;
    ClientWorldProjection full_projection;
    std::vector<WorldCommandOptionDto> available_commands;
    std::vector<WorldEventDto> event_feed;
    std::vector<WorldFrontendHintDto> frontend_hints;
    std::vector<std::string> warning_keys;
};
```

### 8.3 `ClientCommandRequest`

```cpp
struct ClientCommandRequest {
    std::string session_id;
    std::string client_id;
    uint64_t client_sequence = 0;
    uint64_t known_projection_version = 0;
    ClientSubmitMode submit_mode = ClientSubmitMode::Unknown;
    std::string option_id;
    std::optional<WorldCommandDto> command;
    ClientSelectionContext selection_context;
};
```

### 8.4 `ClientCommandResponse`

```cpp
struct ClientCommandResponse {
    std::string session_id;
    uint64_t accepted_client_sequence = 0;
    uint64_t base_projection_version = 0;
    uint64_t new_projection_version = 0;
    bool requires_full_refresh = false;
    WorldCommandResultDto result;
    WorldProjectionPatchDto projection_patch;
    std::vector<WorldCommandOptionDto> available_commands;
    std::vector<WorldEventDto> event_feed;
    std::vector<WorldExperienceDto> experiences;
    std::vector<WorldFrontendHintDto> frontend_hints;
    std::vector<std::string> warning_keys;
};
```

### 8.5 `ClientRefreshRequest`

```cpp
struct ClientRefreshRequest {
    std::string session_id;
    std::string client_id;
    uint64_t known_projection_version = 0;
    std::vector<ClientProjectionScope> requested_scopes;
    std::string requested_layer_key;
    int viewport_center_x = 0;
    int viewport_center_y = 0;
};
```

### 8.6 `ClientWorldProjection`

```cpp
struct ClientWorldProjection {
    uint64_t projection_version = 0;
    std::string actor_key;
    std::string active_layer_key;
    std::vector<WorldCellPatchDto> visible_cells;
    std::vector<WorldEntityPatchDto> visible_entities;
    std::vector<InventoryPatchDto> inventories;
    std::vector<KnowledgePatchDto> knowledge;
    std::vector<AreaEffectPatchDto> area_effects;
    std::vector<std::string> safe_summary_keys;
};
```

### 8.7 `ClientSelectionContext`

```cpp
struct ClientSelectionContext {
    WorldCommandTargetDto target;
    std::string selected_actor_key;
    std::string selected_entity_id;
    std::string selected_inventory_id;
    std::string selected_area_id;
};
```

说明：

```text
selection_context 只是客户端告诉后端“玩家当前关注哪里”。
它不是规则真相。
后端必须重新校验 target 是否存在、是否可见、是否可达、是否可执行。
```

## 9. 协议流程

### 9.1 Bootstrap

```text
ClientBootstrapRequest
-> ClientSessionGateway.bootstrap
-> 构建安全 full_projection
-> 返回 ClientBootstrapResponse
```

验收：

```text
返回 projection_version。
返回初始 available_commands。
不包含 hidden truth。
不依赖具体 UI。
```

### 9.2 Command

```text
ClientCommandRequest(option_id 或 WorldCommandDto)
-> ClientCommandGateway.validate
-> option_id materialize 为 WorldCommandDto
-> WorldCommandPipeline.execute
-> ClientProjectionAdapter 生成 patch / full refresh hint
-> 返回 ClientCommandResponse
```

验收：

```text
所有行为仍走 WorldCommandPipeline。
失败 / blocked / interrupted 也返回安全事件和新的 available_commands。
响应不要求客户端自己猜状态变化。
```

### 9.3 Refresh

```text
ClientRefreshRequest
-> 校验 session
-> 根据 scopes 返回 full_projection 或 scoped projection
```

验收：

```text
版本不匹配时可恢复。
客户端可以放弃本地 patch 队列。
```

### 9.4 Reset

```text
ClientResetRequest
-> 只允许开发或玩家确认后的正式入口
-> 创建新 session / world seed
-> 返回 bootstrap response
```

Reset 不属于普通世界规则，不生成普通采集/移动/教学经验。

## 10. 代码级自动化验证流程

P53 实现时必须建立 fake client harness，模拟以下流程：

```text
1. bootstrap 世界。
2. 读取 available_commands。
3. 提交 Wait option。
4. 应用 command response patch。
5. refresh 投影。
6. 提交一个带过期 projection_version 的 command，验证需要 full refresh。
7. 提交一个非法 option_id，验证 blocked/failed 且不污染状态。
```

这套测试不需要真实 UI，不需要浏览器，不需要 CSS。

## 14. P53 验收标准

```text
有 client_protocol 后端 target。
有 Client DTO validate。
有 JSON codec 或等价 wire codec。
有 bootstrap / command / refresh gateway。
有 fake client harness。
有 projection_version 和 full refresh 测试。
有 available_commands 只能来自后端的测试。
有禁止内容硬编码、禁止 JSON 读取、禁止知识注入、禁止 runtime 直改的架构扫描。
不包含任何玩家界面设计或 UI 实现。
```

## 15. 和 P54 的关系

P54 才开始设计玩家界面。

P54 必须基于 P53 的结果来设计：

```text
玩家看到什么。
玩家怎么选择格子和对象。
玩家怎么理解 available_commands。
玩家怎么看到采集、制作、教学、NPC 行动、野兽行动的结果。
手机横屏如何布局。
```

P53 不回答这些 UI 问题，只保证 P54 不需要重新发明通信协议。
