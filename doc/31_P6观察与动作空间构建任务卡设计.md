# 31_P6观察与动作空间构建任务卡设计

## 1. 设计目标

P6 的目标是在 P5 已通过的 Agent 数据契约之上，建立两个最小构建器：

```text
ObservationBuilder：从当前状态和允许可见输入构建 AgentObservation。
ActionSpaceBuilder：从 AgentObservation 构建 AgentActionSpace。
```

一句话：

```text
P6 不让 Agent 变聪明，P6 只让 Agent 的观察和可选动作不再完全手写。
```

P6 完成后，系统应能证明：

```text
先驱者 Agent 可以从 unknown fruit 初始状态构建观察。
ActionSpaceBuilder 可以基于观察生成 Eat / Explore 等候选动作。
测试可以手动选择 Eat 候选，生成 AgentIntent，再通过 P5 AgentCommandAdapter 转成 CommandEnvelope。
CommandEnvelope 仍由 RulePipeline 权威结算。
蜘蛛遇火、狼呼叫群体仍可通过观察和动作空间表达，但不新增真实 resolver。
Observation 不泄露真实隐藏效果，只表达主观置信度、风险、证据和可传授提示。
```

P6 的核心价值：

```text
P5 证明 Agent 可以表达意图。
P6 证明 Agent 的输入和可选动作可以由系统稳定构建。
P7 才开始做最简单规则策略。
P8 才开始做多 Agent tick 调度。
```

## 2. 为什么 P6 必须在 P7/P8 前做

如果没有 P6，后续策略会出现严重问题：

```text
AgentPolicy 会直接读取 GameState。
测试会继续手写 observation，无法证明真实状态到观察的过滤链。
ActionSpace 会变成策略内部临时拼出来的数组。
玩家、蜘蛛、狼、族群会各自生成不同格式动作。
后续强化学习缺少稳定 observation/action_space 地基。
```

P6 要把这条链路固定住：

```text
GameState + AgentBinding + VisibilityInput + CognitionState
    -> AgentObservation
    -> AgentActionSpace
    -> 手动选择候选
    -> AgentIntent
    -> CommandEnvelope
    -> RulePipeline
```

注意：

```text
P6 仍然不实现自动“选择”。
P6 只生成“能看到什么”和“能尝试什么”。
```

## 3. 本阶段解决什么问题

P6 解决五个问题：

```text
1. 观察构建：AgentObservation 不再只能由测试手写。
2. 可见性输入：当前没有地图/感知系统时，用显式 visible id 输入控制可见范围。
3. 认知投影：从 CognitionState 读取主观 claim，转为 observation 置信度。
4. 动作空间构建：从 observation 生成 Eat / Flee / CallGroup 等候选动作。
5. 闭环验证：unknown fruit 从真实状态构建观察和动作空间后，仍可走 P3/P5 规则闭环。
```

P6 必须支持的最小场景：

```text
Pioneer unknown fruit：从 UnknownFruitFixture 初始状态构建观察，生成 Eat candidate。
Pioneer learned fruit：执行一次 eat 后，从 CognitionState 构建更高 edible confidence 的观察。
Spider fire threat：从 threat seed 构建 Fire threat observation，生成 Flee candidate。
Wolf pack signal：从 social/pack seed 构建 CallGroup candidate，command_supported=false。
Consumed object：已 consumed 对象不能生成 Eat candidate。
```

## 5. P6 与 P0-P5 的关系

P6 可以依赖：

```text
foundation：StrongId、Result、ErrorDetail、Tick、StateVersion。
command：ActionTarget、ActionTargetType、ActionTargetRole。
object：ObjectStore、WorldObject、ObjectDefinition、ObjectRuntimeFlag。
state：GameState、ActorStateStore，只读。
cognition：CognitionState、CognitionClaim、CognitionKey。
agent：AgentBinding、AgentObservation、AgentActionSpace、AgentIntent、AgentCommandAdapter。
rules：UnknownFruitFixture，仅用于 P6 集成测试。
pipeline：RulePipeline，仅用于 P6 集成测试。
```

P6 不应依赖：

```text
replay / protocol，除非只是全量测试已有依赖。
frontend / HTTP / WebSocket。
完整 config registry。
expression evaluator。
Agent runtime / policy。
```

关键边界：

```text
P6 的 builder 可以读 GameState。
P6 的 builder 不能改 GameState。
P6 的 builder 不能执行规则。
P6 的 action space 只是主观候选集合。
RulePipeline 仍可能拒绝任何候选动作。
```

## 6. P6 最小闭环定义

### 6.1 输入

```text
GameState：UnknownFruitFixture::createInitialState()。
AgentBinding：agent_player_001 -> actor_001。
VisibleObjectIds：obj_unknown_fruit_001。
ThreatSeeds：可为空。
CognitionState：初始为空。
```

### 6.2 构建观察

```text
ObservationBuilder 读取 GameState。
确认 actor_001 存在。
确认 obj_unknown_fruit_001 存在且未 consumed。
生成 AgentObservedObject。
known_edible_confidence 初始为 0.0 或来自 CognitionState。
known_usable_confidence 初始为 0.0。
risk_confidence 初始为 0.0 或来自 harmful claim。
evidence_count 来自 cognition claim。
不写入真实 hunger_delta / health_delta / effect_kind。
```

### 6.3 构建动作空间

```text
ActionSpaceBuilder 只读取 AgentObservation。
对可交互对象生成 Eat candidate。
Eat candidate.command_supported = true。
Eat candidate.required_target_types = Object。
reason_key = action.try_eat_observed_object 或类似稳定 key。
对 Fire threat 生成 Flee candidate。
对 Pack/Social signal 生成 CallGroup candidate，command_supported=false。
```

### 6.4 手动选择并结算

```text
P6 集成测试手动选择 Eat candidate。
构造 AgentIntent。
AgentIntent.command_supported_snapshot 从 candidate.command_supported 复制。
AgentCommandAdapter 转 CommandEnvelope。
RulePipeline 执行。
断言 P3 语义结果稳定。
```

注意：

```text
“手动选择候选”不是 Policy。
P6 不能新增 decide()。
P7 才设计 RuleBasedPolicy。
```

## 7. 类型与中文翻译

| 英文类型 | 中文名 | P6 是否实现 | 说明 |
|---|---|---:|---|
| ObservationBuilder | 观察构建器 | 是 | 从只读状态和可见性输入构建 AgentObservation |
| ObservationBuildRequest | 观察构建请求 | 是 | 包含 binding、state、visible ids、threat seeds |
| ObservationBuildResult | 观察构建结果 | 是 | 包含 observation 和 trace / warnings |
| VisibilityInput | 可见性输入 | 是 | P6 临时显式可见对象列表，未来由地图感知替代 |
| ObservedThreatSeed | 威胁观察种子 | 是 | P6 临时威胁输入，用于 fire / social 测试 |
| ObservationBuildTrace | 观察构建轨迹 | 是 | 调试哪些对象被纳入、跳过、来自哪些 claim |
| ActionSpaceBuilder | 动作空间构建器 | 是 | 从 AgentObservation 生成 AgentActionSpace |
| ActionSpaceBuildRequest | 动作空间构建请求 | 是 | 包含 observation 和 builder 选项 |
| ActionSpaceBuildResult | 动作空间构建结果 | 是 | 包含 action_space 和 trace / warnings |
| ActionSpaceBuildTrace | 动作空间构建轨迹 | 是 | 调试候选动作来源和阻断原因 |
| InteractionAvailabilityHint | 交互可用提示 | 可选 | 如 P5 结构不足，给 observed object 增加可交互提示 |
| AgentPolicy | 智能体策略 | 否 | P6 不实现 |
| AgentRuntime | 智能体运行时 | 否 | P6 不实现 |

## 9. 核心接口草案

### 9.1 VisibilityInput

```cpp
struct VisibilityInput {
    std::vector<foundation::ObjectId> visible_object_ids;
    std::vector<ObservedThreatSeed> threat_seeds;

    foundation::Result<void> validateBasic() const;
};
```

说明：

```text
P6 暂时没有地图和感知范围系统。
visible_object_ids 是测试和上层系统显式传入的可见对象列表。
未来地图/感知系统完成后，可以由 PerceptionSystem 生成 VisibilityInput。
```

### 9.2 ObservedThreatSeed

```cpp
struct ObservedThreatSeed {
    foundation::EntityId source_id;
    AgentThreatType threat_type = AgentThreatType::Unknown;
    double severity = 0.0;
    double confidence = 0.0;

    foundation::Result<void> validateBasic() const;
};
```

说明：

```text
P6 用 threat seed 表达火源、社交信号等外部观察输入。
它不是事件，也不是规则结果。
它只用于构造 observation。
```

### 9.3 ObservationBuildRequest

```cpp
struct ObservationBuildRequest {
    AgentBinding binding;
    const state::GameState* state = nullptr;
    VisibilityInput visibility;
    bool include_cognition_claims = true;

    foundation::Result<void> validateBasic() const;
};
```

校验要求：

```text
binding.validateBasic 必须通过。
state 不能为 nullptr。
state->validateBasic 必须通过。
visibility.validateBasic 必须通过。
primary_actor_id 必须存在于 state.actor_store。
```

### 9.4 ObservationBuildTrace

```cpp
struct ObservationBuildTrace {
    std::vector<foundation::ObjectId> included_object_ids;
    std::vector<foundation::ObjectId> skipped_object_ids;
    std::vector<std::string> reason_keys;
};
```

说明：

```text
trace 只用于调试和测试。
trace 不影响规则结果。
trace 不替代 EventRecord。
```

### 9.5 ObservationBuildResult

```cpp
struct ObservationBuildResult {
    AgentObservation observation;
    ObservationBuildTrace trace;
    std::vector<foundation::ErrorDetail> warnings;

    foundation::Result<void> validateBasic() const;
};
```

### 9.6 ObservationBuilder

```cpp
class ObservationBuilder {
public:
    foundation::Result<ObservationBuildResult> build(const ObservationBuildRequest& request) const;
};
```

构建规则：

```text
读取 state.metadata.state_version 和 current_tick。
observation.agent_id 来自 binding.agent_id。
observation.observer_actor_id 来自 binding.primary_actor_id。
只处理 visibility.visible_object_ids 中的对象。
对象不存在：返回错误或 warning，P6 建议返回错误，便于测试严格。
对象 consumed/depleted：可写入 trace.skipped_object_ids，不生成可交互 observed object。
对象存在且可见：生成 AgentObservedObject。
CognitionState 中同 subject + object definition + eat/action claim 会影响 confidence / evidence_count。
不从 ObjectEdibleProfile 暴露真实 hunger_delta / health_delta / effect_kind。
ThreatSeed 转成 AgentObservedThreat。
```

### 9.7 ActionSpaceBuildRequest

```cpp
struct ActionSpaceBuildRequest {
    AgentObservation observation;
    bool include_explore_candidates = true;
    bool include_wait_candidate = false;

    foundation::Result<void> validateBasic() const;
};
```

说明：

```text
ActionSpaceBuilder 只读取 observation。
P6 不允许 ActionSpaceBuilder 读取 GameState。
```

### 9.8 ActionSpaceBuildTrace

```cpp
struct ActionSpaceBuildTrace {
    std::vector<foundation::ActionId> generated_action_ids;
    std::vector<std::string> reason_keys;
};
```

### 9.9 ActionSpaceBuildResult

```cpp
struct ActionSpaceBuildResult {
    AgentActionSpace action_space;
    ActionSpaceBuildTrace trace;
    std::vector<foundation::ErrorDetail> warnings;

    foundation::Result<void> validateBasic() const;
};
```

### 9.10 ActionSpaceBuilder

```cpp
class ActionSpaceBuilder {
public:
    foundation::Result<ActionSpaceBuildResult> build(const ActionSpaceBuildRequest& request) const;
};
```

构建规则：

```text
observation.validateBasic 必须通过。
action_space.agent_id = observation.agent_id。
action_space.state_version = observation.state_version。
每个可交互 observed object 生成 Eat candidate。
Eat candidate.action_id = eat。
Eat candidate.intent_type = AgentIntentType::Eat。
Eat candidate.required_target_types = Object。
Eat candidate.command_supported = true。
Fire threat 生成 Flee candidate。
Flee candidate.command_supported = true，但 P6 测试不提交 RulePipeline。
Social / pack threat seed 可生成 CallGroup candidate。
CallGroup candidate.command_supported = false。
所有 candidate action_id 在同一 ActionSpace 内不能重复；如果需要多个对象，用 eat_obj_xxx 这类确定性 ID。
```

注意：

```text
P6 只生成候选，不做选择。
P6 不计算 expected utility。
P6 不实现 action mask。
P6 不接强化学习。
```
