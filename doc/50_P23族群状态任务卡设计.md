# P23 族群状态任务卡设计

## 1. 阶段定位

P23 是《先驱者》从“个体学习闭环”进入“族群状态”的第一阶段。

P15-P22 已经完成或验证：

```text
P15：个体认知。
P16：个体记忆。
P17：记忆压缩和检索。
P18：知识形成。
P19：知识冲突与修正。
P20：知识传播。
P21：学习闭环编排。
P22：H5 中文对话框最小可玩入口。
```

P23 要解决的是：

```text
一个族群如何以可验证、可复盘、可投影的方式表达“成员、士气、信任、分工、凝聚和分裂风险”。
```

一句话：

```text
P23 让“个体知道什么、族群有多少人、成员怎么分工、整体是否稳定”变成后端可测试的 TribeState。
```

P23 是族群状态基础，不是共同作战、族群战争、文明能力或专家 NPC 阶段。

---

## 2. 本阶段目标

P23 必须新增 `tribe` 模块，提供族群状态的最小可编译、可测试、可扩展实现。

它必须做到：

1. 定义族群状态最小类型：`TribeState`、`TribeMember`、`TribeMemberRole`、`TribeMorale`、`TribeTrust`、`TribeSplitRisk`。
2. 支持成员加入、离开、角色变更、信任变化、士气变化、分裂风险计算。
3. 支持从安全输入构造 `TribeStateInput`，不能读取 raw GameState、SaveGame 或隐藏真相。
4. 输出 `TribeStateResolveResult`，包含更新后状态、状态变更草稿、投影和 trace。
5. 提供 `TribeProjection`，供后续 H5、Agent、文明能力系统读取摘要。
6. 分裂风险必须是确定性评分，第一版禁止随机分裂。
7. 所有 `Result::ok(dto)` 必须通过 `validateBasic()`。
8. 只做族群状态基础，为 P24 共同作战、P25 族群冲突、P26 文明能力提供输入。

核心原则：

```text
P23 只建立族群状态，不做战斗结算。
P23 只消费安全摘要，不读世界真相。
P23 输出状态和投影，不直接解锁文明能力。
P23 的分裂风险是确定性结果，不用随机数。
P23 不实现 1.0 后扩展玩法。
```

---

## 3. 玩家体验映射

P23 本身不一定直接改变 H5 界面，但它决定后续玩家是否能感受到“我不是一个人在学习，而是在带领一个族群”。

### 3.1 玩家未来感受

```text
我教会同伴红果可以吃。
族群开始拥有基础采集知识。
如果食物压力降低，族群士气会上升。
如果同伴不断受伤、饥饿、知识分配不均，族群信任下降，分裂风险上升。
不同成员可以有不同职责：探索者、采集者、守护者、教师。
```

### 3.2 P23 需要证明的体验链

P23 的测试必须能证明：

```text
成员不是字符串列表，而是有角色、状态和信任的族群成员。
士气不是随便写死，而由输入变化确定性影响。
信任变化会影响凝聚力和分裂风险。
角色分工会进入 TribeProjection。
分裂风险可以解释为什么上升或下降。
```

## 5. 本阶段不做什么

P23 明确禁止：

```text
不做共同作战，这是 P24。
不做族群战争，这是 P25。
不做文明能力解锁，这是 P26。
不做对象反应，这是 P27。
不做危险探索压力，这是 P28。
不做存档 SaveManager。
不做 H5 新界面。
不做 HTTP / WebSocket / JSON API。
不做专家 NPC 教学。
不做 NPC 识别玩家谋害。
不做随机分裂。
不读 raw GameState。
不读 hidden truth。
不让前端推导族群状态。
不让 AgentRuntime 或 Policy 成为规则权威。
```

P23 可以为后续保留字段和投影，但不能提前实现后续玩法。

## 7. 数据结构设计

### 7.1 TribeMemberRole

族群成员角色。

建议枚举：

```cpp
enum class TribeMemberRole {
    Unknown,
    Pioneer,
    Explorer,
    Gatherer,
    Guardian,
    Teacher,
    Learner,
    Healer,
    Crafter,
    Elder,
    Child,
    TestOnly
};
```

说明：

```text
P23 只记录角色，不实现角色行为 AI。
Guardian 不代表已能共同作战。
Teacher 不代表专家 NPC。
```

### 7.2 TribeCohesionState

族群凝聚状态。

建议枚举：

```cpp
enum class TribeCohesionState {
    Unknown,
    Stable,
    Watchful,
    Tense,
    Fracturing,
    Splintered,
    TestOnly
};
```

说明：

```text
Stable：稳定。
Watchful：需要观察。
Tense：紧张。
Fracturing：接近分裂。
Splintered：已分裂状态摘要，但 P23 不生成新族群。
```

### 7.3 TribeStateChangeKind

族群状态变化类型。

建议枚举：

```cpp
enum class TribeStateChangeKind {
    Unknown,
    MemberAdded,
    MemberRemoved,
    MemberRoleChanged,
    MoraleChanged,
    TrustChanged,
    SplitRiskChanged,
    CohesionChanged,
    KnowledgeLinked,
    TestOnly
};
```

### 7.4 TribeMember

字段建议：

```text
member_id: EntityId
role: TribeMemberRole
active: bool
trust: double 0..1
morale: double 0..1
known_knowledge_ids: list<KnowledgeId>
joined_tick: Tick
updated_tick: Tick
reason_keys: list<string>
```

校验：

```text
member_id 不为空。
trust/morale 必须在 0..1。
Unknown/TestOnly 默认不允许出现在正式 DTO。
known_knowledge_ids 不允许重复。
reason_keys 不允许 forbidden key。
```

### 7.5 TribeMorale

族群士气摘要。

字段建议：

```text
overall: double 0..1
food_pressure: double 0..1
safety_pressure: double 0..1
recent_success: double 0..1
recent_loss: double 0..1
reason_keys: list<string>
```

说明：

```text
overall 是结果摘要。
pressure 字段是安全输入摘要，不读真实库存。
```

### 7.6 TribeTrust

族群信任摘要。

字段建议：

```text
leader_trust: double 0..1
member_trust: double 0..1
teaching_fairness: double 0..1
knowledge_conflict_pressure: double 0..1
reason_keys: list<string>
```

说明：

```text
leader_trust 可以表达对先驱者或当前领导者的信任。
teaching_fairness 表示知识传播是否不均。
knowledge_conflict_pressure 表示错误知识/冲突知识压力。
```

### 7.7 TribeSplitRisk

分裂风险摘要。

字段建议：

```text
risk: double 0..1
cohesion_state: TribeCohesionState
resource_pressure: double 0..1
trust_pressure: double 0..1
knowledge_conflict_pressure: double 0..1
casualty_pressure: double 0..1
reason_keys: list<string>
```

推荐计算：

```text
risk = clamp(
  resource_pressure * 0.30
+ trust_pressure * 0.30
+ knowledge_conflict_pressure * 0.25
+ casualty_pressure * 0.15
)
```

推荐状态阈值：

```text
0.00 - 0.24: Stable
0.25 - 0.44: Watchful
0.45 - 0.64: Tense
0.65 - 0.84: Fracturing
0.85 - 1.00: Splintered
```

P23 不根据风险随机创建新族群。

### 7.8 TribeState

字段建议：

```text
tribe_id: TribeId 或 StrongId
display_key: string
members: list<TribeMember>
population_total: int
active_population: int
morale: TribeMorale
trust: TribeTrust
split_risk: TribeSplitRisk
shared_knowledge_ids: list<KnowledgeId>
updated_tick: Tick
version: Version
reason_keys: list<string>
```

如果当前 foundation 没有 `TribeId`，P23 第一版可新增 StrongId：

```cpp
struct TribeIdTag {};
using TribeId = StrongId<TribeIdTag>;
```

位置建议：

```text
backend/include/pathfinder/tribe/tribe_types.h
```

不要回头大改 foundation，除非现有基础类型已经提供 TribeId。

### 7.9 TribeStateInput

字段建议：

```text
tribe_id
input_tick
member_events
knowledge_events
morale_delta
trust_delta
resource_pressure
safety_pressure
knowledge_conflict_pressure
casualty_pressure
reason_keys
```

说明：

```text
P23 使用安全摘要输入，不读 GameState。
resource_pressure 是 0..1 摘要，不是库存真相。
casualty_pressure 是 0..1 摘要，不展开战斗结算。
```

### 7.10 TribeStateResolveResult

字段建议：

```text
decision
updated_state
state_changes
projection
trace
reason_keys
warning_keys
```

### 7.11 TribeProjection

字段建议：

```text
tribe_id
display_key
population_summary
role_summary_lines
morale_summary
trust_summary
split_risk_summary
shared_knowledge_count
warning_keys
```

说明：

```text
Projection 只给前端/AI 看摘要。
不要把完整 KnowledgeClaim 塞进 TribeProjection。
```

### 7.12 TribeTrace

字段建议：

```text
trace_key
input_tick
steps
matched_policy_keys
reason_keys
```

用于解释：

```text
为什么士气变了。
为什么信任变了。
为什么分裂风险上升。
```

## 8. 服务与接口

### 8.1 TribeStateResolver

核心入口：

```cpp
class TribeStateResolver {
public:
    Result<TribeStateResolveResult> resolve(
        const TribeState& current,
        const TribeStateInput& input,
        const TribeStateOptions& options = {}) const;
};
```

职责：

```text
应用成员事件。
应用士气变化。
应用信任变化。
重新计算 population_total / active_population。
重新计算 split_risk。
生成 state_changes。
生成 projection。
生成 trace。
```

### 8.2 TribeProjectionBuilder

```cpp
class TribeProjectionBuilder {
public:
    Result<TribeProjection> buildProjection(const TribeState& state) const;
};
```

职责：

```text
把 TribeState 转为安全摘要。
不暴露隐藏知识。
不返回 raw member 内部大对象。
```

### 8.3 TribeStateValidator

可以是 free function 或 DTO 自带 `validateBasic()`。

必须校验：

```text
id 不为空。
人口不为负。
active_population <= population_total。
ratio 字段 0..1。
枚举不为 Unknown/TestOnly。
成员 id 不重复。
知识 id 不重复。
reason/warning 不包含 forbidden key。
```

## 11. 必测故事

### 11.1 族群创建和成员加入

输入：

```text
空 TribeState。
加入 pioneer、companion。
pioneer role=Pioneer。
companion role=Learner。
```

期望：

```text
population_total=2。
active_population=2。
role_summary 包含 Pioneer/Learner。
state_changes 包含 MemberAdded。
```

### 11.2 成员角色变化

输入：

```text
companion 从 Learner 改为 Gatherer。
```

期望：

```text
角色更新。
state_changes 包含 MemberRoleChanged。
projection role_summary 更新。
```

### 11.3 士气上升

输入：

```text
recent_success 上升。
food_pressure 下降。
```

期望：

```text
morale.overall 上升。
state_changes 包含 MoraleChanged。
trace 解释 success/food_pressure。
```

### 11.4 信任下降

输入：

```text
leader_trust_delta < 0。
teaching_fairness 下降。
```

期望：

```text
trust.leader_trust 下降。
trust.member_trust 或 split risk 受影响。
state_changes 包含 TrustChanged。
```

### 11.5 分裂风险上升

输入：

```text
resource_pressure=0.7。
trust_pressure=0.6。
knowledge_conflict_pressure=0.5。
casualty_pressure=0.2。
```

期望：

```text
split_risk.risk 确定性上升。
cohesion_state=Tense 或 Fracturing，按阈值确定。
state_changes 包含 SplitRiskChanged / CohesionChanged。
无随机数。
```

### 11.6 族群知识摘要链接

输入：

```text
shared_knowledge_ids 添加 red_berry/eat/restore_hunger 的 KnowledgeId。
```

期望：

```text
TribeState.shared_knowledge_ids 包含该 id。
Projection.shared_knowledge_count 增加。
不暴露 KnowledgeClaim 全文。
```

## 12. 安全与边界测试

必须测试拒绝：

```text
tribe_id 为空。
member_id 为空。
ratio < 0 或 > 1。
active_population > population_total。
Unknown/TestOnly 枚举进入正式 DTO。
重复 member_id。
重复 shared_knowledge_ids。
reason_keys 包含 hidden_truth / raw_state / GameState / SaveGame。
输入中出现 random_seed 或 probability_split 之类随机分裂语义。
```

P23 forbidden keys：

```text
hidden_truth
real_effect
true_trait
object_truth
raw_state
GameState
SaveGame
AgentRuntime
Policy
random_split
probability_split
```
