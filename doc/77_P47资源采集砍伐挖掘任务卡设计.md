# P47 资源采集、砍伐、挖掘任务卡设计

## 1. 阶段定位

P47 是 V2.0 开放世界网格路线中的“资源节点交互与材料产出”阶段。

它接在以下阶段之后：

```text
P43：统一世界 Command 管线。
P44：世界网格基础与探索投影。
P45：背包、容器、掉落物与物品归属系统。
P46：世界生成器与资源分布。
```

P46 解决“世界上有什么”。

P47 解决“玩家或 NPC 如何通过 Command 消费 ResourceNode，并让产物以正确归属进入世界”。

P47 为以下阶段打基础：

```text
P48：制作与世界反应接入。
P49：探索行为产生知识。
P50：教学与无知 NPC 基础行动。
P51：Agent 网格目标执行。
P52：野兽网格生态与遭遇。
P53：H5 V2 地图界面。
P54：存档复盘与调试门禁。
```

P47 的本质不是“点树给木头”的临时玩法，而是建立 ResourceNode -> WorldCommand -> P45 OnMap 产物 -> 事件/投影 的正式闭环。

## 2. 为什么现在做

P46 已经生成了：

```text
WorldCellRuntime。
WorldResourceNodeRuntime。
GroundItem。
GenerateWorld Command。
```

但如果没有 P47：

```text
树不能砍。
石头不能采。
浆果丛不能采。
ResourceNode 的 remaining_charges 永远不会变化。
P48 制作没有材料来源。
P49 认知没有采集经验来源。
NPC 后续无法学习“哪里能采到什么”。
```

所以 P47 必须在 P48/P49/P50 之前完成。

## 3. 本阶段目标

P47 完成后，系统必须具备：

```text
1. WorldCommandKind::Gather / Chop / Mine / Dig 能真实处理 ResourceNode。
2. 命令能验证 actor、node、距离、可见性、action 类型、工具要求和剩余次数。
3. 采集成功后 ResourceNode remaining_charges 减少。
4. charges 归零后 ResourceNode 进入 Depleted 状态。
5. 产物生成到世界地图 OnMap，且必须通过 P45 的 IWorldEntityLocationPort::spawnEntityOnMap 建立归属。
6. 产物不能直接塞进背包，不能直接 push inventory entries。
7. CommandResult 返回 changed_cells、changed_entities、state_deltas、events、projection patch。
8. 失败命令不修改 ResourceNode、不生成产物、不消耗工具。
```

## 4. 本阶段不解决的问题

| 不做内容 | 原因 | 后续阶段 |
|---|---|---|
| 制作、合成、反应链 | P47 只获取材料，不组合材料 | P48 |
| 采集经验写入认知/记忆/知识 | P47 只产出事件和 trace，不接知识 | P49 |
| NPC 自主选择采集目标 | P47 只提供 Command 能力 | P50/P51 |
| 工具耐久、锋利度消耗 | P47 只检查工具存在，耐久属于工具/状态规则 | P48/P51 后续补强 |
| 资源再生时间 | P47 只支持 Available -> Depleted | P54 后或生态阶段 |
| 自动进背包 | P47 MVP 产物落地 OnMap，玩家/NPC 通过 P45 Pickup 进入背包 | P47.1/P51 |
| 多人争抢同一节点并发锁 | 单线程 Command 先做状态复验 | P54/服务器并发阶段 |

## 5. 核心原则

### 5.1 ResourceNode 是采集权威

Gather/Chop/Mine/Dig 只能作用于 ResourceNode。

禁止：

```text
对普通 GroundItem 执行 gather。
把树做成普通 item 后直接 pickup。
从 command 参数直接给玩家木头。
```

### 5.2 Command 是唯一入口

前端、玩家、NPC、测试都必须提交 WorldCommand。

禁止：

```text
H5 直接调用 ResourceHarvestService。
NPC 直接改 remaining_charges。
测试直接把 output 塞进背包当采集成功。
```

### 5.3 P45 归属接口是产物落地门禁

P47 采集产物必须通过 P45 的归属接口建立 `OnMap` 或 `InInventory` 状态，不能绕过 runtime 直接改数组。

P47 MVP 采用：

```text
采集产物统一生成 OnMap。
生成路径：ResourceHarvestApplier -> IWorldEntityLocationPort::spawnEntityOnMap。
背包获取由后续 Pickup Command 完成。
```

### 5.4 失败不改状态

任何失败：

```text
actor missing。
node missing。
node depleted。
action mismatch。
tool missing。
too far。
not visible。
output spawn failed。
```

都不能：

```text
减少 remaining_charges。
生成产物。
改变 node state。
```

### 5.5 先 validate，再 apply

P47 必须分两段：

```text
validate：只读，生成 draft。
apply：唯一写入，落地产物和更新 node。
```

这和 P45/P46 保持一致，避免半失败状态。

### 5.6 产物 ID 必须确定性

同一个 command_id、node_id、output_index 产生稳定 output entity_id。

建议：

```text
harvest_<command_id>_<node_id>_<output_entity_key>_<index>
```

不能使用真实时间、随机数或全局递增。

### 5.7 最小版只支持单节点单命令

P47 不做批量采集，不做区域采集。

后续 Agent 可以连续提交多个命令，但 P47 单命令只处理一个 ResourceNode。

## 6. 模块划分

| 模块 | 中文含义 | 职责 | 输入 | 输出 | 禁止事项 |
|---|---|---|---|---|---|
| world_harvest types | 采集类型层 | 定义 Harvest DTO/Failure/Output | Command 参数、ResourceNode | Draft/Result | 不依赖前端 |
| ResourceHarvestService | 采集服务 | validate + apply 编排 | HarvestRequest | HarvestResult | 不直接改背包数组 |
| ResourceHarvestValidator | 采集校验器 | 检查 actor/node/distance/action/tool/charges | runtime + request | HarvestDraft | 不写状态 |
| ResourceHarvestApplier | 采集应用器 | 生成产物、更新 ResourceNode、事件 | HarvestDraft | state changes | 不跳过 P45 |
| GatherCommandHandler | Command 接入 | Gather/Chop/Mine/Dig 转 HarvestRequest | WorldCommandDto | WorldCommandExecutionResult | 不自己写采集规则 |
| HarvestProjectionBridge | 投影桥 | 输出 changed cells/entities/deltas/events | HarvestResult | projection patch | 不给前端隐藏规则 |

建议路径：

```text
backend/include/pathfinder/world_harvest/
backend/src/world_harvest/
backend/tests/unit/world_harvest/
backend/tests/integration/world_harvest/
```

建议 target：

```text
pathfinder_world_harvest
pathfinder_tests_world_harvest
pathfinder_tests_world_harvest_integration
```

## 7. 核心枚举

### 7.1 ResourceHarvestKind

```cpp
enum class ResourceHarvestKind {
    Unknown,
    Gather,
    Chop,
    Mine,
    Dig
};
```

中文：采集动作类型。

映射：

```text
WorldCommandKind::Gather -> Gather
WorldCommandKind::Chop -> Chop
WorldCommandKind::Mine -> Mine
WorldCommandKind::Dig -> Dig
```

### 7.2 ResourceHarvestFailureKind

```cpp
enum class ResourceHarvestFailureKind {
    None,
    InvalidRequest,
    ActorMissing,
    NodeMissing,
    NodeNotVisible,
    NodeTooFar,
    NodeDepleted,
    ActionMismatch,
    ToolMissing,
    OutputInvalid,
    OutputSpawnFailed,
    RuntimeConflict
};
```

### 7.3 HarvestOutputLocationPolicy

```cpp
enum class HarvestOutputLocationPolicy {
    Unknown,
    DropOnMap,
    PreferInventory
};
```

P47 MVP 只实现 `DropOnMap`。

`PreferInventory` 预留，不能半实现。

## 8. 核心 DTO

### 8.1 ResourceHarvestRequest

```cpp
struct ResourceHarvestRequest {
    std::string harvest_id;
    std::string actor_key;
    std::string node_id;
    ResourceHarvestKind harvest_kind = ResourceHarvestKind::Unknown;
    HarvestOutputLocationPolicy output_location_policy = HarvestOutputLocationPolicy::DropOnMap;
    std::map<std::string, std::string> parameters;
};
```

### 8.2 ResourceHarvestOutputDraft

```cpp
struct ResourceHarvestOutputDraft {
    std::string entity_id;
    std::string entity_key;
    std::string display_name_key;
    world_runtime::WorldCellCoord coord;
    int quantity = 1;
    bool stackable = true;
    std::string stack_key;
    std::vector<std::string> state_keys;
    std::map<std::string, double> numeric_states;
    std::vector<std::string> tag_keys;
};
```

### 8.3 ResourceHarvestDraft

```cpp
struct ResourceHarvestDraft {
    std::string draft_id;
    ResourceHarvestRequest request;
    world_runtime::WorldResourceNodeRuntime node_before;
    std::vector<ResourceHarvestOutputDraft> output_drafts;
    int charges_to_consume = 1;
    bool will_deplete = false;
    std::vector<std::string> changed_cell_ids;
};
```

### 8.4 ResourceHarvestResult

```cpp
struct ResourceHarvestResult {
    bool ok = false;
    ResourceHarvestFailureKind failure_kind = ResourceHarvestFailureKind::None;

    std::vector<std::string> changed_entity_ids;

```

## 9. Runtime 接口扩展

P46 已有：

```cpp
findResourceNode(node_id)
upsertGeneratedResourceNode(node)
```

P47 需要补强一个明确更新接口，避免拿 snapshot 后绕回 upsert 造成语义不清：

```cpp
virtual foundation::Result<void> updateResourceNodeRuntime(
    const WorldResourceNodeRuntime& node) = 0;
```

如果实现上复用 upsert，也必须在命名或注释中说明这是采集后状态更新，不是生成。

P47 不允许 world_harvest 直接访问 WorldGridRuntime 私有 map。

## 10. Command 参数设计

### 10.1 Gather / Chop / Mine / Dig 通用参数

```text
node_id：必填，目标 ResourceNode。
output_location_policy：可选，P47 只允许 drop_on_map。
```

示例：

```json
{
  "command_kind": "gather",
  "actor_key": "player",
  "target": { "target_kind": "entity", "target_id": "node_berry_bush_surface_1_0" },
  "parameters": {
    "node_id": "node_berry_bush_surface_1_0",
    "output_location_policy": "drop_on_map"
  }
}
```

如果 target 里已经有 node id，也允许 handler 从 target 读取；但最终必须填入 ResourceHarvestRequest.node_id。

## 11. 校验流程

### 11.1 actor 校验

```text
findActor(actor_key) 必须成功。
```

### 11.2 node 校验

```text
findResourceNode(node_id) 必须成功。
node.remaining_charges > 0。
node.node_state_str != Depleted。
```

### 11.3 可见性与距离

P47 MVP：

```text
node.coord 必须在 actor 当前 cell 或相邻 cell。
node.coord 必须对 actor visible。
```

如果 P44 可见性接口不足，至少通过 IWorldEntityLocationPort::isCellVisibleToActor 校验。

### 11.4 action 匹配

```text
ResourceHarvestKind::Gather 只能匹配 node.required_action_key == "gather"。
Chop -> "chop"。
Mine -> "mine"。
Dig -> "dig"。
```

大小写按 content key 精确匹配。

### 11.5 工具校验

如果 `node.required_tool_key` 为空，不需要工具。

如果非空：

```text
actor inventory 中必须存在 entity_key == required_tool_key 的物品。
```

P47 只检查存在，不消耗耐久。

## 12. Apply 流程

成功 apply 必须按顺序：

```text
1. 复验 node 当前仍存在且 charges 足够。
2. 为每个 output_entity_key 创建 output draft。
3. 调用 IWorldEntityLocationPort::spawnEntityOnMap，把产物生成到 node.coord 或 actor.coord。
4. 更新 ResourceNode remaining_charges -= 1。
5. 如果 charges == 0，node_state_str = "Depleted"。
6. 生成 WorldStateDelta 和 WorldEvent。
7. 返回 changed_cell_ids / changed_entity_ids / changed_resource_node_ids。
```

产物默认坐标：

```text
node.coord。
```

如果 node.coord 不可站立也可以先放 node.coord，因为这是地面掉落，不等于 actor 移动。

## 13. 输出规则

P47 MVP 输出规则：

```text
每次采集消耗 1 charge。
每个 output_entity_key 生成 quantity=1。
stack_key = entity_key + ":harvested"。
stackable = true。
tag_keys 至少包含 "harvest_output"。
```

后续 P48/P42 内容系统可把 output quantity、概率、状态、工具加成配置化。

P47 不做随机掉落。

## 14. 事件与投影

成功事件：

```text
event_kind = "ResourceHarvested"
title_text = "资源已采集"
body_text 包含 actor、node、outputs。
coord = node.coord
```

失败事件可选，但 CommandResult 必须有 failure_reason_keys。

state delta：

```text
ResourceNodeUpdated：remaining_charges / state。
HarvestOutputCreated：entity id / entity key / coord。
```

projection patch：

```text
changed_cells：node cell。
changed_entities：产物 entity。
state_deltas：node 更新。
frontend_hints：可选，HighlightCell / FloatingText。
```

## 16. 验收标准

P47 通过条件：

```text
1. 新增 world_harvest 模块。
2. Gather/Chop/Mine/Dig command handler 可注册和执行。
3. ResourceNode 被真实消费 charges。
4. 产物真实生成 OnMap。
5. P45 归属门禁通过。
6. ResourceNode 不变成普通 GroundItem。
7. 失败路径不修改状态。
8. P46 世界生成 + P47 采集端到端测试通过。
9. P43-P47 回归通过。
10. 不实现认知、制作、NPC 自主行动。
```

```text
只实现 P47。

不让前端直接读规则。

```

## 18. 结论

P47 是 V2 世界从“有资源”走向“能获取材料”的关键阶段。

它必须保证：

```text
采集行为真实进入 Command。
资源节点真实被消费。
产物真实落地并有 P45 归属。
失败路径不污染状态。
后续认知、制作、NPC 都能复用同一条链路。
```
