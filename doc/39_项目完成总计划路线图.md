# 《先驱者》项目完成总计划路线图

## 1. 总结论

本路线图不是重新设计，而是在现有 `01-24` 总设计文档与 `P0-P13` 阶段任务卡基础上，继续向“可玩、可审计、可扩展、可交给 AI 稳定实现”的完整游戏推进。

核心原则：

```text
先地基，再玩法。
先可观察，再智能。
先安全输入，再真实数据。
先单体闭环，再族群文明。
先本地工具，再 H5 表现。
先确定性复盘，再训练/AI增强。
```

当前已完成/已设计到：

```text
P0-P4：后端基础、配置命令、状态事件管线、未知对象首条规则、复盘协议前置。
P5-P9：Agent 定义、观察动作空间、运行时、运行记录、复盘锁定、训练样本草稿。
P10-P13：Agent 历史投影、调试报告、文件导出、本地调试 CLI 前置。
```

后续总路线：

```text
P14：安全输入适配。
P15-P21：认知、记忆、知识、传播学习主线。
P22-P27：族群、文明、对象反应、风险与冲突主线。
P28-P32：存档、复盘、协议、H5 最小可玩闭环。
P33-P38：内容扩展、平衡、工具、AI/RL 接入前置。
P39-P42：Alpha/Beta/发布准备。
```

## 2. 完成标准

项目“完成”不是指所有想法都实现，而是达到下面标准：

```text
玩家可以从无知状态开始探索未知对象。
尝试吃/用/组合会产生反馈。
反馈会形成认知。
认知会进入记忆。
记忆会沉淀为知识。
知识可以传播给个体/族群。
族群行为会影响协作、冲突、分裂、文明能力。
世界对象可以组合反应。
所有关键状态变化有事件。
所有关键命令可复盘。
前端只展示后端投影，不复制规则。
调试工具能解释 Agent 为什么这样行动。
训练/AI 接入不会绕过规则权威。
```

## 4. P13-P14：调试输入链路收口

### P13：Agent 本地调试 CLI 前置

目标：

```text
提供最小 CLI，用内置 fixture 调用 P11/P12，生成本地 txt/md 调试文件。
```

交付：

```text
AgentDebugCliOptions。
AgentDebugCliParser。
AgentDebugFixtureFactory。
AgentDebugCliRunner。
pathfinder_agent_debug_cli。
```

验收：

```text
--help 成功。
unknown_fruit markdown 导出成功。
no_decision dry-run 不写文件。
--json / --load-save / 非法 base_name 失败。
P13/P12/P11/P10 回归通过。
```

### P14：Agent 调试输入适配前置

目标：

```text
定义 CLI 未来如何接入真实但安全的数据来源，不直接读 GameState 或真实存档内部结构。
```

交付：

```text
AgentDebugInputSource。
AgentDebugInputManifest。
AgentDebugInputAdapter。
AgentDebugInputValidationReport。
只读输入包格式草案。
```

边界：

```text
不做 JSON。
不做 SaveManager。
不读完整存档。
不读隐藏真相。
只允许输入 P10/P11/P12 安全投影或测试包。
```

验收：

```text
安全输入包可转换成 AgentDebugReport。
非法输入包失败。
隐藏真相字段失败。
CLI 可选择 input source，但仍只走安全 DTO。
```

## 5. P15-P21：认知、记忆、知识、传播学习主线

### P15：认知系统正式化

目标：

```text
把 P3/P6 的最小 CognitionClaim 扩展为正式认知层。
```

解决问题：

```text
一个生物不关心物体颜色形状，而关心它能否食用、能否使用、使用后效果、风险、证据来源和置信度。
```

交付：

```text
CognitionSubject。
CognitionTarget。
CognitionAspect。
CognitionClaimV2。
CognitionEvidenceRecord。
CognitionConfidenceModel。
CognitionQueryService。
```

验收：

```text
吃一口会生成 edible/risk/effect 相关认知。
用一下会生成 usable/effect 相关认知。
认知不读取 ObjectDefinition hidden truth。
认知只由反馈和事件更新。
```

### P16：记忆系统

目标：

```text
实现短期、中期、长期记忆的基础结构。
```

交付：

```text
MemoryRecord。
MemoryScope：ShortTerm / MidTerm / LongTerm。
MemoryStrength。
MemoryDecayPolicy。
MemoryStore。
MemoryWriteService。
```

验收：

```text
一次尝试行为可以形成短期记忆。
重复证据可以增强记忆。
低强度短期记忆会衰减。
关键事件可晋升中期/长期记忆。
```

### P17：记忆压缩与检索

目标：

```text
让大量经验可以压缩成可用摘要，并能被 Agent 查询。
```

交付：

```text
MemoryCompressionPolicy。
MemorySummary。
MemoryRecallQuery。
MemoryRecallResult。
MemoryDecompressionView。
```

验收：

```text
多次吃同类对象可压缩成“该类对象大概率可食用/有害”的摘要。
Agent Observation 可读取记忆摘要，但不能读取隐藏真相。
记忆压缩可复盘。
```

### P18：知识系统

目标：

```text
从稳定记忆中形成可传播、可引用的知识声明。
```

交付：

```text
KnowledgeClaim。
KnowledgeSource。
KnowledgeScope。
KnowledgeConfidence。
KnowledgeRepository。
KnowledgeFormationService。
```

验收：

```text
长期记忆可以形成知识。
知识保留来源和证据链。
知识可以被 Agent/族群查询。
知识不直接等同真实 ObjectDefinition。
```

### P19：知识冲突与修正

目标：

```text
允许错误知识存在、传播、被证伪、被修正。
```

交付：

```text
KnowledgeConflict。
KnowledgeRevision。
ContradictionDetector。
KnowledgeMergePolicy。
KnowledgeTrustModel。
```

验收：

```text
“可食用”和“有害”知识可共存但形成冲突。
新证据可降低旧知识置信度。
修正过程有事件和复盘记录。
```

### P20：传播系统

目标：

```text
让一个个体把知识传给其他个体或族群。
```

交付：

```text
PropagationChannel。
TeachingAction。
PropagationAttempt。
PropagationResult。
KnowledgeTransferPolicy。
```

验收：

```text
个体可 Teach/Share/Signal。
被传授者获得知识投影而非真实隐藏属性。
传播受信任、距离、语言/文明能力影响。
错误知识也可以传播。
```

### P21：学习闭环

目标：

```text
打通“尝试 -> 反馈 -> 认知 -> 记忆 -> 知识 -> 传播 -> 他人行为变化”。
```

交付：

```text
LearningLoopService。
LearningEventChain。
LearningDebugReport。
LearningReplayTest。
```

验收：

```text
先驱者吃未知果实后形成认知和记忆。
重复尝试后形成知识。
传授给同族后，同族观察中知道该对象可食用/危险。
全链路可导出调试报告。
```

## 10. 每阶段固定门禁

每个 P 阶段必须满足：

```text
1. 有任务卡。
2. 有 context_pack。
3. 有明确允许/禁止文件范围。
4. 有单元测试。
5. 有集成测试。
6. 有边界扫描。
7. 有审核报告。
8. 邮件发送审核结论。
9. 不修无关问题。
10. 不绕过 RulePipeline / CommandEnvelope / Event / Replay 权威链。
```

## 11. 最关键的架构红线

```text
Agent 是决策层，不是规则层。
H5 是表现层，不是规则层。
学习算法不是规则层。
存档不能绕过规则层。
复盘不能重新调用策略。
知识不是隐藏真相。
记忆不是 ObjectDefinition。
传播传的是主观知识，不是真实属性。
所有状态变化必须有事件。
所有关键行为必须可复盘。
```

## 12. 为什么这个计划适合 AI 实现

```text
阶段从底层到上层，不跳跃。
每阶段只改少量文件。
每阶段都有 context_pack 防止上下文爆炸。
每阶段都有禁止范围，避免 AI 乱改。
每阶段都有测试和审计，能及时纠偏。
先做工具链，再做复杂学习，方便后续定位问题。
```
