# 先驱者：记忆系统设计

## 1. 设计目标

记忆系统负责把认知系统输出的 `MemoryCandidate` 沉淀为可保存、可查询、可衰减、可强化、可压缩、可回溯、可整理为知识证据的记忆记录。

记忆系统回答以下问题：

```text
谁经历过？
经历了什么？
这段经历来自哪里？
它和哪个对象、行为、效果、条件有关？
它有多重要？
它是短期、中期、长期还是归档？
它是否被压缩过？
它能否被解压成可解释视图？
它能否支持后续知识形成？
AI 是否可以从它获得长期上下文？
```

核心链路：

```text
CognitionUpdate / MemoryCandidate
  ↓
MemoryResolver
  ↓
MemoryRecord
  ↓
MemoryConsolidation / MemoryDecay / MemoryCompression
  ↓
MemorySummary / MemoryBundle / MemoryReconstructionView
  ↓
KnowledgeEvidenceBundle
  ↓
KnowledgeSystem
```

本系统必须保证：

- 记忆是经历记录，不是最终知识。
- 记忆可以分短期、中期、长期和归档层级。
- 记忆可以被强化、衰减、压缩和解压。
- 记忆压缩必须保留来源引用和可解释性。
- 高风险、首次发现、强反证、文明突破等关键记忆必须保护。
- 记忆可以为知识系统提供结构化证据包。
- 记忆可以为 AI / 强化学习提供长期主观上下文。
- 记忆创建、更新和归档必须可复盘、可测试。

## 2. 不解决的问题

| 问题 | 原因 | 归属文档 |
|---|---|---|
| 创建最终 KnowledgeClaim | 记忆只提供证据材料，不决定稳定知识 | `13_知识系统设计.md` |
| 执行知识传播 | 记忆可记录教学经历，但不传播知识 | `14_传播系统设计.md` |
| 修改族群行为 | 记忆可成为输入，但不直接改变族群策略 | `15_族群状态设计.md` |
| 解锁文明能力 | 记忆可提供证据，但不直接解锁能力 | `16_文明能力设计.md` |
| 替代认知状态 | 记忆记录经历，认知记录当前相信什么 | `11_认知状态设计.md` |
| 前端自行创建记忆 | 记忆必须由规则系统产生 | `20_前后端协议设计.md` |
| 黑盒 AI 语义总结 | 第一版只做结构化压缩和可解释摘要 | 后续 AI 内容工具 |
| 完整模拟人脑遗忘 | 第一版只做工程可控的衰减和整理 | 后续高级认知系统 |

关键边界：

```text
FeedbackEvidence：一次反馈能作为证据。
CognitionState：主体当前相信什么。
MemoryRecord：主体经历过什么。
MemorySummary：多条记忆的结构化摘要。
KnowledgeEvidenceBundle：给知识系统的证据包。
KnowledgeClaim：后续稳定知识断言。
```

## 3. 核心概念

| 概念 | 中文名 | 含义 |
|---|---|---|
| MemoryRecord | 记忆记录 | 一条具体经历的结构化记录 |
| MemoryOwner | 记忆拥有者 | 谁拥有这条记忆 |
| MemorySource | 记忆来源 | 记忆来自反馈、认知、教学、观察等 |
| MemoryContent | 记忆内容 | 记忆中的对象、行为、效果、条件和结果 |
| MemoryStrength | 记忆强度 | 记忆保留、强化和知识权重的基础 |
| MemoryRetentionLevel | 记忆保留层级 | 工作、短期、中期、长期、归档 |
| MemoryCompressionLevel | 记忆压缩等级 | 未压缩、轻度、摘要、统计、归档压缩 |
| MemoryTag | 记忆标签 | 风险、成功、失败、教学、地点等标签 |
| MemoryIndex | 记忆索引 | 按对象、行为、效果、时间、主体查询 |
| MemoryResolver | 记忆解析器 | 把 MemoryCandidate 转成 MemoryRecord 或更新已有记忆 |
| MemoryConsolidation | 记忆整理 | 合并相似记忆、强化重复经验 |
| MemoryDecay | 记忆衰减 | 按时间、重要性和层级降低记忆强度 |
| MemoryCompression | 记忆压缩 | 把多条记忆整理成摘要或统计 |
| MemoryReconstructionView | 记忆解压视图 | 按需要展开摘要和来源链 |
| MemorySummary | 记忆摘要 | 多条相似记忆的结构化总结 |
| MemoryBundle | 记忆包 | 一组相关记忆和摘要的集合 |
| KnowledgeEvidenceBundle | 知识证据束 | 给知识系统判断 KnowledgeClaim 的证据包 |
| MemoryTrace | 记忆解释链 | 说明记忆为什么形成、强化、压缩或衰减 |
| MemorySummaryProjection | 记忆摘要投影 | 给前端、AI 或训练系统的长期上下文摘要 |

### 3.1 记忆、认知、知识的边界

```text
记忆：我经历过什么。
认知：我现在相信什么。
知识：这些经历和认知是否能形成稳定断言。
```

示例：

```text
记忆：我吃过新鲜红浆果，饥饿下降。
认知：新鲜红浆果可能可食用。
知识：新鲜红浆果可作为安全食物来源。
```

### 3.2 统一 MemoryRecord，不拆三套系统

短期、中期、长期记忆不应设计成三套类。

正确设计：

```text
MemoryRecord
  + retention_level
  + strength
  + compression_level
  + decay_policy
  + consolidation_state
```

错误设计：

```text
ShortTermMemoryRecord
MidTermMemoryRecord
LongTermMemoryRecord
```

原因：

```text
三套类会导致存储、查询、事件、压缩和知识证据逻辑重复。
统一 MemoryRecord 才能让记忆在生命周期中自然流转。
```

### 3.3 压缩不是删除

记忆压缩不是直接删除原始经历，也不是黑盒文本总结。

第一版压缩必须是：

```text
结构化摘要
+ 来源记忆 ID
+ 来源事件 ID
+ 代表样本
+ 冲突样本
+ 统计字段
+ 可解压视图
```

## 4. 枚举设计

本阶段复用 `02_通用枚举设计.md` 中已有通用枚举，并在本模块内定义两个局部枚举。

### 4.1 使用的既有枚举

| 枚举 | 中文名 | 本系统用途 |
|---|---|---|
| MemoryType | 记忆类型 | 区分观察、实验、事故、失败、警告、教学等 |
| EvidenceType | 证据类型 | 表达记忆作为知识证据时的来源类型 |
| ConfidenceChangeReason | 置信度变化原因 | 说明记忆强化、衰减、反证和修正原因 |
| CognitionOwnerType | 认知归属类型 | 复用为记忆拥有者类型 |
| KnowledgeScope | 知识范围 | 记忆影响范围和证据范围 |
| KnowledgeStatus | 知识状态 | 证据束建议状态，不直接创建知识 |
| FeedbackReliability | 反馈可靠性 | 记忆来源可靠性 |
| FeedbackIntensity | 反馈强度 | 影响记忆强度 |
| ActionType | 行为类型 | 记忆关联行为 |
| EventType | 事件类型 | 输出 MemoryCreated / MemoryUpdated |
| EventVisibility | 事件可见性 | 控制记忆事件是否展示 |
| EventImportance | 事件重要性 | 控制记忆事件重要级别 |
| StateChangeType | 状态变化类型 | 记忆状态变更草稿使用 Create、Update、Append、Delete |
| RuleStepType | 规则步骤类型 | 使用 `ResolveCognition` 或后续 `ResolveMemory` 预留 |
| ErrorSeverity | 错误严重级别 | 表达记忆错误级别 |

### 4.2 MemoryRetentionLevel（记忆保留层级）

`MemoryRetentionLevel` 是记忆系统模块内枚举，用于描述记忆处于什么生命周期层级。

| 枚举 | 外部字符串 | 中文名 | 含义 |
|---|---|---|---|
| Working | working | 工作记忆 | 当前行动或当前回合临时上下文，通常不持久化 |
| ShortTerm | short_term | 短期记忆 | 刚发生不久，细节完整，容易衰减 |
| MidTerm | mid_term | 中期记忆 | 被重复引用或整理后的经验记忆 |
| LongTerm | long_term | 长期记忆 | 重要、反复验证、强情绪或长期使用的稳定记忆 |
| Archived | archived | 归档记忆 | 低频读取，保留历史、摘要和索引 |

使用规则：

```text
Working 不一定进入 MemoryStore。
ShortTerm 保留完整细节。
MidTerm 开始整理和轻度压缩。
LongTerm 保留强证据、代表样本和证据链。
Archived 可保留摘要、统计和来源引用。
```

### 4.3 MemoryCompressionLevel（记忆压缩等级）

`MemoryCompressionLevel` 是记忆系统模块内枚举，用于描述记忆细节被压缩到什么程度。

| 枚举 | 外部字符串 | 中文名 | 含义 |
|---|---|---|---|
| None | none | 不压缩 | 保留完整结构化细节 |
| Light | light | 轻度压缩 | 保留大部分细节，合并重复字段 |
| Summary | summary | 摘要压缩 | 保留结构化摘要、来源引用和代表样本 |
| Statistical | statistical | 统计压缩 | 保留次数、成功率、冲突率、代表样本 |
| Archived | archived | 归档压缩 | 只保留摘要、索引、关键样本和事件引用 |

区别：

```text
MemoryRetentionLevel 回答：记忆处于短期、中期、长期还是归档。
MemoryCompressionLevel 回答：记忆细节被压缩到什么程度。
```

组合示例：

| 记忆 | Retention | Compression |
|---|---|---|
| 刚吃红浆果成功 | ShortTerm | None |
| 多次吃红浆果成功 | MidTerm | Light |
| 长期使用红浆果作食物 | LongTerm | Summary |
| 上百次重复采集记录 | Archived | Statistical |

### 4.4 MemoryType 使用规则

| MemoryType | 中文 | 本系统用途 |
|---|---|---|
| Observation | 观察记忆 | 记录观察到的对象、环境或实体行为 |
| Experiment | 实验记忆 | 记录主动尝试行为 |
| Accident | 意外记忆 | 记录误触、误食、灾害等非计划经历 |
| Result | 结果记忆 | 记录明确结果 |
| Failure | 失败记忆 | 记录无效、失败、无反应或条件不足 |
| Warning | 警示记忆 | 记录危险、伤害、损失 |
| Teaching | 传授记忆 | 记录教导或被教导 |
| Inference | 推断记忆 | 记录由已有认知推断出的经验 |
| Location | 位置记忆 | 记录资源、危险或区域位置 |
| Social | 社会记忆 | 记录族群反应、传播效果、协作变化 |
| Symbolic | 符号记忆 | 记录符号、文字、遗迹信息 |

## 5. 数据结构

### 5.1 MemoryOwner（记忆拥有者）

`MemoryOwner` 描述记忆属于谁。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| owner_type | CognitionOwnerType | 拥有者类型 | 先驱者、族群、族人等 |
| owner_id | string | 拥有者 ID | EntityId、TribeId、记录 ID 等字符串形式 |
| scope | KnowledgeScope | 影响范围 | Personal、Tribe、Region 等 |

约定：

```text
MemoryOwner 可由 CognitionOwner 转换而来。
Pioneer 的记忆不自动变成 Tribe 的记忆。
```

### 5.2 MemoryRecord（记忆记录）

`MemoryRecord` 是记忆系统的核心持久化结构。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| memory_id | MemoryId | 记忆 ID | 记忆系统创建时分配 |
| owner | MemoryOwner | 拥有者 | 谁拥有这条记忆 |
| memory_types | list<MemoryType> | 记忆类型 | 可多选，例如 Experiment + Result |
| retention_level | MemoryRetentionLevel | 保留层级 | 短期、中期、长期、归档等 |
| compression_level | MemoryCompressionLevel | 压缩等级 | 不压缩、摘要、统计等 |
| source | MemorySource | 来源 | 来自认知、反馈、教学等 |
| content | MemoryContent | 内容 | 结构化经历内容 |
| strength | MemoryStrength | 强度 | 重要性、情绪、保留权重 |
| tags | list<MemoryTag> | 标签 | 风险、成功、失败、地点等 |
| related_cognition_keys | list<CognitionKey> | 相关认知 | 来源或影响的认知条目 |
| related_event_ids | list<EventId> | 相关事件 | FeedbackGenerated、KnowledgeUpdated 等 |
| source_memory_ids | list<MemoryId> | 来源记忆 | 压缩或整理时引用的旧记忆 |
| representative_memory_ids | list<MemoryId> | 代表样本 | 摘要记忆的代表样本 |
| contradiction_memory_ids | list<MemoryId> | 冲突样本 | 与本记忆结论冲突的样本 |
| created_at | GameTime | 创建时间 | 记忆形成时间 |
| last_reinforced_at | GameTime | 最近强化时间 | 最近一次被强化 |
| last_accessed_at | GameTime | 最近访问时间 | 最近一次被读取 |
| decay_state | MemoryDecayState | 衰减状态 | 记忆衰减信息 |
| protection | MemoryProtection | 保护策略 | 是否防止压缩或删除 |
| trace_ids | list<string> | 解释链 | 关联 MemoryTrace |

### 5.3 MemorySource（记忆来源）

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| source_kind | string | 来源类型 | feedback / cognition_update / teaching / observation / external_record |
| source_event_id | EventId | 来源事件 | 通常为 FeedbackGenerated、KnowledgeUpdated 等 |
| source_evidence_id | string | 来源证据 | FeedbackEvidence 或 CognitionUpdate 来源 ID |
| source_update_id | string | 来源更新 | CognitionUpdate.update_id |
| source_candidate_id | string | 来源候选 | MemoryCandidate.candidate_id |
| reliability | FeedbackReliability | 可靠性 | 来源可靠性 |
| evidence_type | EvidenceType | 证据类型 | DirectTest、Observation、TribeReport 等 |

### 5.4 MemoryContent（记忆内容）

`MemoryContent` 保存结构化内容，不能只保存文案。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| subject_kind | string | 主体类型 | object_definition / object_instance / region / trait / action |
| subject_id | string | 主体 ID | 对象定义、对象实例、区域、特性等 |
| action_type | ActionType | 行为类型 | Eat、Use、Observe、Combine 等 |
| effect_summary | string | 效果摘要 | restore_hunger、poison、transform 等 |
| condition_summary | string | 条件摘要 | fresh、decayed、with_fire、rainy 等 |
| result_summary | string | 结果摘要 | 成功、失败、中毒、无效果等 |
| location_id | RegionId | 位置 | 发生区域，可选 |
| actor_id | EntityId | 行动者 | 谁执行或经历 |
| target_ids | list<string> | 目标 ID | 行为目标 |
| numeric_summary | map<string,string> | 数值摘要 | 饥饿变化、伤害等展示摘要 |
| narrative_key | string | 叙事键 | 前端本地化键 |
| narrative_params | map<string,string> | 叙事参数 | 展示用参数 |

注意：

```text
narrative_key 不是规则依据。
规则判断必须使用结构化字段。
```

### 5.5 MemoryStrength（记忆强度）

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| importance | EventImportance | 重要性 | Trace、Normal、Important、Critical 等 |
| intensity | FeedbackIntensity | 反馈强度 | 来源反馈强度 |
| emotional_weight | Ratio | 情绪权重 | 危险、惊讶、损失等带来的权重 |
| survival_weight | Ratio | 生存权重 | 对生存影响程度 |
| novelty_weight | Ratio | 新颖权重 | 是否首次发现或罕见 |
| repetition_weight | Ratio | 重复权重 | 重复验证程度 |
| teaching_weight | Ratio | 教学权重 | 是否参与传授或族群学习 |
| total_strength | Ratio | 总强度 | 综合强度，0 到 1 |

### 5.6 MemoryTag（记忆标签）

`MemoryTag` 是局部结构，不是第一阶段通用枚举。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| tag_key | string | 标签键 | risk、success、failure、food、fire、teaching 等 |
| tag_value | string | 标签值 | 可选细分值 |
| weight | Ratio | 权重 | 标签强度 |
| source | string | 来源 | rule / config / inference / system |

标签约定：

```text
标签用于索引、压缩和投影，不应替代结构化字段。
高频标签后续可提升为正式枚举，但第一阶段先局部化。
```

### 5.7 MemoryDecayState（记忆衰减状态）

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| decay_enabled | bool | 是否衰减 | 是否允许衰减 |
| decay_rate | Ratio | 衰减率 | 每个衰减周期减少幅度 |
| last_decay_at | GameTime | 最近衰减时间 | 最近处理时间 |
| decay_count | Count | 衰减次数 | 衰减过几次 |
| min_strength | Ratio | 最低强度 | 衰减下限 |
| decay_reason | ConfidenceChangeReason | 衰减原因 | 通常为 Decay |

### 5.8 MemoryProtection（记忆保护）

`MemoryProtection` 防止关键记忆被过早压缩或删除。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| protected_from_decay | bool | 防衰减 | 是否不衰减或低衰减 |
| protected_from_compression | bool | 防压缩 | 是否不能压缩细节 |
| protected_from_archive | bool | 防归档 | 是否不能归档 |
| reason_keys | list<string> | 保护原因 | first_discovery、fatal_risk、civilization_breakthrough 等 |
| expires_at | GameTime | 过期时间 | 可选，保护到期时间 |

必须保护的记忆：

```text
首次发现。
首次死亡或重大伤害。
强反证。
文明突破。
族群重大失败。
教学成功或失败关键样本。
唯一对象或关键剧情记忆。
```

### 5.9 MemorySummary（记忆摘要）

`MemorySummary` 是结构化压缩结果。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| summary_id | string | 摘要 ID | 临时或持久摘要标识 |
| owner | MemoryOwner | 拥有者 | 谁的摘要 |
| summary_key | string | 摘要键 | subject + action + effect + condition |
| subject_kind | string | 主体类型 | object_definition 等 |
| subject_id | string | 主体 ID | 对象或区域等 |
| action_type | ActionType | 行为 | 行为类型 |
| effect_summary | string | 效果摘要 | 总结效果 |
| condition_summary | string | 条件摘要 | 总结条件 |
| sample_count | Count | 样本数 | 来源记忆数量 |
| success_count | Count | 成功数 | 支持效果的次数 |
| failure_count | Count | 失败数 | 无效或失败次数 |
| conflict_count | Count | 冲突数 | 矛盾样本数量 |
| risk_count | Count | 风险数 | 风险样本数量 |
| confidence_support | Ratio | 置信支持 | 对知识形成的支持度 |
| source_memory_ids | list<MemoryId> | 来源记忆 | 被摘要引用的记忆 |
| representative_memory_ids | list<MemoryId> | 代表样本 | 可展开展示的样本 |
| contradiction_memory_ids | list<MemoryId> | 冲突样本 | 必须保留的反例 |
| compression_level | MemoryCompressionLevel | 压缩等级 | 摘要压缩等级 |
| created_at | GameTime | 创建时间 | 摘要创建时间 |
| updated_at | GameTime | 更新时间 | 最近更新时间 |

## 7. 记忆生命周期

### 7.1 从候选到短期记忆

```text
1. CognitionResolver 输出 MemoryCandidate。
2. MemoryResolver 读取 MemoryPolicy。
3. 生成 MemoryRecord。
4. retention_level = ShortTerm。
5. compression_level = None。
6. 输出 MemoryCreated 事件草稿。
```

### 7.2 短期到中期

触发条件：

```text
相似记忆重复出现。
记忆被多次访问。
认知状态因该记忆进入 Active。
反馈强度较高。
当前探索阶段结束，需要整理。
```

结果：

```text
retention_level: ShortTerm → MidTerm
compression_level: None → Light
生成或更新 MemorySummary
输出 MemoryUpdated 事件草稿
```

### 7.3 中期到长期

触发条件：

```text
多次重复验证。
高风险或高收益。
参与教学或族群长期使用。
支撑可传授知识。
和文明能力有关。
```

结果：

```text
retention_level: MidTerm → LongTerm
compression_level: Light → Summary
protection 可增强
生成 KnowledgeEvidenceBundle
```

### 7.4 长期到归档

触发条件：

```text
长期未访问。
已经形成稳定知识。
原始细节不再频繁用于决策。
存储或查询需要压缩。
```

结果：

```text
retention_level: LongTerm → Archived
compression_level: Summary → Statistical / Archived
保留代表样本和来源引用
不删除强反证和关键样本
```

### 7.5 记忆强化

记忆强化来源：

```text
重复经历。
被认知系统引用。
被知识系统引用。
被教学或传播使用。
发生强风险或强收益。
玩家或 AI 主动回顾。
```

### 7.6 记忆衰减

衰减原则：

```text
ShortTerm 衰减快。
MidTerm 衰减中等。
LongTerm 衰减慢。
Archived 低频读取，但保留索引。
被保护记忆不衰减或低衰减。
```

### 7.7 压缩和解压

压缩流程：

```text
MemoryRecord 列表
  ↓
MemoryCompressionPolicy
  ↓
MemorySummary
  ↓
MemoryBundle
  ↓
KnowledgeEvidenceBundle
```

解压流程：

```text
MemorySummary / MemoryBundle
  ↓
MemoryReconstructor
  ↓
MemoryReconstructionView
```

## 8. 事件输出

记忆系统只输出事件草稿，不直接写 EventStore。

### 8.1 MemoryCreated

```text
MemoryCandidate 成功创建 MemoryRecord。
```

| 字段 | 值 |
|---|---|
| event_type | MemoryCreated |
| payload | KnowledgeEventPayload 或后续 MemoryEventPayload 预留 |
| visibility | Hidden / PlayerVisible，根据重要性决定 |
| importance | 来自 MemoryStrength.importance |
| source_event_ids | 来源反馈或认知事件 |
| episode_id | 可选 |
| decision_id | 可选 |

### 8.2 MemoryUpdated

触发：

```text
记忆被强化、衰减、压缩、归档、保护状态变化。
```

建议事件：

```text
MemoryUpdated
```

### 8.3 第一阶段 payload 兼容策略

`04_事件系统设计.md` 尚未单独细化 `MemoryEventPayload`，第一阶段采用兼容策略：

```text
MemoryCreated / MemoryUpdated 可以先使用最小 KnowledgeEventPayload 兼容摘要，
或在实现阶段回填 MemoryEventPayload 到 04。
```

如果后续新增 `MemoryEventPayload`，必须同步更新：

```text
02_通用枚举设计.md
04_事件系统设计.md
20_前后端协议设计.md
22_测试策略设计.md
```

### 8.4 事件可见性建议

| 场景 | EventVisibility | EventImportance |
|---|---|---|
| 普通短期记忆创建 | Hidden | Trace / Normal |
| 玩家重要发现 | PlayerVisible | Important |
| 严重中毒或死亡记忆 | PlayerVisible | Critical |
| 记忆整理为长期经验 | PlayerVisible / TribeVisible | Important |
| 大量归档压缩 | DebugOnly | Trace |

## 14. 设计注意事项

研发实现时必须遵守：

```text
不要把 MemoryRecord 当成 KnowledgeClaim。
不要让记忆系统直接传播知识。
不要让前端创建或修改记忆。
不要把短期、中期、长期拆成三套类。
不要压缩后丢失 source_memory_ids。
不要删除高风险、首次发现、强反证、文明突破记忆。
不要让 AI observation 读取完整历史流水账。
不要用自然语言摘要替代结构化字段。
不要让归档记忆失去可追溯性。
```

### 14.1 长期记忆不是知识

```text
长期记忆：我记得这件事发生过，而且它很重要。
知识：根据多条记忆和认知，我们形成一个稳定断言。
```

### 14.2 压缩必须可解释

压缩结果必须能回答：

```text
这个摘要来自哪些记忆？
有没有冲突样本？
有没有代表样本？
是否保留首次发现？
是否保留高风险样本？
```

### 14.3 记忆系统不使用随机数

同一输入必须得到同一输出：

```text
同一组 MemoryCandidate
+ 同一个 MemoryPolicy
+ 同一个 MemoryStore 状态
+ 同一个 GameTime / StateVersion
= 同一个 MemoryResolveResult
```

## 16. 阶段结论

记忆系统负责把“经历”变成可保存、可解释、可整理、可压缩、可回溯的证据材料。

当前设计支持：

- 短期 / 中期 / 长期 / 归档记忆。
- 工作记忆预留。
- 记忆强化和衰减。
- 结构化压缩。
- 可解释解压视图。
- 高风险和关键记忆保护。
- 知识证据束输出。
- 前端回顾和 AI 长期上下文。
