# P50 教学与无知 NPC 基础行动任务卡设计

## 1. 阶段定位

P50 是 V2.0 世界网格架构中的“知识传播到 NPC”阶段。

它接在：

```text
P43：统一 WorldCommand 管线。
P44：二维世界网格和 actor 坐标。
P45：物品归属与背包接口。
P46：世界生成和资源分布。
P47：资源采集、砍伐、挖掘。
P48：制作与世界反应。
P49：世界经验进入认知 / 记忆 / 知识闭环。
```

它为后续阶段提供：

```text
P51：Agent 网格目标执行。
P52：野兽网格生态与遭遇。
P53：H5 V2 地图界面。
P54：存档、复盘与调试门禁。
```

P50 的核心不是让 NPC 已经会完整规划，而是先让“知识能被教给 NPC，并成为 NPC 后续行动的权威输入”。

一句话：

```text
P49 让玩家学会东西；P50 让 NPC 能被教会东西，并能基于已知知识通过最小的一步行动门禁。
```

## 2. 为什么现在做

P49 已经把 `WorldCommandExecutionResult.experiences` 接入 `LearningLoopService`，形成 actor 私有 `KnowledgeClaim`。

如果不做 P50，会出现断层：

```text
玩家学会了砍树得木头，但 NPC 永远不知道。
玩家制作火把成功，但 NPC 无法被教会火把知识。
NPC 即使有 actor_key，也没有自己的知识查询和传播流程。
后续 P51 的目标执行没有“NPC 自己知道什么”的输入。
前端只能看到玩家知识，不能看到同伴知识变化。
```

P50 必须在 P51 前完成，因为 P51 的 Agent 目标执行不能直接读取玩家知识，也不能写死 NPC 知道什么。

P50 也必须在 P52 前完成，因为野兽和野生 NPC 后续也是 Agent，它们学习 / 观察 / 被驯化 / 被威慑都依赖“知识属于哪个主体”。

## 3. 阶段目标

P50 完成后，系统必须具备以下能力：

```text
1. Teach command 能通过 WorldCommandPipeline 进入后端。
2. 教学者和接收者都使用 KnowledgeOwner，不允许全局共享知识。
3. 教学者必须真实拥有可教学 KnowledgeClaim。
4. 接收者必须是存在的 actor / agent / 后续 tribe owner。
5. 教学必须调用 LearningLoopService 或 KnowledgePropagationService，不能复制粘贴 claim。
6. 接收者获得自己的 KnowledgeClaim，owner 必须变成接收者。
7. 教学结果必须输出 events / state_deltas / projection_patch.changed_knowledge。
8. 无知 NPC 默认没有知识；被教后才能查询到知识。
9. NPC 基础行动门禁能判断：某个 NPC 是否知道某个 action/object/effect 关系。
10. P51 可复用 P50 的查询接口作为 Agent 决策输入。
```

## 4. 本阶段不解决的问题

P50 不做以下内容：

```text
不做完整 NPC 自主规划：P51 做。
不做链式目标执行：P51 / P40 后续接。
不做野兽生态和野兽学习：P52 做。
不做 H5 V2 地图 UI：P53 做。
不做族群会议 / 大规模传播：后续 TribeKnowledge 阶段做。
不做专家 NPC 内容包：后续内容扩展阶段做。
不做大世界存档：P54 做。
不让前端决定 NPC 学会了什么。
不允许为了演示写死“同伴学会火把”。
```

P50 只做“教学传播”和“一步基础行动资格判断”。

## 5. 玩家体验映射

P50 对玩家体验的意义：

```text
玩家探索世界 -> 玩家形成知识 -> 玩家选择一个 NPC -> 玩家教学 -> NPC 知识变化可见。
```

这会形成《先驱者》的核心情感：

```text
我不是单纯升级自己，而是在把文明火种传给群体。
```

P50 的最小闭环：

```text
玩家砍树 / 制作 / 吃东西形成知识。
玩家对同伴执行 teach command。
同伴从无知变成拥有该知识。
后端能回答：同伴现在知道“这个对象可以怎样用”。
```

P50 不要求同伴立刻自动连锁行动，但必须给 P51 留出直接可用的数据入口。

## 6. 核心设计原则

### 6.1 知识不共享，只传播

知识必须属于 owner。

禁止：

```text
把玩家 KnowledgeClaim 直接塞进 NPC 列表且 owner 不变。
把所有知识写进全局 map。
NPC 查询时直接读玩家知识。
```

正确流程：

```text
source claim owner = teacher
propagation result created claim owner = recipient
repository.put(recipient_claim)
```

### 6.2 教学是 Command，不是 UI 操作

前端只能发送：

```text
WorldCommandKind::Teach
actor_key = teacher
 target.target_actor_key = recipient
 target.knowledge_key = source knowledge id
```

前端不能发送：

```text
recipient learned claim json
confidence
status
usable_by_ai
```

### 6.3 传播系统是权威

P50 不能自己复制 claim。

必须使用：

```text
LearningLoopService(story_kind = TeachingToRecipient)
```

或：

```text
KnowledgePropagationService
KnowledgePropagationApplier
```

建议 P50 首选 `LearningLoopService`，因为它已经封装了传播、接收者投影、冲突修订和 debug trace。

### 6.4 教学需要世界约束

教学不是远程全图广播。

P50 MVP 至少校验：

```text
teacher actor 存在。
recipient actor 存在。
teacher 和 recipient 在同一 layer。
距离不超过 max_teach_distance，默认 1 或 2。
```

如果当前 runtime 不方便注入，必须通过 port 抽象，不允许在 handler 里绕开 runtime 内部 map。

### 6.5 无知 NPC 不是傻代码，而是空知识状态

无知 NPC 的含义：

```text
NPC actor 存在。
NPC KnowledgeOwner 存在。
NPC repository 中该 owner 没有相关 KnowledgeClaim。
```

禁止写死：

```text
if npc == companion then knows_torch = true
```

### 6.6 基础行动只做门禁，不做规划

P50 的基础行动能力只回答：

```text
这个 NPC 是否知道某个对象 / 动作 / 结果关系？
这个 NPC 是否有足够知识尝试一个单步 command？
```

它不负责：

```text
没材料怎么办。
去哪里找材料。
狼来了怎么办。
连续执行多少步骤。
```

这些属于 P51/P52/P40。

### 6.7 投影必须可被前端消费

教学结果必须进入：

```text
WorldCommandResponseDto.event_feed
WorldCommandResponseDto.result.state_deltas
WorldCommandResponseDto.projection_patch.changed_knowledge
```

前端只展示这些结果，不自己算 NPC 学会了什么。

### 6.8 测试优先证明“不是写死”

测试不能只测火把。

P50 测试至少覆盖：

```text
采集知识。
制作知识。
危险知识。
未知知识 id。
不可教学 claim。
距离过远。
接收者已有相同知识。
NPC 反向教学玩家。
```

## 7. 模块划分

| 模块 | 中文含义 | 职责 | 输入 | 输出 | 禁止事项 |
|---|---|---|---|---|---|
| WorldTeachingTypes | 世界教学类型 | 定义请求、结果、失败码、传播摘要 | teach command / claims | DTO | 不读写 repository |
| WorldActorKnowledgeOwnerResolver | actor 知识 owner 解析器 | actor_key 和 KnowledgeOwner 互转 | actor_key | KnowledgeOwner | 不创建全局知识 |
| WorldTeachingEligibilityService | 教学资格服务 | 校验教师、接收者、距离、claim 可教学 | request / runtime / repo | eligibility result | 不执行传播 |
| WorldTeachingLoopBridge | 教学学习桥 | 把教学请求转成 LearningLoopInput | source claim / recipient | LearningLoopResult | 不直接复制 claim |
| WorldTeachingStorePort | 教学存储端口 | 保存接收者 claim | propagation result | repository writes | 不改世界 runtime |
| WorldTeachingProjectionBridge | 教学投影桥 | 转成 knowledge patch / event / delta | recipient claims | world response parts | 不判断规则 |
| TeachCommandHandler | Teach command 处理器 | 接入 WorldCommandPipeline | WorldCommandDto | WorldCommandExecutionResult | 不写死同伴 |
| NpcBasicActionKnowledgeGate | NPC 基础行动知识门禁 | 查询 NPC 是否知道某单步行为 | actor/action/object/effect | allow/block | 不规划路径 |

## 8. 核心枚举

### 8.1 WorldTeachingFailureKind

中文：世界教学失败原因。

允许 `Unknown`：允许，只用于未初始化。

允许 `TestOnly`：允许，只用于测试。

枚举值：

| 值 | 中文 | 使用场景 |
|---|---|---|
| None | 无失败 | 教学成功 |
| InvalidRequest | 请求非法 | command 缺字段 |
| TeacherMissing | 教师不存在 | actor_key 不存在 |
| RecipientMissing | 接收者不存在 | target_actor_key 不存在 |
| SameOwnerNotAllowed | 不允许自己教自己 | teacher == recipient 且 options 不允许 |
| OutOfRange | 超出教学距离 | 不在交流范围 |
| SourceClaimMissing | 源知识不存在 | knowledge_id 查不到 |
| SourceClaimOwnerMismatch | 源知识不属于教师 | 防止偷教别人知识 |
| ClaimNotTeachable | 知识不可教学 | teaching_profile 不允许 |
| RecipientRejected | 接收者拒绝 | 后续信任/敌对关系 |
| PropagationFailed | 传播失败 | LearningLoop 或 Propagation 失败 |
| StoreFailed | 写入失败 | repository.put 失败 |
| ProjectionFailed | 投影失败 | patch/delta 校验失败 |
| TestOnly | 测试专用 | 生产禁止 |

字符串转换使用 PascalCase，例如 `OutOfRange`。

### 8.2 WorldTeachingDecision

中文：教学决策结果。

| 值 | 中文 | 使用场景 |
|---|---|---|
| Unknown | 未知 | 未初始化 |
| TaughtNewClaim | 教会新知识 | recipient 新增 claim |
| ReinforcedExistingClaim | 强化已有知识 | recipient 已有相同知识 |
| RevisedExistingClaim | 修正已有知识 | recipient 有冲突知识 |
| SkippedAlreadyKnown | 跳过，已知道 | 无需变化 |
| Rejected | 被拒绝 | 信任/关系/不可教学 |
| Failed | 失败 | 系统失败 |
| TestOnly | 测试专用 | 生产禁止 |

### 8.3 NpcActionKnowledgeGateDecision

中文：NPC 单步行动知识门禁结果。

| 值 | 中文 | 使用场景 |
|---|---|---|
| Unknown | 未知 | 未初始化 |
| AllowedByKnowledge | 知识允许 | NPC 知道这个动作可尝试 |
| BlockedNoKnowledge | 缺知识 | NPC 无相关知识 |
| BlockedLowConfidence | 信心不足 | claim 只是低可信假设 |
| BlockedDangerWithoutGoal | 危险且无目标收益 | 风险知识阻止行动 |
| BlockedClaimDisproven | 知识已证伪 | 不允许使用 |
| Failed | 系统失败 | 查询失败 |
| TestOnly | 测试专用 | 生产禁止 |

## 9. 核心 DTO

### 9.1 WorldTeachingRequest

```cpp
struct WorldTeachingRequest {
    std::string request_id;
    world_command::WorldCommandDto source_command;
```

## 10. Teach Command 协议

前端 / Agent / 系统统一发送 `WorldCommandDto`：

```cpp
WorldCommandDto command;
command.command_kind = WorldCommandKind::Teach;
command.command_key = "Teach";
command.source = WorldCommandSource::PlayerInput;
command.actor_key = "player";
command.target.target_kind = WorldCommandTargetKind::Actor;
command.target.target_actor_key = "companion_001";
command.target.knowledge_key = "knowledge_xxx";
```

可选参数：

| 参数 | 中文 | 权威 |
|---|---|---|
| channel | 教学渠道 | 后端可覆盖 |
| trust_hint | 信任提示 | 后端可忽略 |
| max_distance | 最大距离 | 后端配置优先 |

禁止参数：

```text
recipient_status
recipient_confidence
recipient_claim_json
force_teach
learned=true
```

## 11. TeachCommandHandler 流程

```text
1. 解析 command。
2. teacher_actor_key = command.actor_key。
3. recipient_actor_key = command.target.target_actor_key。
4. source_knowledge_id = command.target.knowledge_key。
5. 从 repository 查询 teacher_claims / recipient_claims。
6. WorldTeachingEligibilityService 校验：
   - actor 存在。
   - owner 匹配。
   - 知识存在。
   - 知识属于 teacher。
   - claim.teaching_profile.teachable == true。
   - 距离满足。
7. WorldTeachingLoopBridge 构造 LearningLoopInput：
   - story_kind = TeachingToRecipient。
   - actor = teacher。
   - recipient = recipient。
   - actor_existing_claims = teacher_claims。
   - recipient_existing_claims = recipient_claims。
   - feedback 使用安全教学反馈，不包含隐藏真相。
8. 调用 LearningLoopService。
9. LearningLoopApplier.applyRecipientClaims 得到 recipient_snapshot_after。
10. WorldTeachingStorePort 保存新增/更新 recipient claims。
11. WorldTeachingProjectionBridge 生成 events/state_deltas/projection_patch。
12. 返回 WorldCommandExecutionResult。
```

## 12. 教学反馈如何构造

教学反馈不是“新的世界经验”，而是传播证据。

建议最小字段：

```text
feedback_key = command_id + ':teaching_feedback'
cognition_target = source_claim.subject
memory_subject = source_claim.subject
action_context = Observe 或 Touch
 action_key = 'teach'
effect_key = source_claim.predicate.effect_key
outcome_signals = KnowledgeShared 或 NoVisibleEffect；如果没有专用信号，用 NoVisibleEffect
utility_delta = 0.1
risk_delta = 0.0，危险知识仍由 claim predicate / teaching_profile 表达
source_event_ids = teaching event id
reason_keys = ['p50_teaching', source_claim.knowledge_id]
```

注意：如果现有 `CognitionOutcomeSignal` 没有 `KnowledgeShared`，不要改旧枚举制造大范围影响；P50 可以用 `NoVisibleEffect`，传播结果由 KnowledgePropagationService 决定。

## 13. NPC 基础行动知识门禁

P50 的 `NpcBasicActionKnowledgeGate` 只做“单步行为是否有知识依据”。

匹配顺序：

```text
1. owner = actor。
2. projection_flags.usable_for_action == true。
3. subject_object_key 匹配。
4. action_key 匹配。
5. effect_key 如果请求提供，则必须匹配。
6. target_object_key 如果请求提供，则必须匹配 claim.subject.related_subject_ids 或 predicate target。
7. status 不能是 Deprecated / Disproven。
8. 如果 allow_hypothesis=false，则 Hypothesis / Candidate 不允许。
9. 如果 claim 是高风险知识，且 allow_risk_action=false，则只允许用于避险，不允许主动执行危险动作。
```

返回：

```text
AllowedByKnowledge：P51 可以把该 claim 作为 action candidate。
BlockedNoKnowledge：NPC 目前不知道。
BlockedLowConfidence：知道但不够稳。
BlockedDangerWithoutGoal：知道危险，但当前目标不是处理危险。
```

P50 不执行 command，只给 P51 / command validation 使用。

## 14. 与 P49 的关系

P49 形成玩家或 NPC 自己的知识。

P50 教学必须消费 P49 输出的真实 claim：

```text
P49 learned_claims -> repository -> P50 source_claim -> propagation -> recipient_claim
```

如果 P49 没有产生 claim，P50 不能凭 reason_key 直接教学。

## 15. 与 P51 的关系

P51 需要从 P50 获得：

```text
actor_key -> actionable claims
actor/action/object/effect -> gate result
teaching 后 recipient snapshot
```

P51 不应该重新实现教学，也不应该自己判断 claim 是否可传播。

## 16. 与世界网格的关系

P50 需要最小地图约束：

```text
teacher 和 recipient 都是 WorldActorRuntime。
两者 coord 必须存在。
同 layer。
曼哈顿距离 <= max_teach_distance。
```

如果当前 runtime port 还没有直接获取 actor 的接口，P50 可以新增只读 port：

```cpp
class IWorldActorQueryPort {
public:
    virtual std::optional<world_runtime::WorldActorRuntime> findActor(const std::string& actor_key) const = 0;
};
```

禁止：

```text
在 P50 里直接访问 WorldGridRuntime 的 private map。
为了测试把距离校验写死为永远通过。
```

测试可使用 fake port 模拟 actor 坐标。

## 17. 与 Repository 的关系

P50 可以使用现有 `KnowledgeRepository`：

```text
listByOwner(teacher_owner)
listByOwner(recipient_owner)
find(source_knowledge_id)
put(recipient_claim)
```

但必须校验：

```text
find(source_knowledge_id) 返回的 claim.owner == teacher_owner。
```

如果 claim id 存在但 owner 不匹配，返回 `SourceClaimOwnerMismatch`。

## 18. Projection 输出规范

教学成功至少输出一个 event：

```text
event_kind = 'knowledge_taught'
actor_key = teacher_actor_key
target_entity_id = recipient_actor_key
reason_keys = ['p50_teaching', source_knowledge_id]
```

至少输出一个 state delta：

```text
delta_kind = 'recipient_knowledge_delta'
target_ref = recipient_actor_key
fields['knowledge_id'] = recipient_claim.knowledge_id
fields['source_knowledge_id'] = source_claim.knowledge_id
fields['decision'] = decision
```

projection patch：

```text
changed_knowledge.actor_key = recipient_actor_key
changed_knowledge.op = Update
fields 包含 knowledge_id / subject_id / action_key / effect_key / status / confidence / teachable / usable_by_ai / usable_for_action
```

## 19. 错误处理

失败时不应修改 repository。

| 场景 | result_kind | failure_kind | 是否有 patch |
|---|---|---|---|
| 缺 recipient | Failed | InvalidRequest | 否 |
| source claim 不存在 | Blocked | SourceClaimMissing | 否 |
| source claim 不属于 teacher | Blocked | SourceClaimOwnerMismatch | 否 |
| 不可教学 | Blocked | ClaimNotTeachable | 否 |
| 距离太远 | Blocked | OutOfRange | 否 |
| 传播失败 | Failed | PropagationFailed | 否 |
| 写入失败 | Failed | StoreFailed | 否 |

## 20. 安全边界

禁止泄露：

```text
隐藏对象 trait。
隐藏配方真相。
未投影给 teacher 的知识。
recipient 私有 debug 数据。
前端传入的 claim 内容。
```

教学只能传播 teacher 已拥有且可教学的 `KnowledgeClaim`。

## 22. 验收标准

P50 通过标准：

```text
1. TeachCommandHandler 能注册到 WorldCommandHandlerRegistry。
2. 玩家拥有知识，NPC 无知识，教学后 NPC 获得自己的 claim。
3. recipient claim owner 不是玩家，而是 NPC。
4. 不可教学 claim 不能传播。
5. 距离过远不能传播。
6. 不存在的 knowledge_id 不能传播。
7. NPC 反向教学玩家使用同一套服务。
8. NpcBasicActionKnowledgeGate 能基于 NPC 自己的 claims 判断行动是否允许。
9. 教学结果有 event/state_delta/projection_patch。
10. 不依赖 H5 / dialog / playable。
```
