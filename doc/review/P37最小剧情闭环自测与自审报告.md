# P37 最小剧情闭环自测与自审报告

## 1. 审查结论

P37 已按 `doc/66_P37最小剧情闭环任务卡设计.md` 完成第一版后端最小剧情闭环实现，并完成专项测试、相邻回归、代码静态差异检查和人工自审。

本阶段实现结论：系统现在具备一个可由 H5 入口驱动的“第一天生存”剧情骨架。玩家可以围绕红果、腐烂红果、斧头、木头、磨石、干草、火种、火把、靠近的野兽进行尝试、形成知识、教学、推进时间，并在黄昏/夜晚压力下使用火把反制危险。

P37 仍然不是正式任务系统、不是完整地图、不是完整背包、不是战斗系统。它是把 P15-P36 的认知、知识、传播、对象反应、危险压力、H5 安全投影串起来的最小剧情闭环。

## 2. 实现范围

### 2.1 新增剧情模块

新增 `pathfinder_story` 静态库：

- `backend/include/pathfinder/story/story_types.h`
- `backend/include/pathfinder/story/story_system.h`
- `backend/src/story/story_types.cpp`
- `backend/src/story/story_system.cpp`

模块职责：

- 定义 P37 剧情枚举、DTO、运行时状态、剧情投影。
- 提供 `StoryScenarioRegistry`，输出 `first_day_survival` 场景定义。
- 提供 `StoryRuntimeFactory`，创建最小剧情运行时状态。
- 提供 `StoryProgressionService`，根据回合结果推进时间、目标、危险状态。
- 提供 `StoryProjectionAdapter`，把剧情状态转换为 H5 可展示的安全剧情投影。

### 2.2 H5 可玩壳集成

扩展 `pathfinder_h5_playable`：

- `H5PlayableTurnService` 在每次 bootstrap/turn 后读取对话回合状态，推进剧情状态，再把剧情目标、时间、危险提示、推荐行动合入 `H5GameProjection`。
- `H5PlayableProjectionMapper` 增加 P37 相关对象和效果的中文展示映射。
- `DialogTurnService` 增加 `saveSessionSnapshot()`，用于保存最小剧情状态。

P37 没有改变前端协议结构，仍复用 P32/P33 已有安全 projection。

### 2.3 对话场景内容扩展

扩展默认 H5 对话场景：

- 新增对象：`dry_grass` 干草、`fire_seed` 火种、`torch` 火把、`beast_shadow` 靠近的野兽。
- 新增反馈：
  - `fire_seed + use @ dry_grass -> ignite_fire`
  - `wood + use @ fire_seed -> make_torch`
  - `torch + use @ beast_shadow -> repel_beast`
- 新增中文意图：
  - 用火种点燃干草
  - 制作火把
  - 用火把驱赶野兽

这些内容通过后端场景和反馈模板进入学习链路，不由 H5 前端写死知识结果。

## 3. 架构边界自审

### 3.1 后端仍是唯一权威

人工检查结论：通过。

P37 的剧情推进发生在后端 `H5PlayableTurnService -> StoryProgressionService -> StoryProjectionAdapter`，前端仍只消费 projection。

H5 前端没有被要求新增规则判断，也没有成为火把、野兽、时间阶段、目标完成的裁决者。

### 3.2 剧情模块没有替代认知/知识系统

人工检查结论：通过。

P37 没有绕过 P21 学习闭环直接创建知识。火源、火把、驱赶野兽仍通过 `DialogFeedbackTemplate` 反馈进入现有学习、记忆、知识形成流程。

剧情模块只读取摘要：

- 玩家已有知识效果 key。
- 同伴已有知识效果 key。
- 当前可见对象。
- 已完成动作摘要。
- 最近对话反馈。

它不直接写入知识仓库。

### 3.3 H5 输出仍走安全投影

人工检查结论：通过。

`StoryProjectionAdapter::composeProjection()` 对 `ProjectionAudience::Player` 不输出剧情调试 key；`story_projection_player_safe` 已覆盖玩家投影安全。

P37 合入 H5 projection 的内容是中文目标、时间提示、危险提示和推荐行动，不向玩家输出 raw state、hidden truth、真实 HP、掉落、死亡细节。

### 3.4 剧情状态持久化是最小实现

人工检查结论：通过，但标记为后续改进点。

P37 当前把剧情阶段、时间点、威胁状态、生成物摘要暂存在 `DialogSessionState.debug_keys` 中，并通过 `saveSessionSnapshot()` 保存。

这是 P37 最小闭环可接受方案，因为它避免新增正式存档结构；但后续 P41/P存档扩展时，应把剧情状态迁移到正式可序列化的剧情状态块，避免长期依赖 debug key 承载运行时状态。

### 3.5 没有引入完整战斗系统

人工检查结论：通过。

`beast_shadow` 是危险征兆/危险事件，不是有 HP、技能、掉落和战利品的战斗实体。P37 只验证“火把知识可以反制夜晚危险”的最小闭环。

### 3.6 没有引入正式地图和背包

人工检查结论：通过。

P37 仍是地点式小场景，没有坐标、寻路、碰撞、视野、装备栏、重量、堆叠等系统。

## 5. 人工自审细节

### 5.1 是否符合 `00_设计文档编写要求.md`

结论：基本符合。

- 有阶段设计文档。
- 有 context pack。
- 有枚举、DTO、服务接口、流程、测试和边界说明。
- 实现按后端模块落地，没有让前端成为规则层。
- 自检报告如实记录通过项和失败项。

### 5.2 是否符合 P37 设计目标

结论：第一版最小闭环达成。

已具备：

- 第一天天生存场景定义。
- 红果/腐烂红果/斧头/木头/磨石/干草/火种/火把/野兽材料链。
- 观察、吃、使用、目标使用、等待、教同伴、查看知识。
- 时间阶段推进。
- 黄昏/夜晚危险提示。
- 火把驱赶野兽。
- 关键行动形成知识。
- 同伴教学仍走现有传播链路。
- H5 投影能展示目标、时间、危险和推荐行动。

未做且符合边界：

- 不做正式大地图。
- 不做复杂战斗。
- 不做正式背包。
- 不做自由 NLP。
- 不做完整任务系统。

### 5.3 发现的不足

1. P37 当前剧情状态保存方式偏临时，依赖 `DialogSessionState.debug_keys` 承载剧情状态；后续应迁移到正式剧情状态存档。
2. P37 当前剧情目标判断仍偏摘要化，部分判断通过知识 effect key 和中文反馈辅助识别，后续应接入更正式的事件/反应结果摘要。
3. P37 的“制作火把”目前通过 `wood + use @ fire_seed -> make_torch` 表达，还不是完整材料消耗/产物库存系统。
4. 野兽仍是危险提示和反制事件，不是智能 Agent；这是设计内边界，但后续若进入生态阶段需要升级。
5. 全量测试暴露 P21 旧链路失败，虽然不是 P37 直接引入，但会影响长期质量门禁。
