## 3. 完整修复项复审

### 3.1 P46/P58/P59 region 坐标体系不一致

上一轮问题：

```text
generateInitialWorld 的 region (0,0) 曾生成 [0,15]，而 WorldRegionMath::coordToRegion 认为 region (0,0) 是 [-8,7]。
这会导致 snapshot DTO 不能安全判断 coord 是否属于 region。
```

当前修复：

```text
1. WorldRegionMath 新增 centered region 边界工具：regionMinLocalCoord、regionMaxLocalCoord、regionMinWorldCoord、regionMaxWorldCoord、coordBelongsToRegion。
2. TerrainGenerator / TerrainNoiseFieldBuilder 统一使用 WorldRegionMath 生成 region 内坐标和 regionRuntimeId。
3. WorldGridRuntime::generateInitialWorld 改为 centered region loop，且不反向链接 world_generation target。
4. 新增 region_coord_consistency 测试，验证 -1/0/1 区块每个 cell 都能映射回所属区块。
```

复审结论：通过。

注意：runtime 为避免 target 依赖方向破坏，使用同语义本地 loop；WorldGenerationService 路径继续使用 WorldRegionMath。两者已由测试共同约束。

---

### 3.2 generator_version / content_version runtime 权威字段缺失

上一轮问题：

```text
sealRegion 构造 snapshot 时 generator_version/content_version 为空，DTO 只能临时允许空值。
```

当前修复：

```text
1. WorldRuntimeConfig 新增 generator_version、content_version、worldgen_profile_key 默认值。
2. IWorldRuntime 新增 generatorVersion()、contentVersion()、worldgenProfileKey()。
3. WorldGridRuntime 实现上述接口。
4. WorldRegionLifecycleService::sealRegion 从 runtime 元数据写入 snapshot build context。
5. WorldRegionSnapshotHeader::validateBasic() 强制 generator_version 和 content_version 非空。
6. snapshot_validation 增加缺失版本字段负例。
```

复审结论：通过。

---

### 3.3 DetachSafeOnly 不真正 detach

上一轮问题：

```text
SealMode::DetachSafeOnly 只检查 unload guard 并缓存 snapshot，但没有真正从 runtime 移除 cell/entity/node。
```

当前修复：

```text
1. IWorldRuntime 新增 detachRegion(region_id) 权威接口。
2. WorldGridRuntime::detachRegion 删除区块内 cells、OnMap entities、resource nodes，并清理 generated_regions_ 与 exploration 引用。
3. WorldRegionLifecycleService::sealRegion 在 store.put 成功后调用 runtime.detachRegion。
4. detach 失败不会丢 snapshot，会记录 warning；detach 成功后 lifecycle 保持 CachedSnapshot。
5. 新增 detach_safe_only_detaches 测试：seal -> detach -> restore 全链路验证。
```

复审结论：通过。

---

### 3.4 WorldRegionSaveBridge store key enumeration 未完成

上一轮问题：

```text
WorldRegionSaveBridge::buildSectionDraft() 不知道 store 里有哪些 key，只能返回空 collection 和 warning。
```

当前修复：

```text
1. IWorldRegionStateStore 新增 keys()。
2. InMemoryWorldRegionStateStore 维护 snapshots_ 与 keys_ 双索引，put/remove/clear 同步更新。
3. WorldRegionLifecycleService::cachedRegionKeys() 改为直接返回 store.keys()。
4. WorldRegionSaveBridge::buildSectionDraft() 遍历 store.keys()，find snapshot 后写入 region_snapshots。
5. save_bridge 测试验证空 store 无 warning、 populated store 可枚举并导出 snapshot。
```

复审结论：通过。

### 3.6 DTO coord 属区校验未落地

上一轮问题：

```text
Snapshot 只检查 cell.region_id，没有校验 cell/entity/resource node 的 coord 是否真的属于 header.region_key。
```

当前修复：

```text
1. WorldRegionSnapshot::validateBasic() 对 cells 校验 region_id 和 coordBelongsToRegion。
2. on_map_entities 必须 OnMap，且 coord 必须属于 snapshot region/layer。
3. resource_nodes 必须 node_id 非空、charges 合法，且 coord 必须属于 snapshot region/layer。
4. snapshot_validation 增加 cell out-of-region、entity wrong layer、node out-of-region 负例。
```

复审结论：通过。

## 4. 架构边界复审

### 4.1 Command / Runtime / Region 边界

结论：通过。

说明：本次修复没有新增玩法旁路；区块 detach/restore 属于 runtime/region lifecycle 权威职责，没有让前端或测试绕过 Command 管线完成玩法行为。

### 4.2 target 依赖方向

结论：通过。

说明：修复中曾发现 `world_runtime` 直接调用 `WorldRegionMath::cellsInRegion()` 会造成 link 依赖问题。已改为 runtime 本地 centered coordinate loop，保持 `world_runtime` 不反向依赖 `world_generation`。

### 4.3 save bridge

结论：通过。

说明：当前仍是 in-memory/store-section 桥，不是完整磁盘存档系统；但 P59 范围内“缓存区块能进入 save section draft”已经不是占位实现。
