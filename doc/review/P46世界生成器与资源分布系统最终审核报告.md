## 1. 最终结论

结论：P46 通过最终审核，可以验收。

当前实现已经满足 P46 的核心目标：

```text
可确定性生成世界区域。
生成结果真实落地到 WorldGridRuntime。
资源节点有 runtime 权威存储。
地面物品通过 P45 location port 建立 OnMap 归属。
生成过程不创建玩家背包物品。
同 seed 同配置可复盘。
同一区域重复 apply 被阻止。
Command handler 能真实触发生成并返回投影变化。
```

## 3. 架构审核

### 3.1 P45 归属门禁

通过。

P46 地面物品落地路径为：

```text
GroundItemPlacementRule
-> GeneratedEntityDraft
-> WorldGenerationApplier::applyEntities
-> IWorldEntityLocationPort::spawnEntityOnMap
-> WorldGridRuntime entity OnMap + cell.entity_ids
```

这符合硬性门禁：

```text
P46/P47 生成或采集物品时，必须通过 P45 的归属接口建立 OnMap 或 InInventory 状态，不能绕过 runtime 直接改数组。
```

当前扫描未发现 P46 直接写 `item_locations_` 或 inventory `entries`。

### 3.2 WorldCommand 边界

通过。

扫描结果：

```text
backend/include/pathfinder/world_command
backend/src/world_command
```

未直接包含 `pathfinder/world_generation`。

P46 的 runtime-aware GenerateWorld handler 留在 `world_generation` 模块中，没有污染基础 `world_command` 模块。

### 3.3 确定性

通过。

扫描未发现：

```text
std::time
time(
random_device
std::rand
rand(
srand
```

P46 生成使用 `WorldGenerationRng`，manifest 不再依赖真实时间。

### 3.4 ResourceNode 与 GroundItem 分离

通过。

当前结构：

```text
ResourceNode -> WorldResourceNodeRuntime，不进入 WorldEntityInstance。
GroundItem -> WorldEntityInstance OnMap，通过 P45 location port。
```

测试已覆盖 ResourceNode 不在普通 entity 列表中。

### 3.5 重复生成保护

通过。

当前设计：

```text
generate() 是纯生成。
apply() 成功后 markRegionGenerated。
再次 apply 同一区域返回 RegionAlreadyGenerated。
```

这避免重复生成覆盖玩家改动。

## 5. 当前 P46 能力

P46 完成后，系统具备：

```text
1. 可配置 first_world profile。
2. 确定性 terrain/resource/ground item 生成。
3. region 坐标影响真实世界坐标。
4. runtime cell 真实创建。
5. resource node runtime 权威存储。
6. ground item OnMap 归属闭环。
7. GenerateWorld command handler 真实接入 runtime。
8. projection patch / changed_cells / events 输出。
9. 重复 apply 同 region 阻止。
10. layer_keys 最小版明确只支持 default layer，避免半实现误导。
```

## 6. 非阻塞后续建议

这些不是 P46 验收阻塞，建议进入 P47/P48/P54 时继续补强：

1. `WorldRuntime` 后续可把 region generated manifest 从 `std::set<std::string>` 扩展为带 seed/profile/version 的 manifest 存储。
2. 后续如要支持地下/空中层，应单独设计多 layer 生成，不要在 P46 当前最小版里半接入。
3. P47 采集 ResourceNode 时，必须继续通过 P45 归属接口创建 `InInventory` 或 `OnMap` 产物。
4. 后续存档阶段需要序列化 `resource_nodes` 和 generated region manifest。
5. P47 要补 ResourceNode charge 消耗、depleted 状态和采集产出归属测试。
