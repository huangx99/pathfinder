## 问题闭环检查

### 1. protected_key 安全校验

状态：已通过。

当前 `DefendAction::validateBasic()` 已检查：

```text
protected_key 为空时拒绝。
protected_key 包含 forbidden key 时拒绝。
```

对应测试：

```text
p24_security_protected_key_forbidden
```

### 2. damage/death/kill/loot/hp 禁止语义

状态：已通过。

当前 forbidden token 已包含：

```text
damage
death
kill
loot
hp
```

对应测试覆盖：

```text
p24_security_generic_damage_death_kill_loot
```

这能防止 P24 共同作战阶段提前泄露或实现伤害、死亡、击杀、掉落、血量等 P25/P28 之后才可能处理的语义。

### 3. CombatCoordinationInput 必须带合法 TribeState

状态：已通过。

当前 `CombatCoordinationInput::validateBasic()` 已强制：

```text
tribe_state.validateBasic() 必须通过。
tribe_state.tribe_id 必须等于 input.tribe_id。
```

对应测试：

```text
p24_security_empty_tribe_state
```

这避免上游漏传 `TribeState` 时被误判为 `no_available_member`。

### 4. Explorer 预警流程

状态：已通过。

已新增：

```text
p24_explorer_warns_group_flow
```

测试验证：

```text
Explorer 会生成 AssistActionKind::Warn。
trace 中包含 assist:warn。
```

### 5. threat_weight 生效

状态：已通过。

`threat_weight` 已进入协作评分公式：

```text
coordination_score = avg_member_score + role_bonus - split_risk_penalty - safety_pressure - threat_pressure * threat_weight
```

这让配置项不再是“假配置”。

### 6. KnowledgeId 格式校验

状态：已通过。

以下字段已补 `isValidIdString` 校验：

```text
ThreatPressureSummary.known_counter_knowledge_ids
CombatParticipant.known_knowledge_ids
```

对应测试：

```text
p24_security_invalid_knowledge_id
```

## 架构复审

本次复审确认：

- P24 仍只做同族群内部共同作战协调，不做战斗伤害结算。
- 没有新增死亡、掉落、HP、命中、真实怪物属性逻辑。
- 没有引入 H5、HTTP、SaveManager、AgentRuntime 依赖。
- `CombatCoordinationStateDraft` 仍是状态影响草稿，不直接修改 `TribeState`。
- `MemberCoordinationSummary` 保持成员聚合层，继续遵守 P23 的重复 `member_id` 约束。
- P24 没有提前实现 P25 的族群战争、仇恨、外交、宣战、停战。
