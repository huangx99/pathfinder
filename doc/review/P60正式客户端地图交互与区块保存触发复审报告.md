### 2.2 过期 selection 不再污染 session option 快照

涉及文件：

```text
backend/src/world_map_interaction/client_map_selection_service.cpp
backend/src/client_http/client_http_gateway.cpp
```

修复内容：

```text
1. selection 检测到 projection_version 过期时立即返回 stale / requires_full_refresh。
2. stale selection 不再生成 selected_objects / available_options。
3. HTTP gateway 只有在 selection 非 stale 且不要求 full refresh 时，才写回 session available_commands。
```

复审结论：通过。客户端不能用旧投影点击地图对象来覆盖后端权威 option 快照。

### 2.3 activity window 的 seal delay / recently touched 策略真实生效

涉及文件：

```text
backend/include/pathfinder/world_map_interaction/region_activity_window_service.h
backend/src/world_map_interaction/region_activity_window_service.cpp
```

修复内容：

```text
1. 新增 actor region touch 入口。
2. 记录 region 首次离开 active window 的 tick。
3. region 离开 active window 后先进入 blocked，满足 seal_delay_ticks 后才成为 seal candidate。
4. recently_touched_window_ticks 内的 region 不会被 seal。
5. region sealed 后才按 detach_delay_ticks 进入 detach candidate。
```

复审结论：通过。区块不会因为一次移动或刚交互过就被立即保存/卸载。

## 5. 剩余风险与后续建议

### 5.1 active Agent / danger / ongoing command 仍为后续扩展点

`RegionActivityWindowService::collectBlockedRegionIds()` 当前已经阻止 actor 所在区块被 seal。主动 Agent、危险事件、长耗时命令还没有统一运行时状态，因此代码中保留 TODO。

这不是 P60 阻塞项，因为当前阶段还没有对应权威 runtime 数据源；后续进入 Agent 行动、危险事件或多 tick 建造时，应把这些状态接入 blocked region 计算，而不是在 P60 写内容专属 if。

### 5.2 generator_version / content_version 仍使用默认上下文

CommandGateway lifecycle context 当前填入默认 `first_world / 1.0.0`。短期可用，后续应在 runtime/worldgen 配置版本正式权威化后，改为从 runtime context 读取，避免多内容包/多世界版本时出现快照版本语义不足。

### 5.3 command response patch 与 lifecycle 离屏变化

本次已同步最终 projection version，避免 refresh 版本错位。若后续 lifecycle 会影响当前视野内对象，建议进一步让 lifecycle 输出可合并的 patch 或强制 full refresh。当前 P60 的 seal/restore 主要作用于区块生命周期，现有测试覆盖了移动返回 restore 场景。
