# P53 客户端协议与自动化验证第五次复审报告

## 1. 复审结论

结论：**仍不通过；target 完整匹配已修复，但 WorldCommandDto parameters 仍可绕过 option authority**。

本轮确认第四次复查指出的 target 值伪造问题已修复：

```text
WorldCommandDto 与 snapshot option 匹配时，已比较完整 WorldCommandTargetDto：
target_kind / target_entity_id / target_actor_key / target_item_key / target_inventory_id / target_area_id / knowledge_key / recipe_key / target_coord。
```

同时 bootstrap `client_id mismatch` 自动 erase 旧 session 的问题也已修复为拒绝。

但本轮新增发现：

```text
WorldCommandDto 的 command.parameters 仍不需要匹配 available_commands，也没有“必须为空”的限制。
客户端可以提交与已签发 option 同 kind/key/target 的 WorldCommandDto，但附加任意 parameters。
```

这仍然违反 P53 的核心原则：`available_commands` 是操作权威来源。

## 4. 新增阻塞问题

### 4.1 WorldCommandDto parameters 可注入

严重性：**阻塞**。

位置：

```text
backend/src/client_protocol/client_command_gateway.cpp:52
backend/src/client_protocol/client_command_gateway.cpp:72
backend/include/pathfinder/world_command/world_command_types.h:304
```

问题说明：

```text
WorldCommandOptionDto 当前没有 parameters 字段。
因此 available_commands 无法表达“这个 option 签发了哪些参数”。
ClientCommandGateway 也没有要求 WorldCommandDto.parameters 为空。
只要参数 key 不在当前 unsafe_param_keys denylist 中，就可以随 WorldCommandDto 进入 pipeline。
```

当前代码只拒绝：

```text
recipient_claim_json
knowledge_json
force_learn
force_teach
learned
raw_state
hidden_truth
content_key_rule_hint
```

但项目中已有多个 handler 会读取其他 parameters：

```text
world_inventory: quantity / entity_id / entity_key / drop_x / drop_y / layer_key
world_runtime: seed / region_size / vision_radius / tick_delta
world_generation: world_seed / region_x / region_y / layer_keys
```

这些 key 目前都不是 denylist。

为什么阻塞：

```text
P53 的目标是让客户端只提交后端签发的意图。
如果 WorldCommandDto 可以在签发 option 之外附加参数，客户端仍然可以修改命令语义。
后续只要 available_commands 签发某个带 target 的 pickup / drop / wait / generate / move 类 action，客户端就可能通过 parameters 改数量、坐标、种子、时间步长或 entity_key。
```

必须修复，建议二选一：

```text
方案 A：P53 阶段最小安全策略：Client-submitted WorldCommandDto.parameters 必须为空。
方案 B：扩展 WorldCommandOptionDto，加入 signed_parameters / parameter_schema，并要求 WorldCommandDto.parameters 与签发 option 完全匹配或符合 allowlist schema。
```

若选择方案 A，需要新增测试：

```text
WorldCommandDto(command_kind=Wait, command_key=wait, target=None, parameters[tick_delta]=999) 必须 Blocked。
WorldCommandDto(parameters[quantity]=999) 必须 Blocked。
WorldCommandDto(parameters 为空且匹配 option) 仍可通过。
```
