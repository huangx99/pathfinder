## 3. 核心能力自测

P31 已验证以下能力：

1. 合法 P30 package 可以生成 `GoldenReplayBaseline`。
2. 合法 P30 package 可以执行 `GoldenReplayRunner`，并得到 passed 报告。
3. package hash 被篡改时，报告 `PackageIntegrityInvalid`。
4. command replay 记录事件数量与真实 replay 不一致时，报告 `ReplayMismatch` 和 `EventCountMismatch`。
5. 期望 condition trace 缺失时，报告 `ConditionTraceMissing`。
6. 禁止额外 condition trace 时，报告 `ConditionTraceUnexpected`。
7. golden baseline 确定性签名不一致时，报告 `BaselineMismatch`。
8. target state version 超出 package 范围时，报告 `RestorePlanInvalid`。
9. P31 不修改 `GameState`，只通过 P30 snapshot 和 P4 replay 读取与验证。
10. P31 复用 P27/P30 的 forbidden key 检查，不把隐藏真相类字段写入诊断输出。

## 5. 自测判断

P31 当前达成“真实复盘强化”的最小工程目标：

- 存档包先验完整性由 P30 负责。
- 恢复目标选择由 P30 RestorePlan 负责。
- 真实规则重放由 P4 ReplayRunner 负责。
- P31 只负责组织 replay、收集 condition trace、比较 golden baseline、生成结构化问题。

这个边界是正确的，因为它避免 P31 变成第二套规则引擎，也避免前端或诊断层推导规则结果。

## 6. 当前限制

当前限制是有意保留的：

- P31 仍受 P4 限制，只支持单命令 replay。
- P31 只做内存诊断，不做报告文件导出。
- P31 不做玩家可见 UI 投影。
- P31 还没有把差异定位到具体 state path、event payload、agent decision。

这些能力适合放到 P32、P37 或后续 replay 增强阶段继续做，不应在 P31 里膨胀。
