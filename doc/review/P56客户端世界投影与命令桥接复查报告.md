## 3. 阻塞问题

### 阻塞 1：Bootstrap / Refresh 顶层 projection_version 与 full_projection.projection_version 不一致

位置：

```text
backend/src/client_protocol/client_session_gateway.cpp:49
backend/src/client_protocol/client_session_gateway.cpp:67
backend/src/client_protocol/client_session_gateway.cpp:95
backend/src/client_protocol/client_session_gateway.cpp:112
```

当前逻辑：

```cpp
auto proj_result = projection_adapter_.buildFullProjection(
    session.actor_key, session.layer_key, session.projection_version);

response.projection_version = session.projection_version;
response.full_projection = proj_result.value();
```

问题：

```text
ClientRuntimeProjectionBridge 返回的 full_projection.projection_version 来自 runtime.stateVersion()。
但 ClientBootstrapResponse.projection_version 仍然来自 session.projection_version。
```

在 P56 runtime-backed 模式下，二者可能不同。

典型情况：

```text
WorldGridRuntime initialize 后 stateVersion = 1。
generateInitialWorld 后 stateVersion 增加。
projection bridge 返回 runtime stateVersion。
session 初始 projection_version 仍是 1。
bootstrap 顶层 response.projection_version 仍是 1。
full_projection.projection_version 却是 runtime 版本。
```

风险：

```text
前端如果使用 response.projection_version，下一次 command 会按旧版本提交。
前端如果使用 full_projection.projection_version，又可能与 session gateway 的版本校验不一致。
patch_contract 的 base/new version 会与客户端本地 projection 出现跳版本或错位。
真实 UI 做 patch merge 时容易出现“刚 bootstrap 就 out_of_sync”或 patch base 不匹配。
```

为什么现有测试没拦住：

```text
P56 集成测试只断言 boot.value().projection_version > 0。
没有断言 boot.value().projection_version == boot.value().full_projection.projection_version。
refresh 测试也没有覆盖 bootstrap 顶层 version 与 full projection version 一致性。
```

修复要求：

```text
1. bootstrap 构建 projection 后，必须把 session.projection_version 更新为 full_projection.projection_version。
2. response.projection_version 必须等于 response.full_projection.projection_version。
3. refresh 同理，必须同步 session.projection_version 和 full_projection.projection_version。
4. 新增测试：bootstrap_projection_version_matches_full_projection。
5. 新增测试：refresh_projection_version_matches_full_projection。
6. 新增测试：bootstrap 后用 full_projection.projection_version 提交命令也必须通过。
```

建议修复方式：

```cpp
const auto projection_version = proj_result.value().projection_version;
session.projection_version = projection_version;
response.projection_version = projection_version;
```

refresh 中不能用 `const auto& session = it->second` 后再更新版本，应改成可写引用或先保存 actor/layer。

## 4. 非阻塞风险

### 风险 1：架构扫描太弱

位置：

```text
backend/tests/CMakeLists.txt:5824
backend/tests/CMakeLists.txt:5836
backend/tests/CMakeLists.txt:5842
```

当前扫描只是 grep：

```text
RuntimeBacked
bridge_port_
option_bridge_
```

这些扫描只能证明字符串存在，不能证明生产路径没有 stub fallback。

建议后续增强：

```text
1. 扫描 client_server_runtime_factory.cpp 禁止 StubForProtocolTest / Disabled。
2. 增加运行时测试：factory.projection_bridge->diagnostics(...).bridge_mode == RuntimeBacked。
3. 增加运行时测试：bootstrap visible_cells > 0 且 available_commands 中有 Move。
4. 保留 grep 作为辅助，不作为唯一验收依据。
```

当前因为已有集成测试覆盖真实 factory，所以这不是阻塞项。

### 风险 2：Projection bridge 使用 snapshotForDebug 作为查询源

位置：

```text
backend/src/client_runtime_bridge/client_runtime_projection_bridge.cpp:86
backend/src/client_runtime_bridge/client_runtime_projection_bridge.cpp:126
```

当前实现注释说明这是临时方式。

设计允许短期过渡，但长期不应该让 debug snapshot 成为生产查询接口。

建议加入 TODO 或后续任务：

```text
P57/P58 前补 IWorldRuntimeProjectionQueryPort。
正式 query port 应只返回投影需要的安全数据，不返回完整 debug snapshot。
```

当前 bridge 后续仍通过 WorldProjectionAdapter 做可见性过滤，所以暂不阻塞。

### 风险 3：buildScopedProjection 没有传递真实 reason / scopes 语义

位置：

```text
backend/src/client_protocol/client_projection_adapter.cpp:63
backend/src/client_protocol/client_projection_adapter.cpp:69
```

当前 scoped projection 直接调用 full projection，且 buildFromBridge 固定：

```cpp
req.reason = ClientProjectionBuildReason::Bootstrap;
```

风险：

```text
refresh / command result / reset 的诊断原因都会表现成 Bootstrap。
后续做 viewport、inventory、knowledge scope 时会缺少真实语义。
```

当前 P56 最小目标是 full safe projection，所以不是阻塞。

建议后续修复：

```text
buildFullProjection 可接收 reason 或由 gateway 传入 reason。
buildScopedProjection 应把 requested_scopes 写入 ClientRuntimeViewRequest。
```

## 7. 本次复查结论

P56 主体实现方向正确，已经把客户端从“空 projection + Wait/Inspect stub”推进到“真实地图投影 + Move option + pipeline 移动”。

但由于 `ClientBootstrapResponse.projection_version` / `ClientRefreshResponse.projection_version` 与 `full_projection.projection_version` 可能不一致，这会直接影响真实前端同步和 patch merge。

因此本次结论：

```text
P56 暂不验收。
修复版本号一致性后再复审。
```
