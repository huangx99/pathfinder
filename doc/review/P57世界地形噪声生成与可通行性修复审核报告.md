# P57 世界地形噪声生成与可通行性修复审核报告

## 1. 审核结论

本次审核重点看实现算法。

结论：**不建议验收，存在阻塞问题。**

当前实现已经完成了不少正确方向：

```text
1. 新增了 TerrainNoiseSampler / TerrainNoiseFieldBuilder / TerrainClassifier。
2. terrain 默认从 WeightedRandom 切到 NoiseField。
3. 有 BFS 可通行分析和 deterministic repair。
4. forest / stone_field / water_edge 不再默认阻挡。
5. 相关测试重新编译并通过。
```

但是 P57 的核心目标是“无限区域化噪声地图 + 可通行性修复”。当前实现仍有几处关键算法和架构问题，会影响无限地图、资源分布、区块边界、客户端真实可玩性。

验收状态：**Blocked**。

## 3. 正向评价

### 3.1 噪声采样方向正确

实现位置：

```text
backend/src/world_generation/terrain_noise_sampler.cpp
backend/src/world_generation/terrain_noise_field_builder.cpp
```

优点：

```text
1. TerrainNoiseSampler 使用 world_seed + layer_key + channel_salt 组合 salt。
2. sample 使用 world_x/world_y，而不是 local_x/local_y。
3. 支持 ValueNoise2D / Perlin2D / FractalPerlin2D。
4. 输出 clamp 到 [-1,1]。
5. 同输入可确定性复现。
```

这部分符合 P57 的“确定性噪声场”方向。

### 3.2 地形分类职责拆分清楚

实现位置：

```text
backend/src/world_generation/terrain_classifier.cpp
```

优点：

```text
1. TerrainClassifier 独立负责 terrain threshold -> terrain_key。
2. blocksMovement / movementCost / biomeKindFromTerrain 从 TerrainGenerator 中分离出来。
3. forest / stone_field / water_edge 均不默认阻挡。
4. 派生了 terrain:* / walkable / blocked / biome:* 等 tag。
```

这比旧的逐格权重随机更可扩展。

### 3.3 连通性分析与修复方向正确

实现位置：

```text
backend/src/world_generation/terrain_connectivity_analyzer.cpp
backend/src/world_generation/terrain_connectivity_repairer.cpp
```

优点：

```text
1. 使用 BFS 统计 reachable_from_spawn。
2. 统计 walkable_ratio。
3. 统计 spawn safe radius 阻挡比例。
4. 修复过程按坐标/距离排序，基本具备确定性。
5. 修复后重新 analyze，输出 diagnostics。
```

这是正确方向。

## 4. 阻塞问题

### 阻塞 1：非原点 region 的资源分布仍然以世界原点为中心

问题位置：

```text
backend/src/world_generation/resource_distribution_generator.cpp:40
backend/src/world_generation/resource_distribution_generator.cpp:120
backend/src/world_generation/spawn_safety_planner.cpp:85
```

当前代码固定使用：

```cpp
world_runtime::WorldCellCoord spawn_coord{0, 0, profile.default_layer};
```

这在只有 origin region 的 P46 阶段可以接受，但 P57 已经要求无限 region。

问题后果：

```text
1. region (1000,-1000) 生成时，所有 cell 离 (0,0) 极远。
2. resource rule 的 min/max_distance_from_spawn 会把正常候选全部过滤掉。
3. 非原点 region 的正常资源分布基本失效。
4. SpawnSafetyPlanner 会因为原点附近没有资源，在非原点 region 中走 fallback。
5. 结果是：非原点 region 不是自然资源分布，而是每个 region 被 fallback 塞几个保底资源。
```

这违反 P57 的无限地图目标。

建议修复：

```text
1. 引入 region_anchor_coord / local_spawn_coord 概念。
2. WorldGenerationRequest 或内部 helper 统一计算 region 中心：
   { rx * region_size, ry * region_size, layer }。
3. ResourceDistributionGenerator 对非 origin region 使用 region center 作为局部距离中心。
4. SpawnSafetyPlanner 只有 origin region 才执行“玩家出生保底资源”；普通 region 应走区域资源保底策略，而不是玩家出生保底策略。
5. 新增测试：far_region_has_natural_resource_distribution，不允许只靠 fallback 资源通过。
```

### 阻塞 2：WorldGenerationService 对每个 region 都生成 player spawn draft

问题位置：

```text
backend/src/world_generation/world_generation_service.cpp:237
backend/src/world_generation/world_generation_service.cpp:241
```

当前每次 generate 都加入：

```cpp
player_spawn.spawn_id = "spawn_player_0_0";
player_spawn.coord = world_runtime::WorldCellCoord{0, 0, profile->default_layer};
player_spawn.actor_key = "player";
```

问题后果：

```text
1. 生成 region (1000,-1000) 时，也会输出一个位于 (0,0) 的 player spawn。
2. 这和 region manifest / generated_cell_ids 不属于同一个区域。
3. applier 当前只是记录 delta，没有真的创建 actor，所以测试不一定暴露。
4. 后续如果 spawn applier 变成真实创建 actor，会造成重复 player spawn 或跨 region 错误。
5. patch / changed_cell_ids 会反复包含 origin cell，污染非原点 region 的变更语义。
```

建议修复：

```text
1. 只有 origin region (0,0) 才允许生成 PlayerStart。
2. 非 origin region 不生成 player spawn。
3. 如果后续有 NPC camp / beast nest / landmark，必须使用对应 SpawnPointKind，不得复用 PlayerStart。
4. 新增测试：far_region_does_not_emit_player_spawn_at_origin。
```

### 阻塞 3：客户端 HTTP 工厂用固定半径预生成 21x21 region，违背无限地图目标

问题位置：

```text
backend/src/client_http/client_server_runtime_factory.cpp:33
backend/src/client_http/client_server_runtime_factory.cpp:34
backend/src/client_http/client_server_runtime_factory.cpp:49
backend/src/client_http/client_server_runtime_factory.cpp:53
```

当前代码：

```cpp
config.region_size = 9;
config.initial_region_radius = 10;  // large initial world: 21x21 regions
for (int rx = -initial_region_radius; rx <= initial_region_radius; ++rx) {
    for (int ry = -initial_region_radius; ry <= initial_region_radius; ++ry) {
        ... generate/apply region ...
    }
}
```

问题后果：

```text
1. 这是固定大地图预生成，不是无限地图。
2. 启动时直接生成 441 个 region，对本地测试也许能跑，但不是正确架构。
3. 玩家走到 21x21 外仍然会 OutOfBounds。
4. 后续地图扩大只能继续调大 radius，不可持续。
5. 这会让低级 AI 误以为“无限地图 = 启动时生成一个大固定范围”。
```

P57 设计允许暂时不做完整流式加载调度器，但不允许把世界改成固定大小地图。

建议修复：

```text
1. 客户端启动只生成 origin region + 玩家视野需要的邻近 region，最多 3x3。
2. 走到 region 边缘时可以先返回明确 reason：adjacent_region_not_loaded。
3. 如果本阶段想更进一步，可在 move option 生成或 move handler 前触发邻接 region generation。
4. 禁止把 initial_region_radius=10 当作无限地图替代方案。
5. 新增测试：client_runtime_does_not_depend_on_large_fixed_region_radius。
```

### 阻塞 4：P57 和旧 generateInitialWorld 使用不同 region 坐标模型，混用后会留下旧地形残留

问题位置：

```text
backend/src/world_runtime/world_grid_runtime.cpp:87
backend/src/world_runtime/world_grid_runtime.cpp:90
backend/src/world_generation/terrain_noise_field_builder.cpp:17
backend/src/world_generation/terrain_noise_field_builder.cpp:25
```

旧 `generateInitialWorld` 对一个 region 的 cell 使用：

```text
cx = 0..size-1
world_x = rx * size + cx
```

P57 `TerrainNoiseFieldBuilder` 对一个 region 的 cell 使用：

```text
cx = -size/2 .. +size/2
world_x = rx * size + cx
```

例如 `region_size = 9`：

```text
旧 region (0,0)：x = 0..8
P57 region (0,0)：x = -4..4
```

问题后果：

```text
1. client_server_runtime_factory 先 generateInitialWorld，再 apply P57 region。
2. P57 不是完全覆盖旧 runtime 地图，而是覆盖另一套坐标范围。
3. 旧 cells 可能残留在边界或其他区域。
4. old terrain 和 P57 noise terrain 会混在同一个 runtime。
5. 移动和投影可能看到旧地形残留，不是纯 P57 地图。
```

建议修复：

```text
1. 不要先用 generateInitialWorld 生成大地图再用 P57 覆盖。
2. 拆出“创建 player actor / 初始化 exploration”和“生成 stub terrain”。
3. P57 客户端启动应走 WorldGenerationService + Applier 作为唯一地形来源。
4. 如果保留 generateInitialWorld，必须统一 region 坐标模型。
```

### 阻塞 5：相邻 region 无缝测试没有真正测试无缝

问题位置：

```text
backend/tests/unit/world_generation/world_generation_test.cpp:1270
backend/tests/unit/world_generation/world_generation_test.cpp:1298
backend/tests/unit/world_generation/world_generation_test.cpp:1301
```

当前测试只断言：

```text
region (0,0) 存在 x=7。
region (1,0) 存在 x=8。
```

注释里也写了：

```text
We can't require identical terrain ... but we can assert coordinates exist.
```

这不能证明：

```text
1. 噪声使用 world_x/world_y。
2. 相邻 region 没有 local coordinate seam。
3. terrain 过渡自然。
```

建议修复：

```text
1. 直接用 TerrainNoiseSampler 比较同一 world coordinate 在不同构造路径下的采样一致性。
2. 或生成 overlapping diagnostic strip，例如 region edge 外扩 1 格，验证同 world coord 分类一致。
3. 增加架构扫描：禁止 TerrainNoiseFieldBuilder 使用 local_x/local_y 作为 sample 坐标。
```

### 阻塞 6：first_world 噪声 scale 太小，地图会偏噪点化，不像自然区域

问题位置：

```text
backend/src/world_generation/world_generation_service.cpp:34
backend/src/world_generation/world_generation_service.cpp:37
backend/src/world_generation/world_generation_service.cpp:41
```

当前 profile 注释：

```text
Scales reduced so even a 9x9 region shows multiple noise periods for visual variety.
```

实际配置：

```text
Elevation scale = 4.0
Moisture scale = 3.5
Roughness scale = 3.0
ResourceRichness scale = 2.5
DangerPressure scale = 4.5
```

这会带来：

```text
1. 单个 9x9 区域内变化很多，但无限地图整体会变成高频噪点。
2. 森林、水边、石地不容易形成大块自然地貌。
3. 这和 P57 原设计的 scale 24/32/18 方向相反。
```

建议修复：

```text
1. profile 不应为了一个小视口强行缩小噪声尺度。
2. 使用大尺度噪声形成 biome，大尺度之外可叠加小尺度 detail noise。
3. 建议至少拆成：biome_noise(scale 24~48) + detail_noise(scale 4~8)。
4. 当前先把 elevation/moisture scale 提高到 16~32，roughness/detail 可保留较小。
```

## 5. 非阻塞问题

### 非阻塞 1：repair 没有统一的 setWalkableTerrain helper

问题位置：

```text
backend/src/world_generation/terrain_connectivity_repairer.cpp
```

同样的修复代码重复出现多次：

```text
terrain_key = repair_terrain
blocks_movement = false
movement_cost = ...
biome_kind = ...
tag blocked -> walkable
terrain:* 替换
```

建议抽成 helper：

```cpp
static bool convertToRepairTerrain(GeneratedCellDraft& cell, const std::string& repair_terrain);
```

避免后续某处漏改 tag 或 movement_cost。

### 非阻塞 2：warning 语义会丢失“修复前状态”

`TerrainConnectivityRepairer` 最后重新 analyze 后再判断 `spawn_cell_walkable`，所以如果 spawn 原本 blocked 但修好了，最终不一定保留 `worldgen_spawn_cell_was_blocked`。

建议：

```text
记录 pre_diag，并把 pre_diag 的关键问题作为 before_repair warning。
```

### 非阻塞 3：TerrainGenerator legacy 和 TerrainClassifier 的阻挡规则不完全一致

问题位置：

```text
backend/src/world_generation/terrain_generator.cpp:41
backend/src/world_generation/terrain_classifier.cpp:8
```

旧 `TerrainGenerator::blocksMovement` 认为 `water` 阻挡，但 P57 classifier 认为 `deep_water/mountain/blocked` 阻挡。短期不影响 NoiseField，但 WeightedRandom 兼容模式和正式模式规则不一致。

建议：

```text
WeightedRandom 兼容路径也调用 TerrainClassifier 的 movement rule，避免双轨规则。
```

## 7. 总体判断

当前 P57 实现是“局部地形生成算法雏形成立”，但还不是“可验收的无限地图地形生成系统”。

可以认可的部分：

```text
Noise sampler / TerrainClassifier / BFS analyzer / repairer 方向正确。
```

必须修复后再验收的部分：

```text
1. 资源与 spawn safety 不能再固定使用 (0,0) 作为所有 region 的中心。
2. 非 origin region 不能输出 player spawn at origin。
3. 客户端启动不能用固定 21x21 region 预生成冒充无限地图。
4. P57 不能和旧 generateInitialWorld 的坐标模型混用。
5. seam 测试必须真正验证噪声世界坐标连续。
```

审核结论：**Blocked，需修复后复审。**
