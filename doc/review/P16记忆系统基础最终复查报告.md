## 1. 总结论

P16 本轮复查通过。

上次复审剩余的唯一阻塞问题是：`Contradiction` 候选不能匹配并更新被冲突的旧记忆。当前实现已修复该问题：当 candidate 包含 `MemoryKind::Contradiction` 时，只要 owner、subject 相同且旧记录未 Forgotten，即允许匹配旧记录，不再要求 memory kind 必须有交集。

这意味着下面的核心场景已经成立：

```text
旧记忆：Success
新证据：Failure + Contradiction
结果：更新旧记忆，而不是创建孤立记忆
```

复查确认：

- 旧记录会被更新。
- `contradiction_count` 会增加。
- `MemoryKind::Contradiction` 会追加到旧记录。
- `memory.contradiction_observed` reason key 会写入。
- 因包含 `Contradiction`，记忆可以按 P16 晋升规则进入更稳定层级。
- `MemoryIdFactory` 也补上了 `evidence_ref.validateBasic()`，上轮非阻塞建议已处理。
- hidden truth 大小写集成测试已补齐 `source_event_id`，测试语义更纯粹。

因此，P16 作为“记忆系统基础层”已达到当前阶段设计要求，可以进入下一阶段。

## 3. 上轮阻塞项复查

### BLOCKER-RECHECK-1：Contradiction 匹配旧记忆

复查结论：已修复，通过。

当前实现位置：`backend/src/memory/memory_writer.cpp:257`

关键逻辑：

```cpp
bool candidate_has_contradiction = ...;
if (candidate_has_contradiction) {
    return true;
}
```

该逻辑位于 owner、subject、Forgotten 过滤之后，因此含义是：

```text
同 owner + 同 subject + 未遗忘 + 新候选包含 Contradiction => 可以匹配旧记忆
```

这正是 P16 需要的冲突记忆语义。

### 3.1 写入更新路径

当前写入更新路径会执行：

- `updated.contradiction_count += 1`
- 如旧记录没有 `Contradiction`，追加 `MemoryKind::Contradiction`
- 追加 `memory.contradiction_observed`
- 后续进入 promotion 检查

对应位置：`backend/src/memory/memory_writer.cpp:406`

### 3.2 测试覆盖

已新增测试：旧 `Success` 记录 + 新 `Failure/Contradiction` 候选。

测试确认：

- 结果走 `updated_records`，不是 `created_records`。
- 更新的是旧 memory id。
- `contradiction_count == 1`。
- 包含 `MemoryKind::Contradiction`。
- 包含 `memory.contradiction_observed`。
- `reinforcement_count == 1`。

对应位置：`backend/tests/unit/memory/memory_writer_test.cpp:293`

还新增 `CognitionUpdateDecision::Conflicted` 到 candidate contradiction 的测试。

对应位置：`backend/tests/unit/memory/memory_writer_test.cpp:342`

## 5. 阶段边界复查

P16 当前仍保持阶段边界：

- 不做记忆压缩。
- 不做复杂召回。
- 不做 `MemorySummary` / `MemoryBundle`。
- 不做知识系统。
- 不做传播系统。
- 不接 H5/API/HTTP/WebSocket/JSON。
- 不接 SaveManager。
- 不做强化学习 reward/done。
- 不让 AgentRuntime 或 Policy 直接写 MemoryStore。
- 不读取 `ObjectDefinition hidden truth` 或 `GameState` 原始隐藏状态。

结论：阶段边界通过。
