# P32 协议投影扩展自测与自审报告

## 1. 本次结论

P32 已完成实现、自测和人工自审，可以作为 P33 H5 正式最小界面的后端安全投影地基。

本次实现的定位没有改变：P32 只负责把后端权威结果整理成 H5 可读、安全、中文友好的 projection，不执行规则、不修改状态、不裁决行动。

## 2. 本次实现范围

### 2.1 新增模块

新增 `pathfinder_h5_projection` 模块：

- `backend/include/pathfinder/h5_projection/projection.h`
- `backend/src/h5_projection/projection.cpp`

模块依赖：

- `pathfinder_foundation`
- `pathfinder_protocol`
- `pathfinder_replay_diagnostics`

### 2.2 新增协议类型

扩展 `ProtocolPayloadType`：

- `H5GameProjection`
- 字符串：`h5_game_projection`

用途：后续 P33 H5 不直接读取调试字符串，而是通过协议拿到安全 projection。

### 2.3 新增投影能力

P32 当前支持以下投影结构：

- 场景标题与场景摘要。
- 对象卡片：名称、描述、安全标签、知识徽章、可行动作。
- 行动条：尝试吃、尝试使用、观察、教学、组合、规避、协作、调试复盘。
- 知识面板：玩家知识、同伴/接收者知识。
- 认知面板：目标、认知方面、认知结论、信心区间。
- 条件提示：来自 P27 的安全条件摘要。
- 危险提示：来自 P29 的安全危险摘要和应对方式。
- 族群状态：士气、信任、分裂风险、协作、冲突摘要。
- 复盘状态：来自 P31 的黄金复盘安全诊断摘要。
- 投影 issue：截断、权限隐藏、安全拒绝等可展示问题。

## 3. 人工自审重点

### 3.1 架构边界检查

已检查 P32 代码没有引入以下越界行为：

- 不调用规则 resolver。
- 不执行 pipeline。
- 不调用 applier 修改状态。
- 不依赖 `GameState` 或 raw world state。
- 不输出 raw replay log 或 raw save package。
- 不让 H5 根据对象名、标签或文本自行推导行动结果。

当前 `ActionProjection.enabled` 只保留后端输入值，不在 P32 内重新计算。

### 3.2 安全输出检查

已检查并补强：

- Player audience 不允许请求 DebugPanel。
- Player audience 不允许 `include_debug_trace=true`。
- Player projection 会清空 `debug_keys`。
- Player projection 会隐藏不可见的 `replay_status`。
- Player projection 现在会递归拒绝 `SafeTextKind::DebugSummary`，避免调试文本藏进对象名称、知识行、危险提示、族群提示或 issue。
- DeveloperDebug audience 可以保留调试摘要，但仍不能输出 forbidden key 或脚本文本。

### 3.3 禁止 key 检查

当前 P32 会拒绝以下高风险 key 或文本片段：

```text
hidden_truth, true_trait, real_effect, raw_state, gamestate,
actual_hp, true_hp, hp_delta, death, kill, corpse,
loot, drop, random_damage, frontend_unlock, direct_state,
raw_replay_log, raw_save_package, internal_probability,
unmasked_condition, secret_config,
<script, javascript:, onerror=, onclick=, <iframe
```

自审判断：这套拦截偏严格，可能误伤少量英文普通词，但 P32 是前端安全投影层，安全优先是合理的。

## 4. 自审发现并修复的问题

### 4.1 列表上限原先没有真正截断

问题：设计中有 `max_items_per_section`，但初版实现只是校验请求参数，没有对输出列表执行截断。

修复：

- 新增 `truncateVector()`。
- 对 scene/object/action/knowledge/cognition/condition/danger 等列表执行截断。
- 截断时设置 `projection.header.truncated = true`。
- 截断时写入 `ProjectionIssueKind::Truncated`。

新增测试：

- `h5_projection_truncation`

### 4.2 脚本类文本原先没有拦截

问题：只拦截了玩法隐藏信息 key，未拦截 `<script`、`javascript:`、事件属性等 H5 高风险文本。

修复：

- forbidden 列表新增 `<script`、`javascript:`、`onerror=`、`onclick=`、`<iframe`。

新增测试：

- `h5_projection_script`

### 4.3 Player 深层 DebugSummary 原先可能漏出

问题：`ProjectionSanitizer::validateSafeText()` 能拒绝 Player 的 `DebugSummary`，但 `validateProjection()` 原先没有递归检查所有嵌套 `SafeTextProjection`。

风险：调试摘要可能被错误塞进对象名称、知识行、危险提示等普通字段。

修复：

- 新增递归 audience 文本检查。
- 覆盖 ObjectCard、Action、Knowledge、Cognition、Condition、Danger、Tribe、Replay、Issue 等嵌套文本。
- Player 递归拒绝 `SafeTextKind::DebugSummary`。
- DeveloperDebug 仍允许 DebugSummary，但必须通过 forbidden key 检查。

新增测试：

- `h5_projection_player_debug_text`
- `h5_projection_developer_debug_text`

## 6. 当前合理性判断

### 6.1 合理的部分

- P32 没有变成第二套规则系统，职责边界清晰。
- H5 后续只需要渲染 projection，不需要解析调试字符串。
- 中文展示和稳定 key 同时存在，适合后续 UI、本地化和测试。
- Player/DeveloperDebug audience 分离清楚，能支持玩家界面和开发调试界面共用同一套投影协议。
- P31 复盘信息只转成安全摘要，不暴露 raw replay log。
- 截断、权限隐藏、危险文本拒绝都有 issue 或错误输出，便于 H5 给玩家/开发者解释。

### 6.2 当前刻意保留的限制

- P32 builder 当前主要做 validate + pass-through，暂不从 P22/P29/P31 的复杂对象自动生成完整 UI projection。
- `ProjectionMode::Delta` 已预留，但当前 composer 仍输出 full 结构；后续需要增量同步时再扩展，不影响当前架构。
- `H5ProtocolProjectionAdapter` 当前只构造 envelope，暂不序列化 projection body；这是因为现有 `ProtocolEnvelope` 还没有通用 body 容器。
- P32 不负责 H5 页面布局，P33 才做正式最小界面。

## 7. 对 P33 的要求

P33 实现时必须遵守：

- H5 只能消费 `H5GameProjection`。
- H5 不得直接读取 raw state、debug key、replay raw log 或保存包原文。
- H5 不得根据对象名、标签、文本自行推断行动结果。
- H5 的按钮 enabled/disabled 必须来自 P32 projection。
- H5 的失败原因必须来自后端安全 condition/danger/issue 摘要。
- H5 的中文界面优先展示 `zh_cn`，但测试和协议仍以稳定 key 为准。

## 8. 自审结论

P32 代码结构合理，边界符合既有设计，可以进入 P33。

本次自审不是只跑测试，已额外检查实现是否越界、是否形成第二套规则系统、Player 安全过滤是否递归、P31 复盘是否只投影安全摘要、H5 是否会被迫解析调试字符串。发现的问题已修复并补充测试。
