## 关键问题闭环

### 1. P24 状态草稿合并

状态：已通过。

P25 已正确承接 P24：

```text
csd.morale_delta -> morale_delta
csd.trust_delta -> trust_delta
csd.safety_pressure -> safety_pressure_delta
csd.casualty_pressure -> loss_pressure_delta
csd.knowledge_conflict_pressure -> split_risk_delta
```

P25 不再覆盖 P25 自己的 outcome delta。

### 2. TruceOffered 语义

状态：已通过。

停战不再只依赖“对方 strength 很低”，现在会读取：

```text
source_loss_pressure
target_loss_pressure
resource_pressure
对方是否继续 Raid/Pursue
```

测试已强化为必须断言：

```text
outcome == TruceOffered
```

### 3. Projection 双方压力

状态：已通过。

`ConflictProjection` 已合并 source/target 两边状态压力，避免只看 source 导致前端误读。

### 4. 关系方向校验

状态：已通过。

`target_relation_to_source` 已校验方向：

```text
target -> source
```

方向错误会拒绝。

### 5. Forbidden key 误伤

状态：已通过。

不再使用裸 `hp` 子串误伤普通 key。

### 6. stance 组合校验

状态：已通过。

`Avoid + Aggressive`、`Retreat + Aggressive`、`NegotiateTruce + Aggressive` 已被拒绝。

### 7. 威慑 / 撤退方向性

状态：已通过。

`Intimidated` 现在会把压力落到被威慑方：

```text
retreat_pressure_delta > 0
safety_pressure_delta > 0
morale_delta < 0
```

`ForcedRetreat` 现在会把压力落到被迫撤退方：

```text
retreat_pressure_delta > 0
safety_pressure_delta > 0
morale_delta < 0
```

新增测试：

```text
p25_intimidated_applies_pressure_to_losing_tribe_flow
p25_forced_retreat_applies_retreat_pressure_to_forced_side_flow
```

## P26 进入条件

P26 设计时必须承接 P25 输出，但不能重做 P25。

P26 只能读取：

```text
ConflictResolutionResult。
ConflictProjection。
ConflictTrace。
TribeRelationDraft。
ConflictStateDraft。
```

P26 禁止：

```text
绕过 P25 直接推导冲突胜负。
读取 hidden truth。
读取 raw GameState / SaveGame。
从前端推导能力解锁。
把一次冲突成功直接当作文明能力。
```

P26 可以使用 P25 的安全摘要作为能力证据，例如：

```text
多次成功威慑 -> 群体威慑能力候选。
多次低损撤退 -> 组织撤退能力候选。
资源冲突中稳定保护采集 -> 资源防卫能力候选。
停战反复成功 -> 初级冲突降级能力候选。
```

但它们必须进入 evidence / readiness candidate，不得直接解锁能力。
