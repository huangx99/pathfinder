1. `MemoryCompressionService::compress()` 在记录不足时返回 `Skipped`，不再携带非法 plan，`MemoryCompressionResult::validateBasic()` 已可通过。

但复审发现 **1 个仍需修复的公共接口边界问题**：

```text
MemoryCompressionPlanner::planForOwnerSubject(empty_records, key)
返回 Result::ok(plan)，但该 plan 无法通过 MemoryCompressionPlan::validateBasic()。
```

这不影响当前 `MemoryCompressionService::compress()` 的 skipped 结果，但它仍然违反 P17 任务卡中“所有 public DTO 都必须 validateBasic”的工程约束。建议修复这一处后再标记 P17 最终通过。

## 3. 新发现问题

### ISSUE-1：MemoryCompressionPlanner 空输入返回 ok 但 plan 非法

严重级别：**P1，建议作为最终通过前必修。**

位置：

- `backend/src/memory/memory_compression.cpp:114`
- `backend/src/memory/memory_compression.cpp:115`
- `backend/src/memory/memory_summary.cpp:298`

问题描述：

当 `filtered.size() < options.min_records_to_summarize` 时，`MemoryCompressionPlanner::planForOwnerSubject()` 会构造并返回一个 `MemoryCompressionPlan`。

如果 `filtered.size() == 0`：

- `plan.source_memory_ids` 为空。
- `plan.representative_memory_ids` 为空。
- `plan.validateBasic()` 失败。
- 但函数仍然返回 `Result<MemoryCompressionPlan>::ok(plan)`。

额外探针输出：

```text
planner_empty_ok=1 plan_validate_ok=0 source_count=0
```

为什么需要修：

- `MemoryCompressionPlanner` 是 P17 暴露的 public service。
- `MemoryCompressionPlan` 是 P17 public DTO。
- P17 任务卡明确要求：所有 public DTO 都必须 `validateBasic()`。
- `Result::ok` 返回一个不可校验 DTO，会给后续 AI 实现造成错误示范。

建议修复方式：

优先方案：

```text
当 filtered.size() < min_records_to_summarize 时，Planner 不返回 ok(plan)。
改为 Result::fail 或调整 Planner 接口，让 skipped 由 CompressionService 独立处理。
```

更推荐的最小改法：

```text
MemoryCompressionPlanner 只负责生成“可执行压缩计划”。
filtered.size() 不足时返回 fail，错误/原因 key 使用 insufficient_records。
MemoryCompressionService 在调用 planner 前或捕获该错误后转换为 Skipped 结果。
```

如果不想改接口：

```text
新增 plan 状态字段，例如 is_executable 或 plan_decision。
并让 MemoryCompressionPlan::validateBasic() 支持 skipped plan。
```

但当前阶段不建议增加复杂度。

必须补测试：

```text
planner_empty_records_does_not_return_invalid_ok_plan
```

验收条件：

```text
planner.planForOwnerSubject(empty, key) 不能返回 ok + invalid plan。
```
