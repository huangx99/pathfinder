# P57 世界地形噪声生成与可通行性修复复审报告

## 1. 复审结论

复审结论：**可以验收 P57 本轮修复。**

本次复审重点对照上一份审核报告中的阻塞项逐条检查。当前实现已经把 P57 的主要风险从“固定原点/伪无限/旧地形混用”修到了可接受状态：

```text
1. 非原点 region 的资源距离锚点已经改为 region center。
2. PlayerStart 只在 origin region 生成。
3. 客户端启动路径不再调用旧 generateInitialWorld 生成随机旧地形。
4. HTTP factory 不再预生成 21x21 固定大地图，只预热 origin 附近少量 region。
5. 噪声尺度已经从过小参数调整为更适合连续地貌的参数。
6. 相邻 region 的噪声连续性测试比上一版更真实。
```

验收状态：**Pass with Notes**。

注意：P57 通过不代表无限大地图流式加载已经完成。当前通过的是“无限 region 生成器基础能力 + 客户端起步地图可玩性修复”。真正的按需区块加载、远距离探索缓存、前端视野滚动，仍应放到后续 V2 世界地图阶段继续做。

## 3. 上轮阻塞项复审

### 3.1 阻塞 1：非原点 region 资源仍以世界原点为中心

复审结果：**已修复。**

检查位置：

```text
backend/src/world_generation/resource_distribution_generator.cpp:40
backend/src/world_generation/resource_distribution_generator.cpp:51
backend/src/world_generation/resource_distribution_generator.cpp:124
backend/src/world_generation/resource_distribution_generator.cpp:134
backend/src/world_generation/spawn_safety_planner.cpp:85
backend/src/world_generation/spawn_safety_planner.cpp:108
```

当前资源分布和 ground item 分布已经使用：

```text
region_center = region_coord * region_size
```

不再固定用 `(0,0)` 计算距离。

这意味着：

```text
1. region (1000,-1000) 这类远距离区块不会因为离原点太远而被错误过滤。
2. resource rule 的 min/max distance 现在是相对于当前 region 的局部中心生效。
3. spawn safety fallback 也会在当前 region 中心附近补资源，不会跨区块补到原点。
```

### 3.2 阻塞 2：每个 region 都生成 PlayerStart

复审结果：**已修复。**

检查位置：

```text
backend/src/world_generation/world_generation_service.cpp:238
backend/src/world_generation/world_generation_service.cpp:240
```

当前逻辑明确限制：

只有 region_coord == (0,0) 时才生成 spawn_player_0_0。

非 origin region 不再重复输出玩家出生点，避免后续 applier 真实创建 actor 后产生重复玩家或跨 region 脏变更。

### 3.3 阻塞 3：HTTP factory 预生成固定 21x21 region

这仍不是完整无限地图流式加载，但 P57 设计允许本阶段只加载少量 region。只要后续 V2 地图阶段接入按需生成，就不会破坏架构。

### 3.4 阻塞 4：旧 generateInitialWorld 与 P57 地形混用

复审结果：**客户端主路径已修复。**

检查位置：

```text
backend/src/client_http/client_server_runtime_factory.cpp:31
backend/src/client_http/client_server_runtime_factory.cpp:57
backend/src/client_http/client_server_runtime_factory.cpp:59
backend/src/client_http/client_server_runtime_factory.cpp:73
```

当前客户端服务启动流程是：

```text
1. world_runtime.initialize(config) 清空并初始化 runtime。
2. WorldGenerationService 生成 region。
3. WorldGenerationApplier apply 到 runtime。
4. terrain apply 后再 setupPlayerActor。
```

没有再先调用 `WorldGridRuntime::generateInitialWorld` 生成旧随机地形，因此不会在客户端世界里残留旧 terrain。

保留风险：

```text
backend/src/world_runtime/world_command_runtime_bridge.cpp
backend/src/world_runtime/world_grid_runtime.cpp
```

旧 runtime command handler 里仍保留 `generateInitialWorld` 路径，主要用于旧测试和旧命令流。它目前不是 P57 客户端主路径，但后续如果要彻底统一世界生成，应在后续阶段把旧 GenerateWorld runtime bridge 标记为 legacy 或迁移到 WorldGenerationService。

## 4. 新增观察项

### 4.1 客户端 region_size 与 profile/test 不一致

状态：**非阻塞，但建议后续统一。**

检查位置：

```text
backend/src/client_http/client_server_runtime_factory.cpp:33
backend/src/world_generation/world_generation_service.cpp:67
doc/87_P57世界地形噪声生成与可通行性修复任务卡设计.md:644
doc/87_P57世界地形噪声生成与可通行性修复任务卡设计.md:648
```

当前客户端 factory 使用：

```text
region_size = 9
```

而 P57 profile/test 主体使用：

```text
region_size = 16
min_reachable_cells_from_spawn = 120
```

如果单个 region 只有 9x9，也就是 81 格，那么 `reachable_from_spawn >= 120` 在单 region 诊断中天然不可能满足。

当前这不会阻塞 P57，因为客户端预热 3x3 region 后整体仍可移动，且测试已经覆盖基础可玩路径。但后续进入正式 V2 世界地图时，建议统一为：

```text
1. 客户端 factory 使用 P57 profile 默认 region_size 16；或
2. connectivity policy 根据 region_size 动态裁剪 min_reachable_cells_from_spawn；或
3. 诊断明确区分 single-region reachable 与 multi-region reachable。
```
