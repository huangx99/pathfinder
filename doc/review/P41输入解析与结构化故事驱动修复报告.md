# P41 输入解析与结构化故事驱动修复报告

## 1. 修复结论

本轮继续修复 P41 自审遗留风险：H5 输入解析、故事推进、执行状态中存在用中文文本或回复文案驱动规则的问题。

修复后，核心规则链路改为：

- 玩家输入 → `DialogIntentParser::parseWithScenario` 根据 `DialogScenario` 配置解析对象、目标、动作。
- 对话反馈 → `DialogTurnServiceResult` 输出结构化 `intent` 与 `observed_effect_key`。
- H5 故事推进 → `StoryEvaluationContext` 使用结构化 `last_object_key / last_target_object_key / last_action_key / last_effect_key`。
- 故事生成物 → `StorySuggestedActionDefinition.generated_object_keys` 由效果配置驱动，不再因为文本里出现“火把/火”就伪造物品。
- 执行状态/打断 → 根据结构化对象、目标、效果、威胁状态判断，不再扫描回复文本里的“野兽/危险/斧头/木头”。

## 2. 主要改动

### 2.1 输入解析配置化

涉及文件：

- `backend/include/pathfinder/h5_dialog/dialog_intent.h`
- `backend/src/h5_dialog/dialog_intent.cpp`
- `backend/include/pathfinder/h5_dialog/dialog_types.h`
- `backend/src/h5_dialog/dialog_types.cpp`
- `backend/src/h5_dialog/dialog_scenario.cpp`
- `backend/tests/unit/h5_dialog/dialog_intent_test.cpp`

改动内容：

- 新增 `DialogIntentParser::parseWithScenario(input_text, scenario)`。
- `DialogScenarioObject` 新增：
  - `default_action`
  - `input_aliases`
- `DialogScenarioActionTemplate` 新增：
  - `object_key`
  - `target_object_key`
- 输入解析不再直接写死“红果、火把、野兽”等中文词，而是读取场景对象别名、显示名、快捷动作模板。
- “制作火把”通过动作模板解析为：`wood + use -> fire_seed`。
- “用火把驱赶野兽”通过动作模板解析为：`torch + use -> beast_shadow`。
- “使用靠近的野兽”仍解析为：`beast_shadow + use`，不会误判成火把驱赶。

### 2.2 对话服务输出结构化回合结果

涉及文件：

- `backend/include/pathfinder/h5_dialog/dialog_turn_service.h`
- `backend/src/h5_dialog/dialog_turn_service.cpp`

改动内容：

- `DialogTurnServiceResult` 新增：
  - `intent`
  - `observed_effect_key`
- H5 层不再从回复文本反推本回合发生了什么，而是直接读取对话服务的结构化结果。

### 2.3 故事系统改为效果驱动

涉及文件：

- `backend/include/pathfinder/story/story_types.h`
- `backend/src/story/story_types.cpp`
- `backend/src/story/story_system.cpp`
- `backend/tests/unit/story/story_test.cpp`

改动内容：

- `StorySuggestedActionDefinition` 新增 `generated_object_keys`。
- `StoryEvaluationContext` 新增结构化字段：
  - `last_intent_kind`
  - `last_action_key`
  - `last_object_key`
  - `last_target_object_key`
  - `last_effect_key`
- 故事生成物改为根据 `required_effect_key -> generated_object_keys` 推进。
- 删除故事推进中“只要文本包含火把/火就生成 torch/camp_fire”的规则。
- 新增 `story_no_text_magic` 测试，验证即使文本写“用火把驱赶野兽”，没有结构化效果也不会凭空生成火把或故事事件。

### 2.4 H5 执行状态改为结构化判断

涉及文件：

- `backend/include/pathfinder/h5_playable/playable_turn_service.h`
- `backend/src/h5_playable/playable_turn_service.cpp`

改动内容：

- `buildResponse` 接收 `turn_intent` 与 `observed_effect_key`。
- `buildStoryContext` 使用结构化 intent/effect 填充故事上下文。
- `playableExecutionPlan` 不再读取回复文本里的“野兽/危险/斧头/木头”。
- `playableInterrupts` 不再读取回复文本里的“野兽/危险”。
- `toneFromDialog` 不再通过“形成/缓解/同伴听取/不适/危险”等中文回复词决定语气，改为根据 `DialogTurnDecision`、`intent`、`observed_effect_key` 判断。

## 3. 自审结果

### 3.1 已修复的问题

- 修复了输入解析层写死中文对象名的问题。
- 修复了故事层根据中文文本生成火把/火堆的问题。
- 修复了 H5 执行状态根据回复文本推断野兽/木头/斧头的问题。
- 修复了 H5 反馈语气根据回复文案关键词判断的问题。
- 补充了结构化上下文校验，防止新增字段绕过 forbidden key 检查。

### 3.2 当前仍保留的合理内置配置

以下内容仍在 C++ 默认场景中，这是“内置内容包”，不是解析器/规则层写死：

- 默认物品：红果、腐烂红果、斧头、木头、火种、火把、靠近的野兽等。
- 默认反馈效果：`restore_hunger / poison / cut_wood / make_torch / repel_beast`。
- 默认动作模板：制作火把、点燃火源、用火把驱赶野兽。

这些内容后续可迁移到 JSON/内容包；本轮目标是先保证规则执行链不再从中文文本硬推结果。

## 5. 残留风险

### 5.1 内容包仍未外置

默认内容仍写在 `dialog_scenario.cpp` 与 `story_system.cpp`。这不是本轮规则硬编码问题，但后续如果要做到“加毒蘑菇像喝水一样简单”，应继续推进 JSON/内容包外置。

### 5.2 故事层仍有具体默认场景 key

`torch / camp_fire / beast_shadow` 等 key 仍出现在第一天生存场景逻辑里。这属于默认场景逻辑，不属于输入解析硬编码。长期方案是把故事目标、威胁、生成物、反制关系全部配置化。

### 5.3 语气判断仍是轻量映射

`toneFromDialog` 已经不读中文文本，但仍是基于少量 effect key 的轻量映射。后续可以扩展为 `effect_key -> tone` 配置表。
