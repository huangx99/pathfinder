# P47 资源采集砍伐挖掘系统最终审核报告

更新时间：2026-05-21
审核结论：通过，可以提交推送

## 1. 审核范围

本次审核覆盖 P47 资源采集、砍伐、采矿、挖掘系统：

- `world_harvest` 模块边界。
- `Gather/Chop/Mine/Dig` command handler。
- `WorldResourceNodeRuntime` charge 消费。
- P45 `spawnEntityOnMap` 归属接入。
- P46 资源节点到 P47 采集的端到端流程。
- 失败路径不污染状态。
- policy 参数边界。
- 事件 tick 与 command context 对齐。

## 2. 初审发现并已修复的问题

### 2.1 `PreferInventory` 半实现

已修复：

- P47 只允许 `DropOnMap`。
- `PreferInventory` 会失败，不会静默掉落到地图。
- 测试覆盖失败不扣 charge、不生成产物。

### 2.2 空产物节点成功采集

已修复：

- `output_entity_keys` 为空会返回 `OutputInvalid`。
- output key 为空会返回 `OutputInvalid`。
- 失败不扣 charge、不生成产物。

### 2.3 事件 tick 固定为 0

已修复：

- command handler 转发成功事件时统一写入 `context.currentTick()`。
- 测试覆盖 `ResourceHarvested` 与 `NodeDepleted` 的 tick。

### 2.4 未知 policy 被静默当作 DropOnMap

已修复：

- service 只接受 `DropOnMap`。
- `Unknown`、`PreferInventory` 都失败。
- command handler 明确解析缺省、`drop_on_map`、`prefer_inventory`、未知字符串。
- 未知字符串不再静默成功。

## 3. 架构审核结论

### 3.1 通过项

- P47 没有把采集逻辑塞入基础 `world_command`。
- `world_command` 与 `world_generation` 没有反向依赖 `world_harvest`。
- 产物通过 P45 `IWorldEntityLocationPort::spawnEntityOnMap` 建立 `OnMap` 状态。
- 产物不会直接进入 actor inventory。
- 采集后可继续通过 P45 pickup 进入背包。
- 资源节点使用 `WorldResourceNodeRuntime` 作为权威状态。
- output id 基于 command id、entity key、index，保持确定性。
- 未使用真实时间或随机数。
- 失败路径覆盖 actor missing、node missing、depleted、action mismatch、too far、not visible、tool missing、bad output、bad policy。

### 3.2 保留风险

- `apply()` 当前仍是“先 spawn output，再 update resource node”。在现有内存 runtime 下 `updateResourceNodeRuntime()` 基本不会失败，风险可接受。
- 代码已写注释：未来如果接入持久化、多 runtime 或事务型世界状态，需要引入事务或补偿机制。
- 此风险不阻塞 P47，因为 P47 的目标是资源节点到 OnMap 产物闭环，不是事务系统。

## 4. 测试结果

已运行专项测试：

```bash
cmake -S backend -B build/backend
cmake --build build/backend --target pathfinder_tests_world_harvest pathfinder_tests_world_harvest_integration -j 2
ctest --test-dir build/backend -R '^(world_harvest_|world_command_(gather|chop|mine|dig|harvest)|world_generation_)' --output-on-failure
```

结果：79/79 通过。

已运行 P43-P47 相关回归：

```bash
cmake --build build/backend -j 2
ctest --test-dir build/backend -R '^(architecture_world_command_no_runtime_dependency|architecture_world_command_no_inventory_dependency|world_command_|world_runtime_|world_inventory_|world_generation_|world_harvest_)' --output-on-failure
```

结果：165/165 通过。

## 5. 最终结论

P47 现在达到验收标准：

- 能通过 command 采集、砍伐、采矿、挖掘资源节点。
- 能消费资源节点 charge。
- 能生成真实 OnMap 产物。
- 能通过 P45 归属系统拾取产物。
- 不绕过背包与地图归属权威接口。
- 失败路径不会污染状态。
- 参数错误不会被静默修正。
- 测试不是空测，覆盖了核心闭环和多个异常路径。

结论：P47 审核通过，可以提交推送。
