## 5. 非阻塞建议

### P1：独立 Deprecate 策略可后续补明确触发测试

当前旧知识被废弃的核心玩法路径已经成立：

```text
Specialize -> old claim Deprecated -> new claim supersedes old claim
```

但 `KnowledgeResolutionStrategy::Deprecate` / `KnowledgeRevisionDecision::DeprecatedClaim` 的独立触发路径还没有专门测试覆盖。当前不阻塞 P19，因为“过宽旧知识被更精确知识替代”的主路径已经通过 Specialized claim 实现。

建议后续补一条专门测试：

```text
强冲突但不适合创建 specialized claim，且 allow_deprecated=true 时，是否应进入 DeprecatedClaim。
```

如果设计上决定 P19 不需要独立 DeprecatedClaim，也可以在后续文档中明确：废弃动作主要作为 Specialize 的伴随结果。

### P1：Service early return 可统一走 result guard helper

主流程已在 `backend/src/knowledge/knowledge_revision.cpp:853` 增加最终 `validateBasic()` 防线。早退分支当前探针验证为合法，但为了长期维护，建议后续抽一个 helper：

```text
returnValidRevisionResult(result)
```

所有 ok 返回统一经过该 helper，避免后续新增字段时漏掉校验。
