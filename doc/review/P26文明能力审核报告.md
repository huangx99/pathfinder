# P26 文明能力审核报告

## 1. 审核结论

结论：通过。

P26 文明能力模块已完成主体实现、阻断问题修复和全量测试验证。当前版本满足 P26 阶段进入 P27 的最低要求。

本次复审确认：

```text
1. 文明能力不再依赖 bool unlocked、civilization_level、frontend_unlock 或 stage-only 捷径。
2. condition_type 已加白名单，stage/未知/hidden_truth 等非法条件类型会被拒绝。
3. 空 required_conditions 会被拒绝，避免配置层无条件解锁。
4. Projection、Trace、EventDraft、EffectDraft、CapabilityState 等输出 DTO 补齐 forbidden key 校验。
5. resolver 生成的 Stable 能力会产生正向 stability，effect strength 不再为 0。
6. SafeForaging 流程测试现在验证能力确实进入 Stable，而不是只验证存在。
7. P18-P26 相关回归和全量测试均通过。
```

非阻断建议仍保留：后续可拆出 `civilization_resolver.h/cpp`，让 state DTO 与 resolver 编排边界更清晰。

```text

```

已验证的核心故事：

```text
IdentifyEdible 可以从知识摘要形成 Candidate/Emerging。
IdentifyDanger 可以从危险知识形成 Candidate/Emerging。
SafeForaging 在前置能力、食物/危险知识、传播覆盖、成功率、误解率满足并重复实践后进入 Stable。
ConflictDeescalation 从 P25 安全冲突摘要形成 Candidate/Emerging。
能力被冲突阻断时 maturity 不会丢失，只改变 usability。
Stable + Usable 能力才产生 effect draft。
Stage 从能力集合推导，不反向解锁能力。
```

### 4.3 红线复核

当前满足：

```text
不读取 raw GameState / SaveGame。
不读取 hidden truth 决定能力。
不重算 P25 冲突胜负。
不直接修改 TribeState / WorldState。
不把 stage 当成能力来源。
不允许空条件能力定义。
不允许未知或 stage condition_type。
不允许 frontend_unlock / hidden_truth 等危险 key 进入 P26 DTO 输出。
Lost / Blocked 能力不会产生正常 effect draft。
Stable / Institutionalized + Usable 才产生 effect draft。
```
