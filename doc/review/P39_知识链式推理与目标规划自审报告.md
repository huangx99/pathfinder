# P39 知识链式推理与目标规划自审报告

## 实现范围

- 新增 `pathfinder_agent_reasoning` target，包含类型、语义表、目标生成、知识候选、前置链解析、评分、总推理、执行适配和安全投影。
- 新增内置 `EffectSemantics` 表，覆盖 `restore_hunger`、`poison`、`no_visible_effect`、`cut_wood`、`restore_sharpness`、`ignite_fire`、`make_torch`、`repel_beast`、`provide_warmth`、`build_house`。
- `AgentReasoner` 只从 `KnowledgeClaim`、`WorldSnapshot`、`AgentNeedState` 和受限条件事实生成计划，不读取隐藏对象真相。
- `PlanPreconditionResolver` 支持 `torch`、`wood_processed`、`axe.sharpness`、`house` 链式准备，并限制 `max_depth`、`max_plan_steps`、`max_total_expansions`。
- `AgentAutonomyService` 的同伴火把分支已改为调用 P39 推理层，P38 旧行为语义保留为执行适配结果。

## 任务卡约束自检

- 未新增 `if hunger then eat red_berry`、`if cold then build_house`、`if beast then use_torch` 主逻辑；具体行动来自 `KnowledgeClaim.predicate.effect_key` 与语义亲和匹配。
- 规划器不直接修改 `WorldSnapshot`；执行适配仅生成 `AgentAutonomyResult` / `WorldChange`，仍交给 P38 applier 结算。
- 前置检查通过 P27 `ConditionExpressionEvaluator` 的 `condition:test:eq:*` 安全事实适配完成，避免散落 raw state 条件表达式。
- H5 投影只输出中文目标、下一步和阻塞原因，不输出 `utility_score`、`raw_state`、`hidden_truth`。
- 循环依赖、搜索超限、不可准备前置都会返回阻塞计划和中文诊断。

## 测试覆盖

- `backend/tests/unit/agent_reasoning/agent_reasoning_test.cpp` 覆盖任务卡 12 个单测项、P28 reaction rule 前置适配和 H5 安全投影。
- 已回归 `world_interaction`，确认 P38 世界交互、火把、野兽测试继续通过。

## 已知边界

- 已修复：P39 不再仅靠私有 fixture 映射处理 `make_torch` 前置，已通过 reaction rule adapter 读取 P28 `coreP28Rules()` 的产物与对象模式，推导 `wood_processed + camp_fire -> torch`。后续只需把 adapter 的规则来源从 fixture 扩展为正式内容包。
- 建房仍是规划语义与测试 fixture，不包含建筑 UI、占地、耐久、施工时间模拟。
- 多 Agent 分工未实现，符合任务卡本阶段边界。
