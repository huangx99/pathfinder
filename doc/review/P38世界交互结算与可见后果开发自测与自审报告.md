# P38 世界交互结算与可见后果开发自测与自审报告

## 1. 审核结论

P38 已按 `doc/67_P38世界交互结算与可见后果任务卡设计.md` 接入现有 H5 对话与投影架构，未另起炉灶。

本次实现把“动作产生知识”推进为“动作同时改变世界状态、状态可见、Agent 可根据状态产生行为反馈”的最小可玩闭环。玩家现在可以在 H5 中看到资源消耗、工具耐久、材料加工、火堆点燃、火把制作、野兽威胁变化、野生 Agent 避火等后果。

## 2. 本次实现范围

### 2.1 世界状态结算

- 复用 `DialogSessionState.object_runtime_states` 作为 P38 最小世界状态来源。
- 食物、材料、火把、火堆通过 `quantity` 表示数量或存在性。
- 斧头继续通过 `sharpness` 表示工具状态。
- 木头新增 `processed` 表示已处理材料数量。
- 靠近的野兽通过 `threat_level`、`observed_fire`、`knows_fire_danger` 与运行时标签表达威胁状态。

### 2.2 动作后果

- `吃红果 / 吃腐烂红果 / 吃苦叶` 会消耗对应数量。
- `用斧头砍木头` 会消耗未处理木头、增加已处理木材、降低斧头锋利度。
- `用磨石打磨斧头` 会恢复斧头锋利度。
- `用火种点燃干草` 会消耗干草、生成火堆，并让野兽观察到火。
- `制作火把` 需要已处理木材，会生成可用火把。
- `用火把驱赶野兽` 需要已有火把，会结算野兽退去。

### 2.3 Agent 行为

- 野兽不再只是静态物体，而是最小野生 Agent：它有状态、会观察火、会基于“火危险”的本能知识避开火。
- 同伴不直接改世界，而是在满足知识与资源条件后，通过结算层体现协助守夜。
- 本阶段坚持 Agent 边界：Agent 只表现为意图/反应结果，真正状态修改仍由对话结算服务统一执行。

### 2.4 H5 可见反馈

- 对象卡显示当前状态，例如剩余数量、已处理木材、斧头锋利度、火堆是否点燃、火把数量、野兽威胁状态。
- 行动按钮根据后端投影禁用部分未满足条件的操作，例如没有火把时不能执行驱赶。
- `feedback_not_found` 不再泄漏给玩家，失败会显示中文解释。
- 前端只展示后端投影，不在前端推导规则结果。

## 3. 关键文件

## 5. 自审结论

### 5.1 架构一致性

通过。P38 没有绕过现有学习、对话、投影架构，而是在 `DialogTurnService` 内完成动作结算，在 `H5PlayableProjectionMapper` 内完成玩家投影。

### 5.2 权威来源

通过。世界状态权威在后端 `DialogSessionState.object_runtime_states`，H5 前端不裁决、不推导、不修改规则结果。

### 5.3 Agent 边界

基本通过。野兽和同伴的行为没有直接由前端或独立系统改状态，而是由结算服务统一执行。当前仍是最小 Agent 行为，不是完整 Agent 行为管线。

### 5.4 可玩性

通过最小闭环。玩家可以：

1. 吃东西形成知识并消耗资源。
2. 砍木头获得加工材料。
3. 点燃干草生成火堆。
4. 制作火把。
5. 等待让危险推进。
6. 观察野兽因火产生反应。
7. 使用火把驱赶野兽。
8. 教同伴，让知识进入同伴侧。

## 6. 已知限制

- P38 仍然是 H5 最小闭环，不是完整开放世界。
- 野生 Agent 行为目前只实现“野兽避火”这一条最小行为，后续可扩展为正式 Agent 意图队列。
- 世界事件当前通过本回合 `state_summary_lines` 展示，后续可以升级为结构化 `WorldEventProjection`。
- 资源数量仍在初始化代码中声明，后续应迁移到配置化对象/反应数据。
- 同伴自动协助逻辑已经有结算入口，但完整“NPC 主动教学、野生 NPC 自主行动、多 Agent 冲突”应放到后续阶段继续扩展。

## 8. 复查修正记录

### 8.1 用户反馈问题

用户反馈“现在完全没效果”。复查后确认：后端世界状态已经变化，对象卡也有状态变化，但 P38 要求玩家反馈必须直接呈现 `player_feedback_zh_cn`。初版实现把世界变化主要放进场景摘要，H5 最近反馈仍然以学习反馈为主，所以玩家会感觉动作没有产生世界后果。

### 8.2 修正内容

- 将 P38 世界变化与 Agent 事件插入主 `reply_text`。
- H5 最近反馈现在直接显示：
  - `世界变化：`
  - `变化：木头被斧头砍开...`
  - `变化：斧头的锋利度下降...`
  - `变化：干草被火种点燃...`
- 避免使用 `- ` 列表前缀，因为 H5 安全文本提取会过滤列表行，导致世界变化被误删。
- 新增测试要求 P38 世界变化必须出现在 H5 `reply_text.zh_cn`，不是只出现在场景摘要或对象卡。

### 8.3 复查测试

命令：

```bash
cmake --build build/backend --target pathfinder_tests_h5_playable pathfinder_h5_playable_server_exe -j2
ctest --test-dir build/backend -R "h5_playable" --output-on-failure
```

结果：16/16 通过。

### 8.4 真实接口验证

真实请求 `用斧头砍木头` 后，H5 API 返回的 `reply_text.zh_cn` 已包含：

```text
世界变化：
变化：木头被斧头砍开，变成了可以继续加工的材料。
变化：斧头的锋利度下降了。当前锋利度：2。
```

真实请求 `用火种点燃干草` 后，H5 API 返回的 `reply_text.zh_cn` 已包含：

```text
世界变化：
变化：干草被火种点燃，一处小火堆出现了。
变化：火光让树林里的影子变得犹豫。野生生物已经观察到火。
```

### 8.5 复查结论

这次用户指出的问题成立。初版虽然有世界状态变化，但没有严格满足 P38 “玩家反馈直接可见”的要求。当前已修正为：动作结果、世界变化、对象状态三层同时可见。
