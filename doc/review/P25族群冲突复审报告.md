## P0 问题：冲突结果没有正确作用到失败方

状态：必须修复。

位置：

```text
backend/src/tribe/tribe_conflict.cpp:837
backend/src/tribe/tribe_conflict.cpp:845
backend/src/tribe/tribe_conflict.cpp:872
```

当前 `ConflictStateDraftBuilder::buildForTribe()` 对 `Intimidated` 的处理是：

```cpp
case ConflictOutcomeKind::Intimidated:
    if (own_intent.intent_kind == ConflictIntentKind::Intimidate) {
        draft.morale_delta += 0.03;
        draft.safety_pressure_delta -= 0.05;
        draft.trust_delta += 0.02;
    }
    break;
```

这只奖励“发起威慑的一方”，但没有给“被威慑的一方”增加任何撤退压力、安全压力、损失压力或士气影响。

`ForcedRetreat` 也类似：

```cpp
case ConflictOutcomeKind::ForcedRetreat:
    if (own_intent.intent_kind == ConflictIntentKind::Retreat) {
        ...
    }
    break;
```

如果 `ForcedRetreat` 是因为强度差距触发，而不是弱势方主动选择 `Retreat`，当前弱势方也可能拿不到撤退压力。

这会导致：

```text
A 强势威慑 B。
结果是 Intimidated。
A 获得正向状态变化。
B 却没有明显承受“被威慑 / 被逼退”的状态压力。
```

为什么这是 P0：

```text
P25 的核心体验是“不同族群可以敌对、威慑、撤退”。
如果威慑/撤退结果没有作用到承受方，冲突结果就只是一条 outcome 文本，不能可靠推动族群状态变化。
这会影响后续 P26 文明能力、P31 投影、P37 训练 observation 对冲突后果的读取。
```

修复要求：

```text
1. P25 必须明确 outcome 的方向：
   - source_intimidates_target
   - target_intimidates_source
   - source_forced_retreat
   - target_forced_retreat
   或至少在 buildForTribe() 中通过 own_intent/opponent_intent/strength gap 判断自己是施压方还是承压方。
2. Intimidated：
   - 施压方可获得 morale/trust/safety 的正向变化。
   - 承压方必须获得 retreat_pressure_delta 或 safety_pressure_delta，并可降低 morale/trust。
3. ForcedRetreat：
   - 被迫撤退方必须获得 retreat_pressure_delta。
   - 如果不是主动 Retreat，而是强度差距触发，也必须能落到弱势方。
4. Projection 的 visible_pressure_level 应能从承压方压力反映出来。
```

建议实现方式：

```text
方案 A：在 ConflictResolutionResult 增加 winner/loser 或 pressured_tribe_id。
方案 B：新增内部 ConflictOutcomeDirection，进入 ConflictStateDraftBuilder。
方案 C：扩展 ConflictStateDraftBuilder 参数，传入 source_strength / target_strength / is_source。
```

第一版建议用方案 B，保持 DTO 稳定，内部先解决方向判断。

## P1 问题：新增测试断言偏弱

状态：需要修复。

### 1. TruceOffered 测试没有断言 TruceOffered

位置：

```text
backend/tests/integration/p25/tribe_conflict_flow_test.cpp:604
backend/tests/integration/p25/tribe_conflict_flow_test.cpp:648
```

当前断言：

```cpp

```

这个断言太弱。即使结果是 `Standoff`、`Intimidated`、`ForcedRetreat` 也会通过。

```cpp
assert(dto.outcome_kind == ConflictOutcomeKind::TruceOffered);
```

### 2. Projection target pressure 测试没有验证 target 压力真的参与

位置：

```text
backend/tests/integration/p25/tribe_conflict_flow_test.cpp:653
backend/tests/integration/p25/tribe_conflict_flow_test.cpp:709
```

当前断言：

```cpp
assert(!dto.projection.visible_pressure_level.empty());
```

这个断言永远很容易通过，因为 `visible_pressure_level` 正常情况下会是 `none/low/moderate/high`。

必须改成至少验证：

```cpp
assert(dto.projection.state_pressure_changed);
assert(dto.projection.visible_pressure_level == "low" ||
       dto.projection.visible_pressure_level == "moderate" ||
       dto.projection.visible_pressure_level == "high");
assert(std::abs(dto.target_state_draft.retreat_pressure_delta) > 0.0 ||
       std::abs(dto.target_state_draft.loss_pressure_delta) > 0.0 ||
       std::abs(dto.target_state_draft.safety_pressure_delta) > 0.0);
```

### 3. trust_delta not overwritten 测试没有锁定预期值

位置：

```text
backend/tests/integration/p25/tribe_conflict_flow_test.cpp:547
backend/tests/integration/p25/tribe_conflict_flow_test.cpp:600
```

当前断言：

```cpp
assert(std::abs(dto.source_state_draft.trust_delta) <= 0.05);
```

这仍然允许很多非预期结果通过。

如果该场景期望 `Standoff`，建议断言：

```cpp

assert(std::abs(dto.source_state_draft.trust_delta - 0.03) < 0.001);
```
