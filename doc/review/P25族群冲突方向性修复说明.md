# P25 族群冲突方向性修复说明

## 修复结论

已修复 P25 复审中指出的核心问题：

```text
Intimidated / ForcedRetreat 这类带方向性的冲突结果，现在会把压力正确落到承受方。
```

同时已增强原本偏弱的测试断言，避免“结果不是 Unknown 就算通过”这类假通过。

---

## 修复内容

### 1. 状态草稿支持方向性判断

修改位置：

```text
backend/include/pathfinder/tribe/tribe_conflict.h
backend/src/tribe/tribe_conflict.cpp
```

`ConflictStateDraftBuilder::buildForTribe()` 新增参数：

```text
own_strength
opponent_strength
```

用途：

```text
在 outcome 只有 Intimidated / ForcedRetreat 这种摘要结果时，仍能判断当前族群是施压方还是承压方。
```

### 2. Intimidated 现在会影响被威慑方

修复后：

```text
施压方：士气/信任略升，安全压力下降。
承压方：撤退压力上升，安全压力上升，士气/信任下降。
```

这保证了“威慑成功”不是只有一条 outcome 文本，而会进入族群状态影响草稿。

### 3. ForcedRetreat 现在会影响被迫撤退方

修复后：

```text
主动或被迫撤退方：撤退压力上升，安全压力上升，士气/信任下降。
优势方：安全压力可下降。
```

即使撤退不是由显式 Retreat intent 触发，而是由 strength gap 触发，弱势方也会收到撤退压力。
