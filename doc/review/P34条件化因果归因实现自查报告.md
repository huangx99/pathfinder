# P34 条件化因果归因实现自查报告

## 1. 实现结论

本次已经把 H5 最小可玩中的红果/腐烂红果学习逻辑，从“单纯重复次数确认”改成了“条件化知识 + 短期暴露窗口 + 替代解释压制确认”。

核心变化：

```text
知识确认 != 重复次数
知识确认 = 重复经验 + 条件范围 + 时间窗口 + 替代解释排除
```

现在短时间内连续吃多个浆果后，再吃腐烂红果，不会立刻把“腐烂红果有毒”确认为绝对知识。

---

## 2. 主要改动

### 2.1 条件化知识

红果和腐烂红果现在通过对象状态条件区分：

```text
red_berry{新鲜状态} + eat -> restore_hunger
red_berry{腐烂状态} + eat -> poison
```

玩家中文看到的是：

```text
红果：吃（新鲜状态）后可以缓解饥饿
腐烂红果：吃（腐烂状态）后会让身体不适或中毒
```

避免再次出现两个知识都显示成“红果”的问题。

### 2.2 会话暴露记录

新增最小暴露记录：

```cpp
DialogCausalExposureRecord
```

记录：

```text
对象、对象族类、动作、结果、发生 tick、剂量、对象状态、归因原因
```

当前用于判断：

```text
近期是否短时间内吃了多个同类浆果。
```

### 2.3 时间窗口

当前最小实现：

```text
短期浆果暴露窗口 = 6 tick
等待一会 = 推进 8 tick
```

含义：

```text
短时间内吃太多浆果，会成为腐烂红果中毒的替代解释。
等待一会后，旧的短期剂量解释会自然衰减。
```

### 2.4 替代解释压制确认

当系统判断存在：

```text
短时间内吃了多个浆果 + 当前吃腐烂红果后不适
```

会给反馈打上：

```text
causal_ambiguous_dose_window
```

学习系统看到这个 reason 后，会把知识状态最高压到：

```text
Hypothesis / 经验假设
```

不会直接变成：

```text
Active / 已确认
```

### 2.5 等待行为

新增中文指令：

```text
等待
等待一会
休息
过一会
等一会
等一下
```

作用：

```text
推进时间，让短期暴露窗口消退。
```

等待不会清空记忆，也不会清空知识。

## 3. 自查结果

### 3.1 红果与腐烂红果是否仍混淆

结论：未发现混淆。

验证点：

```text
新鲜红果知识投影为“红果”。
腐烂状态知识投影为“腐烂红果”。
```

### 3.2 腐烂红果是否还能一次确认

结论：不会一次确认。

验证点：

```text
第一次吃腐烂红果 -> Candidate
第二次 -> Hypothesis
短期混杂下第三次仍最多 Hypothesis
```

### 3.3 短期吃太多是否会影响归因

结论：会。

验证点：

```text
吃红果 x3 后立刻吃腐烂红果 x3，腐烂红果不会直接 Active。
```

### 3.4 等待是否能排除短期影响

结论：能。

验证点：

```text
等待一会后，旧的红果短期暴露不再作为强替代解释。
之后继续测试腐烂红果，可以重新积累较干净的证据。
```

### 3.5 教学传播是否仍可用

结论：可用。

验证点：

```text
H5 学习和教同伴测试通过。
P22 教学链路测试通过。
```

### 3.6 是否泄露调试 key

结论：未发现前端 JSON 泄露原始调试知识表达。

验证点：

```text
h5_playable_codec
h5_playable_frontend_gate
```

仍然通过。

## 5. 关键测试场景

本次重点覆盖：

```text
1. 吃红果 x3 -> 新鲜红果缓解饥饿成为 Active。
2. 立刻吃腐烂红果 x3 -> 腐烂红果有害最多 Hypothesis，不会 Active。
3. 等待一会 -> 短期暴露窗口消退。
4. 等待后继续吃腐烂红果 x3 -> 腐烂红果可以重新积累较干净证据并 Active。
5. 红果 Active 后，腐烂红果学习不会降低新鲜红果知识。
6. H5 JSON 仍不暴露 red_berry / poison / raw_state 等调试信息。
```

## 6. 仍未实现但已预留的内容

以下内容仍属于后续扩展，不在本次最小实现里：

```text
1. 每个物品族类从 JSON 配置 exposure_window_ticks。
2. 剂量单位从固定 1.0 扩展为 bite / piece / gram。
3. 个体体质影响，如狼、人、蜘蛛、外星人不同反应。
4. 概率性结果，如腐烂食物有概率中毒。
5. 长期慢性暴露，如长期喝污水生病。
6. NPC 基于族群知识判断玩家是否投毒。
7. 制备来源和合成物品的完整因果追踪。
```

本次代码没有把这些写死，后续可以继续沿着当前的：

```text
暴露记录 -> reason key -> 学习状态压制 -> 条件化知识投影
```

扩展。

---

## 7. 自查结论

本次实现符合 P34 最小目标。

它不是完整真实世界因果系统，但已经把当前最危险的问题修掉了：

```text
不会再因为单纯重复或短期混杂，直接生成过度确定的知识。
```

同时保留了后续扩展入口：

```text
时间窗口、剂量、对象状态、替代解释、知识条件、传播保真。
```
