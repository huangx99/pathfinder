## 3. 已实现玩法闭环

- 吃红果：资源减少，反馈显示世界变化，知识形成。
- 吃腐烂红果：资源减少，状态后果进入反馈，因果归因仍保留。
- 砍木头：木头减少，处理木材增加，斧头锋利度下降。
- 点燃干草：干草减少，火堆生成，野兽观察到火。
- 制作火把：消耗处理木材，火把生成。
- 等待：威胁推进。
- 野兽避火：野生 Agent 观察火后产生 `AvoidHazard` 行动。
- 同伴协助：同伴具备知识和火把条件时可执行 `HoldTorch` 协助。
- H5 对象卡：读取 `WorldProjectionMapper` 的对象状态补丁显示当前状态。

## 4. 关键修复

### 4.1 废除临时 `debug_keys` 世界事件承载

初版用 `debug_keys` 承载 `world.event:` / `agent.event:`，这不符合 P38。

整改后：

- 世界变化由 `WorldChange` 表示。
- 玩家反馈由 `WorldProjectionPatch.player_feedback_lines_zh_cn` 表示。
- 对象状态由 `WorldObjectProjectionPatch` 表示。
- `debug_keys` 不再作为 P38 世界事件的权威来源。

### 4.2 失败动作不进入学习

初版容易出现“世界条件不满足但学习仍可能发生”的风险。

整改后：

- `WorldInteractionService::resolve()` 在学习前执行。
- 世界交互失败会返回 `InteractionFailureKind`。
- `DialogTurnService` 将失败转为中文拒绝，不进入学习闭环。

### 4.3 前端不裁决规则

H5 前端仍然只渲染后端投影。

对象状态来自：

```text
WorldSnapshot -> WorldProjectionMapper -> WorldObjectProjectionPatch -> H5PlayableProjectionMapper -> H5 projection
```

### 4.4 修复野兽一直“潜伏未近身”

本次复查发现一个不符合 P38 设计的问题：`DialogTurnService` 在等待和普通玩家动作后，曾经给 `AgentAutonomyService` 传入伪造的 `observed_object_keys = {"fire"}`。

这会导致：

- 即使玩家没有点火、没有火堆、没有火把，野兽也会执行 `AvoidHazard`。
- 威胁每次刚推进就被火光压回去。
- H5 对象卡长期显示 `状态：潜伏未近身`。
- 玩家看到“野生 Agent 会本能避开火”，但世界里并没有真实火源，规则来源不可信。

整改后：

- `DialogTurnService` 不再伪造火源观察。
- `AgentAutonomyService` 只在真实火源存在时避让：`camp_fire.quantity > 0` 或 `torch.quantity > 0` 或真实观察事件带入 `fire`。
- `WorldSnapshotAdapter::writeBackToDialogSession()` 同步威胁阶段时会清理过期的 `avoiding_fire` 标签，避免旧状态残留。
- 新增 H5 回归测试 `h5_playable_p38_beast_no_fake_fire`，确保无火等待会推进到观察、靠近、对峙，不会出现“观察到火光”文案。

```text
第 1 次等待：靠近的野兽 当前状态：状态：正在观察；野生 Agent：会本能避开火。
```


```text

```

结论：野兽现在不是固定潜伏；没有火会靠近，有真实火才会避让。

## 9. 追加复查：野兽攻击与按钮忙碌态

### 9.1 问题

试玩反馈指出：

- 无火源连续等待后，野兽只推进到对峙，但不会真正攻击玩家或造成失败后果。
- H5 点击任意按钮时，所有按钮都会同时置灰/闪烁，玩家会误以为所有操作都被触发或界面异常。

### 9.2 修复

后端修复：

- `AgentAutonomyService` 在野兽无真实火源可避让、威胁达到 `100` 时执行 `AttackTarget`。
- 攻击产生正式 `WorldChange`，而不是 UI 文案假效果。
- `WorldChangeApplier` 会把威胁阶段写成 `ThreatEventPhase::Failed`，并给 `beast_shadow` 添加 `failed`、`attacked` 标签。
- `WorldSnapshotAdapter` 与对象卡状态摘要支持显示 `状态：已经袭击营地`。

前端修复：

- `frontend/h5_playable/app.js` 增加 `activeBusyKey`。
- 请求中只给当前点击按钮添加 `is-busy` 状态，不再把所有按钮统一 disabled。
- 其他可用按钮在请求期间保持视觉稳定，避免“全部闪烁”的错觉。

结果：20/20 通过。

新增/加强覆盖：

- 第 4 次等待必须出现 `野兽冲入营地` 和 `第一夜失败`。
- 对象卡必须显示 `状态：已经袭击营地`。
