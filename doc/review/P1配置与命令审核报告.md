## 9. 非阻塞建议

### 9.1 ConfigLoader 路径安全可以后续增强

当前逻辑：

```text
拒绝绝对路径。
拒绝任何包含 .. 的路径。
```

判断：

```text
P1 可接受。
```

后续建议：

```text
P2/P3 可以改为 filesystem canonical 后确认路径仍在 root_path 内。
当前简单拒绝策略更安全，不阻塞。
```

### 9.2 Result<T> 默认构造限制仍存在

当前 `Result<T>::fail(...)` 仍要求 `T{}` 可构造。

判断：

```text
P1 不阻塞。
```

后续建议：

```text
进入更复杂不可默认构造类型前，考虑把 Result<T> 改为 std::optional<T> 或 std::variant。
```

### 9.3 ConfigValidationReport 与 CommandValidationReport 暂不合并

判断：

```text
P1 允许重复少量报告结构。
```

后续建议：

```text
等 P2 event/state/reporting 都出现后，再考虑抽象 common ValidationReport。
不要现在过早抽象。
```
