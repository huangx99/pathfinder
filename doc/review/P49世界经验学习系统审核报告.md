# P49 世界经验学习系统审核报告

## 1. 审核结论

P49 当前审核结论：通过。

本阶段实现已经把 V2 世界 Command 产生的 `WorldExperienceDto` 接回旧认知 / 记忆 / 学习 / 知识闭环，符合 P49 设计目标：

```text
WorldCommandExecutionResult.experiences
-> WorldExperienceExtractor
-> WorldKnowledgeTemplateResolver
-> WorldSafeFeedbackBuilder
-> LearningLoopService
-> KnowledgeRepository
-> WorldKnowledgeProjectionBridge
-> events / state_deltas / projection_patch
```

审核中发现并修复了一个阻塞级风险：`EffectDefinitionContent.risk_score` 是内容侧 0-10 风险分，而 `LearningSafeFeedbackInput.risk_delta` 要求 0-1。原实现直接传入 `risk_score`，遇到毒蘑菇 / 中毒 / 高风险效果会导致反馈校验失败，进而无法形成危险知识。现已改为 `risk_score / 10.0` 并 clamp 到 0-1，同时补充高风险学习测试。

## 2. 已核对的架构要求

### 2.1 没有另起炉灶

P49 没有新写一套知识系统，而是复用已有学习闭环：

```text
LearningSafeFeedbackInput
LearningLoopService
LearningLoopApplier
KnowledgeRepository
KnowledgeClaim
```

P49 只是适配世界经验和旧学习系统之间的边界。

### 2.2 没有前端 / H5 依赖

已扫描：

```text
backend/include/pathfinder/world_learning
backend/src/world_learning
```

未发现 `h5`、`dialog`、`playable` 等前端耦合。

### 2.3 没有内容写死

生产代码中未发现 `red_berry`、`torch`、`wood`、`beast`、`wolf`、`mushroom` 等具体游戏内容 key。

测试中允许使用 `torch`、`berry_bush`、`toxic_mushroom` 作为测试内容。

### 2.4 没有绕过世界归属 / runtime

已扫描 P49 生产代码，未发现：

```text
spawnEntity
destroyEntity
setEntityStackData
cells_
entries
parameters["knowledge"]
```

P49 不直接改世界 runtime、不直接改背包、不绕过 P45 归属系统。

### 2.5 没有让前端伪造知识

P49 只从 `WorldCommandExecutionResult.experiences` 学习。

已有测试覆盖：即使请求参数里传入 `knowledge_key`，只要没有真实 experience，就不会形成知识。

## 3. 本次修复内容

### 3.1 高风险效果归一化

修改位置：

```text
backend/src/world_learning/world_safe_feedback_builder.cpp
```

修复逻辑：

```text
risk_delta = clamp(effect.risk_score / 10.0, 0.0, 1.0)
```

修复意义：

```text
毒蘑菇、中毒、野兽伤害、火焰灼伤等高风险经验不会因为 risk_delta 超范围而被学习系统拒绝。
NPC 后续也能学习“这个行为危险”，而不是完全学不到。
```

### 3.2 新增高风险测试

新增测试：

```text
world_learning_high_risk_effect_is_normalized
```

覆盖内容：

```text
toxic_mushroom + eat -> poison
risk_score = 8
risk_delta = 0.8
outcome = HealthWorsened
最终仍能经过 LearningLoopService 形成 KnowledgeClaim
```

## 5. 自我审核发现的非阻塞风险

### 5.1 失败命令暂时跳过学习

当前 P49 对失败 command 默认跳过，不形成正向知识。

这符合 P49 MVP 设计，但后续阶段如果要让 NPC 学会：

```text
缺材料所以不能制作
斧头钝了所以不能砍树
目标太远所以不能交互
```

需要在后续阶段把失败经验接成 `Blocked / MissingInput / StateMismatch / Risk` 类记忆，而不是在 P49 直接扩展。

### 5.2 action_key 到 cognition action context 仍是小型映射

`WorldSafeFeedbackBuilder` 目前把 `eat/use/craft/gather/chop/mine` 映射成已有 cognition action context。

这不是内容写死，但后续如果 command 类型继续增长，建议迁移到 JSON / registry 驱动的 action semantics，避免长期维护 if 分支。

### 5.3 P49 只负责世界经验转知识，不负责教学和 NPC 自主行动

P49 生成的 `KnowledgeClaim` 已保留 `teachable / usable_by_ai / usable_for_action` 等字段。

但“NPC 是否会主动使用这些知识”属于 P50/P51/P40 目标执行链，不属于 P49。

## 6. 最终判断

P49 当前可以验收。

理由：

```text
1. 设计要求的世界经验到知识闭环已经接通。
2. 没有绕开旧认知、记忆、学习、知识系统。
3. 没有前端依赖。
4. 没有具体内容写死在生产代码。
5. 没有绕过 runtime / inventory 归属系统。
6. 高风险经验学习的阻塞问题已修复并有测试覆盖。
7. P49 专项与相关旧系统回归全部通过。
```
