## 2. 自审中发现并修正的问题

### 2.1 问题：Story 曾反向依赖 H5 Dialog

初版修复为了复用 `DialogScenario.suggested_action_templates`，让 `pathfinder_story` 链接了 `pathfinder_h5_dialog`。

这是不合格的分层：

- `H5 Dialog` 是前端对话原型层。
- `Story` 是剧情/目标层。
- 高层表现模块不应该成为剧情核心模块的依赖来源。

如果保留这个依赖，后续会导致：

- Story 无法独立于 H5 使用。
- 配置目录迁移时依赖方向混乱。
- 未来其他前端或服务端剧情系统也会被 H5 原型层绑住。

### 2.2 修正方案

已修正：

- 移除 `pathfinder_story -> pathfinder_h5_dialog` 链接。
- 移除 `story_system.cpp` 对 `dialog_scenario.h` 的 include。
- 在 `StoryScenarioDefinition` 中新增 `suggested_actions`。
- 新增 `StorySuggestedActionDefinition`。
- Story 投影从自身 `scenario.suggested_actions` 生成建议动作。

相关文件：

- `backend/CMakeLists.txt`
- `backend/include/pathfinder/story/story_types.h`
- `backend/src/story/story_types.cpp`
- `backend/src/story/story_system.cpp`

修正后分层恢复为：

- Story 管自己的剧情建议动作。
- H5 Dialog 管自己的对话场景动作模板。
- 二者当前内容一致，但不互相反向依赖。
- 后续更合理的方向是抽公共 content/config 包，而不是让 Story 依赖 H5。

## 5. 残留风险

### 5.1 仍未迁移到外部 JSON

当前 `DialogScenario` 和 `StoryScenarioDefinition` 仍是 C++ 内置配置。

这次修复只是把分散逻辑收敛到配置结构，不是完整内容包系统。

后续仍建议做：

- `DialogScenarioCatalog` 支持 JSON / 配置包加载。
- `StoryScenarioRegistry` 支持 JSON / 配置包加载。
- 对象、反馈、建议动作、威胁知识统一进入内容目录。

### 5.2 H5 输入词表仍是原型层逻辑

中文输入解析仍在 H5 Dialog 原型层，后续应该从对象别名 / 行为别名配置读取。

这不阻塞本次修复，因为本轮目标是处理默认知识和 effect_key 分散风险。
