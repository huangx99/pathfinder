# P40 动态目标执行与外界打断系统复审修复报告

## 结论

已按 `doc/review/P40动态目标执行与外界打断系统审核报告.md` 的 P0/P1 问题完成整改，建议复审。

## P0 修复

### P0-1 H5 execution_status 未接入 GoalExecutionSystem

已修复：

- `H5PlayableTurnService::buildResponse` 现在构造最小 `ExecutionTickRequest`。
- H5 turn 流程调用 `GoalExecutionSystem::tick`。
- `ExecutionResult.projection` 被映射到 `H5PlayableExecutionStatus`。
- `H5PlayableResponse.execution_status` 在 bootstrap / wait / 普通 turn 中实际填充，`visible == true`。
- 前端仍只消费后端 `execution_status` 字段，不从回复文本推导执行状态。

覆盖测试：

- `h5_execution_status_visible_when_goal_execution_runs`
- `h5_playable_codec` 断言 `execution_status` 非空、visible、包含 current_goal / active_step。

### P0-2 内部阻塞子目标未进入目标栈

已修复：

- `ExecutionResult` 新增 `generated_subgoals`。
- `GoalExecutionSystem::tick` 处理内部阻塞后，会暂停原 active frame，并将首个子目标转为 `GoalFrame` 入栈。
- 入栈子目标保留 `parent_frame_id`，同时 `requires_reasoning == true`，等待 P39 给出正式计划。

覆盖测试：

- `internal_blocker_pushes_subgoal_frame`
- `goal_execution_dull_axe` 断言钝斧阻塞生成 `RestoreToolState` 子目标，目标栈新增子目标帧。

### P0-3 恢复未验证 active_driver_state

已确认并加强测试：

- `GoalStackManager::resumeFrame` 从 `PausedExecutionContext.driver_state` 恢复 `active_driver_state`，并移除 paused context。
- 测试断言恢复后 active driver 仍是 `ChopWood`。

覆盖测试：

- `resume_restores_driver_state`
- `goal_execution_resume`

## P1 修复

### P1-1 材料状态投影不可信

已补测试证明真实 H5/world adapter 状态链路可用：

- 使用 `DialogScenarioCatalog::defaultScenario` 与 `WorldSnapshotAdapter::fromDialogSession` 构造真实 `WorldSnapshot`。
- 验证 `axe.sharpness == 0` 会被同步为 `dull` tag。
- 验证 `dry_grass` 的 runtime `wet` 状态会进入 P40 材料状态判断，并触发 `blocked_by_state`。

覆盖测试：

- `material_tags_from_dialog_runtime_snapshot`

- 增加核心规则测试，确保 torch / wood_processed / restore_sharpness 链路不会产生 `unknown_material`。

覆盖测试：

- `reaction_material_resolver_no_unknown_material_for_core_rules`
