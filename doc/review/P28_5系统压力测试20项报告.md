## 2. 20 项测试覆盖了什么

### 01. 合成到知识到族群到认知完整链

测试：

```text
火 + 干树枝 -> 火把
火把生成知识草稿
知识转 KnowledgeClaim
族群共享该知识
玩家形成红果可食用认知
```

目的：验证“合成 -> 知识 -> 族群 -> 认知”能连起来。

### 02. 生成物继续可用

测试：

```text
火把作为运行态对象被使用。
使用火把后 fuel 从 100 降到 90。
```

目的：验证生成物不是文本，而是真的能继续进反应系统。

### 03. 缺少材料后补材料重试

测试：

```text
只有石片，没有树枝，构造合成输入失败。
补入干树枝后，石片 + 干树枝 -> 矛 成功。
```

目的：验证缺材料不会假成功，补齐材料后流程能恢复。

### 04. 潮湿树枝不能点火

测试：

```text
火 + 湿树枝 不会变火把。
```

目的：验证条件表达式能阻挡错误合成。

### 05. 错误动作不能触发合成

测试：

```text
对树枝执行 Use，不会触发 Combine 规则。
```

目的：验证动作类型是规则匹配的一部分。

### 06. 角色放反不能合成

测试：

```text
树枝当 Source、火当 Material，不会触发 火 + 树枝 -> 火把。
```

目的：验证对象角色 Source / Material 不是可随便乱填。

### 07. 消耗后的物品不能重复使用

测试：

```text
喝水后 clean_water quantity 变为 0。
再次喝同一份水，不会成功。
```

目的：验证消耗物不会无限复用。

### 08. 水能熄灭火

测试：

```text
清水 + 火源 -> 移除 fire 的 burning 状态。
```

目的：验证 RemoveState 能正确应用到运行态对象。

### 09. 石片 + 树枝能合成矛

测试：

```text
石片 + 干树枝 -> 树枝 TransformObject 为 spear。
```

目的：验证非火把合成也走同一套配置化流水线。

### 10. 藤蔓 + 树枝能生成新物品

测试：

```text
藤蔓 + 干树枝 -> ProduceObject 生成 branch_bundle。
```

目的：验证 ProduceObject 能生成新对象，而不只是 TransformObject。

### 11. 行动者、旁观者、被教学者都能收到知识草稿

测试：

```text
DirectExperience
Observation
Taught
```

目的：验证反应知识可以进入玩家、NPC 观察学习和教学路径。

### 12. 反复教学只强化不重复

测试：

```text
玩家教同伴一条知识。
再教同一条知识。
同伴知识数量不增加，support_count 增加。
```

目的：验证重复教学走强化，不制造重复知识。

### 13. 同一次教学混合新建和强化

测试：

```text
学生已有“鲜红果可缓解饥饿”。
老师同时教“鲜红果可缓解饥饿”和“腐烂红果有毒”。
结果应该是一条强化，一条新建，总数为 2。
```

目的：验证之前修复的“混合新建/强化分类”不会回退。

### 14. 认知反复学习会增强

测试：

```text
玩家两次获得红果可食用证据。
认知 confidence 从 0.3 增加到 0.7。
evidence_count 变为 2。
```

目的：验证认知层能反复积累证据。

### 15. 鲜红果和腐烂红果认知分开

测试：

```text
鲜红果 -> Edible
腐烂红果 -> Harmful
```

目的：验证不会再把两个都混成 red_berry。

### 16. 同伴被教后也能形成认知

测试：

```text
同伴获得“腐烂红果有害”的认知证据。
同伴认知中能查到 harmful claim。
```

目的：验证 NPC / 同伴也能拥有自己的认知。

### 17. 族群知识共享去重

测试：

```text
族群第一次链接 know_torch。
第二次再链接同一个 know_torch。
共享知识数量仍然是 1。
```

目的：验证族群知识不会重复堆积。

### 18. 高知识冲突压力会推动族群分裂风险

测试：

```text
resource_pressure 高。
safety_pressure 高。
knowledge_conflict_pressure 高。
casualty_pressure 高。
trust_delta 为负。
```

结果：

```text
split_risk >= 0.5
cohesion_state 进入 Tense / Fracturing / Splintered 范围。
```

目的：验证族群分裂压力链路有效。

### 19. 分裂压力下成员离开

测试：

```text
MemberRemoved 事件移除一个成员。
同时有较高 knowledge_conflict_pressure。
```

目的：验证族群分裂/离群可以通过成员事件表达。

### 20. 共同作战与族群冲突安全输出

测试：

```text
带共享知识的 tribe_alpha 进入共同作战协调。
tribe_alpha 与 tribe_beta 在高资源/恐惧/生存压力下冲突。
冲突结果必须有效，且公开摘要不能包含 death / loot 等隐藏或不安全词。
```

目的：验证共同作战和族群冲突能接上，同时不泄露不该给前端的内容。

## 3. 本次发现的问题

本次 20 项压力测试中，初始版本有一个测试预期需要修正：

```text
同一个 TribeStateInput 里重复传入同一个 knowledge_id，会被 TribeStateInput 校验拒绝。
```

这不是底层 bug，而是当前设计要求：

```text
同一个输入批次不能有重复 knowledge_id。
跨批次重复链接同一个 knowledge_id 时，族群状态会去重。
```

因此测试 17 已调整为：

```text
第一批次链接 know_torch。
第二批次再次链接 know_torch。
最终 shared_knowledge_ids 仍为 1。
```

这个行为更符合现有设计。
