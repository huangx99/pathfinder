# P21 学习闭环修复说明

## 1. 修复结论

**结论：通过，P21 可以进入 P22 最小 H5 文本可玩阶段。**

本轮修复关闭了原审核报告中的三个阻塞项：

```text
BLOCKER-1：红果完整闭环没有形成知识、没有传播，但返回 Completed。
BLOCKER-2：P20 should_route_to_revision 后，P21 构造给 P19 的接收者修正输入不完整。
BLOCKER-3：P21 测试过弱，不能证明最小可玩学习闭环成立。
```

修复后，红果故事可以真实完成：

```text
反馈捕获 -> 认知更新 -> 记忆写入 -> 记忆压缩 -> 记忆召回 -> 知识形成 -> 知识传播 -> 接收者受影响 -> 报告生成
```

并且 P20 传播冲突路由到 P19 修正时，P21 会构造完整的接收者侧 `KnowledgeRevisionInput`，使接收者旧知识被修正，新知识进入接收者快照。

## 2. 修改范围

### 2.1 P18 小修：知识形成阈值可配置

涉及文件：

```text
backend/include/pathfinder/knowledge/knowledge_claim.h
backend/src/knowledge/knowledge_claim.cpp
backend/src/knowledge/knowledge_formation.cpp
```

修改内容：

```text
KnowledgeFormationOptions 增加 min_hypothesis_confidence。
validateBasic() 校验 min_hypothesis_confidence 范围。
validateBasic() 校验 active_confidence_threshold >= min_hypothesis_confidence。
KnowledgeFormationPlanner 不再硬编码 0.35，而是使用 options.min_hypothesis_confidence。
```

原因：

```text
P21 最小可玩闭环中，一次安全反馈形成的是低置信假说。
低置信假说应该可以进入 Hypothesis，而不是被 P18 硬编码阈值直接丢弃。
P18 默认值仍保持 0.35，不破坏既有阶段语义；P21 只在自己的编排 options 中调低阈值。
```

### 2.2 P21 编排修复

涉及文件：

```text
backend/src/learning/learning_loop.cpp
```

修改内容：

```text
强化 LearningLoopInput owner 校验，拒绝 actor/recipient 快照 owner 混用。
修复 trace/result validateBasic() 对 Unknown/TestOnly failure_reason 的漏检。
写入记忆前，将 candidate owner/subject 强制对齐 P21 的安全输入主体。
MemoryCompressed 只有真实产生 summary 时才标记 Executed，否则 SkippedNoInput。
KnowledgeFormed 使用 P21 专用低阈值，让最小故事能形成 Hypothesis/Active 知识。
KnowledgeFormed trace 正确记录新 knowledge_id。
KnowledgePropagated 允许 Hypothesis/Candidate 在 P21 教学链路中传播，并使用 P21 专用传播阈值。
P20 should_route_to_revision=true 时，P21 从传播计划定位 source claim，并构造完整 recipient revision input。
修复接收者 revision 成功后未正确应用 updated/created claims 的问题。
最终 decision 不再只看流程是否跑完；必要阶段缺失会返回 Failed 或 PartialCompleted。
```

### 2.3 P21 测试强化

涉及文件：

```text
backend/tests/integration/p21/learning_loop_discovery_story_test.cpp
backend/tests/integration/p21/learning_loop_revision_story_test.cpp
backend/tests/integration/p21/learning_loop_security_test.cpp
```

强化内容：

```text
p21_discovery_and_teaching 强断言 KnowledgeFormed、KnowledgePropagated、RecipientInfluenced 必须执行。
p21_discovery_and_teaching 强断言 actor 形成 red_berry/eat/restore_hunger 知识。
p21_discovery_and_teaching 强断言 recipient 得到 owner=agent_b 且 evidence 包含 Teaching 的知识。
p21_revision_and_propagation_route 强断言 P20 assessment.should_route_to_revision=true。
p21_revision_and_propagation_route 强断言 P21 产生 KnowledgeRevised:Routed，而不是 KnowledgeRevised:Failed。
p21_revision_and_propagation_route 强断言接收者冲突知识被收敛，不能保留多个 active/shared 矛盾主张。
p21_recipient_owner_mismatch 强断言 owner mismatch 不允许以正常 Completed 通过。
```

## 3. 核心闭环验证

### 3.1 红果发现与教学闭环

验证故事：

```text
agent_a 尝试 eat red_berry。
安全反馈输入给出 restore_hunger。
P21 形成 actor 知识。
P21 通过 P20 把知识教给 agent_b。
agent_b 得到 Teaching 来源的主观知识。
```

修复后探针结果：

```text
decision=completed ok=1 actor_claims=1 recip_claims=1
feedback_captured:executed:none
cognition_updated:executed:none
memory_written:executed:none
memory_compressed:executed:none
memory_recalled:executed:none
knowledge_formed:executed:none
knowledge_revised:skipped_no_input:none
knowledge_propagated:executed:none
recipient_influenced:executed:none
report_built:executed:none
actor know_agent_a_red_berry_has_effect_10_0 red_berry eat restore_hunger status=active
recip know_agent_b_red_berry_has_effect_10_0 red_berry eat restore_hunger status=hypothesis
```

结论：

```text
BLOCKER-1 已修复。
P21 不再是假 Completed。
最小可玩学习链路已经真实产生知识并传播给接收者。
```

### 3.2 传播冲突路由修正闭环

验证故事：

```text
agent_b 原本相信 red_berry/eat/restore_hunger。
agent_a 传播 red_berry/eat/poison。
P20 判断 should_route_to_revision=true。
P21 把传播冲突路由到 P19 recipient revision。
agent_b 的旧知识被 deprecated，新知识进入 teachable。
```

修复后探针结果：

```text
decision=routed_to_revision claims=2 prop=weakened_recipient_claim
route=1 rec_dec=weakened_recipient_claim
knowledge_propagated:executed:none
knowledge_revised:routed:none
recip know_b_restore restore_hunger status=deprecated
recip know_agent_b_red_berry_has_effect_20_1 poison status=teachable
```

结论：

```text
BLOCKER-2 已修复。
P20 route_to_revision 不再停在传播层，而是能进入 P19 修正并应用到 recipient snapshot。
```

## 6. 后续注意事项

```text
P21 当前仍是后端学习闭环，不是完整游戏 UI。
P22 接入 H5 时只能调用 P21 暴露的安全 DTO，不允许前端或适配层读取 hidden truth。
P22 第一版建议只做中文对话框文本游戏，通过按钮/文本命令触发 eat/use/teach/inspect 等安全动作。
P22 不要把红果效果写死在 UI；对象、动作、反馈仍应从安全配置和后端结果组合出来。
P22 不要引入网络依赖；本地 6006 端口静态 H5/轻服务即可。
```
