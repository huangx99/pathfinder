# P20 知识传播系统任务卡设计

## 1. 阶段定位

P20 是《先驱者》认知、记忆、知识、传播学习主线的第六个核心玩法阶段。

前置阶段已经完成：

```text
P15：认知系统正式化，回答“主体当前相信什么”。
P16：记忆系统基础，回答“一次经历如何被记住、强化、衰减”。
P17：记忆压缩与检索，回答“大量经验如何沉淀成可用摘要”。
P18：知识系统，回答“稳定记忆如何形成可查询知识声明”。
P19：知识冲突与修正，回答“旧知识如何被新证据强化、削弱、特化、废弃、证伪”。
```

P20 解决：

```text
一个个体已经知道某个对象能吃/危险/可用之后，如何把这条主观知识告诉另一个个体或族群。
接收者获得的是被传授的主观知识，不是真实隐藏属性。
传播可以成功、失败、打折、变形。
错误知识也可以传播，后续仍可被 P19 修正。
```

一句话：

```text
P20 让“我知道红果能吃”变成“我能让同族也知道红果能吃”。
```

P20 在主链路中的位置：

```text
探索 → 反馈 → 认知 → 记忆 → 压缩/召回 → 知识形成 → 知识修正 → 知识传播 → 学习闭环 → 族群/文明
```

P20 只做到“主观知识从发送者到接收者的安全转移”，不做完整族群政治、不做文明能力解锁、不做 H5 表现、不做 AI/RL 训练。

---

## 2. 本阶段目标

P20 的目标是让 P18/P19 产出的 `KnowledgeClaim` 可以被安全传递给其他主体，并在接收者侧形成可追溯、可查询、可再次修正的主观知识。

必须做到：

1. 支持 Agent → Agent 的知识传播。
2. 支持 Agent → Tribe 的知识传播草稿，第一版可用 `KnowledgeOwnerKind::Tribe` 表示族群接收者。
3. 支持 Tribe → Agent 的知识下发草稿，第一版不做复杂族群状态。
4. 传播输入只能来自有效 `KnowledgeClaim` 或安全 `KnowledgeProjectionItem`，不能来自 hidden truth。
5. 接收者获得的是新 owner 下的 `KnowledgeClaim`，不是共享同一个对象实例。
6. 接收者知识必须记录 `Teaching` evidence、source owner、source knowledge id、channel、信任/可靠度影响。
7. 传播可以因距离、信任、通道质量、知识可教性、接收者能力不足而失败或降级。
8. 错误知识、低置信知识可以传播，但必须带着较低置信度和来源证据。
9. `Deprecated` / `Disproven` 默认不传播给普通接收者；纠错传播必须通过明确 correction 模式。
10. 所有 `Result::ok(dto)` 返回值必须能通过 `validateBasic()`。
11. 继续保持 hidden truth 防线：不能读取 ObjectDefinition hidden truth，不能读取 raw GameState。

核心原则：

```text
传播传主观知识，不传真实规则。
接收者相信多少，由传播上下文决定。
传播产生新证据链，不复用发送者的所有内部证据。
错误知识也能传播，但必须可追溯、可修正。
P20 不直接改变世界，只改变接收者的知识状态草稿。
```

## 3. 玩家体验映射

P20 对玩家体验的意义：

```text
玩家作为先驱者，不只是自己探索。
玩家探索后得到知识，可以教给其他族群成员。
其他成员因为被教过，下一次面对对象时不再完全无知。
如果玩家教错了，族群也可能犯错。
如果玩家后来修正知识，可以再传播纠错。
```

典型体验：

```text
Agent A 吃红果，形成“红果可食用”。
Agent A 教给 Agent B。
Agent B 遇到红果时，观察里出现“听 A 说可食用，置信度中等”。
Agent B 可以选择相信、谨慎尝试、或忽略。
```

风险体验：

```text
Agent A 误以为某蘑菇可食用。
Agent A 传播给 Agent B。
Agent B 吃后中毒。
P19 修正该知识，P20 后续可传播“纠错知识”。
```

P20 不直接做 UI，但必须为后续 UI 提供明确状态：

```text
这条知识是自己经历来的。
这条知识是别人教的。
传播成功/失败。
接收者相信/怀疑。
传播来源是谁。
传播通道是什么。
传播后置信度是多少。
```

## 5. 本阶段不做什么

P20 明确禁止：

```text
不做完整族群状态。
不做士气、分工、分裂风险。
不做共同作战。
不做族群冲突。
不做文明能力解锁。
不做对象反应。
不做危险系统扩展。
不做 H5/API/HTTP/WebSocket/JSON。
不做 SaveManager。
不做 RL reward/done。
不让 AgentRuntime 或 Policy 直接写 KnowledgeRepository。
不从 ObjectDefinition hidden truth 推导传播内容。
不从 GameState raw state 推导传播内容。
不复制发送者完整 MemoryRecord / MemorySummary 给接收者。
不物理删除 KnowledgeClaim。
不把 Deprecated / Disproven 默认传播给普通视图。
不做自由文本 AI 自动生成知识。
```

P20 可以为 P21 学习闭环保留事件、trace、debug 字段，但不能提前实现 P21 的完整链路编排。

## 7. 枚举设计

### 7.1 KnowledgePropagationChannel（知识传播通道）

中文：知识传播通道。

| 枚举 | 字符串 | 中文 | 用途 |
|---|---|---|---|
| Unknown | unknown | 未知 | 非法默认值 |
| DirectTeaching | direct_teaching | 直接教学 | 明确把知识教给接收者 |
| Demonstration | demonstration | 示范 | 通过行为示范让接收者学习 |
| WarningSignal | warning_signal | 警告信号 | 告知危险、不要做某事 |
| SharedObservation | shared_observation | 共享观察 | 多个主体共同见证经验 |
| TribeInstruction | tribe_instruction | 族群指令 | 族群层向成员下发知识 |
| Correction | correction | 纠错传播 | 传播对旧错误知识的修正 |
| ExternalRecord | external_record | 外部记录 | 后续预留，第一版不接入自由外部文本 |
| TestOnly | test_only | 测试专用 | 默认禁止 |

规则：

```text
Unknown/TestOnly 默认禁止进入正式 DTO。
ExternalRecord 第一版只保留枚举，不接入自由文本。
Correction 可传播修正后的有效 claim，但不能直接把 Disproven 当行动知识传播。
```

### 7.2 KnowledgePropagationDecision（知识传播决策）

中文：知识传播决策。

| 枚举 | 字符串 | 中文 | 用途 |
|---|---|---|---|
| Unknown | unknown | 未知 | 非法默认值 |
| Skipped | skipped | 跳过 | 没有可传播知识或目标为空 |
| Rejected | rejected | 拒绝 | 输入非法或安全边界失败 |
| Failed | failed | 传播失败 | 条件不足，未产生接收者知识 |
| CreatedRecipientClaim | created_recipient_claim | 创建接收者知识 | 接收者新获得一条知识 |
| UpdatedRecipientClaim | updated_recipient_claim | 更新接收者知识 | 接收者已有相关知识，被教学证据更新 |
| ReinforcedRecipientClaim | reinforced_recipient_claim | 强化接收者知识 | 教学证据支持接收者已有知识 |
| WeakenedRecipientClaim | weakened_recipient_claim | 削弱接收者知识 | 教学证据与接收者已有知识冲突 |
| CorrectionDelivered | correction_delivered | 纠错已传达 | 接收者获得修正知识或旧知识降权 |
| TestOnly | test_only | 测试专用 | 默认禁止 |

规则：

```text
Rejected 不作为 Result::ok 的成功结果。
Skipped/Failed 可以作为 Result::ok，但 DTO 必须 validateBasic 通过。
Updated/Reinforced/Weakened 可以在第一版内部调用 P19 修正逻辑，也可以生成 patch 交给后续应用。
```

### 7.3 KnowledgePropagationTrustBand（传播信任等级）

中文：传播信任等级。

| 枚举 | 字符串 | 中文 | 用途 |
|---|---|---|---|
| Unknown | unknown | 未知 | 非法默认值 |
| Distrusted | distrusted | 不信任 | 几乎不接受知识 |
| Low | low | 低信任 | 接收但置信度大幅下降 |
| Medium | medium | 中等信任 | 正常接收但保留怀疑 |
| High | high | 高信任 | 接收者较强相信 |
| Authority | authority | 权威 | 族群导师/权威来源，第一版仍不能越过证据链 |
| TestOnly | test_only | 测试专用 | 默认禁止 |

规则：

```text
TrustBand 只影响接收者初始置信度，不代表真实正确。
Authority 也不能绕过 hidden truth 或直接创建 Established 真理。
```

### 7.4 KnowledgePropagationFailureReason（传播失败原因）

中文：传播失败原因。

| 枚举 | 字符串 | 中文 | 用途 |
|---|---|---|---|
| Unknown | unknown | 未知 | 非法默认值 |
| NoTransferableClaim | no_transferable_claim | 无可传播知识 | 来源没有可传播 claim |
| SourceInvalid | source_invalid | 来源非法 | 发送者或来源知识非法 |
| TargetInvalid | target_invalid | 目标非法 | 接收者非法 |
| SameOwner | same_owner | 同一主体 | 发送者和接收者相同且不允许自传播 |
| ClaimNotTeachable | claim_not_teachable | 知识不可教 | 状态、置信度或 flags 不允许传播 |
| ClaimDeprecated | claim_deprecated | 知识已废弃 | 默认不传播 Deprecated |
| ClaimDisproven | claim_disproven | 知识已证伪 | 默认不传播 Disproven |
| TrustTooLow | trust_too_low | 信任过低 | 接收者不接受 |
| ChannelTooWeak | channel_too_weak | 通道过弱 | 距离/噪声/表达不足 |
| ConflictWithRecipient | conflict_with_recipient | 与接收者旧知识冲突 | 需要 P19 修正路径 |
| SecurityRejected | security_rejected | 安全拒绝 | hidden truth / raw state / TestOnly 等 |
| TestOnly | test_only | 测试专用 | 默认禁止 |

规则：

```text
FailureReason 主要写入 warning_keys/reason_keys。
SecurityRejected 应优先 Result::fail，不应伪装成普通 Failed。
```

## 8. DTO 设计

所有 DTO 必须实现：

```cpp
pathfinder::foundation::Result<void> validateBasic() const;
```

任何 `Result::ok(dto)` 返回的 DTO 必须能通过 `validateBasic()`。

### 8.1 KnowledgePropagationOptions

中文：知识传播选项。

```cpp
struct KnowledgePropagationOptions {
    double min_source_confidence = 0.35;
    double min_transfer_score = 0.25;
    double direct_teaching_weight = 0.85;
    double demonstration_weight = 0.70;
    double warning_weight = 0.75;
    double shared_observation_weight = 0.65;
    double tribe_instruction_weight = 0.80;
    double correction_weight = 0.90;
    double max_created_confidence = 0.80;
    double max_shared_confidence = 0.70;
    bool allow_agent_to_agent = true;
    bool allow_agent_to_tribe = true;
    bool allow_tribe_to_agent = true;
    bool allow_same_owner = false;
    bool allow_deprecated = false;
    bool allow_disproven = false;
    bool allow_correction = true;
    bool allow_create_recipient_claim = true;
    bool allow_update_recipient_claim = true;
    size_t max_claims_per_attempt = 8;
    size_t max_evidence_refs = 50;

    pathfinder::foundation::Result<void> validateBasic() const;
};
```

规则：

```text
所有 double 必须在 0..1。
max_claims_per_attempt 不能为 0。
max_evidence_refs 不能为 0。
allow_deprecated/allow_disproven 默认 false。
Correction 模式可以传播“修正后的有效 claim”，但不是传播 Disproven 作为行动知识。
```

中文：知识传播结果。

```cpp

    std::vector<KnowledgeStateChangeDraft> state_changes;
    std::vector<std::string> reason_keys;
    std::vector<std::string> warning_keys;

    bool ok() const;
    pathfinder::foundation::Result<void> validateBasic() const;
};
```

规则：

```text
decision 不能 Unknown/TestOnly。
Rejected 不作为 ok 成功结果。
Skipped/Failed 可以没有 created_claims/updated_claims。
CreatedRecipientClaim 必须有 created_claims。
Updated/Reinforced/Weakened/CorrectionDelivered 通常必须有 updated_claims 或 applied_patches。
所有 created_claims/updated_claims/event/state_change 必须 validateBasic。
Result::ok(result) 必须 validateBasic 通过。
```

## 10. 传播评分算法

### 10.1 基础公式

P20 第一版使用确定、可测试的简单公式：

```text
channel_weight = 根据 channel 选择 options 中的权重。
source_confidence = source_claim.confidence.confidence。
trust = context.trust_score。
distance = context.distance_factor。
quality = context.channel_quality。
noise_penalty = 1.0 - context.noise_factor。

transfer_score = clamp(
    source_confidence * channel_weight * trust * distance * quality * noise_penalty,
    0.0,
    1.0
)
```

接收者新知识置信度：

```text
created_confidence = min(transfer_score, options.max_created_confidence)

如果 source_claim.status == Candidate/Hypothesis：
    created_confidence *= 0.75

如果 channel == WarningSignal：
    relation_type 为 HasRisk / IsDangerousUnder 时不惩罚；其他 relation_type *= 0.8

如果 channel == Correction：
    created_confidence = min(transfer_score, options.correction_weight)
```

### 10.2 信任等级到默认分数

如果调用方没有明确传 `trust_score`，可按 `trust_band` 取默认值：

| trust_band | 默认 trust_score |
|---|---:|
| Distrusted | 0.10 |
| Low | 0.30 |
| Medium | 0.55 |
| High | 0.80 |
| Authority | 0.90 |

注意：

```text
Authority 只是高信任，不是真理。
Authority 传播的错误知识仍可被 P19 修正。
```

### 10.3 状态映射

接收者新知识 status 建议：

| created_confidence | status |
|---|---|
| < 0.10 | Candidate |
| 0.10 - 0.34 | Hypothesis |
| 0.35 - 0.69 | Shared |
| >= 0.70 | Active |

规则：

```text
P20 不主动创建 Teachable，除非后续多次传播/亲身经历增强。
P20 不主动创建 Operational。
Shared 表示“来自传播的可用主观知识”。
```

## 11. 传播决策流程

### 11.1 输入校验

校验顺序：

```text
1. options.validateBasic。
2. attempt.validateBasic。
3. propagator.owner 与 source_claim.claim.owner 一致。
4. target_existing_claims 的 owner 与 target.owner 一致。
5. source owner 与 target owner 默认不同。
6. reason/capability/condition keys 不含 hidden truth。
```

### 11.2 筛选可传播 claim

默认可传播：

```text
Active
Teachable
Shared
Operational
```

谨慎传播：

```text
Candidate / Hypothesis 可以传播，但 transfer_score 打折，接收者不能直接得到 Active 高置信知识。
```

默认不可传播：

```text
Deprecated
Disproven
Unknown
TestOnly
```

Correction 例外：

```text
Correction 不传播 Disproven 作为行动知识。
Correction 应传播修正后的有效 claim，或传播“旧知识需要降权”的 patch。
```

### 11.3 匹配接收者已有知识

匹配条件第一版保守处理：

```text
target owner 相同。
subject_id 相同。
predicate.relation_type 相同。
predicate.action_key 相同。
effect_key 相同或安全兼容。
```

如果接收者已有同类知识：

```text
同 effect / 同条件：ReinforcedRecipientClaim。
不同 effect / 条件冲突：WeakenedRecipientClaim 或 route_to_revision。
无匹配：CreatedRecipientClaim。
```

### 11.4 创建接收者新知识

创建规则：

```text
owner = target.owner。
subject = source_claim.subject。
predicate = source_claim.predicate。
conditions = source_claim.conditions。
confidence = 基于 transfer_score 计算。
status = 基于 created_confidence 映射。
evidence_refs 只新增 Teaching evidence。
related_knowledge_ids 包含 source_knowledge_id。
trace_ids 可记录 propagation trace。
reason_keys 包含 received_by_propagation / propagated_from_source。
```

Teaching evidence 必须包含：

```text
evidence_kind = KnowledgeEvidenceKind::Teaching。
supports_claim = true。
reliability = transfer_score。
confidence_delta = created_confidence。
source_summary_id 可以为空。
source_memory_ids 默认不复制。
memory_evidence_refs 默认不复制。
```

### 11.5 更新接收者已有知识

如果接收者已有知识，P20 可以生成 patch：

```text
支持：追加 Teaching evidence，提高 confidence。
冲突：追加 Teaching evidence，但 supports_claim=false 或 route_to_revision。
纠错：优先 route_to_revision，让 P19 处理 weakening/specialization/deprecation。
```

第一版建议：

```text
简单同 effect 支持：P20 直接生成 patch。
不同 effect 或条件冲突：P20 生成 should_route_to_revision=true，由调用方或 P21 编排调用 P19。
```

不要在 P20 内部复杂重写 P19 逻辑。

## 12. 事件与状态变更草稿

P20 只生成事件草稿，不直接写 EventStore。

建议 event_key：

| 事件 key | 触发 |
|---|---|
| knowledge.propagation_attempted | 发生传播尝试 |
| knowledge.propagation_failed | 传播失败 |
| knowledge.claim_transferred | 接收者获得新知识 |
| knowledge.claim_teached | 发送者成功教出知识 |
| knowledge.claim_received | 接收者收到知识 |
| knowledge.claim_transfer_reinforced | 传播强化接收者旧知识 |
| knowledge.claim_transfer_conflicted | 传播与接收者旧知识冲突 |
| knowledge.correction_delivered | 纠错传播完成 |

状态变更建议：

| change_key | 含义 |
|---|---|
| knowledge.state_change.propagation_created | 接收者新增知识 |
| knowledge.state_change.propagation_updated | 接收者知识被传播证据更新 |
| knowledge.state_change.propagation_failed | 传播失败记录 |

规则：

```text
事件草稿必须 validateBasic。
Skipped/Failed 可以不生成 KnowledgeEventDraft，避免空 knowledge_id。
如果生成 failed event，必须有合法 knowledge_id 或单独事件类型支持无 knowledge_id；第一版建议不生成 failed event draft。
```

## 13. Repository / Projection 规则

### 13.1 Repository

P20 不要求 Repository 直接新增写入接口。

调用方流程：

```text
KnowledgePropagationService::propagate -> result.created_claims / updated_claims
调用方显式 repo.put(claim)
```

规则：

```text
PropagationService 不直接持有或写入 KnowledgeRepository。
Repository 只负责保存 validateBasic 通过的 claim。
```

### 13.2 Projection

P20 不要求普通 Projection 直接显示完整传播证据链。

但接收者通过 query/projection 应能看到：

```text
subject / predicate / confidence / status。
reason_keys 可显示 received_by_propagation。
projection_flags 根据 status 设置。
```

普通 Projection 仍不得输出：

```text
Deprecated
Disproven
hidden truth
完整 memory evidence refs
```

## 14. 安全边界

必须禁止：

```text
ObjectDefinition hidden truth。
raw GameState。
SaveGame / SaveManager。
AgentTickRecord raw data。
HTTP / WebSocket / JSON。
RL reward/done。
AgentRuntime / Policy 直接写 Repository。
自由文本 AI 生成 claim。
```

传播只能使用：

```text
已 validateBasic 的 KnowledgeClaim。
安全 KnowledgeProjectionItem。
调用方显式传入的 trust/distance/channel DTO。
```

不能：

```text
传播时重新读取真实对象属性。
根据对象真实 edible_profile 创建知识。
把 source claim 的全部 memory 复制给 target。
让接收者知识直接变成 Established 真理。
```

## 16. 验收标准

P20 完成必须满足：

```text
1. Agent A 的有效 KnowledgeClaim 可以传播给 Agent B。
2. Agent B 获得的是 owner=B 的新 KnowledgeClaim，不是 source claim 原对象。
3. 接收者 claim 证据类型为 Teaching，并记录 source knowledge id。
4. 传播置信度受 trust/channel/distance/noise 影响。
5. 错误或低置信知识可以传播，但接收者置信度不会被抬成真理。
6. Deprecated/Disproven 默认不能传播。
7. 不同 effect 的接收者旧知识不会被 P20 静默覆盖。
8. PropagationService 不直接写 Repository。
9. 所有 ok result validateBasic 通过。
10. 不读取 hidden truth/raw GameState/Save/API/RL。
```

玩家体验验收：

```text
A 知道红果可食用。
A 教给 B。
B 查询知识时能看到“红果可食用，来源是教学，置信度受信任影响”。
```

这就是 P20 最小成立标准。

## 18. AI 实现注意事项

1. 不要把 P20 做成社交系统；P20 只做知识传播。
2. 不要读取真实对象属性；传播来源只能是已有主观知识。
3. 不要把发送者 claim 原样改 owner 后复制；必须重新生成接收者 claim、evidence、id、confidence。
4. 不要复制发送者完整 memory evidence refs 给接收者。
5. 不要让 Authority trust 变成真理来源。
6. 不要默认传播 Deprecated/Disproven。
7. 不要在 PropagationService 里直接写 Repository。
8. 不要在 P20 内部重写 P19 的复杂冲突修正；冲突时应生成 patch 或 route_to_revision。
9. 不要返回 ok + invalid DTO。
10. 不要为了测试方便打开 TestOnly 默认通道。
