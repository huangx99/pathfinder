# P49 世界经验学习风险头脑风暴

更新时间：2026-05-21
阶段：P49
用途：实现前风险扫描

## 1. 风险结论

P49 最大风险不是代码难，而是边界容易写错：

```text
把一次 experience 当成稳定知识。
绕过旧学习系统直接写 KnowledgeClaim。
为了测试写死火把 / 红果。
让前端或 JSON 直接决定运行时知识。
把失败经验误认为正向知识。
泄露隐藏属性。
```

所以 P49 必须以“适配器”身份存在。

## 2. 玩法场景压力测试

### 2.1 红果 / 毒蘑菇 / 腐烂物

场景：

```text
玩家吃普通红果恢复饥饿。
玩家吃腐烂红果中毒。
玩家吃毒蘑菇中毒。
玩家吃过多普通红果也可能不适。
```

风险：

```text
如果 P49 只按 subject_object_key + eat 归并，会把普通红果和腐烂红果混成同一知识。
如果不保留 condition_keys，就无法表达“腐烂状态”“过量窗口”“新鲜状态”。
```

要求：

```text
P49 必须保留 condition_keys。
P49 不能在模板缺条件时自己猜。
```

### 2.2 制作火把

场景：

```text
木头 + 火源 -> 火把。
火源可能是催化物，不消耗。
木头消耗。
火把进入背包。
```

风险：

```text
只记录 torch + craft -> make_torch，会丢失输入材料知识。
只记录 wood + craft -> make_torch，会丢失产物知识。
```

要求：

```text
P49 MVP 至少记录模板声明的知识。
后续支持一个 reaction 多 knowledge template。
```

### 2.3 斧头砍木头与磨斧头

场景：

```text
斧头锋利时砍树成功。
斧头钝了砍树失败。
磨石可恢复锋利。
```

风险：

```text
失败不能变成“斧头不能砍树”。
状态条件必须保留 sharp/dull。
```

要求：

```text
P49 失败经验默认不形成正向知识。
状态不匹配只形成失败记忆或跳过。
```

### 2.4 房屋建造链

场景：

```text
寒冷 -> 需要房屋。
房屋需要木头。
木头需要砍树。
砍树需要斧头锋利。
斧头钝了需要打磨。
```

风险：

```text
P49 如果只生成文本，P51 无法做链式推理。
```

要求：

```text
知识必须保留 action_key、effect_key、subject、target、conditions。
```

### 2.5 NPC 观察学习

场景：

```text
NPC 看玩家制作火把。
NPC 没亲手做，但有低置信观察知识。
```

风险：

```text
观察和亲身体验混同，NPC 过早确信。
```

要求：

```text
source_kind 必须区分 DirectExperience / Observation / Taught。
P49 MVP 只做 DirectExperience，但枚举必须预留。
```

### 2.6 野兽学习火危险

场景：

```text
狼接近火把被驱退。
狼后续本能避开火。
```

风险：

```text
把野兽写死成怕火，而不是它的 Agent 经验。
```

要求：

```text
P49 输出必须支持 Agent owner。
P52 再接野兽 experience source。
```

### 2.7 高级文明科技

场景：

```text
高级文明发现合金配方、能量护盾、星门。
```

风险：

```text
如果 P49 只能处理 eat/use/craft，就不够。
```

要求：

```text
experience_kind 可扩展。
action_key/effect_key/template 泛化，不依赖动作枚举写死。
```

### 2.8 魔法世界

场景：

```text
火焰符文 + 魔力水晶 -> 火球术。
区域魔法改变地形。
```

风险：

```text
知识主体可能不是物品，而是区域、法术、环境。
```

要求：

```text
P49 MVP 支持 object/action/effect。
后续通过 KnowledgeSubjectKind 扩展 area/spell/environment。
```

### 2.9 机械危机

场景：

```text
机器人被电磁脉冲瘫痪。
玩家学会 EMP 对机械有效。
```

风险：

```text
目标是敌人类型，不是物品。
```

要求：

```text
target_object_key 可为空。
后续支持 target_actor_key / target_agent_species。
```

### 2.10 群体知识

场景：

```text
多个成员都学会某反应，族群形成公共知识。
```

风险：

```text
P49 直接写全族知识，会跳过传播和社会学习。
```

要求：

```text
P49 只写 actor/agent 私有知识。
族群知识由 P50 传播和后续 tribe aggregation 形成。
```

## 3. 工程风险

### 3.1 旧学习系统接口过重

风险：

```text
LearningLoopInput 需要 cognition/memory/knowledge 多个字段，P49 构造错误会导致测试脆弱。
```

缓解：

```text
新增 WorldSafeFeedbackBuilder 单测。
先让 builder 输出可验证 DTO，再调用 service。
```

### 3.2 模板解析不稳定

风险：

```text
unordered map 遍历导致不同机器模板选择不同。
```

缓解：

```text
候选模板 key 排序。
reason_keys 优先。
fallback 结果排序。
```

### 3.3 缺模板被当失败阻断 command

风险：

```text
命令成功，但学习失败导致玩家操作失败。
```

缓解：

```text
P49 是 post-command。
TemplateMissing 默认 warning/skipped，不回滚世界状态。
```

### 3.4 写死内容

风险：

```text
为了测火把，代码里写 torch/wood。
```

缓解：

```text
测试可以使用这些 key。
生产代码禁止出现具体内容 key。
审核用 rg 检查 world_learning 目录。
```

### 3.5 伪造前端知识

风险：

```text
前端传 parameters[knowledge_key] 让角色学会。
```

缓解：

```text
P49 不读取前端传入的知识结论。
只读取 command_result.experiences 和 ContentRegistry。
```

## 4. 实现策略建议

P49 最小实现建议：

```text
第一步：WorldLearningTypes + enum/string/validation。
第二步：ExperienceExtractor 从 command result 提取 experiences。
第三步：TemplateResolver 解析 knowledge templates。
第四步：SafeFeedbackBuilder 构造 LearningLoopInput。
第五步：WorldKnowledgeLearningService 调 LearningLoopService。
第六步：把 learned claims 放入 KnowledgeRepository。
第七步：ProjectionBridge 输出 knowledge deltas/events。
第八步：集成 P48 craft story 测试。
```

## 5. 审核重点

最终审核必须检查：

```text
world_learning 目录没有具体内容 key 写死。
P49 不依赖 H5。
P49 不直接改 WorldRuntime。
P49 成功和失败经验区分。
P49 无模板不产生假知识。
P49 形成的知识可 teachable / usable_by_ai。
P49 测试覆盖 P47/P48 经验。
```
