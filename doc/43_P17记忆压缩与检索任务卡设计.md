# P17 记忆压缩与检索任务卡设计

## 1. 阶段定位

P17 是《先驱者》认知、记忆、知识、传播学习主线的第三个核心玩法阶段。

P15 解决：

```text
主体当前相信什么。
认知来自什么安全反馈。
认知的置信度、风险、效用、可传授程度是多少。
```

P16 解决：

```text
一次经历是否会被记住。
记忆是短期、中期还是长期。
重复经历如何强化。
普通记忆如何衰减、淡化、遗忘。
高风险、关键、冲突、可教学记忆如何被保护。
```

P17 解决：

```text
大量同类记忆如何形成安全摘要。
Agent 或后续系统如何按条件检索相关记忆。
长期记忆如何被整理为可供 P18 知识系统读取的证据候选。
如何在不丢失原始记忆、不读取隐藏真值的前提下减少上下文压力。
```

P17 不解决：

```text
记忆变成正式知识。
知识置信度与知识冲突修正。
知识传播给族群。
族群共享记忆。
H5 记忆界面。
存档持久化。
强化学习 reward/done。
```

这些留给后续阶段：

- P18：知识系统。
- P19：知识冲突与修正。
- P20：传播系统。
- P21：学习闭环。

一句话：

```text
P17 让“我记住了很多次经历”变成“我能安全、稳定、可解释地找回相关经历和摘要”。
```

---

## 2. 本阶段目标

P17 的目标是建立一个可扩展、可审计、对 AI 友好的记忆整理与检索层。

它必须做到：

1. 从 P16 `MemoryRecord` 生成 `MemorySummary`。
2. `MemorySummary` 必须保留来源 `MemoryId`，不能丢失追溯链。
3. 支持按 owner、subject、memory kind、scope、importance、teaching、contradiction、risk 方向做基础召回。
4. 支持确定性排序，不能依赖随机或未定义容器顺序。
5. 支持返回 `MemoryRecallResult`，其中可以同时包含原始记忆和摘要。
6. 支持压缩计划 `MemoryCompressionPlan`，但 P17 第一版不删除原始 `MemoryRecord`。
7. 支持输出压缩事件草稿和状态变更草稿，但不直接改 GameState。
8. 保持 hidden truth 防线，不读取 ObjectDefinition hidden truth，不读取 GameState raw state。
9. 给 P18 知识系统留下稳定的“证据候选”接口，但不创建 `KnowledgeClaim`。

核心原则：

```text
P17 的压缩是安全投影，不是销毁原始记忆。
P17 的检索是记忆召回，不是知识判断。
P17 的摘要是证据候选，不是事实真理。
```

## 3. 玩家体验映射

P17 对玩家体验的意义：

```text
玩家多次吃红果恢复需求：系统不必每次都翻所有原始记忆，而能召回“红果多次食用成功”的摘要。
玩家曾经吃蓝果中毒：危险摘要能被优先召回，避免重复犯错。
玩家遇到相反结果：冲突记忆能被召回，后续 P18/P19 可判断是否形成不稳定知识。
玩家准备传授经验：可教学记忆能被检索出来，供 P20 传播使用。
```

这一步不是 UI，也不是 AI 策略本身，而是核心玩法链路的检索基础：

```text
探索 → 反馈 → 认知 → 记忆 → 压缩/召回 → 知识 → 传播 → 文明
```

P17 只做到“压缩/召回”。

## 5. 本阶段不做什么

P17 明确禁止：

```text
不做 KnowledgeClaim。
不做 KnowledgeRepository。
不做知识置信度归并。
不做知识冲突修正。
不做知识传播。
不做族群共享记忆。
不做 H5 接口。
不做 HTTP / WebSocket / JSON。
不做 SaveManager。
不做强化学习 reward/done。
不让 AgentRuntime 或 Policy 直接写 MemorySummaryStore。
不删除 P16 原始 MemoryRecord。
不读取 ObjectDefinition hidden truth。
不读取 GameState 原始隐藏状态生成摘要。
```

P17 可以定义“给 P18 使用的证据候选字段”，但不能实现知识系统核心逻辑。

## 9. 枚举设计

所有枚举必须实现：

```cpp
std::string toString(Enum value);
Result<Enum> enumFromString(const std::string& value);
```

未知字符串必须 `Result::fail`，不能静默返回 `Unknown`。

### 9.1 MemorySummaryKind

中文：记忆摘要类型。

```cpp
enum class MemorySummaryKind {
    Unknown,
    OwnerSubjectPattern,
    RiskPattern,
    TeachingPattern,
    ContradictionPattern,
    LongTermPattern,
    TestOnly
};
```

| 枚举 | 中文 | 含义 |
|---|---|---|
| Unknown | 未知 | 非法或未初始化 |
| OwnerSubjectPattern | 拥有者对象模式 | 同一 owner + subject 的经验摘要 |
| RiskPattern | 风险模式 | 高风险、事故、警告类摘要 |
| TeachingPattern | 教学模式 | 可传授记忆摘要 |
| ContradictionPattern | 冲突模式 | 包含冲突证据的摘要 |
| LongTermPattern | 长期模式 | 长期稳定记忆摘要 |
| TestOnly | 测试专用 | 仅测试使用 |

### 9.2 MemoryCompressionLevel

中文：记忆压缩等级。

```cpp
enum class MemoryCompressionLevel {
    Unknown,
    None,
    LightSummary,
    StatisticalSummary,
    ArchivedProjection,
    TestOnly
};
```

| 枚举 | 中文 | 含义 |
|---|---|---|
| None | 不压缩 | 原始记忆 |
| LightSummary | 轻摘要 | 保留代表样本和关键计数 |
| StatisticalSummary | 统计摘要 | 记录成功/失败/风险/冲突数量 |
| ArchivedProjection | 归档投影 | 只读归档摘要；P17 不删除原始记忆 |
| TestOnly | 测试专用 | 仅测试使用 |

P17 第一版建议只真正实现 `LightSummary` 和 `StatisticalSummary`。

```cpp
enum class MemoryCompressionDecision {
```

中文：记忆召回模式。

## 10. DTO 设计

### 10.1 MemorySummaryKey

中文：记忆摘要键。

```cpp
struct MemorySummaryKey {
    MemoryOwner owner;
    MemorySubject subject;
    MemorySummaryKind summary_kind = MemorySummaryKind::Unknown;
    std::vector<MemoryKind> memory_kinds;

    bool operator==(const MemorySummaryKey& other) const = default;
```

规则：


### 10.2 MemorySummary

中文：记忆摘要。

```cpp
struct MemorySummary {
    MemorySummaryId summary_id;
    std::string schema_version = "memory_summary.v1";
    MemorySummaryKey key;
    MemoryCompressionLevel compression_level = MemoryCompressionLevel::Unknown;
    MemoryImportance highest_importance = MemoryImportance::Unknown;
    MemoryStrengthBand aggregate_strength_band = MemoryStrengthBand::Unknown;
    double aggregate_strength = 0.0;
    size_t source_memory_count = 0;
    size_t success_count = 0;
    size_t failure_count = 0;
    size_t warning_count = 0;
    size_t accident_count = 0;
    size_t contradiction_count = 0;
    size_t teaching_candidate_count = 0;
    bool has_protected_source = false;
    bool has_long_term_source = false;
    pathfinder::foundation::Tick first_observed_tick;
    pathfinder::foundation::Tick last_observed_tick;
    pathfinder::foundation::Tick summarized_tick;
    std::vector<pathfinder::foundation::MemoryId> source_memory_ids;
    std::vector<pathfinder::foundation::MemoryId> representative_memory_ids;
    std::vector<MemoryEvidenceRef> evidence_refs;
    std::vector<std::string> reason_keys;
    std::vector<std::string> warning_keys;

    Result<void> validateBasic() const;
};
```

关键约束：

```text
summary_id 非空。
schema_version 必须为 memory_summary.v1。
source_memory_ids 必须非空。
source_memory_count 必须等于 source_memory_ids.size()。
representative_memory_ids 必须非空，且必须来自 source_memory_ids。
evidence_refs 必须非空，保留来源追踪。
aggregate_strength 必须在 [0,1]。
compression_level 不能 Unknown。
highest_importance 不能 Unknown。
不能包含 hidden truth key。
```

## 12. 压缩规则

### 12.1 可压缩记录

默认可压缩：

```text
scope == MidTerm 或 LongTerm。
lifecycle == Active 或 Protected。
strength >= 0.20。
evidence_refs 非空。
owner + subject 与 summary key 匹配。
```

默认不可压缩：

```text
ShortTerm。
Forgotten。
Unknown scope / lifecycle。
TestOnly，除非 allow_test_only=true。
含 hidden truth 字段。
```

### 12.2 分组规则

P17 第一版只实现最小稳定分组：

```text
owner + subject + summary_kind
```

不要做复杂语义聚类，不做向量召回，不做自然语言 embedding。

### 12.3 代表样本选择

代表样本排序：

```text
protect_from_forgetting=true 优先
importance 高优先
strength 高优先
last_touched_tick 新优先
memory_id 升序兜底
```

最多保留 `max_representative_records` 个代表样本。

### 12.4 聚合强度

建议公式：

```text
aggregate_strength = clamp(
  max(strength) * 0.50 + average(strength) * 0.30 + repetition_bonus * 0.20,
  0.0,
  1.0
)
```

其中：

```text
repetition_bonus = min(source_memory_count / 10.0, 1.0)
```

P17 只要求确定性，不追求复杂数值平衡。

### 12.5 统计字段

按 `MemoryRecord.memory_kinds` 统计：

```text
Success -> success_count
Failure -> failure_count
Warning -> warning_count
Accident -> accident_count
Contradiction -> contradiction_count
teaching_candidate=true 或 TeachingRelated -> teaching_candidate_count
```

## 13. 召回规则

### 13.1 记录过滤

`MemoryRecallService` 必须支持：

```text
owner 必选。
subject 可选。
mode 必选。
memory_kinds 可选。
scope 可选。
min_importance 可选。
teaching_only 可选。
contradiction_only 可选。
include_forgotten 默认 false。
```

### 13.2 mode 行为

| mode | 过滤逻辑 |
|---|---|
| ExactSubject | 必须 subject 匹配 |
| OwnerRecent | owner 匹配，按 last_touched_tick 排序 |
| ByMemoryKind | 至少一个 memory kind 匹配 |
| ImportantOrCritical | importance >= Important |
| TeachingCandidates | teaching_candidate 或 TeachingRelated |
| RiskRelated | Warning / Accident / Critical |
| ContradictionRelated | Contradiction 或 contradiction_count > 0 |
| LongTermOnly | scope == LongTerm 或 summary.has_long_term_source |

### 13.3 relevance_score

P17 第一版使用确定性简化评分：

```text
score = 0
+ strength_score * 0.35
+ importance_score * 0.25
+ recency_score * 0.15
+ mode_match_score * 0.20
+ teaching_or_risk_bonus * 0.05
```

要求：

```text
最终 clamp 到 [0,1]。
不能使用随机数。
不能依赖系统时间。
只能使用 Tick、强度、枚举、计数等安全字段。
```

### 13.4 排序

排序必须稳定：

```text
主排序按 query.sort。
次排序 relevance_score desc。
再按 last_touched_tick 或 summarized_tick desc。
最后 ID asc。
```

## 23. 最终说明

P17 不是为了“把记忆变成知识”，而是为了让后续知识系统拿到稳定、可追溯、低上下文压力的记忆证据。

正确完成 P17 后，P18 才能安全地回答：

```text
这些多次出现的记忆，是否足以形成一条知识？
这条知识是否稳定？
是否存在冲突证据？
能不能传授给别人？
```
