# P46 世界生成器与资源分布任务卡设计

## 1. 阶段定位

P46 是 V2.0 开放世界网格路线中的“世界生成器与资源分布”阶段。

它接在以下阶段之后：

```text
P42：JSON 配置内容目录与加载管线。
P43：统一世界 Command 管线前置。
P44：世界网格基础与探索投影。
P45：背包、容器、掉落物与物品归属系统。
```

它为以下阶段打基础：

```text
P47：资源采集、砍伐、挖掘。
P48：制作与世界反应接入。
P49：探索行为产生知识。
P50：教学与无知 NPC 基础行动。
P51：Agent 网格目标执行。
P52：野兽网格生态与遭遇。
P53：H5 V2 地图界面。
P54：存档、复盘与调试门禁。
```

P46 的本质不是“随机摆几个物品”，而是建立一个可复盘、可配置、可扩展的世界生成权威层。

P46 完成后，世界里的地形、资源节点、初始对象、危险分布、出生区，不再由 C++ 临时写死，而是由 seed + JSON 内容配置 + 生成规则稳定产生。

## 2. 为什么现在做

当前已经有：

```text
P43：世界行为统一走 WorldCommand。
P44：二维世界网格、坐标、layer_key、探索投影。
P45：物品归属、背包、地面 stack、容器和掉落物基础。
```

如果没有 P46，后续 P47 会遇到根本问题：

```text
采集什么？
在哪里采集？
树、石头、浆果、矿脉、野兽从哪里来？
玩家出生点附近如何保证有最小生存闭环？
同一 seed 是否能复盘同一个世界？
资源枯竭后地图状态如何保存？
```

所以 P46 必须先于 P47。

P46 要解决的是“世界上有什么，以及它们为什么在那里”。

P47 才解决“玩家或 NPC 怎么采集这些东西”。

## 3. 本阶段一句话目标

P46 要建立一个可配置、可复盘、可测试的世界生成器，把 JSON worldgen/resource 配置转换成 V2 WorldGrid 上的地形、资源节点、初始实体和危险分布，为采集、制作、探索、NPC、生物生态提供真实地图内容。

## 4. 玩家体验目标

P46 本身不是玩家直接看到的玩法按钮，但它决定玩家进入地图后的第一分钟是否成立。

P46 完成后，玩家体验应该是：

```text
我醒来后不是站在空白地图上。
周围有可理解的地形差异：草地、树林、石地、水边等。
附近有基础生存资源：红果、木头、石片、干草、火种等。
远一点有更高价值或更危险的资源：矿石、兽迹、洞穴、毒蘑菇等。
危险不是凭空出现，而是有生态分布和区域规则。
同一个 seed 进入世界，资源分布可以稳定复现。
```

P46 不是要做“完整大世界”，而是要让后续玩法有稳定落点。

## 5. 本阶段不解决的问题

P46 不做以下内容：

| 不做内容 | 原因 | 后续阶段 |
|---|---|---|
| 采集、砍伐、挖掘动作结算 | P46 只生成资源，不消费资源 | P47 |
| 制作、合成、反应链 | P46 只生成原料和初始对象，不处理配方 | P48 |
| 探索产生知识 | P46 只提供可探索对象，不写入知识 | P49 |
| NPC 自主行动 | P46 可生成 NPC 出生点，但不让 NPC 决策 | P50/P51 |
| 野兽移动、攻击、学习 | P46 可生成野兽巢穴/危险区，不实现生态行为 | P52 |
| H5 地图界面 | P46 只提供后端世界内容，前端不直接读生成配置 | P53 |
| 长期存档格式完整兼容 | P46 要预留 generation manifest，不做完整存档 | P54 |
| 无限大世界流式加载 | P46 先做区域生成接口和确定性，后续扩展流式 | P54 之后 |

## 6. 核心设计原则

### 6.1 seed + 配置决定世界，不靠临时代码摆放

世界生成必须由：

```text
world_seed
worldgen_profile_key
biome/resource JSON 配置
确定性随机算法
```

共同决定。

禁止实现者在 C++ 中写死：

```text
如果 x==1 y==0 就生成红果。
如果 seed==1 就放木头。
固定出生点旁边永远摆同样对象。
```

可以有 bootstrap safety rule，但必须写成配置或明确的 generator policy。

### 6.2 生成的是世界状态，不是玩法结果

P46 只生成：

```text
地形。
资源节点。
初始实体。
危险标记。
出生点。
区域标签。
```

P46 不生成：

```text
玩家已经采集到的物品。
玩家已经学会的知识。
NPC 已经完成的行动。
制作结果。
```

这些必须由后续 Command 和规则系统产生。

### 6.3 生成必须可复盘

同一组输入：

```text
world_seed
generator_version
content_version
worldgen_profile_key
region_coord
```

必须生成同一结果。

后续存档和复盘只要记录 generation manifest，就能判断世界是否可重建。

### 6.4 WorldGrid 是运行时权威，JSON 是内容模板

JSON 不直接等于运行时状态。

JSON 定义：

```text
有哪些地形。
哪些资源可以分布在哪些地形。
密度、权重、最小距离、稀有度。
```

WorldGenerator 读取 JSON，生成 WorldGridRuntime 中的 cell/entity/resource node。

运行后状态变更以 WorldGrid/Inventory 为权威。

### 6.5 P45 物品归属接口是所有物品落地的门禁

P46/P47 生成或采集物品时，必须通过 P45 的归属接口建立 `OnMap` 或 `InInventory` 状态，不能绕过 runtime 直接改数组。

这条是硬性架构门禁，不是建议。

具体要求：

```text
P46 生成地面物品：必须形成 OnMap 归属。
P46 生成资源节点：节点本身属于 WorldGrid，不等于玩家背包物品。
P47 采集产出物品：必须通过 P45 transfer / spawn-to-map / spawn-to-inventory 这类归属接口。
P47 采集失败：不能先改背包再回滚。
后续制作/掉落/拾取：也必须继续沿用 P45 归属接口。
```

禁止实现：

```text
直接 push_back 到 inventory entries。
直接把 world entity 的 location_kind 改成 InInventory。
直接改 WorldGrid entity 数组后不登记 item_locations_。
同一个 entity_id 同时出现在地图和背包。
为了通过测试在生成器里给玩家塞初始物品。
```

如果 P46 需要生成“初始掉落物”，它只能生成地面物品；如果后续需要生成“初始背包物品”，必须由专门 command 经过 P45 归属接口完成，并在 manifest/trace 里记录来源。

### 6.6 前端不能读取 worldgen JSON

P46 不能让 H5 直接读 JSON 来画地图或推断对象。

前端只能读取：

```text
WorldProjectionDto
WorldProjectionPatchDto
WorldEventDto
available_commands
```

P46 对前端的影响，必须通过 P44/P43 投影和 Command response 表达。

### 6.7 资源节点和普通物品要分清

P46 需要区分：

```text
地面物品：可以拾取，例如石片、掉落红果。
资源节点：需要后续采集命令，例如浆果丛、树、矿脉。
生态/危险源：后续可能生成野兽或区域事件，例如狼穴、蜘蛛巢、毒雾区。
```

不能把所有东西都生成成可直接 pickup 的普通物品。

### 6.8 出生区必须支持最小闭环

P46 不是完全随机。

出生区必须能配置最小保证：

```text
一定范围内有可食用探索对象。
一定范围内有木/石/草这类基础材料。
一定范围内有低强度危险征兆，但不立刻秒杀玩家。
```

这不是写死结果，而是 generator safety policy。

### 6.9 所有生成对象必须有 layer_key

V2 是二维地图 + 虚拟 Z 层。

任何生成结果必须携带：

```text
x
y
layer_key
```

禁止只用 x/y。

后续地下、空中、水面、建筑内部都依赖 layer_key。

## 7. P46 要做什么

P46 要做 8 件事。

### 7.1 定义 WorldGenerator 模块

新增世界生成模块，负责从配置和 seed 生成区域。

建议路径：

```text
backend/include/pathfinder/world_generation/
backend/src/world_generation/
backend/tests/unit/world_generation/
backend/tests/integration/world_generation/
```

建议 target：

```text
pathfinder_world_generation
pathfinder_tests_world_generation
pathfinder_tests_world_generation_integration
```

### 7.2 定义生成输入 DTO

生成输入必须包含：

```cpp
struct WorldGenerationRequest {
    std::string world_id;
    uint64_t world_seed;
    std::string generator_version;
    std::string content_version;
    std::string worldgen_profile_key;
    WorldRegionCoord region_coord;
    int region_size = 16;
    std::vector<std::string> enabled_layer_keys;
};
```

中文说明：

```text
world_seed：世界种子，同 seed 同配置必须稳定。
generator_version：生成器版本，未来算法变化时用于存档兼容。
content_version：内容包版本，资源配置变化会影响生成。
worldgen_profile_key：生成方案，例如 first_world、forest_start、dry_plains。
region_coord：要生成哪个区域。
region_size：区域大小。
enabled_layer_keys：本次生成哪些虚拟层，例如 surface、underground、air。
```

### 7.3 定义生成输出 DTO

生成输出必须包含：

```cpp
struct WorldGenerationResult {
    bool ok = false;
    WorldGenerationFailureKind failure_kind = WorldGenerationFailureKind::None;
    WorldGenerationManifest manifest;
    std::vector<GeneratedCellDraft> cell_drafts;
    std::vector<GeneratedEntityDraft> entity_drafts;
    std::vector<GeneratedResourceNodeDraft> resource_node_drafts;
    std::vector<GeneratedSpawnPointDraft> spawn_point_drafts;
    std::vector<GeneratedDangerZoneDraft> danger_zone_drafts;
    std::vector<world_command::WorldEventDto> events;
};
```

注意：

```text
P46 输出 draft，不直接让前端使用。
真正落地到 WorldGridRuntime 需要通过 applier 或 command handler。
```

### 7.4 定义地形生成

P46 至少支持基础地形：

```text
plain：平地。
forest：树林。
stone_field：石地。
water_edge：水边。
blocked：不可通行阻挡。
```

地形生成要支持：

```text
权重。
噪声阈值。
邻近平滑。
出生点保护。
区域标签。
```

P46 不需要做复杂真实生态，但要预留 biome。

### 7.5 定义资源节点生成

资源节点不是普通物品。

建议新增：

```cpp
struct ResourceNodeRuntime {
    std::string node_id;
    std::string resource_key;
    world_runtime::WorldCellCoord coord;
    ResourceNodeKind node_kind;
    ResourceNodeState state;
    int remaining_charges = 1;
    int max_charges = 1;
    std::string required_action_key;
    std::string required_tool_key;
    std::vector<std::string> output_entity_keys;
    std::vector<std::string> tag_keys;
};
```

中文说明：

```text
node_id：资源节点实例 id。
resource_key：资源配置 key，例如 berry_bush、tree_pine、stone_outcrop。
coord：节点所在格子。
node_kind：节点类型，例如植物、树、矿、土、危险源。
state：可采、枯竭、隐藏、封锁等。
remaining_charges：还能采几次。
required_action_key：需要什么动作，例如 gather/chop/mine/dig。
required_tool_key：需要什么工具，例如 axe/pickaxe，也可以为空。
output_entity_keys：后续 P47 采集产出哪些物品。
```

P46 只生成节点，不执行采集。

### 7.6 定义初始对象生成

P46 也可以生成少量可直接拾取的地面物品。

例如：

```text
stone_flake：石片。
dry_grass：干草。
fallen_branch：掉落树枝。
red_berry_loose：掉落红果。
```

这些必须走 P45 地面 item/entity 归属模型。

要求：

```text
生成的地面物品必须是 OnMap。
必须有 coord。
必须有 quantity/stack_key/stackable/state_keys。
不能默认进入玩家背包或营地库存。
```

### 7.7 定义出生区保障

P46 必须有 `SpawnSafetyPolicy`。

建议：

```cpp
struct SpawnSafetyPolicy {
    int safe_radius = 2;
    int basic_food_min_count = 1;
    int basic_material_min_count = 2;
    int tool_hint_min_count = 1;
    int immediate_threat_max_count = 0;
    std::vector<std::string> guaranteed_resource_keys;
    std::vector<std::string> forbidden_danger_keys;
};
```

这不是写死红果或木头，而是配置化保障。

例如 first_world profile 可以配置：

```text
出生点半径 2 内至少有：
1 个低风险食物资源。
1 个木质材料来源。
1 个石质材料来源。
0 个主动攻击危险。
```

### 7.8 定义生成接入方式

P46 不允许外部直接调用一堆函数乱改 WorldGrid。

推荐接入方式：

```text
WorldCommandKind::GenerateWorld 或 GenerateRegion
    -> WorldCommandPipeline
    -> GenerateWorldCommandHandler
    -> WorldGenerationService
    -> WorldGenerationApplier
    -> WorldGridRuntime / InventoryRuntime / ResourceNodeRuntime
    -> ProjectionPatch / Event / Trace
```

如果 P44 已有 `generateInitialWorld`，P46 要重构它：

```text
P44 generateInitialWorld 只保留最小启动能力。
P46 后生成逻辑迁移到 world_generation。
WorldGridRuntime 不再自己写复杂资源生成规则。
```

## 8. 核心枚举

### 8.1 WorldGenerationFailureKind

```cpp
enum class WorldGenerationFailureKind {
    None,
    InvalidRequest,
    ContentMissing,
    ProfileNotFound,
    RegionAlreadyGenerated,
    InvalidRegionCoord,
    InvalidLayer,
    DeterminismMismatch,
    ApplyFailed,
    RuntimeConflict
};
```

中文说明：

| 枚举 | 中文 | 使用场景 |
|---|---|---|
| None | 无错误 | 成功 |
| InvalidRequest | 请求无效 | 缺少 world_id、region_size 非法等 |
| ContentMissing | 内容缺失 | JSON 内容包未加载或缺少资源定义 |
| ProfileNotFound | 生成方案不存在 | worldgen_profile_key 无效 |
| RegionAlreadyGenerated | 区域已生成 | 重复生成同一区域且不允许覆盖 |
| InvalidRegionCoord | 区域坐标非法 | region coord 不合法 |
| InvalidLayer | 层非法 | layer_key 不在允许列表 |
| DeterminismMismatch | 确定性不一致 | 测试同 seed 生成不同结果 |
| ApplyFailed | 落地失败 | draft 应用到 runtime 失败 |
| RuntimeConflict | 运行时冲突 | cell/entity id 冲突等 |

## 9. 核心模块划分

| 模块 | 中文含义 | 职责 | 输入 | 输出 | 禁止事项 |
|---|---|---|---|---|---|
| WorldGenerationService | 世界生成服务 | 调度地形、资源、实体、出生点生成 | GenerationRequest、content registry | GenerationResult | 不直接绕过 applier 改 runtime |
| TerrainGenerator | 地形生成器 | 生成 cell terrain、movement、biome tags | seed、region、profile | cell drafts | 不生成采集结果 |
| ResourceDistributionGenerator | 资源分布生成器 | 根据 biome 和资源配置放置资源节点 | cell drafts、resource config | resource node drafts | 不把资源直接塞进玩家背包 |
| EntityPlacementGenerator | 实体摆放生成器 | 生成可拾取物、初始对象、标记物 | entity config、terrain | entity drafts | 不写死红果/石片坐标 |
| SpawnSafetyPlanner | 出生区保障器 | 确保出生区最小可玩资源 | spawn policy、drafts | 修正后的 drafts | 不制造已学会知识 |
| WorldGenerationApplier | 生成落地器 | 把 drafts 应用到 WorldGrid/Inventory/ResourceNode runtime | GenerationResult | runtime state changes | 不跳过校验 |
| WorldGenerationTrace | 生成追踪 | 记录 seed、profile、roll、结果 | generation steps | manifest/trace | 不存运行时秘密给前端 |
| WorldGenerationProjectionBridge | 投影桥接 | 输出 changed cells/entities/events | applied changes | ProjectionPatch | 不让前端读 JSON |

## 10. P46 与现有系统关系

### 10.1 与 P42 JSON 的关系

P42 提供内容目录与加载管线。

P46 应读取或预留读取：

```text
worldgen profile。
terrain definitions。
biome definitions。
resource node definitions。
entity spawn definitions。
danger zone definitions。
spawn safety policy。
```

如果当前 JSON schema 还不完整，P46 可以先实现 C++ DTO 和测试内配置，但必须保留 ContentRegistry 接口，不允许把最终内容写死在生成器内部。

### 10.2 与 P43 Command 的关系

P46 的生成动作必须可以被 Command 触发和追踪。

最小支持：

```text
GenerateWorld：生成初始世界。
GenerateRegion：生成指定区域，后续流式地图使用。
```

P46 不要求前端手动点“生成世界”，但后端 bootstrap 也应走同一套生成服务。

### 10.3 与 P44 WorldGrid 的关系

P46 生成的 cell draft 最终落到 WorldGridRuntime。

P46 不应该让 WorldGridRuntime 自己决定复杂资源分布。

WorldGridRuntime 负责保存运行时状态。
WorldGenerator 负责决定初始生成内容。

### 10.4 与 P45 Inventory 的关系

P46 生成地面物品时必须符合 P45：

```text
location_kind = OnMap。
coord 必须存在。
quantity/stack_key/stackable/state_keys 必须明确。
不能同时进入 inventory。
不能默认归玩家、同伴、营地。
```

P45 当前仍有 partial pickup 归属问题未验收，因此 P46 设计可以推进，但实现时要注意：

```text
不要依赖 P45 未修好的 partial pickup 行为。
P46 只生成 OnMap 地面对象和资源节点。
不要在 P46 中补丁式绕过 P45。
```

### 10.5 与 P47 采集的关系

P46 生成 ResourceNodeRuntime。

P47 消费 ResourceNodeRuntime。

边界：

```text
P46：树在哪里、有几次可砍、需要什么 action/tool、可能产出什么。
P47：玩家/NPC 提交 chop command 后，检查工具、消耗 charge、生成木头进入背包或地面。
```

## 11. 最小生成闭环

P46 最小闭环只要求生成一个可玩出生区。

建议 first_world profile：

```text
玩家出生在 surface:0:0。
半径 2 内：
- 至少一个 food 类型资源节点，例如 berry_bush。
- 至少一个 wood 类型资源节点，例如 fallen_branch_node 或 young_tree。
- 至少一个 stone 类型资源节点，例如 loose_stone_node。
- 至少一个 hint 对象，例如 strange_tracks 或 cold_wind_marker。
半径 3-5 内：
- 可出现低强度危险源，例如 beast_tracks。
- 可出现更高价值资源，例如 ore_chip 或 cave_entrance。
```

注意：

```text
这里的 red_berry、wood、stone 不能写死在 C++。
它们应该来自 profile 的 guaranteed_resource_tags，例如 food_basic、wood_basic、stone_basic。
```

## 12. 生成配置草案

P46 可以先定义 JSON 目标结构，不一定一次接完整内容包。

示例：

```json
{
  "worldgen_profiles": [
    {
      "key": "first_world",
      "region_size": 16,
```

## 13. Command 接口设计

### 13.1 GenerateWorld Command

输入参数：

```text
world_id
world_seed
worldgen_profile_key
region_radius
layer_keys
```

行为：

```text
创建或重建初始区域。
生成出生区。
应用地形、资源节点、初始实体。
返回 changed_cells、changed_entities、events、trace。
```

失败：

```text
profile_not_found。
content_missing。
region_already_generated。
apply_failed。
```

### 13.2 GenerateRegion Command

输入参数：

```text
world_id
region_x
region_y
layer_key
worldgen_profile_key
```

行为：

```text
生成指定区域。
如果已经生成，默认返回 blocked。
后续可支持 force_regenerate，但 P46 不建议开放。
```

## 15. 验收标准

P46 通过条件：

```text
1. 存在独立 world_generation 模块。
2. 同 seed + 同 profile + 同 region 生成结果稳定。
3. 生成结果全部携带 layer_key。
4. 出生区满足最小资源保障。
5. 资源节点与地面物品区分清楚。
6. 地面物品符合 P45 OnMap 归属规则。
7. GenerateWorld / GenerateRegion 至少一个命令路径接入 WorldCommandPipeline。
8. 投影返回 changed_cells / changed_entities / events。
9. 生成 trace/manifest 能记录 seed、profile、content version、generator version。
10. P46 不实现采集、制作、知识、NPC 行动。
```

## 16. 风险与门禁

### 16.1 风险：把资源节点当普通物品

错误实现：

```text
树直接变成背包里的 wood。
浆果丛直接变成 red_berry item。
矿脉直接变成 stone item。
```

正确实现：

```text
树是 ResourceNode。
浆果丛是 ResourceNode。
矿脉是 ResourceNode。
采集动作在 P47 才产出物品。
```

### 16.2 风险：出生区保障写死

错误实现：

```cpp
if (x == 0 && y == 1) spawn red_berry;
```

正确实现：

```text
SpawnSafetyPolicy 根据 guaranteed_resource_tags 选择候选资源。
```

### 16.3 风险：破坏 P45 归属

错误实现：

```text
生成时直接给玩家背包塞 10 个木头。
生成时让同一个 entity 同时 OnMap 和 InInventory。
```

正确实现：

```text
P46 只生成 OnMap 地面物品或 ResourceNode。
玩家获得物品必须通过 P47/P48/Pickup Command。
```

### 16.4 风险：不可复盘随机

错误实现：

```cpp
std::random_device
std::time(nullptr)
全局 rand()
```

正确实现：

```text
基于 world_seed + region_coord + generator_version 的 deterministic RNG。
```

## 18. 给研发 AI 的硬性注意事项

```text
1. 不允许把红果、木头、石头固定坐标写死在 C++。
2. 不允许前端读取 worldgen JSON。
3. 不允许 P46 直接产出玩家背包物品。
4. 不允许 P46 实现采集、制作、教学、NPC 行动。
5. 所有生成对象必须有 layer_key。
6. 同 seed 同配置必须生成同结果。
7. 资源节点和地面物品必须分开。
8. 地面物品必须符合 P45 OnMap 归属规则。P46/P47 生成或采集物品时，必须通过 P45 的归属接口建立 `OnMap` 或 `InInventory` 状态，不能绕过 runtime 直接改数组。
9. 生成过程必须能写入 manifest/trace。
10. 如果 P45 partial pickup 仍未最终验收，P46 不能绕过 P45 修背包，只能生成世界内容。
```

## 20. 结论

P46 要做的是：

```text
让世界可以被稳定生成，而不是靠 demo 写死摆物品。
```

它是 P47 采集、P48 制作、P49 认知、P51 Agent、P52 野兽生态的地图内容地基。

P46 不追求大世界规模，而追求：

```text
确定性。
可配置。
可复盘。
归属正确。
资源节点和物品分离。
出生区最小闭环成立。
```
