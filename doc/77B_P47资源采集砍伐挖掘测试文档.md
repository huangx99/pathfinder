# P47 资源采集、砍伐、挖掘测试文档

## 1. 目的

本文档定义 P47 的测试矩阵。

测试重点不是只看 command succeeded，而是验证：

```text
ResourceNode 被正确消费。
产物通过 P45 OnMap 归属落地。
失败路径不污染状态。
Command / projection / events 完整。
```

## 3. P45 归属测试

必须覆盖：

```text
world_harvest_output_uses_p45_on_map_location
world_harvest_output_not_in_actor_inventory
world_harvest_output_cell_contains_entity_id
world_harvest_output_can_be_picked_up_by_p45_pickup
```

关键断言：

```text
runtime.findEntity(output_id).is_ok()
entity.location_kind == OnMap
entity.coord.has_value()
cell.entity_ids contains output_id
inventory_runtime.findInventory(player_inventory) 中没有 output，直到 pickup command 执行
```

## 5. P46 -> P47 端到端测试

```text
world_generation_then_gather_berry_bush
world_generation_then_chop_tree_with_tool
world_generation_then_gather_loose_stone
```

流程：

```text
1. GenerateWorld。
2. 找到对应 ResourceNode。
3. 提交 Gather/Chop/Mine/Dig。
4. 验证 node charge、output OnMap、events、projection。
```

## 6. 禁止空跑测试

以下测试无效：

```text
只检查 result.ok。
只遍历 outputs 但不 assert outputs 非空。
只检查 state_deltas 数量，不检查 runtime 状态。
只创建 fake output，不经过 P45 location port。
只测 service，不测 command handler。
```

