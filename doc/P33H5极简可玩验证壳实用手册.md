# 先驱者 H5 极简可玩验证壳实用手册

版本：P33 当前可玩验证壳  
访问端口：1999  
界面语言：中文  
当前定位：手机可访问的文本探索原型，不是最终正式 UI。

---

## 1. 当前这个界面能做什么

当前 H5 界面已经可以完成一个最小可玩闭环：

```text
看见物体 -> 尝试行动 -> 得到反馈 -> 形成知识 -> 查看知识 -> 教给同伴 -> 同伴获得知识 -> 重开再试
```

它验证的是“先驱者”的核心体验：

- 开局你不知道物体有什么用。
- 你通过吃、使用、观察等尝试获得经验。
- 系统把经验转成知识。
- 你可以把知识传给同伴。
- 普通物体和特殊状态物体会形成不同知识，例如红果和腐烂红果不会互相覆盖。

---

## 2. 怎么进入游戏

电脑上服务已经启动在：

```text
http://192.168.31.24:1999/
```

如果通过内网穿透访问，请让穿透目标指向：

```text
127.0.0.1:1999
```

或：

```text
192.168.31.24:1999
```

如果页面打不开，优先检查：

1. 服务是否还在监听 1999。
2. 手机和电脑是否在同一个局域网。
3. 内网穿透目标端口是否还是 1999。
4. 手机浏览器是否缓存旧页面，必要时刷新或重新打开地址。

## 3. 界面区域说明

### 3.1 顶部区域

顶部显示：

- 游戏名：先驱者。
- 当前投影模式。
- 当前回合数。
- 重开按钮。

回合数会随着你的行动增加，用来确认系统确实处理了你的操作。

### 3.2 场景区域

当前场景是：

```text
林地边缘
```

场景区域会告诉你当前能看见什么、当前处于什么环境。

当前可见物体有：

- 红果
- 腐烂红果
- 苦叶
- 石片

### 3.3 最近反馈

这里显示你刚才做了什么，以及发生了什么。

例如：

- 你吃了红果。
- 身体的需求有所缓解。
- 你吃了腐烂红果。
- 你感到不适，健康状况变差了。
- 你使用了石片。
- 你感觉到这个工具有某种用途。

这个区域是你判断“操作有没有生效”的第一位置。

### 3.4 你能看见的东西

这里以卡片方式显示当前物体。

每张物体卡片包含：

- 物体名称。
- 简短描述。
- 可点击行动按钮。
- 已知状态标签。

当前物体按钮：

| 物体 | 可用按钮 | 作用 |
|---|---|---|
| 红果 | 吃红果 | 尝试吃普通红果 |
| 腐烂红果 | 吃腐烂红果 | 尝试吃腐烂状态的红果 |
| 苦叶 | 吃苦叶 | 尝试吃苦叶 |
| 石片 | 使用石片 | 尝试使用石片 |

注意：你一开始不是不知道物体形态，而是不知道它有什么用。当前界面会显示物体名，但不会提前告诉你具体效果。

### 3.5 我的知识

这里显示玩家已经形成的知识。

知识展示格式是中文形式：

```text
物体 -> 行动 -> 效果
状态
```

例如：

```text
红果 → 吃 → 可以缓解饥饿
已确认
```

当前可能形成的知识：

| 知识 | 中文含义 |
|---|---|
| 红果 → 吃 → 可以缓解饥饿 | 红果吃后可以缓解饥饿 |
| 腐烂红果 → 吃 → 会让身体不适或中毒 | 腐烂红果吃后有危险 |
| 苦叶 → 吃 → 暂时没有明显效果 | 苦叶吃后当前没有明显效果 |
| 石片 → 使用 → 可能有工具用途 | 石片可以作为工具使用的线索 |

### 3.6 同伴知识

这里显示同伴当前知道什么。

同伴不会自动知道你的经验。你必须点击“教同伴”，同伴才会获得你已经知道的知识。

同伴通过你传授获得的知识，通常会先是：

```text
经验假设
```

这表示：

- 你亲自体验过，所以你更确定。
- 同伴只是听你讲，所以同伴先把它当作假设。

### 3.7 提示与危险

这里显示条件提示、危险提示和系统提示。

例如，当你知道腐烂红果有毒后，这里可能提示：

```text
你已经有经验表明，某些相似食物可能会让身体不适。
再次尝试前，先观察或回忆已有知识。
```

这个区域不是调试信息，而是未来正式 UI 中“危险认知提醒”的雏形。

### 3.8 底部输入与快捷按钮

底部有两种操作方式：

1. 点击快捷按钮。
2. 手动输入中文指令。

当前全局快捷按钮：

| 按钮 | 作用 |
|---|---|
| 观察 | 查看周围情况 |
| 教同伴 | 把你的知识传给同伴 |
| 查看知识 | 查看你的知识和同伴知识 |
| 查看同伴 | 重点查看同伴知道什么 |
| 重开 | 重置当前会话 |

## 4. 推荐试玩流程

### 4.1 基础学习流程

按这个顺序试：

```text
观察
吃红果
查看知识
教同伴
查看同伴
```

你应该看到：

- 你形成“红果可以缓解饥饿”的知识。
- 同伴一开始不知道。
- 点击教同伴后，同伴获得这条知识。

### 4.2 危险知识流程

按这个顺序试：

```text
吃腐烂红果
查看知识
教同伴
查看同伴
```

你应该看到：

- 你形成“腐烂红果会让身体不适或中毒”的知识。
- 危险提示区域出现相关提醒。
- 教同伴后，同伴也知道腐烂红果有危险。

### 4.3 红果与腐烂红果区别验证

按这个顺序试：

```text
吃红果
吃红果
吃红果
吃腐烂红果
吃腐烂红果
查看知识
```

正确结果应该是：

```text
红果 → 吃 → 可以缓解饥饿：已确认
腐烂红果 → 吃 → 会让身体不适或中毒：已确认
```

红果知识不会因为你吃了腐烂红果而变成假设。

### 4.4 工具线索流程

按这个顺序试：

```text
使用石片
查看知识
```

你应该看到：

```text
石片 → 使用 → 可能有工具用途
```

这表示当前版本已经支持“使用物品形成知识”，不只是吃东西。

### 4.5 无明显效果流程

按这个顺序试：

```text
吃苦叶
查看知识
```

你应该看到：

```text
苦叶 → 吃 → 暂时没有明显效果
```

这类知识很重要，因为它代表“尝试后没有明显收益”也会变成认知。

## 5. 当前支持的中文指令

你可以点击按钮，也可以手动输入。

当前建议输入：

```text
观察
吃红果
吃腐烂红果
吃苦叶
使用石片
教同伴
查看知识
查看同伴
帮助
重开
```

部分相近说法也可能被识别，例如：

```text
看看周围
吃一口红果
用石片
我知道什么
同伴知道什么
```

如果系统没有理解，建议先使用按钮。

## 6. 知识状态是什么意思

当前界面会把知识状态显示成中文。

常见状态：

| 状态 | 含义 |
|---|---|
| 候选判断 | 证据很少，只是初步判断 |
| 经验假设 | 有一定经验，但还不够稳定 |
| 已确认 | 当前足够可靠，可以当作有效知识 |
| 可传授 | 可靠度更高，适合传播给别人 |
| 已分享 | 已经被传播过 |
| 旧知识 | 已经被新经验替代，不应继续作为当前判断 |
| 已否定 | 已经被证明不成立 |

当前 H5 页面主要会看到：

- 已确认
- 经验假设
- 候选判断

## 7. 当前版本已经验证的核心规则

### 7.1 前端不写死规则

H5 页面不会自己判断：

- 红果能不能吃。
- 腐烂红果有没有毒。
- 石片有没有用。
- 同伴应该学会什么。

这些都由后端规则系统裁决，前端只负责显示后端给出的安全投影。

### 7.2 普通物体和状态物体分开学习

当前已修正：

```text
红果
腐烂红果
```

会形成两条不同知识，不会互相覆盖。

也就是说：

```text
红果能吃
```

和：

```text
腐烂红果有毒
```

可以同时成立。

### 7.3 同伴不会自动同步知识

你学会了，不代表同伴学会。

必须点击：

```text
教同伴
```

同伴才会获得知识。

### 7.4 教给同伴后，同伴通常先是假设

你是亲身经历者，所以你的知识更可靠。  
同伴是听你讲，所以同伴会先形成经验假设。

## 8. 当前还没有的功能

当前 P33 只是极简可玩验证壳，所以暂时没有：

- 正式地图。
- 角色移动。
- 背包。
- 生命值 UI。
- 饥饿值 UI。
- 物品合成 UI。
- 多 NPC 对话。
- 战斗界面。
- 存档读档界面。
- 复杂任务系统。
- 正式美术。
- 音效和动画。

这些不是 bug，而是当前阶段边界。

## 9. 如果你遇到问题

### 9.1 按钮点不了

先刷新页面。  
如果还不行，重新打开地址。

当前已经修过一次按钮禁用问题，新页面使用了版本号防缓存。

### 9.2 页面打不开

检查服务是否在：

```bash
ss -ltnp '( sport = :1999 )'
```

当前服务应该监听：

```text
0.0.0.0:1999
```

### 9.3 知识看起来不对

建议先点击：

```text
重开
```

然后按推荐流程重新测试。

如果还不对，记录你的操作顺序，例如：

```text
吃红果 -> 吃腐烂红果 -> 吃红果 -> 查看知识
```

这样方便定位是学习、修正、传播还是展示问题。

## 10. 当前最适合测试什么

当前最值得测试的是这 5 件事：

1. 点击物体按钮是否有反馈。
2. 行动后我的知识是否更新。
3. 教同伴后同伴知识是否更新。
4. 红果和腐烂红果是否能同时保留正确知识。
5. 重开后知识是否清空。

如果这 5 件事都稳定，说明 P33 的“最小可玩闭环”成立。

---

## 11. 一句话总结

当前 H5 已经能玩一个最小文本版“先驱者”：你能看见物体，尝试它们，获得知识，把知识教给同伴，并验证知识不会被错误覆盖。下一阶段 P34 应该在这个基础上做端到端 MVP，而不是推翻当前结构。
