# P53 客户端协议测试文档

## 1. 测试目标

P53 测试只验证客户端协议，不验证玩家界面。

本阶段不需要浏览器，不需要页面，不需要 CSS，不需要手机横屏测试。

P53 必须用代码证明：

```text
客户端能 bootstrap。
客户端能拿到安全 projection。
客户端能拿到 available_commands。
客户端能提交 option_id。
客户端能接收 command response。
客户端能应用或拒绝 patch。
客户端版本失配时能触发 full refresh。
客户端不能注入规则、知识或内容。
```

## 2. 单元测试

### 2.1 枚举与 DTO

```text
client_protocol_enum_roundtrip
client_bootstrap_request_validate
client_bootstrap_response_validate
client_command_request_validate
client_command_response_validate
client_refresh_request_validate
client_world_projection_validate
client_selection_context_validate
```

验收：

```text
Unknown 非法。
TestOnly 只允许测试。
session_id / client_id / command_id 规则明确。
OptionId 模式必须有 option_id。
WorldCommandDto 模式必须有 command。
WorldCommand target 坐标必须携带 layer_key。
```

### 2.2 Codec

```text
client_protocol_codec_roundtrip_bootstrap
client_protocol_codec_roundtrip_command
client_protocol_codec_rejects_unknown_enum
client_protocol_codec_rejects_hidden_debug_fields
```

验收：JSON 往返不丢字段，不接受 hidden truth / raw_state 字段。

### 2.3 Projection version

```text
client_patch_accepts_matching_base_version
client_patch_rejects_stale_base_version
client_patch_requires_full_refresh_when_flagged
client_refresh_restores_projection_version
```

验收：版本不匹配不能局部合并。

### 2.4 Available command

```text
client_available_commands_returned_by_backend
client_command_option_id_materializes_world_command
client_rejects_unknown_option_id
client_rejects_expired_option_id
client_does_not_generate_command_from_entity_key
```

验收：客户端测试只能提交后端给出的 option_id。

### 2.5 安全请求

```text
client_request_rejects_knowledge_json
client_request_rejects_recipient_claim_json
client_request_rejects_force_learn
client_request_rejects_content_key_rule_hint
```

验收：客户端不能传知识结论或规则暗示。

## 3. 集成测试

### 3.1 最小协议闭环

```text
client_protocol_bootstrap_command_patch_refresh
```

流程：

```text
1. fake client 调 bootstrap。
2. 后端返回 projection_version=1 和 available_commands。
3. fake client 选择 Wait option。
4. command gateway materialize WorldCommandDto。
5. WorldCommandPipeline 执行。
6. 返回 projection_patch，new_projection_version=2。
7. fake client refresh，拿到 version=2 full projection。
```

### 3.2 失败命令闭环

```text
client_protocol_failed_command_returns_safe_response
```

验收：失败命令也返回 result、event_feed、warning_keys、available_commands，不污染 runtime。

### 3.3 版本失配

```text
client_protocol_stale_projection_requires_refresh
```

验收：known_projection_version 过期时，不执行或执行后要求 full refresh，具体策略必须稳定且可复盘。

### 3.4 option_id 过期

```text
client_protocol_expired_option_is_blocked
```

验收：过期 option_id 不会执行成旧命令。

### 3.5 不依赖 UI

```text
client_protocol_tests_run_without_frontend_files
```

验收：删除或不构建 frontend 目录时，P53 后端协议测试仍可运行。

## 4. 架构扫描

### 4.1 禁止 H5V2 命名

```bash
rg -n "H5V2|h5_v2|h5-v2" backend/include/pathfinder/client_protocol backend/src/client_protocol backend/tests/unit/client_protocol backend/tests/integration/client_protocol -S
```

结果必须无匹配。

### 4.2 禁止内容硬编码

```bash
rg -n "red_berry|berry|torch|fire|wolf|wood|companion|beast_shadow|mushroom" backend/include/pathfinder/client_protocol backend/src/client_protocol -S
```

结果必须无匹配。

### 4.3 禁止读取内容 JSON

```bash
rg -n "content/core|recipes|reaction_rules|knowledge_templates|ContentRegistry" backend/include/pathfinder/client_protocol backend/src/client_protocol -S
```

结果必须无匹配；如果后端确需读取内容，只能在已有内容/规则系统中读取，client_protocol 不能直接读取。

### 4.4 禁止知识注入

```bash
rg -n "KnowledgeClaim|recipient_claim_json|knowledge_json|force_learn|force_teach|learned=true" backend/include/pathfinder/client_protocol backend/src/client_protocol -S
```

结果必须无匹配。

### 4.5 禁止 runtime 直改

```bash
rg -n "\.actors\[|\.entities\[|\.inventories\[|\.resource_nodes\[|createOrUpdateGeneratedCell|upsertGeneratedResourceNode" backend/include/pathfinder/client_protocol backend/src/client_protocol -S
```

结果必须无匹配。

### 4.6 禁止 UI 文件作为 P53 验收

```bash
find frontend -maxdepth 2 -type f | rg "client_protocol|P53" || true
```

P53 不应要求新增 frontend 文件。

## 5. 必跑命令

P53 实现后建议命令：

```bash
cmake --build build/backend --target pathfinder_tests_client_protocol pathfinder_tests_client_protocol_integration -j 2
ctest --test-dir build/backend -R '^(client_protocol_|architecture_client_protocol_)' --output-on-failure
ctest --test-dir build/backend -R '^(world_command_|world_runtime_|world_inventory_|world_generation_|world_harvest_|world_reaction_|world_learning_|world_teaching_|world_agent_execution_|world_beast_ecology_)' --output-on-failure
```

## 6. 验收清单

```text
[ ] 没有 UI / CSS / DOM 实现作为 P53 内容。
[ ] DTO 不绑定 H5。
[ ] fake client 可以 bootstrap。
[ ] fake client 可以提交 option_id。
[ ] option_id 由后端 materialize 成 WorldCommandDto。
[ ] command response 包含 result / patch / events / available_commands。
[ ] projection_version 失配会触发 full refresh。
[ ] 失败 command 不污染状态。
[ ] client_protocol 不读取 JSON 内容包。
[ ] client_protocol 不写死内容 key。
[ ] client_protocol 不传 KnowledgeClaim。
[ ] client_protocol 不直接修改 runtime。
[ ] P43-P52 回归通过。
```
