# P49 世界经验学习测试文档

更新时间：2026-05-21
阶段：P49
用途：指导实现和验收测试

## 1. 测试目标

P49 测试要证明：

```text
世界 Command 产生的 experience 可以安全进入学习闭环。
知识模板来自 ContentRegistry，不是代码写死。
学习结果写入 actor 知识仓库。
失败、缺模板、不安全经验不会生成假知识。
采集和制作两条 V2 主线都能形成可查询知识。
```

## 2. 单元测试

### 2.1 world_learning_enum_roundtrip

验证：

```text
WorldLearningSourceKind
WorldLearningExperienceKind
WorldLearningFailureKind
WorldKnowledgeLearningDecision
```

要求：

```text
toString / fromString 往返成功。
Unknown 合法解析但 validate 不允许作为业务输入。
TestOnly 默认 validate 拒绝，测试 options 可允许。
```

### 2.2 world_learning_extracts_experience_from_command_result

构造：

```text
WorldCommandExecutionResult 内含 2 条 WorldExperienceDto。
```

断言：

```text
Extractor 稳定提取 2 条。
按 experience_id 排序或保持 command result 顺序。
空 experiences 返回 NoExperience/Skipped。
```

### 2.3 world_learning_resolves_template_from_reason_keys

构造：

```text
experience.reason_keys = [reaction_key, knowledge_make_torch]
ContentRegistry 有 knowledge_make_torch。
```

断言：

```text
Resolver 优先命中 reason_keys。
不会 fallback 到其他模板。
```

### 2.4 world_learning_resolves_template_by_action_effect_fallback

构造：

```text
reason_keys 无 template。
experience.subject_entity_key = berry_bush
action_key = gather
effect_key = gather_red_berry
ContentRegistry 中有匹配模板。
```

断言：

```text
Resolver 能通过 subject/action/effect 匹配。
多个候选按 key 稳定排序。
```

### 2.5 world_learning_skips_missing_template_without_world_mutation

构造：

```text
experience 有 effect，但 registry 没有模板。
```

断言：

```text
result.ok = true 或 partial ok。
decision = SkippedNoTemplate。
learned_claims 为空。
knowledge repository size 不变。
世界 runtime 不变。
```

### 2.6 world_learning_rejects_unsafe_experience

构造：

```text
experience.reason_keys 或 condition_keys 含 forbidden truth/debug/raw key。
```

断言：

```text
result 包含 UnsafeExperience。
不调用 LearningLoopService。
不写 KnowledgeRepository。
```

### 2.7 world_learning_builds_safe_feedback

构造：

```text
experience + knowledge template + effect learning。
```

断言：

```text
LearningSafeFeedbackInput.validateBasic() 通过。
action_key/effect_key/condition_keys/source_event_ids 完整。
不包含隐藏字段。
```

### 2.8 world_learning_direct_experience_forms_claim

构造：

```text
DirectExperience。
模板 teachable=true usable_by_ai=true。
```

断言：

```text
LearningLoopService 返回 knowledge_formation_result。
WorldKnowledgeLearningService 输出 learned_claims。
KnowledgeRepository 可 query 到 claim。
claim owner 是 actor。
```

### 2.9 world_learning_repeated_experience_reinforces_claim

构造：

```text
同一 actor 重复两次相同 experience。
第二次传入第一次后的 existing claims。
```

断言：

```text
不会生成无界重复垃圾 claim。
结果为 Reinforced 或通过 revision/formation 更新。
confidence/status 不下降。
```

### 2.10 world_learning_no_frontend_knowledge_input

构造：

```text
command.parameters["knowledge_key"] = 某个模板。
但 command_result.experiences 不含该模板。
```

断言：

```text
P49 不读取这个前端参数。
不会形成伪造知识。
```

## 3. 集成测试

### 3.1 world_learning_craft_experience_uses_p48_knowledge_template

流程：

```text
建立 ContentRegistry：objects/effects/reactions/knowledge_templates。
给 actor 背包材料。
执行 P48 Craft。
取 command result experiences。
执行 P49 learnFromCommandResult。
```

断言：

```text
P48 产物生成成功。
P49 命中 reaction.knowledge_templates。
KnowledgeRepository 里出现制作知识。
knowledge delta 包含 action=craft/effect=make_torch。
```

### 3.2 world_learning_harvest_experience_uses_p47_output

流程：

```text
生成 resource node。
执行 Gather/Chop。
P47 输出 experience。
P49 学习。
```

断言：

```text
形成资源采集知识。
不需要 H5。
不写死具体节点。
```

### 3.3 world_learning_command_result_merges_learning_events

流程：

```text
command result 有原世界事件。
P49 产生 learning events。
```

断言：

```text
学习事件 event_kind 明确。
source command event 不丢失。
projection_patch 包含 knowledge 变化。
```

### 3.4 world_learning_missing_template_does_not_fail_craft

流程：

```text
P48 Craft 成功。
删除 knowledge template。
P49 学习。
```

断言：

```text
物品仍已生成。
P49 skipped。
不会回滚 P48。
```
