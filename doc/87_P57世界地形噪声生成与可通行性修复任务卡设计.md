# P57 世界地形噪声生成与可通行性修复任务卡设计

## 0. 文档状态

阶段编号：P57。

阶段名称：世界地形噪声生成与可通行性修复。

设计目标：把 P46 的“逐格权重随机地图”升级为“可复盘、可配置、可通行、可扩展、无限区域化”的噪声地形生成系统，解决当前地图大面积障碍、出生点附近移动受阻、地图缺少自然区域感的问题。P57 不能把世界设计成固定大小地图，`region_size` 只能表示区块尺寸，世界本身由无限多个 region 组成。

当前决策：正式客户端 UI 原计划放在 P57，但当前可玩性最大阻塞是地图生成质量太差。没有可通行地图，前端 UI 再好也只能看到“没有可用移动命令”。因此正式 UI 阶段后移，本阶段优先修复世界生成器。

本文档遵守：

```text
doc/00_设计文档编写要求.md
```

前置阶段：

```text
P43 统一 WorldCommand 管线。
P44 世界网格 Runtime 与探索投影。
P45 背包容器与物品归属。
P46 世界生成器与资源分布。
P47 资源采集砍伐挖掘。
P53 客户端协议。
P54 本地 Client HTTP 网关。
P56 客户端世界投影与命令桥接。
```

本阶段不是另起炉灶，而是对 P46 地形生成层做架构级补强。

---

## 1. 问题定义

当前 P46 地形生成的核心问题是：

```text
每个格子独立按权重随机 terrain_key。
```

这会导致：

```text
1. 地图没有连续地貌，森林、水边、石地像随机噪点。
2. 障碍格可能随机挤在出生点附近。
3. 玩家移动时频繁出现“没有可用移动命令”。
4. 前端看起来像移动坏了，实际是地形生成不支持基础探索体验。
5. 后续资源、野兽、NPC、区域事件缺少稳定地貌基础。
```

玩家体验上的表现是：

```text
玩家刚进入世界，想上下左右探索，却发现大量方向不能走。
玩家不知道是输入失败、前端失败，还是地形阻挡。
地图看起来不像自然世界，而像随机障碍数组。
```

这会直接破坏开放世界探索游戏最基础的体验。

---

## 2. 本阶段目标

P57 的目标不是做完整大世界生态，而是建立一套可长期扩展的地形生成骨架。

本阶段必须做到：

```text
1. 地形由确定性噪声生成，而不是逐格独立随机。
2. 同 world_seed + region + generator_version 输出稳定。
3. 不同 seed 输出有明显差异。
4. 出生点附近一定可通行。
5. 区域内可通行比例达标。
6. 出生点到区域内主要方向至少存在基础通路。
7. 世界不能有固定边界，任意 region_coord 包括负坐标和远距离坐标都必须可生成。
8. region_size 只是区块尺寸，不是世界大小，也不能被前端当成地图边界。
9. 地形噪声必须按 world_x/world_y 采样，保证相邻 region 边界连续。
10. 地形结果仍然通过 P46 draft -> applier -> WorldGridRuntime 落地。
8. 前端仍然只消费后端 projection 和 available_commands，不参与地形判断。
9. 资源分布后续能基于 terrain / biome / noise tags 扩展。
```

本阶段完成后，玩家在最小客户端里应该能稳定移动，不再被随机障碍困在出生点。

---

## 3. 本阶段不解决的问题

P57 不做以下内容：

```text
1. 不做最终美术地图表现。
2. 不做正式客户端 UI 重构。
3. 不做完整无限区块流式加载调度器；但本阶段必须把生成器设计成无限地图兼容，不能留下固定世界大小假设。
4. 不做真实水流、温度、风向、季节模拟。
5. 不做复杂地下、天空、洞穴生成。
6. 不做完整生态链。
7. 不做寻路系统。
8. 不做 NPC 长距离迁移。
9. 不做动态地形破坏。
10. 不让前端自己判断哪里能走。
```

这些能力可以在 P58+ 或 V2 后续大世界阶段扩展。

## 4. 设计原则

### 4.1 后端权威

地形是否存在、是否阻挡、移动代价、biome、terrain tag，全部以后端生成结果为准。

前端只能显示投影和提交后端给出的 option_id。

禁止：

```text
前端根据颜色、terrain_key、entity_key 自己推导能不能走。
前端自己生成 Move option。
前端自己绕过 available_commands 移动玩家。
```

### 4.2 确定性优先

任何生成结果必须由以下输入唯一决定：

```text
world_seed
region_coord
layer_key
generator_version
worldgen_profile_key
content_version
```

禁止使用：

```text
当前时间。
全局随机数。
std::random_device 直接参与地形结果。
容器遍历不稳定顺序。
```

### 4.3 可通行优先于自然感

地图自然很重要，但原型阶段更重要的是可玩。

如果噪声生成结果导致出生点被困，本阶段必须修复地形，而不是保留“真实随机结果”。

### 4.4 噪声是基础，不是唯一规则

柏林噪声用于生成连续地貌，但生成后必须经过规则约束：

```text
出生安全区。
可通行比例。
连通性。
主通路。
障碍上限。
资源最低保障。
```

### 4.5 配置优先

地形阈值、噪声尺度、障碍比例、出生安全半径都必须能进入 profile 配置结构。

本阶段可以先用内置 profile，但结构必须允许后续 JSON 化。


### 4.6 无限地图优先

本游戏目标是开放世界无限地图，不是固定尺寸关卡。

P57 必须明确：

```text
world_id：代表一个世界存档。
region_coord：代表一个可独立生成、独立落地、独立缓存的区块坐标。
region_size：只代表单个区块边长，例如 16x16，不代表世界宽高。
world_x/world_y：无限二维世界坐标，可以为负数，也可以非常大。
layer_key：虚拟 Z 层，例如 surface / underground / sky，必须参与生成 seed。
```

禁止出现：

```text
world_width。
world_height。
max_world_x / max_world_y。
固定地图数组作为权威世界。
移动到 region 边缘就等于世界边界。
前端根据当前 projection 大小判断世界边界。
```

允许本阶段暂时只加载一个或少数 region 做测试，但生成器、seed、坐标、manifest、测试都必须按无限 region 设计。

### 4.7 区块边界连续

无限地图最容易出问题的是区块缝合。

P57 噪声必须用全局世界坐标采样：

```text
sample(world_seed, world_x, world_y, layer_key, channel_salt)
```

不能只用 region 内局部坐标：

```text
sample(local_x, local_y)
```

否则相邻 region 的边界会断裂，森林、水域、山脉都会出现明显切缝。

P57 至少要保证：

```text
region (0,0) 右边缘和 region (1,0) 左边缘来自同一套连续噪声场。
region (0,0) 下边缘和 region (0,1) 上边缘来自同一套连续噪声场。
负坐标 region 也稳定，例如 region (-1,0)、region (0,-1)。
```

## 5. 头脑风暴：后续复杂场景会遇到什么

本阶段设计必须覆盖未来复杂地图需求，至少不能把路堵死。

### 5.1 石器时代出生地

玩家出生在草地边缘，附近有浆果、树、石头、草。

要求：

```text
出生点附近安全。
四周有可探索空间。
资源不要全部贴脸，也不要远到找不到。
```

### 5.2 森林与野兽

森林区域可以有野兽、树木、阴影、危险脚印。

要求：

```text
森林应该成片，而不是随机散点。
森林不应全部阻挡移动。
森林移动代价可以更高。
```

### 5.3 河流与水边

水域应该连续，水边可以长草、出现鱼、泥土、芦苇。

要求：

```text
水域可以阻挡。
水边不一定阻挡。
河流不能把出生点完全切断。
```

### 5.4 山脉与矿石

高地和石地可以生成矿石、洞穴入口、危险巢穴。

要求：

```text
山脉可以局部阻挡。
石地可通行但移动代价高。
矿石资源跟石地噪声相关。
```

### 5.5 魔法污染区

未来魔法世界可能有腐化地、法阵区、禁魔区。

要求：

```text
area_effect 可以基于地形 tag 扩展。
污染不一定阻挡移动，但可以影响状态。
噪声层应支持特殊区域 mask。
```

### 5.6 机械危机区

机械文明遗迹、金属废墟、巡逻机器人区域。

要求：

```text
废墟应该形成区域。
机器巡逻依赖可通行路径。
不能生成机器人无法离开的封闭区域，除非明确是设计目的。
```

### 5.7 高级文明城市

后期城市有道路、建筑、门、广场、地下层。

要求：

```text
本阶段噪声不直接生成城市。
但地形 pipeline 要能被后续结构化生成器替换或叠加。
```

### 5.8 地下洞穴

虚拟 Z 层的地下洞穴需要通路、矿脉、危险巢穴。

要求：

```text
layer_key 必须参与 seed。
不同 layer 的同 x/y 可以有不同地形。
连通性检查必须按 layer 执行。
```

### 5.9 空中单位

天空层可能有云、浮岛、飞行生物。

要求：

```text
P57 不做天空移动。
但 profile 结构不能假设只有 surface。
```

### 5.10 玩家建造后改变地图

玩家砍树、挖矿、建房会改变地形/对象分布。

要求：

```text
生成器只负责初始世界。
运行时改动不能通过重新生成覆盖。
同 region 已生成后不能重复 apply 破坏玩家建设成果。
```

## 6. 总体方案

P57 新增“噪声地形生成 + 可通行性修复”流水线。

建议流水线：

```text
WorldGenerationRequest
-> WorldgenProfile
-> RegionSeed
-> NoiseFieldBuilder
-> TerrainClassifier
-> TerrainDraftBuilder
-> SpawnSafetyTerrainPass
-> ConnectivityRepairPass
-> TerrainGenerationDiagnostics
-> ResourceDistributionGenerator
-> SpawnSafetyPlanner
-> WorldGenerationApplier
-> WorldGridRuntime
```

其中 P57 主要替换/增强：

```text
TerrainGenerator
WorldgenProfile
WorldGenerationManifest diagnostics
测试与审计
```

不改变：

```text
WorldGenerationService::generate 主入口。
WorldGenerationResult draft 输出结构。
WorldGenerationApplier 落地边界。
Client projection / command 协议。
WorldCommandPipeline。
```

## 7. 新增与调整的数据结构

### 7.1 TerrainGenerationMode

新增枚举：

```cpp
enum class TerrainGenerationMode {
    Unknown,
    WeightedRandom,
    NoiseField,
    TestOnly
};
```

中文说明：

```text
Unknown：未知，不允许作为有效配置运行。
WeightedRandom：旧的逐格权重随机模式，只保留测试或兼容。
NoiseField：P57 默认模式，使用噪声场生成地形。
TestOnly：测试专用。
```

注意：

```text
P57 后 first_world profile 必须默认使用 NoiseField。
WeightedRandom 只能用于兼容测试，不应作为正式默认。
```

### 7.2 NoiseChannelKind

新增枚举：

```cpp
enum class NoiseChannelKind {
    Unknown,
    Elevation,
    Moisture,
    Temperature,
    Roughness,
    ResourceRichness,
    DangerPressure,
    TestOnly
};
```

中文说明：

```text
Elevation：高度，用于平地、水域、山地判断。
Moisture：湿度，用于草地、森林、水边判断。
Temperature：温度，P57 可先预留。
Roughness：粗糙度，用于石地、山地、难走地形。
ResourceRichness：资源丰度，后续资源密度可读取。
DangerPressure：危险压力，后续野兽巢穴、污染区可读取。
```

### 7.3 NoiseAlgorithmKind

新增枚举：

```cpp
enum class NoiseAlgorithmKind {
    Unknown,
    ValueNoise2D,
    Perlin2D,
    FractalPerlin2D,
    TestOnly
};
```

中文说明：

```text
ValueNoise2D：值噪声，实现简单，适合作为保底。
Perlin2D：柏林噪声，连续自然。
FractalPerlin2D：多 octave 柏林噪声，P57 推荐默认。
```

实现建议：

```text
如果研发担心 Perlin 实现时间，可以先实现 deterministic value noise + fractal octave。
但接口命名必须保留 Perlin/FractalPerlin 扩展位。
```

### 7.4 NoiseChannelConfig

新增结构：

```cpp
struct NoiseChannelConfig {
    NoiseChannelKind channel = NoiseChannelKind::Unknown;
    NoiseAlgorithmKind algorithm = NoiseAlgorithmKind::FractalPerlin2D;
    double scale = 24.0;
    int octaves = 3;
    double persistence = 0.5;
    double lacunarity = 2.0;
    double bias = 0.0;
    double weight = 1.0;
    uint64_t salt = 0;
};
```

中文说明：

```text
channel：噪声通道类型。
algorithm：使用什么噪声算法。
scale：噪声尺度，越大区域越平滑。
octaves：分形层数。
persistence：每层振幅衰减。
lacunarity：每层频率增长。
bias：结果偏移。
weight：组合权重。
salt：通道独立盐值，避免高度和湿度完全同步。
```

### 7.5 TerrainThresholdRule

新增结构：

```cpp
struct TerrainThresholdRule {
    std::string terrain_key;
    double min_elevation = -1.0;
    double max_elevation = 1.0;
    double min_moisture = -1.0;
    double max_moisture = 1.0;
    double min_roughness = -1.0;
    double max_roughness = 1.0;
    int priority = 0;
    std::vector<std::string> tag_keys;
};
```

中文说明：

```text
terrain_key：输出地形，例如 plain / forest / stone_field / water_edge / blocked。
min/max_elevation：高度区间。
min/max_moisture：湿度区间。
min/max_roughness：粗糙度区间。
priority：多条规则命中时优先级，越高越优先。
tag_keys：给 cell 添加的标签。
```

注意：

```text
terrain_key 不能写死在前端。
terrain_key 到 blocks_movement / movement_cost / biome 的映射仍在后端。
```

### 7.6 TerrainConnectivityPolicy

新增结构：

```cpp
struct TerrainConnectivityPolicy {
    bool enabled = true;
    int spawn_safe_radius = 3;
    double min_walkable_ratio = 0.72;
    double max_blocked_ratio_in_spawn_radius = 0.10;
    int min_reachable_cells_from_spawn = 80;
    bool carve_cardinal_corridors = true;
    int corridor_half_width = 1;
    std::vector<std::string> repair_preferred_terrain_keys;
};
```

中文说明：

```text
enabled：是否启用连通性修复。
spawn_safe_radius：出生安全半径。
min_walkable_ratio：区域整体最小可通行比例。
max_blocked_ratio_in_spawn_radius：出生点附近最大阻挡比例。
min_reachable_cells_from_spawn：从出生点至少能到达多少格。
carve_cardinal_corridors：是否向上下左右挖基础通路。
corridor_half_width：通路半宽。
repair_preferred_terrain_keys：修复阻挡时优先改成什么地形，例如 plain。
```

### 7.7 TerrainGenerationDiagnostics

新增结构：

```cpp
struct TerrainGenerationDiagnostics {
    int total_cells = 0;
    int walkable_cells = 0;
    int blocked_cells = 0;
    int reachable_from_spawn = 0;
    double walkable_ratio = 0.0;
    double blocked_ratio_in_spawn_radius = 0.0;
    bool spawn_cell_walkable = false;
    bool connectivity_repaired = false;
    int repaired_cells = 0;
    std::vector<std::string> warning_keys;
};
```

中文说明：

```text
total_cells：总格子数。
walkable_cells：可通行格子数。
blocked_cells：阻挡格子数。
reachable_from_spawn：从出生点可达格子数。
walkable_ratio：可通行比例。
blocked_ratio_in_spawn_radius：出生安全区阻挡比例。
spawn_cell_walkable：出生点是否可通行。
connectivity_repaired：是否执行过修复。
repaired_cells：被修复的格子数量。
warning_keys：诊断警告。
```

诊断应写入 manifest trace 或单独字段。

如果不想改 public DTO 太多，本阶段可以先通过：

```text
manifest.trace_roll_keys / warning_keys 风格字段记录摘要。
测试直接读取 TerrainGenerator::Result diagnostics。
```

## 8. WorldgenProfile 调整

`WorldgenProfile` 新增字段：

```cpp
TerrainGenerationMode terrain_generation_mode = TerrainGenerationMode::NoiseField;
std::vector<NoiseChannelConfig> noise_channels;
std::vector<TerrainThresholdRule> terrain_threshold_rules;
TerrainConnectivityPolicy connectivity_policy;
```

first_world 默认建议：

```text
terrain_generation_mode = NoiseField
region_size = 16  # 区块尺寸，不是世界大小
spawn_safe_radius = 3
min_walkable_ratio = 0.75
max_blocked_ratio_in_spawn_radius = 0.05
min_reachable_cells_from_spawn = 120
repair_preferred_terrain_keys = ["plain"]
```

first_world 噪声通道建议：

```text
Elevation: FractalPerlin2D, scale 32, octaves 3, salt 101
Moisture: FractalPerlin2D, scale 24, octaves 3, salt 202
Roughness: FractalPerlin2D, scale 18, octaves 2, salt 303
ResourceRichness: FractalPerlin2D, scale 14, octaves 2, salt 404
DangerPressure: FractalPerlin2D, scale 28, octaves 2, salt 505
```

first_world 地形规则建议：

```text
water_edge：elevation < -0.55，blocked=false，movement_cost=2
plain：默认地形，blocked=false，movement_cost=1
forest：moisture > 0.25 且 elevation 在正常区间，blocked=false，movement_cost=2
stone_field：roughness > 0.45 或 elevation > 0.45，blocked=false，movement_cost=2
blocked：elevation > 0.78 且 roughness > 0.45，blocked=true，movement_cost=99
```

注意：

```text
森林不能默认阻挡。
石地不能默认阻挡。
水边不应默认阻挡。
真正阻挡只能是 blocked / mountain / deep_water 等明确阻挡地形。
```

## 9. 新增服务与类

### 9.1 TerrainNoiseSampler

职责：给定 world 坐标和 channel config，返回稳定噪声值。

建议接口：

```cpp
class TerrainNoiseSampler {
public:
    explicit TerrainNoiseSampler(uint64_t world_seed);

    double sample(
        int world_x,
        int world_y,
        const std::string& layer_key,
        const NoiseChannelConfig& config) const;
};
```

要求：

```text
输出范围建议归一化到 [-1, 1]。
同输入必须稳定。
不同 channel salt 必须产生不同图案。
layer_key 必须参与 hash。
不能依赖外部随机状态。
```

### 9.2 TerrainNoiseField

职责：保存一个 region 的多通道噪声采样结果。

建议结构：

```cpp
struct TerrainNoiseSample {
    int x = 0;
    int y = 0;
    std::string layer_key;
    double elevation = 0.0;
    double moisture = 0.0;
    double roughness = 0.0;
    double resource_richness = 0.0;
    double danger_pressure = 0.0;
};

struct TerrainNoiseField {
    std::vector<TerrainNoiseSample> samples;
};
```

### 9.3 TerrainNoiseFieldBuilder

职责：根据 request/profile 构建 region 噪声场。

建议接口：

```cpp
class TerrainNoiseFieldBuilder {
public:
    static TerrainNoiseField build(
        const WorldGenerationRequest& request,
        const WorldgenProfile& profile);
};
```

### 9.4 TerrainClassifier

职责：把 `TerrainNoiseSample` 转成 terrain_key / tags / biome。

建议接口：

```cpp
class TerrainClassifier {
public:
    static GeneratedCellDraft classify(
        const WorldGenerationRequest& request,
        const WorldgenProfile& profile,
        const TerrainNoiseSample& sample,
        const std::string& region_id);
};
```

规则：

```text
1. 收集所有命中的 TerrainThresholdRule。
2. 选择 priority 最高的规则。
3. 没有命中时使用 plain。
4. 根据 terrain_key 填 blocks_movement / movement_cost / biome_kind。
5. 把 rule.tag_keys 写入 cell.tag_keys。
```

### 9.5 TerrainConnectivityAnalyzer

职责：分析生成后的地形是否能走。

建议接口：

```cpp
class TerrainConnectivityAnalyzer {
public:
    static TerrainGenerationDiagnostics analyze(
        const std::vector<GeneratedCellDraft>& cells,
        const world_runtime::WorldCellCoord& spawn_coord,
        const TerrainConnectivityPolicy& policy);
};
```

算法：

```text
1. 建立 coord -> cell 索引。
2. 找 spawn_coord。
3. BFS / flood fill 四方向扩展。
4. 不进入 blocks_movement=true 的格子。
5. 统计 reachable_from_spawn、walkable_ratio、spawn_radius blocked ratio。
```

### 9.6 TerrainConnectivityRepairer

职责：对不合格地形进行最小修复。

建议接口：

```cpp
class TerrainConnectivityRepairer {
public:
    static TerrainGenerationDiagnostics repair(
        std::vector<GeneratedCellDraft>& cells,
        const world_runtime::WorldCellCoord& spawn_coord,
        const TerrainConnectivityPolicy& policy);
};
```

修复顺序：

```text
1. 强制 spawn cell 为 plain / 可通行。
2. 强制 spawn_safe_radius 内大部分格子可通行。
3. 如果启用 carve_cardinal_corridors，向上下左右挖基础通路。
4. 重新 BFS。
5. 如果 reachable 仍不足，从 spawn 向最近不可达可通行区域挖窄通道。
6. 如果 blocked ratio 仍过高，按离 spawn 近到远把部分 blocked 改为 plain。
```

注意：

```text
修复必须 deterministic。
修复不能随机选格。
修复优先级必须稳定，例如按距离、x、y 排序。
修复只能发生在生成阶段，不能覆盖已生成 region 的运行时改动。
```

## 11. 地形阻挡规则

P57 必须修正一个重要认知：

```text
资源丰富不等于阻挡。
森林不等于阻挡。
石地不等于阻挡。
水边不等于阻挡。
```

建议阻挡规则：

```text
plain：blocks_movement=false, movement_cost=1
forest：blocks_movement=false, movement_cost=2
stone_field：blocks_movement=false, movement_cost=2
water_edge：blocks_movement=false, movement_cost=2
blocked：blocks_movement=true, movement_cost=99
mountain：blocks_movement=true, movement_cost=99
deep_water：blocks_movement=true, movement_cost=99
```

如果未来要做“树挡路”，应通过资源节点或 entity 的 blocks_movement 单独处理，不能把整个 forest terrain 都设为不可走。

## 13. 与移动命令的关系

移动命令仍然由后端生成。

P57 改善的是：

```text

玩家按方向键后，前端更少看到“没有可用移动命令”。
```

但 P57 不允许：

```text

为了让移动成功，Move handler 忽略 blocks_movement。
为了让测试通过，把所有格子强制改成 plain。
```

正确做法是：

```text
生成阶段保证地图本身有合理可通行空间。
移动阶段继续严格执行 runtime cell blocks_movement。
```
