# 先驱者：知识系统设计

## 1. 设计目标

知识系统负责把认知状态、记忆证据和交互候选整理成稳定、可引用、可传播、可更新、可反证、可复盘的知识断言。

知识系统回答以下问题：

```text
这条认知是否足够稳定？
哪些记忆和证据支持它？
有没有冲突证据？
它在什么条件下成立？
它属于个人、族群、区域还是文明？
它是否可行动？
它是否可传授？
它是否已经被更精确的知识替代？
AI 是否可以基于它做决策？
```

核心链路：

```text
KnowledgePromotionHint
+ KnowledgeEvidenceBundle
+ KnowledgeCandidate
  ↓
KnowledgeResolver
  ↓
KnowledgeClaim
  ↓
KnowledgeStore / KnowledgeIndex / KnowledgeProjection
  ↓
PropagationSystem / TribeState / CivilizationCapability / AI Observation
```

本系统必须保证：

- 知识不是单次反馈直接生成。
- 知识不是简单标签，例如 `edible=true`。
- 知识必须表达对象、行为、效果、条件、证据、置信度、范围和状态。
- 知识可以错误、争议、废弃、证伪和被更精确知识替代。
- 知识必须保留证据来源，能追溯到记忆、认知和事件。
- 知识不能偷看世界真实规则，只能由证据和配置策略形成。
- 知识可以为传播、族群行为、文明能力和 AI 决策提供稳定输入。

## 2. 不解决的问题

| 问题 | 原因 | 归属文档 |
|---|---|---|
| 执行传播 | 知识系统只判断可传授，不执行传播 | `14_传播系统设计.md` |
| 改变族群行为 | 知识可作为输入，但不直接修改族群状态 | `15_族群状态设计.md` |
| 解锁文明能力 | 知识可作为前置条件，但不直接解锁 | `16_文明能力设计.md` |
| 修改世界对象真实状态 | 知识是主观/群体认知，不是物理状态 | 世界对象 / 规则管线 |
| 创建原始记忆 | 记忆由记忆系统负责 | `12_记忆系统设计.md` |
| 直接读取真实特性 | 知识只能来自证据，不能偷看 TraitDefinition 真值 | `07_特性系统设计.md` |
| 前端创建知识 | 前端只能展示事件和投影，不能生成 KnowledgeClaim | `20_前后端协议设计.md` |
| 黑盒 AI 自动编知识 | 第一版只做结构化知识形成，不做自由生成 | 后续 AI 内容工具 |

关键边界：

```text
CognitionEntry：主体当前相信什么。
MemoryRecord / MemorySummary：主体经历过什么。
KnowledgeEvidenceBundle：哪些记忆可以作为证据。
KnowledgeClaim：系统认可的一条可引用知识断言。
PropagationResult：这条知识是否被传播成功。
CivilizationCapability：知识长期稳定后可能解锁的能力。
```

## 3. 核心概念

| 概念 | 中文名 | 含义 |
|---|---|---|
| KnowledgeClaim | 知识断言 | 一条结构化知识，例如“新鲜红浆果可食用” |
| KnowledgeOwner | 知识拥有者 | 谁拥有这条知识，例如先驱者、族群、区域记录 |
| KnowledgeSubject | 知识主体 | 知识描述的对象、区域、特性、组合或行为 |
| KnowledgePredicate | 知识谓词 | 知识表达的关系，例如可食用、有毒、可燃、可转化 |
| KnowledgeCondition | 知识条件 | 知识成立的上下文条件 |
| KnowledgeEffect | 知识效果 | 行为导致的结果摘要和效果引用 |
| KnowledgeEvidence | 知识证据 | 支撑或反对知识的证据引用 |
| KnowledgeConfidence | 知识置信度 | 综合证据后的置信信息 |
| KnowledgeResolver | 知识解析器 | 创建、更新、反证、废弃 KnowledgeClaim |
| KnowledgeContradictionResolver | 知识冲突解析器 | 处理冲突证据和条件化修正 |
| KnowledgeGeneralization | 知识泛化 | 从实例知识推广到类型、区域或普遍规律 |
| KnowledgeSpecialization | 知识特化 | 将过宽知识拆成更具体条件知识 |
| KnowledgeStore | 知识存储 | 保存 KnowledgeClaim |
| KnowledgeIndex | 知识索引 | 按主体、行为、效果、条件、状态查询 |
| KnowledgeProjection | 知识投影 | 给前端、AI、传播系统的主观知识视图 |
| KnowledgeTrace | 知识解释链 | 说明知识为什么创建、更新、反证或废弃 |

### 3.1 知识的基本表达

知识必须优先表达为：

```text
主体 + 行为 + 效果 + 条件 + 证据 + 置信度 + 范围 + 状态
```

示例：

```text
red_berry + Eat + restore_hunger + state:fresh + confidence:0.86 + scope:Personal + status:Teachable
red_berry + Eat + poison + state:decayed + confidence:0.74 + scope:Personal + status:Active
fire + UseWith(dry_branch) + transform_to_torch + condition:dry + confidence:0.82
fire + Near(wolf) + reduce_threat + condition:burning/night + confidence:0.77
```

### 3.2 知识不是标签

错误：

```text
red_berry.edible = true
```

正确：

```text
subject = red_berry
action = Eat
effect = restore_hunger
condition = state:fresh
risk = low
confidence = 0.86
status = Teachable
```

原因：

```text
标签无法表达条件、反例、证据、范围和修正。
```

### 3.3 知识必须条件化

当出现反例时，优先做条件化修正，而不是粗暴推翻所有旧知识。

示例：

```text
旧知识：红浆果可食用。
反例：腐烂红浆果导致中毒。
修正：新鲜红浆果可食用；腐烂红浆果有中毒风险。
```

## 4. 枚举设计

### 4.2 KnowledgeRelationType（知识关系类型）

`KnowledgeRelationType` 是知识系统模块内枚举，用于描述知识谓词表达的关系。

| 枚举 | 外部字符串 | 中文名 | 含义 |
|---|---|---|---|
| HasEffect | has_effect | 具有结果 | 某行为会产生某效果 |
| HasRisk | has_risk | 具有风险 | 某行为或对象存在风险 |
| IsUsableFor | is_usable_for | 可用于 | 对象可用于某用途 |
| IsEdibleUnder | is_edible_under | 条件可食用 | 在条件下可食用 |
| IsDangerousUnder | is_dangerous_under | 条件危险 | 在条件下危险 |
| ReactsWith | reacts_with | 可反应 | 两个或多个对象会发生反应 |
| TransformsInto | transforms_into | 转化为 | 行为导致对象转化 |
| Produces | produces | 产出 | 行为产出资源或对象 |
| Prevents | prevents | 防止 | 对象或行为降低某风险 |
| Requires | requires | 需要条件 | 行为或效果需要某条件 |
| FailsUnder | fails_under | 条件失败 | 在某条件下无效或失败 |
| LocatedAt | located_at | 位于 | 资源、危险或对象位置知识 |
| Indicates | indicates | 指示 | 符号、迹象或环境线索指示某事 |

### 4.3 KnowledgeConfidenceBand（知识置信区间）

`KnowledgeConfidenceBand` 是模块内枚举，用于展示和策略判断。

| 枚举 | 外部字符串 | 中文名 | 建议范围 |
|---|---|---|---|
| Unknown | unknown | 未知 | 0.00 - 0.09 |
| Weak | weak | 较弱 | 0.10 - 0.34 |
| Plausible | plausible | 可信假设 | 0.35 - 0.54 |
| Reliable | reliable | 较可靠 | 0.55 - 0.74 |
| Strong | strong | 强可信 | 0.75 - 0.89 |
| Established | established | 已确立 | 0.90 - 1.00 |

注意：

```text
KnowledgeConfidenceBand 不替代 Ratio confidence。
它是展示、传播、AI 决策阈值的辅助分层。
```

## 5. 数据结构

### 5.1 KnowledgeOwner（知识拥有者）

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| owner_type | CognitionOwnerType | 拥有者类型 | Pioneer、Tribe、TribeMember 等 |
| owner_id | string | 拥有者 ID | EntityId、TribeId 等字符串形式 |
| scope | KnowledgeScope | 知识范围 | Personal、Tribe、Region 等 |

### 5.2 KnowledgeSubject（知识主体）

`KnowledgeSubject` 描述知识讲的是谁。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| subject_kind | string | 主体类型 | object_definition / object_instance / trait / region / action / combination |
| subject_id | string | 主体 ID | ObjectDefinitionId、ObjectId、TraitId、RegionId 等 |
| subject_definition_id | ObjectDefinitionId | 对象定义 | 可选，类型级知识优先使用 |
| related_subject_ids | list<string> | 相关主体 | 组合或反应对象 |
| relation_group_key | string | 关系组键 | 多对象知识的稳定键 |

约定：

```text
可传播知识优先使用 ObjectDefinitionId，而不是 ObjectId。
ObjectId 只适合实例知识，例如“这棵树已被采集”。
组合知识必须通过 related_subject_ids 表达。
```

## 7. 知识生命周期

### 7.1 创建假设知识

```text
1. KnowledgePromotionHint.ready_for_knowledge = true。
2. KnowledgeEvidenceBundle 提供足够证据。
3. KnowledgeResolver 生成 KnowledgeClaim。
4. status = Hypothesis 或 Active。
5. 输出 KnowledgeCreated 事件草稿。
```

### 7.2 从假设到有效

```text
支持证据增加。
冲突证据为 0 或可解释。
confidence 达到 active_min。
status: Hypothesis → Active。
```

### 7.3 从有效到可传授

```text
confidence 达到 teachable_min。
证据链可解释。
条件足够清晰。
风险警告完整。
teaching_profile.teachable = true。
status: Active → Teachable。
```

### 7.4 共享、可行动、已确立

```text
传播系统成功传播后：Teachable → Shared。
族群可稳定使用后：Shared → Operational。
长期使用和文明沉淀后：Operational → Established。
```

注意：

```text
Shared、Operational、Established 通常需要 14、15、16 阶段系统推动。
知识系统只预留状态和候选输出。
```

### 7.5 冲突与修正

```text
1. 新证据进入。
2. KnowledgeContradictionResolver 检测冲突。
3. 判断冲突类型。
4. 若是 MissingCondition，则特化旧知识。
5. 若是真正相反证据，则标记 Disputed 或 Disproven。
6. 若新知识更准确，则旧知识 Deprecated。
7. 输出 KnowledgeContradicted 或 KnowledgeUpdated。
```

### 7.6 废弃与证伪

```text
Deprecated：旧知识被更准确知识替代，但保留历史。
Disproven：旧知识被证据证明错误，不应再推荐。
```

废弃和证伪都不能物理删除 KnowledgeClaim。

### 9.9 配置限制

禁止配置做以下事情：

```text

直接删除旧知识。

```

## 11. 校验规则

### 11.1 KnowledgeClaim 校验

| 规则 | 错误级别 |
|---|---|
| knowledge_id 为空 | RuleError |
| owner 为空 | RuleError |
| subject.subject_id 为空 | RuleError |
| predicate.relation_type 为空 | RuleError |
| predicate.effect_summary 为空 | RuleError |
| conditions 为空且知识不是无条件知识 | RuleError |
| confidence 不在 0 到 1 | RuleError |
| status 与 confidence 阈值严重不符 | RuleError |
| Teachable 但 teaching_profile.teachable=false | RuleError |
| Disproven 但 usable_for_action=true | RuleError |
| Deprecated 但 usable_for_teaching=true | RuleError |

### 11.2 证据校验

| 规则 | 错误级别 |
|---|---|
| KnowledgeClaim 无证据 | RuleError |
| evidence_refs 无 source_memory_ids 且无 source_bundle_id | RuleError |
| conflict_count > 0 但无冲突样本 | RuleError |
| Contradiction 证据被忽略 | RuleError |
| SystemValidation 进入玩家可见知识 | DataError |

### 11.3 冲突校验

| 规则 | 处理 |
|---|---|
| 同条件相反效果 | 标记 Disputed 或 Disproven |
| 条件缺失 | 优先 Specialization |
| 范围不匹配 | 拆分 scope 或 Region 条件 |
| 过度泛化 | 降级或 Deprecated |
| 旧知识被替代 | 保留 superseded_by_knowledge_id |

### 11.4 泛化校验

| 规则 | 错误级别 |
|---|---|
| 样本不足却提升到 Tribe | RuleError |
| 有冲突却提升到 Universal | RuleError |
| 没有区域样本却创建 Region 知识 | RuleError |
| 由 ObjectId 直接泛化且无 ObjectDefinitionId | RuleError |

## 14. 设计注意事项

研发实现时必须遵守：

```text
不要把 KnowledgeClaim 简化成标签。
不要把单次反馈直接变成稳定知识。
不要让知识系统读取隐藏真实规则。
不要忽略条件字段。
不要删除旧知识，必须保留废弃和证伪历史。
不要忽略冲突证据。
不要从 ObjectId 直接泛化成 Universal。
不要让前端创建或修改 KnowledgeClaim。
不要让 AI 使用未掌握的知识。
```

### 14.1 条件优先原则

冲突出现时优先检查条件：

```text
新鲜 / 腐烂。
干燥 / 潮湿。
白天 / 夜晚。
晴天 / 雨天。
燃烧 / 熄灭。
区域 A / 区域 B。
```

### 14.2 证据可追溯原则

每条知识必须能追溯：

```text
MemoryId
MemorySummary
KnowledgeEvidenceBundle
EventId
CognitionKey
```

### 14.3 可复盘原则

同一输入必须得到同一输出：

```text
同一个 KnowledgeStore
+ 同一组 KnowledgePromotionHint
+ 同一组 KnowledgeEvidenceBundle
+ 同一组 KnowledgeCandidate
+ 同一个 KnowledgePolicy
= 同一个 KnowledgeResolveResult
```

知识系统不使用随机数。
