# 先驱者：认知状态设计

## 1. 设计目标

认知状态系统负责记录“某个主体当前如何理解世界”。

它回答以下问题：

```text
谁知道？
知道哪个对象或行为？
认为它有什么效果？
在什么条件下成立？
有多相信？
是否存在误解？
是否需要继续验证？
这份认知能否进入记忆或知识系统？
AI 或前端应该看到怎样的主观世界？
```

在《先驱者》中，认知不是视觉揭示，不关心对象颜色、形态、清晰度。

认知关注的是：

```text
能否食用。
能否使用。
食用后有什么影响。
使用后有什么结果。
在哪些条件下成立。
风险是什么。
证据是否可靠。
是否能教给别人。
```

核心链路：

```text
FeedbackEvidence
  ↓
CognitionResolver
  ↓
CognitionState / CognitionEntry / CognitionUpdate
  ↓
MemoryCandidate
  ↓
KnowledgeClaim
  ↓
Propagation / Civilization
```

本系统必须保证：

- 认知是主观状态，不等于世界真实规则。
- 认知可以错误、冲突、衰减、修正。
- 认知可以按主体隔离，先驱者知道不代表族群知道。
- 认知可以按条件细化，避免“所有红色果实都能吃”这种错误泛化无法修正。
- 认知更新可复盘、可解释、可测试。
- 认知投影可服务前端、AI 决策和强化学习。
- 后续记忆、知识、传播、文明系统可以稳定消费认知输出。

## 2. 不解决的问题

| 问题 | 原因 | 归属文档 |
|---|---|---|
| 生成最终记忆 | 认知状态只记录当前理解，不负责记忆沉淀 | `12_记忆系统设计.md` |
| 生成最终 KnowledgeClaim | 认知状态只提供候选和置信变化，不合并稳定知识 | `13_知识系统设计.md` |
| 知识传播 | 认知可判断是否可教，但不执行传播 | `14_传播系统设计.md` |
| 族群行为改变 | 认知可成为输入，但不直接改族群策略 | `15_族群状态设计.md` |
| 文明能力解锁 | 认知可提供前置条件，但不负责解锁 | `16_文明能力设计.md` |
| 世界真实规则推导 | 认知只能基于证据更新，不能偷看真实规则 | 配置 / 交互 / 特性系统 |
| UI 动画表现 | 认知系统可输出投影，不定义动画 | `20_前后端协议设计.md` |
| RL 最终 reward | 认知可提供 observation，不定义奖励函数 | 后续 AI 训练系统 |

关键边界：

```text
FeedbackEvidence：一次反馈能作为证据。
CognitionState：主体当前相信什么。
Memory：主体经历过什么。
KnowledgeClaim：被沉淀、可引用、可传播的知识断言。
```

示例：

```text
第一次吃红浆果恢复饥饿。

FeedbackEvidence：red_berry + eat + restore_hunger + fresh
CognitionEntry：我认为新鲜红浆果可能能缓解饥饿
MemoryCandidate：我吃过一次红浆果，饥饿下降
KnowledgeCandidate：新鲜红浆果可食用
KnowledgeClaim：多次验证后形成稳定知识
```

## 3. 核心概念

| 概念 | 中文名 | 含义 |
|---|---|---|
| CognitionOwner | 认知拥有者 | 谁拥有这份认知，例如先驱者、族群、族人 |
| CognitionState | 认知状态 | 某个主体当前所有认知条目的集合 |
| CognitionEntry | 认知条目 | 对一个“主体 + 行为 + 效果 + 条件”的理解 |
| CognitionKey | 认知键 | 用于唯一定位一条认知的组合键 |
| CognitionBelief | 认知信念 | 当前相信的结论和置信度 |
| CognitionUncertainty | 认知不确定性 | 尚未确认、需要继续验证的部分 |
| CognitionMisconception | 错误认知 | 当前可能错误或已被反证的理解 |
| CognitionUpdate | 认知更新 | 一次证据进入后产生的认知变化 |
| CognitionResolver | 认知解析器 | 根据证据更新认知状态 |
| CognitionUpdatePolicy | 认知更新策略 | 配置化的置信度、状态转换和冲突处理规则 |
| CognitionProjection | 认知投影 | 按主体认知过滤后的主观世界视图 |
| CognitionTrace | 认知解释链 | 解释某次认知为何变化 |
| MemoryCandidate | 记忆候选 | 认知系统输出给记忆系统的候选 |
| KnowledgePromotionHint | 知识晋升提示 | 认知系统输出给知识系统的候选信号 |

### 3.1 认知条目的基本表达

认知条目必须优先表达：

```text
对象 / 对象类型
+ 行为
+ 效果
+ 条件
+ 证据
+ 置信度
```

例如：

```text
red_berry + eat + restore_hunger + state:fresh + confidence:0.35
red_berry + eat + poison + state:decayed + confidence:0.72
fire + use_with dry_branch + transform_to_torch + dry + confidence:0.80
```

### 3.2 主观认知与真实规则分离

```text
真实规则属于配置、特性和交互规则。
认知状态属于主体当前理解。
```

同一个世界对象可以同时存在不同主体的认知：

```text
先驱者：红浆果可能可食用，置信度 0.4
族群：红浆果未知，置信度 0.0
某族人：听说红浆果能吃，置信度 0.2
系统真实规则：新鲜红浆果可食用，腐烂红浆果有风险
```

## 4. 枚举设计

本阶段不新增通用枚举，优先使用 `02_通用枚举设计.md` 中已定义的认知相关枚举。

### 4.1 使用的既有枚举

| 枚举 | 中文名 | 本系统用途 |
|---|---|---|
| CognitionOwnerType | 认知归属类型 | 表达认知属于先驱者、族群、族人、外部记录等 |
| KnowledgeScope | 知识范围 | 表达认知影响范围，个人、小群体、族群、区域、文明等 |
| KnowledgeStatus | 知识状态 | 表达认知条目当前生命周期状态 |
| MemoryType | 记忆类型 | 输出 MemoryCandidate 时建议记忆类型 |
| EvidenceType | 证据类型 | 表达认知更新所依赖的证据来源 |
| ConfidenceChangeReason | 置信度变化原因 | 表达置信度为什么变化 |
| FeedbackReliability | 反馈可靠性 | 从反馈证据继承可靠性 |
| FeedbackIntensity | 反馈强度 | 影响认知更新权重 |
| ActionType | 行为类型 | 表达认知和行为的关系 |
| EventType | 事件类型 | 输出认知相关事件草稿时使用 |
| EventVisibility | 事件可见性 | 控制认知变化事件是否展示 |
| EventImportance | 事件重要性 | 控制认知变化事件展示优先级 |
| ErrorSeverity | 错误严重级别 | 表达认知更新失败级别 |
| RuleStepType | 规则步骤类型 | 使用 `ResolveCognition` |
| StateChangeType | 状态变化类型 | 认知状态变更草稿使用 Create、Update、Delete、Append |

### 4.2 CognitionOwnerType 使用规则

| 值 | 中文 | 使用方式 |
|---|---|---|
| Pioneer | 先驱者 | 玩家角色个人认知，默认核心认知主体 |
| Tribe | 族群 | 族群共享认知，不等于所有个体都亲自验证 |
| TribeMember | 族群成员 | 单个族人或小群体认知 |
| ExternalRecord | 外部记录 | 石碑、符号、遗迹记录中的信息来源 |
| Environment | 环境线索 | 环境稳定规律提供的认知来源 |
| System | 系统 | 测试、初始化、校验使用，不应直接给玩家当叙事知识 |

注意：

```text
Pioneer 认知不能自动同步到 Tribe。
Tribe 认知不能自动视为每个 TribeMember 的亲身经验。
ExternalRecord 需要解释和验证，不能直接变成 Established。
```

### 4.3 KnowledgeStatus 在认知状态中的使用

认知状态可以复用 `KnowledgeStatus`，但含义偏向“认知条目状态”，不是最终知识状态。

| 状态 | 中文 | 认知层含义 |
|---|---|---|
| Draft | 草稿 | 刚收到证据，尚未整理 |
| Hypothesis | 假设 | 初步相信，但证据不足 |
| Active | 有效 | 当前可用于个人判断 |
| Teachable | 可传授 | 已足够清楚，可建议进入知识系统 |
| Shared | 已共享 | 已有传播迹象，但不代表完全稳定 |
| Operational | 可行动 | 已能驱动稳定行为策略 |
| Established | 已确立 | 长期稳定，接近常识或制度前置 |
| Disputed | 有争议 | 存在冲突证据 |
| Deprecated | 已废弃 | 被更精确认知替代 |
| Disproven | 已证伪 | 已被证据证明错误 |

### 4.4 EvidenceType 到认知更新的建议映射

| EvidenceType | 中文 | 默认影响 |
|---|---|---|
| DirectTest | 直接测试 | 中等提升或降低置信度 |
| RepeatedTest | 重复测试 | 明显提升置信度 |
| Observation | 观察证据 | 小到中等提升，取决于可解释性 |
| Accident | 意外证据 | 对风险认知权重高，对正向用途权重低 |
| TribeReport | 族群报告 | 初始较弱，后续需验证 |
| ExternalRecord | 外部记录 | 初始较弱，但可开启假设 |
| Inference | 推断证据 | 较弱，不能直接 Established |
| Contradiction | 反证 | 降低置信度，可能进入 Disputed / Disproven |
| LongTermUse | 长期使用 | 大幅提升，可推动 Operational / Established |
| SystemValidation | 系统验证 | 仅测试和调试，不进入玩家叙事认知 |

### 4.5 ConfidenceChangeReason 使用规则

| 原因 | 中文 | 典型来源 |
|---|---|---|
| NewObservation | 新观察 | 第一次观察到对象反应 |
| SuccessfulTest | 成功实验 | 行为结果支持当前认知 |
| FailedTest | 失败实验 | 行为结果不支持当前认知 |
| RepeatedConfirmation | 重复确认 | 多次结果一致 |
| ContradictoryEvidence | 矛盾证据 | 新证据和旧认知冲突 |
| TeachingAccepted | 传授被接受 | 后续传播系统使用 |
| TeachingFailed | 传授失败 | 后续传播系统使用 |
| TribeUsageSuccess | 族群使用成功 | 后续族群系统使用 |
| TribeUsageFailure | 族群使用失败 | 后续族群系统使用 |
| EnvironmentalChange | 环境变化 | 条件变化导致适用性变化 |
| Decay | 记忆衰减 | 长时间未验证 |
| Correction | 修正 | 添加条件、替换旧条目或修正错误泛化 |

## 5. 数据结构

### 5.1 CognitionOwner（认知拥有者）

`CognitionOwner` 描述认知属于谁。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| owner_type | CognitionOwnerType | 拥有者类型 | 先驱者、族群、族人等 |
| owner_id | string | 拥有者 ID | EntityId、TribeId、RegionId 或记录 ID 的字符串形式 |
| scope | KnowledgeScope | 影响范围 | Personal、Tribe、Region 等 |
| display_name_key | string | 显示名键 | 前端或日志使用 |

约定：

```text
owner_type 决定 owner_id 应解释成什么 ID。
owner_id 对外使用字符串，C++ 内部应在上下文中转为强类型 ID。
```

### 5.2 CognitionState（认知状态）

`CognitionState` 是某个主体当前所有认知的集合。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| owner | CognitionOwner | 认知拥有者 | 谁的认知状态 |
| entries | map<CognitionKey, CognitionEntry> | 认知条目 | 当前全部认知 |
| misconceptions | list<CognitionMisconception> | 错误认知 | 已发现或疑似错误理解 |
| uncertainty_index | list<CognitionUncertainty> | 不确定索引 | 需要继续验证的点 |
| source_event_ids | list<EventId> | 来源事件 | 影响过该状态的事件 |
| state_version | StateVersion | 状态版本 | 跟随权威状态版本记录 |
| updated_at | GameTime | 更新时间 | 最近更新时间 |

注意：

```text
CognitionState 可以持久化。
CognitionState 不能存储完整世界对象快照。
CognitionState 不应保存真实规则答案。
```

### 5.3 CognitionKey（认知键）

`CognitionKey` 用于唯一定位一条认知。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| subject_kind | string | 主体类型 | object_definition / object_instance / trait / region / action |
| subject_id | string | 主体 ID | ObjectDefinitionId、ObjectId、TraitId 等字符串形式 |
| action_type | ActionType | 行为类型 | Eat、Use、Observe、Combine 等 |
| effect_summary | string | 效果摘要 | restore_hunger、poison、transform 等 |
| condition_key | string | 条件键 | fresh、decayed、with_fire、region:forest 等 |

设计原则：

```text
优先使用 ObjectDefinitionId 形成可传播认知。
ObjectId 只用于实例级认知，例如“这一棵树已经被采集过”。
condition_key 必须存在，未知条件使用 unknown，不允许空字符串。
```

示例：

```text
object_definition:red_berry + Eat + restore_hunger + state:fresh
object_definition:red_berry + Eat + poison + state:decayed
object_definition:dry_branch + Use + transform_to_torch + with:fire
```

### 5.4 CognitionEntry（认知条目）

`CognitionEntry` 表示主体对一条认知键的当前理解。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| key | CognitionKey | 认知键 | 认知对象 |
| belief | CognitionBelief | 认知信念 | 当前相信的内容 |
| status | KnowledgeStatus | 状态 | Draft、Hypothesis、Active 等 |
| evidence_refs | list<CognitionEvidenceRef> | 证据引用 | 支撑或反对该认知的证据 |
| uncertainty | CognitionUncertainty | 不确定性 | 条件、样本数、冲突等 |
| first_seen_at | GameTime | 首次形成时间 | 第一次形成该认知 |
| last_updated_at | GameTime | 最近更新时间 | 最近一次证据影响 |
| update_count | Count | 更新次数 | 被证据更新过几次 |
| promotion_hint | KnowledgePromotionHint | 晋升提示 | 是否建议进入知识系统 |
| trace_ids | list<string> | 解释链 | 关联认知 trace |

### 5.5 CognitionBelief（认知信念）

`CognitionBelief` 描述当前主体相信什么。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| statement | string | 认知陈述 | 人类可读摘要，不作为规则判断唯一依据 |
| confidence | Ratio | 置信度 | 0.0 到 1.0 |
| reliability | FeedbackReliability | 可靠性 | 从证据综合得出 |
| positive_evidence_count | Count | 支持证据数 | 支持该认知的证据数量 |
| negative_evidence_count | Count | 反对证据数 | 反对该认知的证据数量 |
| last_change_reason | ConfidenceChangeReason | 最近变化原因 | 为什么变化 |
| confidence_delta_last | Delta | 最近变化值 | 最近一次变化量 |
| is_actionable | bool | 是否可用于行动 | 是否可影响主体决策 |
| is_teachable_candidate | bool | 是否可传授候选 | 是否建议进入知识系统 |

约定：

```text
confidence 是认知强度，不是客观真理概率。
statement 只用于解释和调试，规则判断必须使用 key 和结构化字段。
```

### 5.6 CognitionEvidenceRef（认知证据引用）

`CognitionEvidenceRef` 记录认知条目引用过哪些证据。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| evidence_id | string | 证据 ID | 来自 FeedbackEvidence |
| source_event_id | EventId | 来源事件 | 通常是 FeedbackGenerated |
| evidence_type | EvidenceType | 证据类型 | DirectTest、Observation 等 |
| reliability | FeedbackReliability | 可靠性 | 证据可靠性 |
| intensity | FeedbackIntensity | 强度 | 反馈强度 |
| confidence_delta | Delta | 置信变化建议 | 该证据带来的建议变化 |
| supports_belief | bool | 是否支持 | true 支持，false 反对 |
| condition_summary | string | 条件摘要 | 证据发生条件 |
| created_at | GameTime | 产生时间 | 证据时间 |

### 5.7 CognitionUncertainty（认知不确定性）

`CognitionUncertainty` 描述认知尚不稳定的地方。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| unknown_conditions | list<string> | 未知条件 | 不知道哪些条件会改变结果 |
| missing_evidence_types | list<EvidenceType> | 缺失证据 | 还缺什么证据类型 |
| required_repeat_count | Count | 所需重复次数 | 距离稳定还需多少次 |
| current_repeat_count | Count | 当前重复次数 | 已有多少次一致结果 |
| conflict_count | Count | 冲突数 | 有多少冲突证据 |
| needs_controlled_test | bool | 需要控制测试 | 是否需要更严格实验 |
| uncertainty_summary | string | 不确定摘要 | 供调试和前端解释 |

示例：

```text
知道红浆果能吃，但不确定腐烂后是否安全。
知道火能驱狼，但不确定雨天是否有效。
```

### 5.8 CognitionMisconception（错误认知）

`CognitionMisconception` 表示错误或疑似错误理解。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| misconception_id | string | 错误认知 ID | 临时唯一标识 |
| owner | CognitionOwner | 拥有者 | 谁产生误解 |
| original_key | CognitionKey | 原始认知 | 被怀疑错误的认知 |
| corrected_key | CognitionKey | 修正认知 | 可选，更精确的新认知 |
| reason | ConfidenceChangeReason | 原因 | 通常为 ContradictoryEvidence 或 Correction |
| status_before | KnowledgeStatus | 修正前状态 | 原状态 |
| status_after | KnowledgeStatus | 修正后状态 | Disputed、Deprecated、Disproven 等 |
| evidence_refs | list<CognitionEvidenceRef> | 证据引用 | 反证或修正证据 |
| correction_summary | string | 修正摘要 | 说明如何修正 |

示例：

```text
错误认知：所有红色浆果都能吃。
修正认知：新鲜红浆果可食用，腐烂红浆果有毒性风险。
```

### 5.9 CognitionUpdate（认知更新）

`CognitionUpdate` 表示一次输入证据导致的认知变化。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| update_id | string | 更新 ID | 本次更新唯一标识 |
| owner | CognitionOwner | 拥有者 | 更新谁的认知 |
| source_evidence_id | string | 来源证据 | 来自 FeedbackEvidence |
| affected_keys | list<CognitionKey> | 影响认知键 | 被影响的条目 |
| created_entries | list<CognitionEntry> | 新建条目 | 新产生的认知 |
| updated_entries | list<CognitionEntry> | 更新条目 | 已存在认知变化后结果 |
| deprecated_entries | list<CognitionKey> | 废弃条目 | 被替代或失效的旧认知 |
| misconceptions | list<CognitionMisconception> | 错误认知变化 | 新发现或修正的误解 |
| memory_candidates | list<MemoryCandidate> | 记忆候选 | 输出给记忆系统 |
| promotion_hints | list<KnowledgePromotionHint> | 知识晋升提示 | 输出给知识系统 |
| event_drafts | list<EventDraft> | 事件草稿 | 交给事件系统创建事件 |
| trace | CognitionTrace | 解释链 | 为什么这样变化 |
| errors | list<ErrorDetail> | 错误 | 更新失败或异常 |

### 5.10 MemoryCandidate（记忆候选）

`MemoryCandidate` 是认知系统给记忆系统的输出，不是最终记忆。

| 字段 | 类型 | 中文名 | 含义 |
|---|---|---|---|
| candidate_id | string | 候选 ID | 临时标识 |

## 7. 认知状态流转

### 7.1 新证据形成假设

```text
1. FeedbackEvidence 进入 CognitionResolver。
2. 根据 subject、action、effect、condition 生成 CognitionKey。
3. CognitionEntry 不存在。
4. 创建新 CognitionEntry。
5. 初始状态设为 Draft 或 Hypothesis。
6. 根据证据类型和可靠性设置 confidence。
7. 输出 MemoryCandidate 和 KnowledgePromotionHint。
```

示例：

```text
第一次吃红浆果恢复饥饿。
状态：Hypothesis
置信度：0.25 - 0.40
```

### 7.2 重复验证提升置信度

```text
1. 新证据命中已有 CognitionKey。
2. 证据支持旧认知。
3. positive_evidence_count 增加。
4. confidence 上升。
5. status 从 Hypothesis → Active → Teachable。
6. 达到阈值后输出 ready_for_knowledge=true。
```

### 7.3 反证导致争议

```text
1. 新证据与已有 CognitionEntry 冲突。
2. negative_evidence_count 增加。
3. contradiction_detected=true。
4. status 转为 Disputed。
5. 输出 CognitionMisconception。
6. 如果能识别条件差异，创建更精确的新 CognitionKey。
```

示例：

```text
旧认知：red_berry + Eat + restore_hunger + condition:unknown
新证据：red_berry + Eat + poison + state:decayed
修正：新鲜可食用，腐烂有风险
```

### 7.4 条件化修正

```text
1. 发现冲突不一定说明旧认知完全错误。
2. 先检查 condition_summary 是否不同。
3. 如果条件不同，旧条目降级为 Deprecated 或 Disputed。
4. 新建两个更精确条目。
5. 输出 Correction。
```

设计原则：

```text
优先修正条件，不要粗暴删除旧认知。
```

### 7.5 认知衰减

第一阶段只预留，不强制实现自动衰减。

```text
长时间未验证的低置信认知，可以降低 confidence。
衰减原因使用 ConfidenceChangeReason.Decay。
```

### 7.6 个人认知到族群认知

认知状态系统只输出提示，不直接传播。

```text
Pioneer CognitionEntry 达到 Teachable
  ↓
KnowledgePromotionHint.ready_for_teaching = true
  ↓
后续 KnowledgeSystem / PropagationSystem 处理
```

## 8. 事件输出

认知状态是规则状态的一部分，因此重要变化必须有事件草稿。

### 8.1 第一阶段事件兼容策略

由于 `02_通用枚举设计.md` 暂未定义独立 `CognitionUpdated` 事件，第一阶段采用兼容策略：

| 场景 | 事件草稿 | 可见性 |
|---|---|---|
| 认知形成但未稳定 | KnowledgeUpdated | Hidden / DebugOnly |
| 认知达到可行动 | KnowledgeUpdated | PlayerVisible |
| 认知达到可传授 | KnowledgeUpdated | PlayerVisible / TribeVisible |
| 发现矛盾证据 | KnowledgeContradicted | PlayerVisible |
| 发现对象用途或风险 | TraitDiscovered | PlayerVisible |
| 形成记忆候选 | 不直接输出 MemoryCreated | 交给记忆系统 |
| 创建最终知识 | 不直接输出 KnowledgeCreated | 交给知识系统 |

### 8.2 事件草稿字段建议

| 字段 | 来源 |
|---|---|
| event_type | KnowledgeUpdated / KnowledgeContradicted / TraitDiscovered |
| actor_id | CognitionContext.actor_id |
| command_id | CognitionContext.command_id |
| source_event_ids | CognitionContext.source_event_ids |
| visibility | 根据 status 和 importance 决定 |
| importance | 根据 confidence_delta、risk、contradiction 决定 |
| payload | KnowledgeEventPayload 或兼容摘要 |
| episode_id | CognitionContext.episode_id，可选 |
| decision_id | CognitionContext.decision_id，可选 |

### 8.3 注意事项

```text
认知系统不直接写 EventStore。
认知系统只输出 event_drafts。
如果后续新增 CognitionUpdated，必须更新 02、04、20、21。
```

| effect_summary 为空 | RuleError |
| condition_key 为空 | RuleError |
| ObjectId 与 ObjectDefinitionId 混用 | RuleError |

### 11.2 认知状态校验

| 规则 | 错误级别 |
|---|---|
| confidence 不在 0 到 1 | RuleError |
| positive_evidence_count 为负 | RuleError |
| negative_evidence_count 为负 | RuleError |
| status 与 confidence 阈值严重矛盾 | RuleError |
| Disproven 条目仍 is_actionable=true | RuleError |
| Deprecated 条目仍 ready_for_teaching=true | RuleError |
| SystemValidation 进入玩家可见认知 | DataError |

### 11.3 配置校验

| 规则 | 错误级别 |
|---|---|
| CognitionUpdatePolicy.id 重复 | DataError |
| priority 缺失 | DataError |
| 阈值不在 0 到 1 | DataError |
| 阈值顺序错误 | DataError |
| contradiction_policy 未知 | DataError |
| ExpressionId 不存在 | DataError |
| policy 试图直接创建 KnowledgeClaim | RuleError |

### 11.4 更新校验

| 规则 | 处理 |
|---|---|
| 证据无 subject_id | 返回 RuleError |
| 证据无法生成 CognitionKey | 返回 RuleError |
| 没有命中策略 | 使用默认保守策略或返回 Warning |
| 同一证据重复应用 | 必须幂等，不重复增加置信度 |
| 冲突无法归因条件 | 标记 Disputed，不强行 Disproven |
| confidence 更新后越界 | clamp 到 0 到 1，并记录 trace |

## 14. 设计注意事项

研发实现时必须遵守：

```text
不要把认知状态当成真实规则。
不要让认知系统读取对象隐藏特性来作弊。
不要因为先驱者知道就自动让族群知道。
不要把 FeedbackEvidence 直接变成 KnowledgeClaim。
不要给 MemoryCandidate 分配 MemoryId。
不要给 KnowledgePromotionHint 分配 KnowledgeId。
不要删除错误认知，应该保留 Disputed / Deprecated / Disproven 历史。
不要让前端自行计算置信度变化。
不要让 AI Observation 读取未认知到的规则。
```

### 14.1 条件化优先原则

当新证据和旧认知冲突时，优先判断是否是条件不同导致：

```text
新鲜可食用，腐烂有毒。
晴天火能驱狼，暴雨中火熄灭。
干树枝可燃，湿树枝难以点燃。
```

不要简单地把旧认知全部删除。

### 14.2 主体隔离原则

```text
Pioneer CognitionState
Tribe CognitionState
TribeMember CognitionState
AI Entity CognitionState
```

这些状态必须可以不同步。

同步必须通过后续传播、教学、观察或长期使用完成。

### 14.3 可复盘原则

同一输入应得到同一输出：

```text
同一个 CognitionState
+ 同一组 FeedbackEvidence
+ 同一个 CognitionUpdatePolicy
+ 同一个 StateVersion
= 同一个 CognitionUpdateResult
```

认知系统不使用随机数。

## 16. 阶段结论

认知状态系统是《先驱者》核心玩法的主观理解层。

它让游戏能够表达：

```text
我不知道。
我猜测。
我试过一次。
我多次验证。
我以前误解了。
我修正了条件。
我知道但族群不知道。
AI 只能根据自己的认知行动。
```

当前设计为后续系统提供稳定接口：

```text
反馈系统 → 认知状态 → 记忆系统 → 知识系统 → 传播系统 → 族群状态 → 文明能力
```

```text
12_记忆系统设计.md
```

记忆系统将负责把认知更新中的重要经历沉淀为可回忆、可衰减、可整理、可用于知识形成的记忆记录。
