# P18 知识系统任务卡设计

## 1. 阶段定位

P18 是《先驱者》认知、记忆、知识、传播学习主线的第四个核心玩法阶段。

前置阶段已经完成：

```text
P15：认知系统正式化，回答“主体当前相信什么”。
P16：记忆系统基础，回答“一次经历是否会被记住、强化、衰减”。
P17：记忆压缩与检索，回答“大量记忆如何安全摘要、可追溯召回”。
```

P18 解决：

```text
稳定、可追溯的记忆摘要，是否足以形成一条知识声明。
这条知识属于谁。
它描述哪个对象、行为、效果和条件。
它有多少证据支撑。
它当前有多可信。
它是否可以被 Agent 查询和后续传播系统使用。
```

一句话：

```text
P18 让“我多次记得红果吃了能恢复需求”变成“我拥有一条可引用、可查询、可传授候选的知识”。
```

P18 在主链路中的位置：

```text
探索 → 反馈 → 认知 → 记忆 → 压缩/召回 → 知识 → 传播 → 文明
```

P18 只做到“知识形成与查询”，不做传播、不做冲突修正、不做文明解锁。

---

## 2. 本阶段目标

P18 的目标是建立一个可扩展、可审计、对 AI 实现友好的知识形成层。

必须做到：

1. 从 P17 `MemorySummary` 和必要的 P16 `MemoryRecord` 引用生成 `KnowledgeClaim`。
2. `KnowledgeClaim` 必须保留来源 `MemorySummaryId`、`MemoryId`、`MemoryEvidenceRef` 追溯链。
3. 支持 `KnowledgeFormationService` 判断摘要是否足以形成知识。
4. 支持 `KnowledgeRepository` 保存和确定性查询知识。
5. 支持基础 `KnowledgeProjection`，让 Agent/后续系统读取“主观知识视图”。
6. 支持基础 confidence 计算，但不做复杂冲突修正。
7. 支持输出知识事件草稿和状态变更草稿，但不直接改 GameState。
8. 保持 hidden truth 防线：不能读取 ObjectDefinition hidden truth，不能读取 raw GameState。
9. 给 P19 冲突修正、P20 传播系统留下稳定接口。

核心原则：

```text
P18 的知识是证据形成的主观断言，不是真实世界属性。
P18 的知识可以不完整，但必须可追溯。
P18 的知识可以是错误假设，但不能偷看真相。
P18 的知识形成要保守，不从单条弱证据直接生成强知识。
```

## 3. 玩家体验映射

P18 对玩家体验的意义：

```text
玩家多次尝试红果后，系统形成“红果 + Eat + restore_need”的知识。
玩家可以看到这不是神谕，而是来自多次经历的总结。
Agent 再遇到红果时，可以查询该知识作为行动依据。
后续 P20 中，玩家可以把该知识传播给族群。
如果后续发现腐烂红果有毒，P19 再处理冲突与条件化修正。
```

P18 的玩家体验目标不是 UI 展示，而是让游戏主线第一次出现：

```text
经历可以沉淀为可复用知识。
```

## 5. 本阶段不做什么

P18 明确禁止：

```text
不做 KnowledgeConflictResolver。
不做 KnowledgeRevision。
不做 KnowledgeMergePolicy。
不做复杂知识冲突修正。
不做知识传播。
不做族群共享知识写入。
不做 H5/API/HTTP/WebSocket/JSON。
不做 SaveManager。
不做 RL reward/done。
不让 AgentRuntime 或 Policy 直接写 KnowledgeRepository。
不删除 MemoryRecord。
不删除 MemorySummary。
不读取 ObjectDefinition hidden truth。
不读取 GameState 原始隐藏状态生成知识。
不从配置真值直接生成 KnowledgeClaim。
```

P18 可以预留冲突字段和传播候选字段，但不能实现 P19/P20 核心逻辑。

## 8. ID 设计

### 8.1 KnowledgeId

使用已有：

```cpp
pathfinder::foundation::KnowledgeId
```

### 8.2 KnowledgeEvidenceId

在 knowledge 模块内定义：

```cpp
struct KnowledgeEvidenceIdTag {};
using KnowledgeEvidenceId = pathfinder::foundation::StrongId<KnowledgeEvidenceIdTag>;
```

### 8.3 KnowledgeTraceId

在 knowledge 模块内定义：

```cpp
struct KnowledgeTraceIdTag {};
using KnowledgeTraceId = pathfinder::foundation::StrongId<KnowledgeTraceIdTag>;
```

### 8.4 ID 生成规则

必须通过工厂生成，不允许业务手写随机 ID。

```cpp
class KnowledgeIdFactory {
public:
    pathfinder::foundation::Result<pathfinder::foundation::KnowledgeId> makeKnowledgeId(
        const KnowledgeOwner& owner,
        const KnowledgeSubject& subject,
        const KnowledgePredicate& predicate,
        pathfinder::foundation::Tick created_tick,
        size_t sequence_index) const;

    pathfinder::foundation::Result<KnowledgeEvidenceId> makeEvidenceId(
        const pathfinder::foundation::KnowledgeId& knowledge_id,
        size_t evidence_index) const;

    pathfinder::foundation::Result<KnowledgeTraceId> makeTraceId(
        const pathfinder::foundation::KnowledgeId& knowledge_id,
        size_t trace_index) const;
};
```

建议格式：

```text
know_{owner_key}_{subject_key}_{predicate_key}_{tick}_{sequence}
kevd_{knowledge_id}_{index}
ktrace_{knowledge_id}_{index}
```

必须满足 `foundation::isValidIdString`。

注意：

```text
KnowledgeId 代表主观知识声明，不代表真实 ObjectDefinition 属性。
同一对象可以有多条知识，例如 fresh red berry 可食用、decayed red berry 有毒。
```

## 10. DTO 设计

所有 DTO 必须实现：

```cpp
pathfinder::foundation::Result<void> validateBasic() const;
```

任何 `Result::ok(dto)` 返回的 DTO 必须能通过 `validateBasic()`。

### 10.1 KnowledgeOwner

中文：知识拥有者。

```cpp
struct KnowledgeOwner {
    KnowledgeOwnerKind kind = KnowledgeOwnerKind::Unknown;
    pathfinder::foundation::EntityId entity_id;
    pathfinder::foundation::TribeId tribe_id;
    pathfinder::foundation::RegionId region_id;
    std::string external_key;

    bool operator==(const KnowledgeOwner& other) const = default;
    pathfinder::foundation::Result<void> validateBasic() const;
};
```

规则：

```text
Agent/Actor 必须有 entity_id。
Tribe 必须有 tribe_id。
Region 必须有 region_id。
ExternalRecord/TestOnly 必须有 external_key。
Unknown 非法。
默认不允许 TestOnly。
```

### 10.2 KnowledgeSubject

中文：知识主体。

```cpp
struct KnowledgeSubject {
    KnowledgeSubjectKind kind = KnowledgeSubjectKind::Unknown;
    std::string subject_id;
    std::string subject_type_key;
    std::vector<std::string> related_subject_ids;
    std::string relation_group_key;
    std::vector<std::string> safe_tags;

    bool operator==(const KnowledgeSubject& other) const = default;
    pathfinder::foundation::Result<void> validateBasic() const;
};
```

规则：

```text
subject_id 必须非空。
Combination 必须有 related_subject_ids 或 relation_group_key。
subject_id、subject_type_key、safe_tags 必须过 hidden truth guard。
```

### 10.3 KnowledgePredicate

中文：知识谓词。

```cpp
struct KnowledgePredicate {
    KnowledgeRelationType relation_type = KnowledgeRelationType::Unknown;
    pathfinder::foundation::ActionId action_id;
    std::string action_key;
    std::string effect_key;
    std::vector<pathfinder::foundation::EffectId> effect_ids;
    std::vector<pathfinder::foundation::TraitId> trait_ids;
    std::string polarity_key;
    std::string value_summary_key;
    std::vector<std::string> safe_tags;

    bool operator==(const KnowledgePredicate& other) const = default;
    pathfinder::foundation::Result<void> validateBasic() const;

```

规则：

```text
relation_type 不能 Unknown。
action_id 或 action_key 至少一个非空。
effect_key、value_summary_key 只能是安全摘要键，不允许 hidden truth 字段名。
P18 不允许直接写 edible_profile、hunger_delta、health_delta 等真实字段。
```

### 10.5 KnowledgeEvidence

中文：知识证据。

```cpp
struct KnowledgeEvidence {
    KnowledgeEvidenceId evidence_id;
    KnowledgeEvidenceKind evidence_kind = KnowledgeEvidenceKind::Unknown;
    pathfinder::memory::MemorySummaryId source_summary_id;
    std::vector<pathfinder::foundation::MemoryId> source_memory_ids;
    std::vector<pathfinder::memory::MemoryEvidenceRef> memory_evidence_refs;
    std::vector<pathfinder::foundation::EventId> source_event_ids;
    bool supports_claim = true;
    double confidence_delta = 0.0;
    double reliability = 1.0;
    pathfinder::foundation::Tick observed_tick;
    std::vector<std::string> reason_keys;

    bool operator==(const KnowledgeEvidence& other) const = default;
    pathfinder::foundation::Result<void> validateBasic() const;
};
```

规则：

```text
evidence_id 必须非空且格式合法。
evidence_kind 不能 Unknown。
source_summary_id 或 source_memory_ids 或 memory_evidence_refs 至少一个非空。
confidence_delta 范围 [-1.0, 1.0]。
reliability 范围 [0.0, 1.0]。
P18 主要生成 supports_claim=true 的证据。
supports_claim=false 只做 P19 预留，不在 P18 主动处理冲突。
```

中文：知识置信度。

```cpp
struct KnowledgeConfidence {

    size_t source_memory_count = 0;
```

规则：

```text

```

## 12. 知识形成算法

### 12.1 输入来源

P18 第一版只支持从 P17 `MemorySummary` 形成知识：

```text
MemorySummary.source_memory_count
MemorySummary.representative_memory_ids
MemorySummary.evidence_refs
MemorySummary.aggregate_strength
MemorySummary.highest_importance
MemorySummary.success_count / failure_count / warning_count / accident_count
MemorySummary.teaching_candidate_count
MemorySummary.has_long_term_source
MemorySummary.key.owner
MemorySummary.key.subject
```

P18 不直接从 ObjectDefinition、GameState 或配置隐藏真值形成知识。

### 12.2 最小形成条件

默认条件：

```text
source_memory_count >= 3
representative_memory_ids 非空
evidence_refs 非空
aggregate_strength >= 0.55
summary.key.summary_kind 不是 Unknown/TestOnly
如果 summary.contradiction_count > 0，默认跳过，留给 P19
```

### 12.3 关系类型推导

P18 不从 hidden truth 推导 relation，只从输入显式安全字段推导：

```text
KnowledgeFormationInput.target_relation
KnowledgeFormationInput.action_key
KnowledgeFormationInput.effect_key
KnowledgeFormationInput.candidate_conditions
```

如果没有足够安全字段：

```text
Planner 返回 fail。
Service 转换为 Skipped，reason_keys 包含 missing_safe_predicate。
```

### 12.4 confidence 计算

第一版建议公式：

```text
base = summary.aggregate_strength * 0.55
sample_bonus = min(source_memory_count / 10.0, 1.0) * 0.20
long_term_bonus = has_long_term_source ? 0.10 : 0.0
teaching_bonus = min(teaching_candidate_count / 5.0, 1.0) * 0.05
risk_penalty = (warning_count + accident_count) > 0 ? 0.05 : 0.0
contradiction_penalty = contradiction_count > 0 ? 0.20 : 0.0
confidence = clamp(base + sample_bonus + long_term_bonus + teaching_bonus - risk_penalty - contradiction_penalty, 0, 1)
```

注意：

```text
warning/accident 不一定阻止风险知识形成。
如果 target_relation 是 HasRisk 或 IsDangerousUnder，warning/accident 可以成为支持证据，而不是 penalty。
```

### 12.5 status 决策

```text
confidence < 0.35 -> Skipped
0.35 <= confidence < active_threshold -> Hypothesis
active_threshold <= confidence < teachable_threshold -> Active
confidence >= teachable_threshold -> Teachable
```

默认：

```text
active_threshold = 0.55
teachable_threshold = 0.75
```

### 12.6 teaching_profile 决策

```text
status == Teachable
AND confidence >= teachable_confidence_threshold
AND evidence_refs 非空
AND predicate 安全
AND 条件说明不为空或 relation 不需要条件
```

风险知识：

```text
HasRisk / IsDangerousUnder 的 teachable=true 时，risk_warning_required 必须 true。
```

## 20. 完成标准

```text

P18 正确完成后，P19 才能处理错误知识、冲突证据和条件化修正；P20 才能把知识传播给族群。
```
