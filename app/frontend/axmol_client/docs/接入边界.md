# Axmol 客户端接入边界

## 目标

把 Axmol 放进前端目录，作为未来替换网页客户端的独立表现层。

## 不做

- 不改后端引擎。
- 不下载或 vendoring Axmol 源码。
- 不接入真实 Runtime。
- 不实现任何玩法规则。
- 不替换现有网页客户端。

## 后续必须补的阶段

1. Axmol 构建脚本和外部 SDK 检测。
2. Client Protocol DTO 镜像或自动生成。
3. LocalRuntimeTransport / HttpTransport 二选一桥接。
4. Projection 到 Tile/Entity 的渲染映射。
5. Command option 到点击/触控输入的映射。
6. 客户端性能测试和手机端打包验证。

## 验收边界

本阶段只验收目录、边界、骨架是否成立：

- 所有新增文件在 `app/frontend/axmol_client/`。
- 后端无新增依赖。
- 前端骨架不含玩法规则。
- 删除 `app/frontend/axmol_client/` 不影响后端构建。
